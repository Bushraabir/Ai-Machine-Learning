## 1. Why Probability Is the Language of ML

Every machine learning algorithm is, at its core, a statement about uncertainty. A classifier does not know the true label — it assigns probabilities. A regression model does not know the true value — it estimates a conditional expectation under a noise model. A generative model does not know the true data distribution — it approximates it.

The bias-variance decomposition in the previous note was derived using expectations. The MLE derivation that produces least squares is a probability argument. The cross-entropy loss is a negative log-likelihood. Regularisation is a prior.

None of this requires exotic mathematics. It requires one framework, understood deeply. This note builds that framework from the ground up and connects every piece to where it reappears in machine learning.

## 2. Probability Spaces and Random Variables

### 2.1 The Probability Space

Formally, a probability space is a triple $(\Omega, \mathcal{F}, P)$:

- $\Omega$ — the **sample space**: the set of all possible outcomes.
- $\mathcal{F}$ — the **sigma-algebra**: the collection of events (subsets of $\Omega$) to which we can assign probability. Closed under complement and countable union.
- $P : \mathcal{F} \to [0,1]$ — the **probability measure**: assigns probabilities to events, satisfying $P(\Omega) = 1$ and countable additivity.

For ML purposes, the formalism matters less than the intuition: $P(A)$ is the long-run fraction of times event $A$ occurs if the experiment is repeated many times.

### 2.2 Random Variables

A **random variable** $X : \Omega \to \mathbb{R}$ is a function from outcomes to real numbers. It is a way to attach numerical values to outcomes.

For a discrete random variable taking values in $\mathcal{X} = {x_1, x_2, \ldots}$, the **probability mass function (PMF)** is:

$$p(x) = P(X = x), \quad p(x) \geq 0, \quad \sum_{x \in \mathcal{X}} p(x) = 1$$

For a continuous random variable, there is no probability mass at individual points. Instead, the **probability density function (PDF)** $f : \mathbb{R} \to \mathbb{R}_{\geq 0}$ satisfies:

$$P(a \leq X \leq b) = \int_a^b f(x), dx, \quad \int_{-\infty}^{\infty} f(x), dx = 1$$

Note: $f(x)$ is not a probability. It is a density. $f(x)$ can exceed 1. Only its integral over a region gives a probability.

The **cumulative distribution function (CDF)** unifies discrete and continuous cases:

$$F(x) = P(X \leq x)$$

## 3. Expectation, Variance, and Moments

### 3.1 Expectation

The **expected value** (mean) of a random variable $X$ is:

$$\mathbb{E}[X] = \begin{cases} \sum_x x \cdot p(x) & \text{discrete} \ \int_{-\infty}^{\infty} x \cdot f(x), dx & \text{continuous} \end{cases}$$

Expectation is a linear operator:

$$\mathbb{E}[aX + bY] = a\mathbb{E}[X] + b\mathbb{E}[Y]$$

for any constants $a, b$ and any random variables $X, Y$ — regardless of whether $X$ and $Y$ are independent. This is used constantly.

The **law of total expectation** (tower property):

$$\mathbb{E}[X] = \mathbb{E}\left[\mathbb{E}[X \mid Y]\right]$$

where $\mathbb{E}[X \mid Y]$ is itself a random variable — a function of $Y$. This appears in deriving EM, in computing marginal likelihoods, and in reinforcement learning (the Bellman equation is an application of the tower property).

### 3.2 Variance and Standard Deviation

$$\text{Var}(X) = \mathbb{E}\left[(X - \mathbb{E}[X])^2\right] = \mathbb{E}[X^2] - (\mathbb{E}[X])^2$$

The second form — $\mathbb{E}[X^2] - (\mathbb{E}[X])^2$ — is often easier to compute. Memorise it.

Variance is not linear. For independent $X, Y$:

$$\text{Var}(aX + bY) = a^2 \text{Var}(X) + b^2 \text{Var}(Y)$$

Standard deviation $\sigma = \sqrt{\text{Var}(X)}$ is in the same units as $X$.

### 3.3 Covariance and Correlation

$$\text{Cov}(X, Y) = \mathbb{E}[(X - \mathbb{E}[X])(Y - \mathbb{E}[Y])] = \mathbb{E}[XY] - \mathbb{E}[X]\mathbb{E}[Y]$$

Covariance measures the tendency of $X$ and $Y$ to move together. It depends on scale.

The **Pearson correlation coefficient** normalises this:

$$\rho(X, Y) = \frac{\text{Cov}(X, Y)}{\sqrt{\text{Var}(X) \cdot \text{Var}(Y)}} \in [-1, 1]$$

$\rho = 1$: perfect positive linear relationship. $\rho = -1$: perfect negative. $\rho = 0$: no linear relationship.

Important: $\rho = 0$ does not imply independence. $X$ and $X^2$ have zero correlation (when $X$ is symmetric around zero) but are clearly dependent.

If $X$ and $Y$ are **independent**, then $\text{Cov}(X, Y) = 0$, but not vice versa.

### 3.4 The Covariance Matrix

For a random vector $X = (X_1, \ldots, X_d)^T$, the **covariance matrix** is:

$$\Sigma = \text{Cov}(X) = \mathbb{E}[(X - \mu)(X - \mu)^T]$$

where $\mu = \mathbb{E}[X]$. The $(i,j)$ entry of $\Sigma$ is $\text{Cov}(X_i, X_j)$. The diagonal entries are the variances.

$\Sigma$ is always:

- **Symmetric**: $\Sigma = \Sigma^T$
- **Positive semidefinite**: $v^T \Sigma v \geq 0$ for all $v \in \mathbb{R}^d$

The covariance matrix is central to PCA, Gaussian distributions, and Mahalanobis distance. Understanding it as the generalisation of variance to multiple dimensions is essential.

## 4. Common Distributions

### 4.1 Bernoulli Distribution

$X \sim \text{Bernoulli}(p)$, $p \in [0,1]$:

$$P(X = 1) = p, \quad P(X = 0) = 1 - p$$ $$\mathbb{E}[X] = p, \quad \text{Var}(X) = p(1-p)$$

Models a single binary outcome — a coin flip, a click, a default. The natural distribution for binary classification labels.

### 4.2 Binomial Distribution

$X \sim \text{Binomial}(n, p)$: the number of successes in $n$ independent Bernoulli$(p)$ trials.

$$P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}$$ $$\mathbb{E}[X] = np, \quad \text{Var}(X) = np(1-p)$$

### 4.3 Categorical Distribution

$X \sim \text{Categorical}(p_1, \ldots, p_k)$, $\sum_j p_j = 1$:

$$P(X = j) = p_j$$

The multi-class generalisation of Bernoulli. The output distribution of a softmax classifier. The observation model in a multinomial naive Bayes.

### 4.4 Gaussian (Normal) Distribution

$X \sim \mathcal{N}(\mu, \sigma^2)$:

$$f(x) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$ $$\mathbb{E}[X] = \mu, \quad \text{Var}(X) = \sigma^2$$

The Gaussian is the most important distribution in statistics and ML. Three reasons:

1. **Central Limit Theorem**: sums of i.i.d. random variables converge to Gaussian regardless of the original distribution. Measurement errors, feature sums, and gradient noise are approximately Gaussian.
    
2. **Maximum entropy**: among all distributions with fixed mean $\mu$ and variance $\sigma^2$, the Gaussian has the highest entropy. It is the least informative distribution given only mean and variance.
    
3. **Conjugacy**: the Gaussian is its own conjugate prior under Gaussian likelihood. Bayesian updates stay Gaussian — closed-form posterior inference.
    

The standard normal $Z \sim \mathcal{N}(0,1)$ has CDF denoted $\Phi(z)$.

### 4.5 Multivariate Gaussian

$X \sim \mathcal{N}(\mu, \Sigma)$, $X \in \mathbb{R}^d$:

$$f(x) = \frac{1}{(2\pi)^{d/2}|\Sigma|^{1/2}} \exp!\left(-\frac{1}{2}(x-\mu)^T \Sigma^{-1} (x-\mu)\right)$$

The term $(x-\mu)^T \Sigma^{-1} (x-\mu)$ is the **Mahalanobis distance** — a distance metric that accounts for the covariance structure of the distribution, unlike Euclidean distance which treats all directions equally.

Key properties of the multivariate Gaussian:

- Marginals of a Gaussian are Gaussian.
- Conditionals of a Gaussian are Gaussian.
- Linear transformations of a Gaussian are Gaussian: if $X \sim \mathcal{N}(\mu, \Sigma)$ then $AX + b \sim \mathcal{N}(A\mu + b, A\Sigma A^T)$.

The shape of the distribution is determined by $\Sigma$:

- $\Sigma = \sigma^2 I$: spherical, all directions equally spread.
- $\Sigma$ diagonal: axis-aligned ellipsoid.
- $\Sigma$ full: rotated ellipsoid, axes along eigenvectors of $\Sigma$.

### 4.6 Exponential Distribution

$X \sim \text{Exp}(\lambda)$, $\lambda > 0$:

$$f(x) = \lambda e^{-\lambda x}, \quad x \geq 0$$ $$\mathbb{E}[X] = \frac{1}{\lambda}, \quad \text{Var}(X) = \frac{1}{\lambda^2}$$

Models waiting times between events in a Poisson process. **Memoryless**: $P(X > s + t \mid X > s) = P(X > t)$.

### 4.7 Beta Distribution

$X \sim \text{Beta}(\alpha, \beta)$, $\alpha, \beta > 0$, $X \in [0,1]$:

$$f(x) = \frac{x^{\alpha-1}(1-x)^{\beta-1}}{B(\alpha,\beta)}, \quad B(\alpha,\beta) = \frac{\Gamma(\alpha)\Gamma(\beta)}{\Gamma(\alpha+\beta)}$$ $$\mathbb{E}[X] = \frac{\alpha}{\alpha+\beta}$$

The Beta distribution is the **conjugate prior for the Bernoulli/Binomial likelihood**. If the prior on $p$ is $\text{Beta}(\alpha, \beta)$ and we observe $k$ successes in $n$ trials, the posterior is $\text{Beta}(\alpha + k, \beta + n - k)$. This is the canonical example of Bayesian updating in closed form.

### 4.8 Dirichlet Distribution

The multivariate generalisation of Beta. $X \sim \text{Dir}(\alpha_1, \ldots, \alpha_k)$, $X \in \Delta^{k-1}$ (the probability simplex):

$$f(x) \propto \prod_{j=1}^k x_j^{\alpha_j - 1}$$

The **conjugate prior for the Categorical/Multinomial**. Used extensively in topic models (LDA) and Bayesian naive Bayes.

### 4.9 The Exponential Family

Many distributions — Gaussian, Bernoulli, Binomial, Poisson, Exponential, Beta, Dirichlet — belong to the **exponential family**:

$$p(x; \eta) = h(x) \exp!\left(\eta^T T(x) - A(\eta)\right)$$

where:

- $\eta$ are the **natural parameters**
- $T(x)$ are **sufficient statistics**
- $A(\eta) = \log \int h(x) e^{\eta^T T(x)} dx$ is the **log-partition function** (ensures normalisation)
- $h(x)$ is the **base measure**

Why the exponential family matters for ML:

1. **Sufficient statistics**: $T(x)$ captures everything in the data relevant to estimating $\eta$ — you do not need the raw data, only $T(\mathcal{D}) = \sum_i T(x_i)$.
2. **MLE has a closed form**: the MLE condition $\mathbb{E}[T(x)] = \frac{1}{n}\sum_i T(x_i)$ (moment matching) always holds.
3. **Conjugate priors exist**: for every exponential family likelihood, there is a conjugate prior in the exponential family — Bayesian updating stays in the family.
4. **GLMs are exponential family**: the Generalised Linear Model framework assumes the output distribution is in the exponential family, which is why the gradient of the cross-entropy loss takes the same form for Gaussian, Bernoulli, and Poisson outputs.

## 5. Joint, Marginal, and Conditional Distributions

### 5.1 Joint Distribution

The **joint distribution** $p(x, y)$ specifies the probability of every combination of values of $X$ and $Y$.

For discrete variables:

$$\sum_x \sum_y p(x, y) = 1$$

### 5.2 Marginal Distribution

The **marginal distribution** of $X$ is obtained by summing (integrating) out $Y$:

$$p(x) = \sum_y p(x, y) \quad \text{(discrete)}, \qquad p(x) = \int p(x, y), dy \quad \text{(continuous)}$$

This is **marginalisation** — computing the distribution of one variable by averaging over all possible values of another. It appears everywhere: computing the evidence in Bayesian inference, deriving the EM algorithm, computing the marginal likelihood for model selection.

### 5.3 Conditional Distribution

$$p(y \mid x) = \frac{p(x, y)}{p(x)}$$

This is defined only when $p(x) > 0$. The conditional distribution of $Y$ given $X = x$ describes the distribution of $Y$ once we know the value of $X$.

The relationship between joint, marginal, and conditional:

$$p(x, y) = p(y \mid x), p(x) = p(x \mid y), p(y)$$

This is the **product rule** (or chain rule) of probability. Extending to $n$ variables:

$$p(x_1, \ldots, x_n) = p(x_1) \prod_{i=2}^n p(x_i \mid x_1, \ldots, x_{i-1})$$

### 5.4 Independence

$X$ and $Y$ are **independent** (written $X \perp Y$) if:

$$p(x, y) = p(x), p(y) \quad \text{for all } x, y$$

Equivalently: $p(y \mid x) = p(y)$ — knowing $X$ provides no information about $Y$.

$X$ and $Y$ are **conditionally independent given $Z$** (written $X \perp Y \mid Z$) if:

$$p(x, y \mid z) = p(x \mid z), p(y \mid z) \quad \text{for all } x, y, z$$

Conditional independence is the foundation of graphical models (Bayesian networks, Markov random fields) and naive Bayes.

## 6. Bayes' Theorem

Bayes' theorem follows directly from the product rule. From $p(x, y) = p(y \mid x) p(x) = p(x \mid y) p(y)$:

$$\boxed{p(\theta \mid \mathcal{D}) = \frac{p(\mathcal{D} \mid \theta), p(\theta)}{p(\mathcal{D})}}$$

Written in words: **posterior $\propto$ likelihood $\times$ prior**.

Each term has a name and a role:

|Term|Name|Meaning|
|---|---|---|
|$p(\theta \mid \mathcal{D})$|Posterior|Belief about $\theta$ after seeing data|
|$p(\mathcal{D} \mid \theta)$|Likelihood|Probability of data given parameters|
|$p(\theta)$|Prior|Belief about $\theta$ before seeing data|
|$p(\mathcal{D})$|Evidence (marginal likelihood)|Normalising constant, $= \int p(\mathcal{D}\mid\theta)p(\theta),d\theta$|

The evidence $p(\mathcal{D}) = \int p(\mathcal{D} \mid \theta) p(\theta), d\theta$ is the marginalised likelihood — it does not depend on $\theta$ and is often intractable to compute. For parameter estimation, we usually only need the posterior up to proportionality.

### 6.1 Bayesian Inference Workflow

1. Specify a prior $p(\theta)$ encoding beliefs before data.
2. Specify a likelihood model $p(\mathcal{D} \mid \theta)$.
3. Apply Bayes' theorem: $p(\theta \mid \mathcal{D}) \propto p(\mathcal{D} \mid \theta), p(\theta)$.
4. Use the posterior for prediction: $p(y^* \mid x^_, \mathcal{D}) = \int p(y^_ \mid x^*, \theta), p(\theta \mid \mathcal{D}), d\theta$.

Step 4 — the **posterior predictive distribution** — averages predictions over all parameter values weighted by their posterior probability. This is full Bayesian inference. It is usually intractable and requires approximation (MCMC, variational inference).

## 7. Maximum Likelihood Estimation

**Maximum Likelihood Estimation (MLE)** answers: what parameter value $\theta$ makes the observed data most probable?

Given i.i.d. data $\mathcal{D} = {x_1, \ldots, x_n}$ from a parametric model $p(x; \theta)$:

$$\hat{\theta}_{\text{MLE}} = \underset{\theta}{\arg\max}\ p(\mathcal{D}; \theta) = \underset{\theta}{\arg\max}\ \prod_{i=1}^n p(x_i; \theta)$$

The product over $n$ terms is numerically unstable and analytically awkward. Taking logarithm (monotone transformation, preserves argmax):

$$\hat{\theta}_{\text{MLE}} = \underset{\theta}{\arg\max}\ \sum_{i=1}^n \log p(x_i; \theta)$$

This is the **log-likelihood**. Maximising log-likelihood is equivalent to minimising **negative log-likelihood (NLL)**, which is the standard form of loss functions in ML.

### 7.1 MLE for the Gaussian

Given $x_1, \ldots, x_n \overset{\text{i.i.d.}}{\sim} \mathcal{N}(\mu, \sigma^2)$:

$$\log p(\mathcal{D}; \mu, \sigma^2) = -\frac{n}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_{i=1}^n (x_i - \mu)^2$$

Taking derivatives and setting to zero:

$$\frac{\partial}{\partial \mu}: \quad \hat{\mu}_{\text{MLE}} = \frac{1}{n}\sum_{i=1}^n x_i = \bar{x}$$

$$\frac{\partial}{\partial \sigma^2}: \quad \hat{\sigma}^2_{\text{MLE}} = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2$$

The MLE recovers the sample mean and (biased) sample variance. The $1/n$ in the variance estimator makes it biased — $\mathbb{E}[\hat{\sigma}^2_{\text{MLE}}] = \frac{n-1}{n}\sigma^2$. The unbiased estimator uses $1/(n-1)$, which is not the MLE.

### 7.2 MLE for Bernoulli

Given $x_1, \ldots, x_n \overset{\text{i.i.d.}}{\sim} \text{Bernoulli}(p)$:

$$\log p(\mathcal{D}; p) = \sum_{i=1}^n x_i \log p + (1 - x_i)\log(1-p)$$

$$\frac{d}{dp}\log p(\mathcal{D}; p) = \frac{\sum x_i}{p} - \frac{n - \sum x_i}{1-p} = 0$$

$$\hat{p}_{\text{MLE}} = \frac{\sum_{i=1}^n x_i}{n} = \bar{x}$$

The fraction of ones in the data. Intuitive — the MLE is the empirical frequency.

### 7.3 MLE and Least Squares

Assume the regression model:

$$y_i = w^T x_i + \epsilon_i, \quad \epsilon_i \overset{\text{i.i.d.}}{\sim} \mathcal{N}(0, \sigma^2)$$

The likelihood is:

$$p(\mathcal{D}; w, \sigma^2) = \prod_{i=1}^n \frac{1}{\sqrt{2\pi\sigma^2}}\exp!\left(-\frac{(y_i - w^Tx_i)^2}{2\sigma^2}\right)$$

The log-likelihood is:

$$\log p(\mathcal{D}; w, \sigma^2) = -\frac{n}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_{i=1}^n (y_i - w^Tx_i)^2$$

Maximising over $w$ is equivalent to minimising $\sum_{i=1}^n (y_i - w^Tx_i)^2$ — the **sum of squared residuals**. Ordinary least squares is MLE under Gaussian noise. This is the probabilistic justification for the squared loss.

### 7.4 MLE and Cross-Entropy

Assume the binary classification model:

$$p(y_i = 1 \mid x_i; w) = \sigma(w^T x_i), \quad \sigma(z) = \frac{1}{1+e^{-z}}$$

The log-likelihood is:

$$\log p(\mathcal{D}; w) = \sum_{i=1}^n \left[y_i \log \sigma(w^T x_i) + (1-y_i)\log(1 - \sigma(w^Tx_i))\right]$$

Minimising the negative log-likelihood is **exactly binary cross-entropy loss**. The cross-entropy loss is not an arbitrary choice — it is the NLL under the Bernoulli model. Minimising it is MLE for logistic regression.

## 8. Maximum A Posteriori Estimation

**Maximum A Posteriori (MAP)** estimation incorporates a prior:

$$\hat{\theta}_{\text{MAP}} = \underset{\theta}{\arg\max}\ p(\theta \mid \mathcal{D}) = \underset{\theta}{\arg\max}\ \left[\log p(\mathcal{D} \mid \theta) + \log p(\theta)\right]$$

MAP is MLE with an added $\log p(\theta)$ term. The prior acts as a regulariser.

### 8.1 MAP with Gaussian Prior = Ridge Regression

Suppose the prior on $w$ is $p(w) = \mathcal{N}(0, \tau^2 I)$. Then:

$$\log p(w) = -\frac{d}{2}\log(2\pi\tau^2) - \frac{1}{2\tau^2}|w|_2^2$$

MAP estimation maximises:

$$-\frac{1}{2\sigma^2}\sum_i (y_i - w^Tx_i)^2 - \frac{1}{2\tau^2}|w|^2_2 + \text{const}$$

Equivalently, minimises:

$$\sum_i (y_i - w^Tx_i)^2 + \frac{\sigma^2}{\tau^2}|w|^2_2$$

This is **Ridge regression** with regularisation parameter $\lambda = \sigma^2 / \tau^2$.

**Interpretation**: a Gaussian prior on $w$ corresponds to L2 regularisation. The prior says: before seeing data, weights are probably small. The regularisation enforces this belief. Stronger prior (smaller $\tau^2$) → larger $\lambda$ → stronger regularisation.

### 8.2 MAP with Laplace Prior = Lasso

Suppose the prior on $w$ is $p(w) = \prod_j \frac{1}{2b} e^{-|w_j|/b}$ (a Laplace prior). Then:

$$\log p(w) = -\frac{1}{b}|w|_1 + \text{const}$$

MAP estimation minimises:

$$\sum_i (y_i - w^Tx_i)^2 + \frac{2\sigma^2}{b}|w|_1$$

This is **Lasso regression** with $\lambda = 2\sigma^2 / b$.

**Interpretation**: the Laplace prior has heavier tails than Gaussian near zero but concentrates mass more tightly at zero. It encourages sparsity — it says weights are probably zero. The resulting Lasso has zero weights for irrelevant features.

### 8.3 MLE vs. MAP vs. Full Bayesian

|Method|Estimate|Uncertainty|Regularisation|Tractable?|
|---|---|---|---|---|
|MLE|Point estimate $\hat{\theta}_{\text{MLE}}$|No|No|Usually yes|
|MAP|Point estimate $\hat{\theta}_{\text{MAP}}$|No|Yes (via prior)|Usually yes|
|Full Bayesian|Posterior distribution $p(\theta \mid \mathcal{D})$|Yes|Yes (via prior)|Often no|

MLE and MAP both produce point estimates. Full Bayesian inference produces a distribution — it quantifies uncertainty. The posterior predictive distribution averages predictions over all plausible parameter values. This is almost always intractable and requires approximation (MCMC, variational inference). For most ML tasks, MAP is the practical compromise.

## 9. Information Theory

Information theory and ML are deeply intertwined. The cross-entropy loss, KL divergence in VAEs, mutual information in feature selection, and the minimum description length principle all come from here.

### 9.1 Entropy

The **Shannon entropy** of a discrete distribution $p$ over $\mathcal{X}$ is:

$$H(p) = -\sum_{x \in \mathcal{X}} p(x) \log p(x)$$

(with the convention $0 \log 0 = 0$). Using natural logarithm gives entropy in nats; $\log_2$ gives bits.

Entropy measures **uncertainty** — the average number of bits needed to describe an outcome. High entropy: distribution is spread out (uniform), outcomes are unpredictable. Low entropy: distribution is concentrated, outcomes are predictable.

Properties:

- $H(p) \geq 0$, with equality iff $p$ is a point mass (one outcome has probability 1).
- $H(p)$ is maximised by the uniform distribution: $H(p) \leq \log|\mathcal{X}|$.
- For a Bernoulli$(p)$ variable: $H = -p\log p - (1-p)\log(1-p)$, maximised at $p = 0.5$.

**ML connection.** In decision tree learning, the splitting criterion is information gain — the reduction in entropy after splitting on a feature. Splitting on a feature that produces pure leaves (low entropy) is preferred.

### 9.2 Cross-Entropy

The **cross-entropy** between true distribution $p$ and predicted distribution $q$ is:

$$H(p, q) = -\sum_x p(x) \log q(x)$$

It measures the expected code length when using $q$ to encode outcomes drawn from $p$. If $q = p$, the cross-entropy equals the entropy: $H(p, p) = H(p)$.

**ML connection.** In classification, the true distribution $p$ over classes is a one-hot vector ($p(y) = 1$ for the correct class, 0 otherwise). The predicted distribution $q$ is the softmax output. The cross-entropy becomes:

$$H(p, q) = -\log q(y_{\text{true}})$$

Minimising cross-entropy loss is equivalent to maximising the log-probability assigned to the correct class. This is exactly MLE for the categorical model.

### 9.3 KL Divergence

The **Kullback-Leibler (KL) divergence** from $q$ to $p$ is:

$$D_{\text{KL}}(p | q) = \sum_x p(x) \log \frac{p(x)}{q(x)} = \mathbb{E}_p!\left[\log \frac{p(x)}{q(x)}\right]$$

Properties:

- $D_{\text{KL}}(p | q) \geq 0$, with equality iff $p = q$ almost everywhere. (This is **Gibbs' inequality** / non-negativity of KL divergence.)
- $D_{\text{KL}}(p | q) \neq D_{\text{KL}}(q | p)$ in general — KL divergence is not symmetric and is not a metric.

**Proof of non-negativity** (using Jensen's inequality). Since $\log$ is concave:

$$D_{\text{KL}}(p | q) = -\mathbb{E}_p!\left[\log \frac{q(x)}{p(x)}\right] \geq -\log \mathbb{E}_p!\left[\frac{q(x)}{p(x)}\right] = -\log \sum_x p(x) \cdot \frac{q(x)}{p(x)} = -\log 1 = 0$$

**Relationship to cross-entropy:**

$$H(p, q) = H(p) + D_{\text{KL}}(p | q)$$

Minimising cross-entropy over $q$ (with $p$ fixed) is equivalent to minimising $D_{\text{KL}}(p | q)$ — they differ only by $H(p)$, which does not depend on $q$.

**ML connections:**

- **Variational Autoencoders (VAE)**: the ELBO objective contains a KL term $D_{\text{KL}}(q_\phi(z|x) | p(z))$ that regularises the encoder to stay close to the prior.
- **RL (policy optimisation)**: PPO clips the policy update to prevent $D_{\text{KL}}(\pi_{\text{old}} | \pi_{\text{new}})$ from being too large.
- **Knowledge distillation**: train a student model to match a teacher's output distribution by minimising KL divergence.
- **Model evaluation**: KL divergence from the data distribution to the model distribution measures how well the generative model fits the data.

### 9.4 Mutual Information

The **mutual information** between $X$ and $Y$ is:

$$I(X; Y) = D_{\text{KL}}(p(x,y) | p(x)p(y)) = \mathbb{E}!\left[\log \frac{p(x,y)}{p(x)p(y)}\right]$$

It measures how much knowing $X$ reduces uncertainty about $Y$. Equivalent formulations:

$$I(X; Y) = H(X) - H(X \mid Y) = H(Y) - H(Y \mid X) = H(X) + H(Y) - H(X, Y)$$

Properties:

- $I(X; Y) \geq 0$, with equality iff $X \perp Y$.
- $I(X; Y) = I(Y; X)$ — symmetric.

**ML connections:**

- **Feature selection**: mutual information between a feature $X_j$ and the label $Y$ measures relevance. Features with $I(X_j; Y) = 0$ are irrelevant.
- **Information bottleneck**: the information bottleneck principle frames representation learning as finding a compressed representation $Z$ that preserves $I(Z; Y)$ while minimising $I(Z; X)$.
- **Self-supervised learning**: maximising mutual information between different views of the same data is the theoretical motivation for contrastive learning (e.g., SimCLR).

## 10. Concentration Inequalities

Concentration inequalities bound the probability that a random variable deviates far from its mean. They are the mathematical tools behind PAC learning bounds.

### 10.1 Markov's Inequality

For any non-negative random variable $X$ and $t > 0$:

$$P(X \geq t) \leq \frac{\mathbb{E}[X]}{t}$$

**Proof.** $\mathbb{E}[X] = \mathbb{E}[X \mathbf{1}_{X \geq t}] + \mathbb{E}[X \mathbf{1}_{X < t}] \geq \mathbb{E}[X \mathbf{1}_{X \geq t}] \geq t \cdot P(X \geq t)$. $\square$

Weak bound, but requires only the first moment. The basis for all tighter inequalities.

### 10.2 Chebyshev's Inequality

For any random variable $X$ with finite mean $\mu$ and variance $\sigma^2$, and any $t > 0$:

$$P(|X - \mu| \geq t) \leq \frac{\sigma^2}{t^2}$$

**Proof.** Apply Markov's inequality to the non-negative random variable $(X - \mu)^2$ and threshold $t^2$. $\square$

Requires only mean and variance. Gives a $1/t^2$ tail bound — heavier than Gaussian tails.

### 10.3 Hoeffding's Inequality

For independent random variables $X_1, \ldots, X_n$ with $X_i \in [a_i, b_i]$ and $\bar{X} = \frac{1}{n}\sum X_i$:

$$P!\left(\bar{X} - \mathbb{E}[\bar{X}] \geq t\right) \leq \exp!\left(-\frac{2n^2 t^2}{\sum_{i=1}^n (b_i - a_i)^2}\right)$$

For i.i.d. variables in $[0,1]$: $P(|\bar{X} - \mu| \geq t) \leq 2e^{-2nt^2}$.

This is the tool used in the PAC learning proof. It gives **exponential** concentration — the probability of large deviations decays exponentially in $n$ and $t^2$.

### 10.4 The Central Limit Theorem

For i.i.d. random variables $X_1, \ldots, X_n$ with mean $\mu$ and variance $\sigma^2 < \infty$:

$$\frac{\bar{X} - \mu}{\sigma / \sqrt{n}} \xrightarrow{d} \mathcal{N}(0, 1) \quad \text{as } n \to \infty$$

The CLT says the sample mean is approximately Gaussian for large $n$, regardless of the original distribution. This justifies using Gaussian models for aggregated quantities and explains why gradient noise in SGD is approximately Gaussian.

The rate of convergence is $O(1/\sqrt{n})$ — matching the sample complexity bounds from PAC learning. This is not a coincidence.

## 11. Worked Examples

### 11.1 Coin Flip: Full Bayesian Treatment

We flip a possibly biased coin $n$ times. Let $p$ be the probability of heads. We observe $k$ heads.

**Prior.** $p \sim \text{Beta}(\alpha, \beta)$ — encoding prior belief that the coin is roughly fair but with uncertainty.

**Likelihood.** $k \mid p \sim \text{Binomial}(n, p)$:

$$p(k \mid p) = \binom{n}{k} p^k (1-p)^{n-k}$$

**Posterior.** By conjugacy (Beta-Binomial):

$$p(p \mid k) \propto p^k(1-p)^{n-k} \cdot p^{\alpha-1}(1-p)^{\beta-1} = p^{\alpha + k - 1}(1-p)^{\beta + n - k - 1}$$

$$p \mid k \sim \text{Beta}(\alpha + k, \beta + n - k)$$

The posterior mean is:

$$\mathbb{E}[p \mid k] = \frac{\alpha + k}{\alpha + \beta + n}$$

When $\alpha = \beta = 1$ (uniform prior), this gives $\frac{k+1}{n+2}$ — **Laplace smoothing**. With large $n$, the posterior mean approaches the MLE $k/n$ — the prior is washed out by data.

This is the Beta-Binomial conjugate pair. The same structure underlies Laplace smoothing in naive Bayes text classification.

### 11.2 Why Cross-Entropy and Not MSE for Classification

Suppose we use MSE loss for a binary classifier: $\mathcal{L} = (y - \hat{p})^2$.

Under the Gaussian noise model, MSE is MLE. But for binary classification, the data is Bernoulli, not Gaussian. Using MSE implicitly assumes Gaussian label noise — which is wrong when $y \in {0, 1}$.

The correct probabilistic model gives cross-entropy. Concretely: MSE has a vanishing gradient when $\hat{p}$ is near 0 or 1 and the prediction is wrong (the sigmoid saturates and MSE's gradient $(\hat{p} - y) \cdot \hat{p}(1-\hat{p})$ vanishes). Cross-entropy does not: its gradient $(\hat{p} - y)$ remains large. Training with MSE on a binary classification problem is slower and often worse.

The principled reason: use the loss that corresponds to the correct noise model for your data.
