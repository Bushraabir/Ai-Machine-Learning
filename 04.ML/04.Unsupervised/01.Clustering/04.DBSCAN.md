## ✦ Motivation: What K-Means Cannot Do

K-Means partitions space by proximity to centroids. Its implicit assumption: clusters are **convex and roughly spherical**. Given two interleaved half-moons, or a ring around a core, K-Means fails categorically — not because of tuning, but because the model is wrong.

```
  K-Means sees this:         Reality:
  
  ●●●  ○○○                   ●●●●●●●●
  ●●●  ○○○                       ○○○
  ●●●  ○○○                   ●●●●●●●●
  
  (Two blobs — fine)         (Ring around core — impossible for K-Means)
```

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) sidesteps this entirely. Its definition of a cluster: **a maximal set of density-connected points**. Shape is irrelevant. What matters is whether a region of space is dense.

**Original paper:** Ester, M., Kriegel, H.-P., Sander, J., & Xu, X. (1996). _A Density-Based Algorithm for Discovering Clusters in Large Spatial Databases with Noise._ KDD-96. One of the most cited papers in data mining.

---

## ✦ The Two Parameters

Everything in DBSCAN is defined in terms of two hyperparameters:

- $\varepsilon$ (epsilon): the neighbourhood radius. A point $q$ is a **neighbour** of $p$ if $d(p, q) \leq \varepsilon$.
- $\text{MinPts}$: the minimum number of points required to form a dense region (including the point itself).

These two numbers are the entire model. There is no objective function being optimised, no iterative update, no convergence criterion — just a graph traversal.

---

## ✦ Definitions

Let $X = {x_1, \ldots, x_n}$ be the dataset and $d(\cdot, \cdot)$ a distance metric.

### $\varepsilon$-Neighbourhood

$$N_\varepsilon(p) = { q \in X : d(p, q) \leq \varepsilon }$$

The set of all points within radius $\varepsilon$ of $p$, including $p$ itself.

### Core Point

A point $p$ is a **core point** if:

$$|N_\varepsilon(p)| \geq \text{MinPts}$$

Its neighbourhood is dense enough to anchor a cluster. Core points are the "interior" of clusters.

### Border Point

A point $p$ is a **border point** if:

$$|N_\varepsilon(p)| < \text{MinPts} \quad \text{but} \quad \exists \text{ core point } q \text{ such that } p \in N_\varepsilon(q)$$

Not dense enough to be a core, but reachable from one. Border points are the "edges" of clusters.

### Noise Point (Outlier)

A point $p$ is a **noise point** if it is neither a core point nor a border point. It belongs to no cluster. This is the key feature absent from K-Means and GMMs — DBSCAN has an explicit "none of the above" category.

### Directly Density-Reachable

Point $q$ is **directly density-reachable** from $p$ (w.r.t. $\varepsilon$, MinPts) if:

1. $p$ is a core point, and
2. $q \in N_\varepsilon(p)$

Note: this relation is **not symmetric**. $q$ may be directly reachable from $p$ without $p$ being directly reachable from $q$ (if $q$ is a border point).

### Density-Reachable

Point $q$ is **density-reachable** from $p$ if there exists a chain of points $p = p_1, p_2, \ldots, p_k = q$ such that each $p_{i+1}$ is directly density-reachable from $p_i$.

Still not symmetric in general.

### Density-Connected

Points $p$ and $q$ are **density-connected** if there exists a point $o$ such that both $p$ and $q$ are density-reachable from $o$.

Density-connectedness **is symmetric**. This is the equivalence relation that defines cluster membership.

### Cluster (Formal Definition)

A **cluster** $C$ (w.r.t. $\varepsilon$, MinPts) is a non-empty subset of $X$ satisfying:

1. **Maximality:** If $p \in C$ and $q$ is density-reachable from $p$, then $q \in C$.
2. **Connectivity:** Every pair of points in $C$ is density-connected.

Equivalently: a cluster is a maximal set of mutually density-connected points. The clusters partition the non-noise points.

## ✦ The Algorithm

```
Input: X, ε, MinPts
Output: cluster assignment for each point ∈ X (or NOISE)

labels = {p: UNVISITED for p in X}
cluster_id = 0

for each point p in X:
    if labels[p] ≠ UNVISITED:
        continue
    
    neighbours = RangeQuery(X, p, ε)   # = N_ε(p)
    
    if |neighbours| < MinPts:
        labels[p] = NOISE              # tentative — may become border later
        continue
    
    cluster_id += 1
    labels[p] = cluster_id
    seed_set = neighbours \ {p}        # points to process
    
    while seed_set is not empty:
        q = seed_set.pop()
        
        if labels[q] == NOISE:
            labels[q] = cluster_id    # border point: was noise, now assigned
        
        if labels[q] ≠ UNVISITED:
            continue                  # already in a cluster
        
        labels[q] = cluster_id
        q_neighbours = RangeQuery(X, q, ε)
        
        if |q_neighbours| ≥ MinPts:   # q is a core point — expand further
            seed_set = seed_set ∪ q_neighbours

return labels
```

This is a **graph BFS/DFS** over the $\varepsilon$-neighbourhood graph, seeded only from core points. Every point is visited at most once (once labelled, it is skipped). The order of iteration does not affect which points are core points or noise — but it can affect which cluster a border point is assigned to when it is reachable from multiple clusters (this is a known ambiguity in the definition).

### Complexity

- **Time:** $O(n \cdot T_{\text{query}})$ where $T_{\text{query}}$ is the cost of a range query.
    - With a KD-tree or ball tree: $T_{\text{query}} = O(\log n)$ for low-dimensional data → $O(n \log n)$ total.
    - In high dimensions (KD-tree degrades): $T_{\text{query}} = O(n)$ → $O(n^2)$ total.
- **Space:** $O(n)$ for labels + $O(n^2)$ for a precomputed distance matrix (the default in sklearn for small $n$).

## ✦ Geometric Intuition: Clusters as Level Sets

DBSCAN is equivalent to finding **connected components of the level set** ${x : \hat{p}(x) \geq \lambda}$ of an implicit density estimator, where the threshold $\lambda$ is determined by $\varepsilon$ and MinPts.

Formally: if you construct a kernel density estimate with a uniform kernel of bandwidth $\varepsilon$:

$$\hat{p}(x) = \frac{1}{n} |N_\varepsilon(x)|$$

then a core point is one where $\hat{p}(x) \geq \frac{\text{MinPts}}{n}$. DBSCAN finds the connected components of the region where $\hat{p} \geq \frac{\text{MinPts}}{n}$.

This connects DBSCAN directly to **density estimation** (cf. [[03.Density Estimation]]) and makes its statistical behaviour analysable. The clusters are the high-density "mountains" of the underlying distribution, separated by "valleys" of low density.

## ✦ Choosing $\varepsilon$ and MinPts

This is the practical crux. Unlike K-Means where $k$ is interpretable, $\varepsilon$ depends on the scale of your data — there is no universal default.

### MinPts

A common heuristic: $\text{MinPts} = 2 \cdot d$ where $d$ is the number of features (dimensionality). At minimum, MinPts $\geq 3$. Larger MinPts → smoother, more robust clustering; smaller MinPts → more clusters, more sensitive to noise.

The paper's authors suggest MinPts $= 4$ as a default for 2D data.

### The $k$-Distance Plot (Knee Method)

The canonical method for choosing $\varepsilon$:

1. For each point $x_i$, compute $d_k(x_i)$ = distance to its $k$-th nearest neighbour, where $k = \text{MinPts} - 1$.
2. Sort these $k$-distances in descending order and plot them.
3. The **knee** (point of maximum curvature) of the resulting curve suggests the appropriate $\varepsilon$.

**Why this works:** Points in dense regions have small $k$-distances; outliers and noise have large $k$-distances. The knee separates them. Values of $\varepsilon$ above the knee include too many points as core points (everything merges); values below miss genuine structure.

```
k-distance
    │  \
    │   \         ← noise points (large k-dist)
    │    \
    │     '---____  ← knee: choose ε here
    │             ‾‾‾‾‾  ← core/cluster points (small k-dist)
    └────────────────────────
                 Points (sorted)
```

In code:

```python
from sklearn.neighbors import NearestNeighbors
import numpy as np
import matplotlib.pyplot as plt

k = 4  # = MinPts - 1
nbrs = NearestNeighbors(n_neighbors=k).fit(X)
distances, _ = nbrs.kneighbors(X)
k_distances = np.sort(distances[:, -1])[::-1]

plt.plot(k_distances)
plt.xlabel('Points sorted by k-distance (desc)')
plt.ylabel(f'{k}-NN distance')
plt.title('k-Distance Plot — look for the knee')
plt.axhline(y=0.5, color='r', linestyle='--', label='candidate ε')
plt.legend()
```

### Scale Sensitivity

$\varepsilon$ is in the **same units as your distance metric**. If you standardise features (zero mean, unit variance), then Euclidean distances are comparable across datasets and a starting point of $\varepsilon \approx 0.3$–$0.5$ is reasonable for 2D–10D data — but always use the k-distance plot.

**DBSCAN does not handle clusters of varying density.** If your data has a dense core surrounded by a sparse ring, no single $\varepsilon$ will correctly identify both. This is the primary motivation for HDBSCAN.

## ✦ DBSCAN Properties

### What DBSCAN Can Do That K-Means Cannot

1. **Arbitrary shape clusters:** Rings, crescents, spirals, fractals — anything connected.
2. **Automatic $k$:** The number of clusters is a result, not a parameter.
3. **Explicit noise detection:** Points not belonging to any dense region are labelled as outliers. This is intrinsic, not post-hoc.
4. **No assumption of cluster shape or size:** Large and small clusters coexist naturally.

### What DBSCAN Cannot Do

1. **Varying density:** A single $\varepsilon$ cannot adapt to regions of different densities.
2. **High dimensions:** The curse of dimensionality makes all $\varepsilon$-neighbourhoods either empty or full — the "crowding problem." KD-trees also degrade. For $d \gtrsim 10$, DBSCAN becomes unreliable.
3. **No probabilistic output:** Hard assignments only, no cluster membership probability.
4. **Parameter sensitivity:** Results can change dramatically with small changes to $\varepsilon$ near the knee.
5. **Border point ambiguity:** A border point reachable from two clusters is assigned to whichever cluster processes it first — non-deterministic over orderings.

## ✦ Worked Example (by hand)

**Data:** 8 points in $\mathbb{R}^2$, with $\varepsilon = 1.5$, MinPts $= 3$.

```
Points: A=(0,0), B=(0,1), C=(1,0), D=(5,5), E=(5,6), F=(6,5), G=(10,10), H=(0,0.5)
```

**Step 1: Compute $N_\varepsilon$ for each point.**

- $N_{1.5}(A) = {A, B, C, H}$ → $|N| = 4 \geq 3$ → **core**
- $N_{1.5}(B) = {A, B, C, H}$ → $|N| = 4 \geq 3$ → **core**
- $N_{1.5}(C) = {A, B, C, H}$ → $|N| = 4 \geq 3$ → **core**  
    _(check: $d(C,H)=d((1,0),(0,0.5))=\sqrt{1+0.25}=1.12 \leq 1.5$ ✓)_
- $N_{1.5}(H) = {A, B, C, H}$ → $|N| = 4 \geq 3$ → **core**
- $N_{1.5}(D) = {D, E, F}$ → $|N| = 3 \geq 3$ → **core**  
    _(check: $d(D,F)=d((5,5),(6,5))=1 \leq 1.5$ ✓)_
- $N_{1.5}(E) = {D, E, F}$ → $|N| = 3 \geq 3$ → **core**
- $N_{1.5}(F) = {D, E, F}$ → $|N| = 3 \geq 3$ → **core**
- $N_{1.5}(G) = {G}$ → $|N| = 1 < 3$ → **noise**

**Step 2: Run BFS from first unvisited core point, say $A$.**

- Seed set = ${B, C, H}$. Assign all to cluster 1.
- Process $B$: core, $N(B) = {A,B,C,H}$, all already in cluster 1.
- Process $C$: core, same neighbourhood.
- Process $H$: core, same neighbourhood.
- Seed set empty. Cluster 1 = ${A,B,C,H}$.

**Step 3: Next unvisited point $D$.**

- Seed set = ${E, F}$. Assign to cluster 2.
- Process $E$: core, $N(E) = {D,E,F}$, all assigned.
- Process $F$: core, same.
- Cluster 2 = ${D,E,F}$.

**Step 4:** $G$ is labelled noise. No further unvisited points.

**Result:** Cluster 1 = ${A,B,C,H}$, Cluster 2 = ${D,E,F}$, Noise = ${G}$.

## ✦ Formal Correctness: The Key Lemma

**Lemma:** Let $p$ be a core point and $C$ be any cluster containing $p$. Then $C$ contains all points density-reachable from $p$.

_Proof._ Suppose $q$ is density-reachable from $p$ via chain $p = p_1, \ldots, p_k = q$. By maximality of $C$ (property 1 of the cluster definition), $p_2$ is directly density-reachable from $p_1 = p \in C$, so $p_2 \in C$. Since $p_2 \in C$ and $p_3$ is directly density-reachable from $p_2$, $p_3 \in C$. By induction, $q \in C$. $\square$

**Corollary:** Each core point uniquely determines its cluster (up to the ambiguity of border points). The clustering is essentially determined by the connected components of the core-point $\varepsilon$-neighbourhood graph.

## ✦ Implementation (NumPy, from scratch)

```python
import numpy as np
from collections import deque

def dbscan(X, eps, min_pts):
    """
    DBSCAN clustering from scratch.
    Returns: labels array of shape (n,), with -1 for noise.
    """
    n = len(X)
    labels = np.full(n, -2, dtype=int)   # -2 = UNVISITED
    cluster_id = 0

    def range_query(idx):
        """Return indices of all points within eps of X[idx]."""
        dists = np.linalg.norm(X - X[idx], axis=1)
        return np.where(dists <= eps)[0].tolist()

    for i in range(n):
        if labels[i] != -2:
            continue

        neighbours = range_query(i)

        if len(neighbours) < min_pts:
            labels[i] = -1   # NOISE (may be reassigned to border later)
            continue

        # i is a core point — start a new cluster
        labels[i] = cluster_id
        queue = deque(neighbours)

        while queue:
            j = queue.popleft()

            if labels[j] == -1:
                labels[j] = cluster_id    # border: was noise, now assigned

            if labels[j] != -2:
                continue                  # already labelled

            labels[j] = cluster_id
            j_neighbours = range_query(j)

            if len(j_neighbours) >= min_pts:  # j is also a core point
                queue.extend(j_neighbours)

        cluster_id += 1

    return labels


# --- Example ---
from sklearn.datasets import make_moons
import matplotlib.pyplot as plt

X, y_true = make_moons(n_samples=300, noise=0.05, random_state=42)
labels = dbscan(X, eps=0.2, min_pts=5)

colors = ['steelblue', 'coral', 'grey']
for lbl, color, marker in zip([0, 1, -1], colors, ['o', 's', 'x']):
    mask = labels == lbl
    label_name = f'Cluster {lbl}' if lbl >= 0 else 'Noise'
    plt.scatter(X[mask, 0], X[mask, 1], c=color, marker=marker,
                s=20, label=label_name, zorder=3)

plt.legend(); plt.title('DBSCAN on two moons'); plt.tight_layout()
```

**Production (sklearn):**

```python
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)
db = DBSCAN(eps=0.3, min_samples=5, metric='euclidean', n_jobs=-1)
labels = db.fit_predict(X_scaled)

n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = (labels == -1).sum()
print(f'{n_clusters} clusters, {n_noise} noise points')

# Core sample mask
core_mask = np.zeros_like(labels, dtype=bool)
core_mask[db.core_sample_indices_] = True
```

## ✦ OPTICS: Generalising DBSCAN

OPTICS (Ordering Points To Identify the Clustering Structure, Ankerst et al. 1999) addresses DBSCAN's inability to handle varying density.

**Key idea:** Instead of fixing $\varepsilon$, OPTICS computes a **reachability plot** over all possible $\varepsilon$ values simultaneously.

Define for each point $p$:

- **Core distance:** $d_{\text{core}}(p) = $ distance to MinPts-th nearest neighbour (if $|N_\varepsilon(p)| \geq \text{MinPts}$; else $\infty$).
- **Reachability distance** of $q$ w.r.t. $p$: $$\text{reach-dist}_{\text{MinPts}}(q, p) = \max(d_{\text{core}}(p),; d(p, q))$$

OPTICS outputs points in a traversal order where reachability-distance is small between consecutive points in the same cluster. Plotting reachability vs. order reveals valleys (clusters) and peaks (cluster boundaries / noise).

Extracting clusters: cut the reachability plot at a threshold $\xi$ (relative steepness) → clusters of varying density are correctly identified.

**OPTICS is strictly more general than DBSCAN** — every DBSCAN clustering is recoverable from the OPTICS ordering at the corresponding $\varepsilon$.

## ✦ HDBSCAN: The Modern Default

HDBSCAN (Hierarchical DBSCAN, Campello et al. 2013) combines OPTICS-style thinking with a proper hierarchy and automatic cluster extraction.

### The Construction

1. **Mutual reachability distance:** $$d_{\text{mreach},k}(p, q) = \max!\left(d_{\text{core},k}(p),; d_{\text{core},k}(q),; d(p,q)\right)$$ This "spreads out" low-density regions — sparse points are pushed far apart, dense points keep their true distances.
    
2. **MST** of the complete graph on $X$ under $d_{\text{mreach}}$ (cf. [[03.Hierarchical]] — single linkage on mutual reachability = the MST).
    
3. **Condensed cluster tree:** Extract the hierarchy from the MST. At each merge, if the resulting cluster has fewer than `min_cluster_size` points, the smaller piece becomes noise; the larger piece continues. This condenses the full dendrogram into a tree of meaningful clusters.
    
4. **Cluster selection:** Each node in the condensed tree has a **stability** score: $$\text{stability}(C) = \sum_{x \in C} \left(\lambda_{\text{death}}(x) - \lambda_{\text{birth}}(C)\right)$$ where $\lambda = 1/\varepsilon$ (inverse distance at which a point joins/leaves). Select clusters by maximising total stability over a valid selection (no cluster and its ancestor simultaneously selected).
    
5. **Soft clustering (optional):** HDBSCAN can output membership probabilities, not just hard assignments — a significant advantage over vanilla DBSCAN.
    

### HDBSCAN vs. DBSCAN

||DBSCAN|HDBSCAN|
|---|---|---|
|Parameters|$\varepsilon$, MinPts|`min_cluster_size`, `min_samples`|
|Varying density|No|Yes|
|Hierarchy|No|Yes (condensed tree)|
|Soft assignments|No|Yes|
|Runtime|$O(n \log n)$|$O(n \log n)$ approx.|
|Stability|Sensitive to $\varepsilon$|More robust|

```python
import hdbscan  # pip install hdbscan
# or: from sklearn.cluster import HDBSCAN  (sklearn >= 1.3)

clusterer = hdbscan.HDBSCAN(min_cluster_size=5, min_samples=3,
                              prediction_data=True)
labels = clusterer.fit_predict(X_scaled)

# Soft cluster membership (probabilities)
soft_clusters = hdbscan.all_points_membership_vectors(clusterer)
```

## ✦ Anomaly Detection with DBSCAN

The noise label $-1$ is not merely a side effect — it is a first-class anomaly detector.

For anomaly detection specifically, rather than running DBSCAN and checking for noise labels, consider:

**Local Outlier Factor (LOF):** A score-based variant. For each point $p$, LOF computes the ratio of the average local density of $p$'s neighbours to $p$'s own local density:

$$\text{LOF}_k(p) = \frac{\frac{1}{|N_k(p)|}\sum_{q \in N_k(p)} \text{lrd}_k(q)}{\text{lrd}_k(p)}$$

where the **local reachability density** is: $$\text{lrd}_k(p) = \left(\frac{\sum_{q \in N_k(p)} \text{reach-dist}_k(p,q)}{|N_k(p)|}\right)^{-1}$$

$\text{LOF} \approx 1$: $p$ has similar density to neighbours → inlier.  
$\text{LOF} \gg 1$: $p$ is in a sparser region than its neighbours → outlier.

LOF is more nuanced than DBSCAN noise labelling because it produces a **continuous score** rather than a binary label, and correctly handles outliers at the boundary of clusters.

```python
from sklearn.neighbors import LocalOutlierFactor
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05)
outlier_labels = lof.fit_predict(X)   # -1 = outlier, 1 = inlier
scores = -lof.negative_outlier_factor_  # higher = more outlier-like
```

## ✦ The Curse of Dimensionality and DBSCAN

In high dimensions ($d \gg 10$), DBSCAN degrades for the same reason as KNN (cf. [[07.k-Nearest Neighbours]]): distances concentrate. As $d \to \infty$:

$$\frac{\max_i d(x, x_i) - \min_i d(x, x_i)}{\min_i d(x, x_i)} \to 0$$

All pairwise distances become approximately equal. The $\varepsilon$-neighbourhood of any point either contains all points or no points — no intermediate density structure survives.

**Practical consequence:** For $d > 10$–$20$, either:

1. Reduce dimensionality first (PCA, UMAP) then run DBSCAN, or
2. Use a domain-specific distance metric that captures meaningful similarity, or
3. Use HDBSCAN with approximate nearest neighbours (faster and partially mitigates the curse).

DBSCAN with Euclidean distance on raw high-dimensional data is generally not meaningful.

## ✦ Connection to AI, ML, and Deep Learning

### Outlier Removal in Training Pipelines

Before training a supervised model on real-world data, DBSCAN (or LOF) is used to remove mislabelled or corrupted training examples. Noise points disproportionately harm models like SVMs (which are margin-based) and overfit small neural networks. Running DBSCAN on the feature space and discarding noise is a standard data cleaning step in industrial ML pipelines.

### Clustering Latent Representations

In self-supervised and representation learning, DBSCAN is applied to embeddings (from BERT, ResNet, VAE latent spaces) to discover emergent structure without labels. Because deep embeddings are not spherical or equal-variance, DBSCAN's shape-agnosticism makes it more appropriate than K-Means for this purpose.

A canonical workflow in NLP:

```
Sentence → SBERT embedding (768-dim) → UMAP (20-dim) → HDBSCAN → topics
```

This pipeline underlies **BERTopic** — one of the most widely used topic modelling frameworks (2022–present).

### Semi-Supervised Learning via Density

DBSCAN's clusters can serve as pseudo-labels for unlabelled data. If a small number of labelled points are known, a cluster containing labelled points can propagate those labels to all unlabelled cluster members — a form of **label propagation** grounded in density rather than graph Laplacian.

### Autonomous Driving: LiDAR Point Cloud Segmentation

In 3D point cloud processing (LiDAR data), DBSCAN is the standard algorithm for **object detection and segmentation**. A LiDAR scan produces $\sim 10^5$ 3D points per frame; DBSCAN identifies connected regions of high point density corresponding to vehicles, pedestrians, and obstacles. The noise label correctly handles the ground plane and distant sparse returns.

The key reason DBSCAN is preferred over K-Means here: the number of objects in the scene is unknown and varies per frame.

### Federated Learning: Client Clustering

In federated learning with non-IID data, DBSCAN is applied to **gradient fingerprints** (the gradient updates submitted by clients) to detect:

1. Byzantine clients (noise points — gradients far from any cluster)
2. Naturally occurring data distribution clusters

This is density-based anomaly detection applied at the system level, not the data level.

### Curriculum Learning

DBSCAN can partition a training set by difficulty: dense regions (many similar examples) → easy; noise points (isolated examples) → hard or mislabelled. A curriculum learning strategy trains first on cluster interiors, progressively including border and noise points.
