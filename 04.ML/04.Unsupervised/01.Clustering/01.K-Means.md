## §1 — The Problem Setup

We are given $n$ unlabelled points ${x_1, \ldots, x_n} \subset \mathbb{R}^d$ and a fixed integer $k$. We want to **partition** these points into $k$ disjoint clusters $C_1, \ldots, C_k$ such that points within the same cluster are similar and points across clusters are dissimilar.

**What does "similar" mean here?** K-Means formalises this as _squared Euclidean distance to the cluster centre_. This is not the only choice — it is an _assumption_ baked into the algorithm. (DBSCAN uses density; GMM uses Mahalanobis distance under a Gaussian model.)

---

## §2 — The Objective: Within-Cluster Sum of Squares

Define cluster centroids $\mu_k \in \mathbb{R}^d$ and cluster assignments $r_{ik} \in {0, 1}$ with $\sum_k r_{ik} = 1$.

$$ J = \sum_{i=1}^{n} \sum_{k=1}^{K} r_{ik} , |x_i - \mu_k|^2 $$

This is the **Within-Cluster Sum of Squares (WCSS)**, also called _inertia_. We want to minimise $J$ over both the assignments ${r_{ik}}$ and the centroids ${\mu_k}$.

**Why is this hard?** Joint minimisation over ${r_{ik}, \mu_k}$ is NP-hard in general ($d \geq 2$, $k \geq 2$). The number of ways to partition $n$ points into $k$ groups is $S(n,k)$ (the Stirling number of the second kind), which grows exponentially. For $n=100, k=10$: astronomically large.

---

## §3 — Lloyd's Algorithm: Coordinate Descent on $J$

The key insight: **if we fix one set of variables, minimising over the other is trivial.**

### 3.1 Derivation

**Step A — Fix centroids ${\mu_k}$, minimise over assignments ${r_{ik}}$:**

$$ r_{ik} = \begin{cases} 1 & k = \arg\min_j |x_i - \mu_j|^2 \ 0 & \text{otherwise} \end{cases} $$

This is just assigning each point to its nearest centroid. Each term in $J$ is independent across $i$, so we minimise pointwise. Assignment is $O(nk)$ per iteration.

**Step B — Fix assignments ${r_{ik}}$, minimise over centroids ${\mu_k}$:**

$$ \frac{\partial J}{\partial \mu_k} = -2 \sum_{i=1}^{n} r_{ik}(x_i - \mu_k) = 0 \implies \mu_k = \frac{\sum_i r_{ik} x_i}{\sum_i r_{ik}} $$

The optimal centroid is the **mean of the assigned cluster**. This is why it's called K-_Means_.

### 3.2 The Algorithm

```
Input: data X ∈ ℝ^{n×d}, number of clusters k
Output: assignments {r_ik}, centroids {μ_k}

1. Initialise μ_1, ..., μ_k  (see §5 for strategies)
2. Repeat until convergence:
   a. ASSIGN:  for each i, set r_{ik} = 1 for k = argmin_j ||x_i - μ_j||²
   b. UPDATE:  for each k, set μ_k = mean of { x_i : r_{ik} = 1 }
3. Return assignments and centroids
```

**Convergence proof:**

- $J$ is non-negative and bounded below by $0$.
- The ASSIGN step never increases $J$ (we replace each assignment with the nearest centroid).
- The UPDATE step never increases $J$ (the mean minimises sum of squared deviations).
- Since $J$ is strictly decreasing at each step (unless already converged) and the number of distinct partitions is finite, the algorithm must terminate.

**However:** It terminates at a _local minimum_, not necessarily the global minimum. The solution depends on initialisation.

**Complexity per iteration:** $O(nkd)$ — scales linearly in all of $n$, $k$, $d$.

## §4 — Worked Example: Two Moons (Intuition)

Consider 6 points in $\mathbb{R}^2$:

```
Points: A=(1,1), B=(2,1), C=(1.5,2),  D=(4,4), E=(5,4), F=(4.5,5)
```

**Iteration 0 — Initialise:** $\mu_1 = A = (1,1)$, $\mu_2 = D = (4,4)$

**Iteration 1 — ASSIGN:**

- $A$: dist to $\mu_1 = 0$, dist to $\mu_2 = \sqrt{18}$ → Cluster 1
- $B$: dist to $\mu_1 = 1$, dist to $\mu_2 = \sqrt{13}$ → Cluster 1
- $C$: dist to $\mu_1 = \sqrt{1.25}$, dist to $\mu_2 = \sqrt{9.25}$ → Cluster 1
- $D, E, F$ → Cluster 2 (closer to $\mu_2$)

**Iteration 1 — UPDATE:** $$\mu_1 = \frac{(1+2+1.5)}{3}, \frac{(1+1+2)}{3} = (1.5, 1.33), \quad \mu_2 = (4.5, 4.33)$$

**Iteration 2 — ASSIGN:** (same) → Converged.

$J = |(1,1)-(1.5,1.33)|^2 + |(2,1)-(1.5,1.33)|^2 + \ldots$ — small and balanced.

> _Professor's marginal note: K-Means "wants" round, equally-sized blobs. Feed it two crescent moons and watch it fail. This is a feature, not a bug — it tells you exactly what assumption is baked in._

## §5 — Initialisation: The Achilles Heel

### 5.1 Random Initialisation

Pick $k$ points uniformly at random from the data. Fast but brittle — can lead to poor local minima, particularly if two initial centroids land in the same true cluster.

### 5.2 K-Means++ (Arthur & Vassilvitskii, 2007)

**Idea:** Spread out the initial centroids probabilistically.

```
1. Choose μ_1 uniformly at random from X.
2. For k = 2, ..., K:
   a. For each x_i, compute D(x_i) = min_{j<k} ||x_i - μ_j||²
   b. Sample μ_k with probability proportional to D(x_i)
3. Run standard K-Means from these initialised centroids.
```

**Theoretical guarantee:** K-Means++ achieves expected cost $O(\log k)$ times the optimal. This is the tightest known efficient approximation ratio.

**Intuition:** Sampling proportionally to $D^2$ means distant points (likely in a new cluster) are selected with high probability.

**Default in sklearn:** `KMeans(init='k-means++')` uses this.

### 5.3 Multiple Restarts

Run K-Means $r$ times from different random initialisations; keep the solution with lowest $J$. The `n_init` parameter in sklearn (default 10) does this.

## §6 — Choosing $k$: The Model Selection Problem

K-Means requires you to specify $k$ in advance. This is a genuine weakness.

### 6.1 Elbow Method

Plot $J(k)$ vs. $k$. As $k$ increases, $J$ always decreases (at $k=n$, $J=0$). Look for the "elbow" — the point of diminishing returns.

```
J(k)
 ↑
 |·
 | ·
 |  ·
 |   ·
 |    · ← elbow at k=3
 |      ·  ·  ·  ·
 +-------------------→ k
  1  2  3  4  5  6
```

**Problem:** The elbow is often not clearly defined. It's a heuristic, not a test.

### 6.2 Silhouette Score

For each point $x_i$, define:

- $a(i)$ = mean distance to all other points in the **same** cluster (cohesion)
- $b(i)$ = mean distance to all points in the **nearest other** cluster (separation)

$$ s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))} \in [-1, 1] $$

- $s(i) \approx 1$: point is well-matched to its cluster, far from others — good.
- $s(i) \approx 0$: point is near a cluster boundary — ambiguous.
- $s(i) < 0$: point is closer to a neighbouring cluster — misassigned.

The **mean silhouette score** over all points is a quality metric. Choose $k$ that maximises it.

**Complexity:** $O(n^2)$ — expensive for large $n$. Approximate versions exist.

### 6.3 Gap Statistic (Tibshirani et al., 2001)

Compare $\log J(k)$ on the data to $\mathbb{E}[\log J(k)]$ on random reference data (uniform in the bounding box). The optimal $k$ is where the gap $\text{Gap}(k) = \mathbb{E}[\log J_{\text{ref}}(k)] - \log J(k)$ is maximised.

**Principle:** If there are no real clusters, $k=1$ will match random data. A large gap signals genuine cluster structure.

## §7 — Limitations: When K-Means Fails

These are not empirical observations — they follow directly from the objective function.

### 7.1 Assumes Spherical, Equally-Sized Clusters

The WCSS objective is equivalent to assuming clusters are generated by **isotropic Gaussians with equal variance**. If true clusters are elongated, nested, or of very different densities:

```
True clusters:        K-Means finds:
  ●●●●●              ●●●● | ●●●●
  ●●●●●    →         ●●●● | ●●●●
  ●●●●●              ●●●● | ●●●●
(elongated)         (wrong split)
```

**Fix:** Gaussian Mixture Models (GMM) with full covariance matrices.

### 7.2 Sensitive to Scale

WCSS uses Euclidean distance. A feature with range $[0, 10000]$ will dominate a feature with range $[0, 1]$. **Always standardise before K-Means.**

### 7.3 Outlier Sensitivity

A single outlier at $(1000, 1000)$ will pull a centroid toward itself. The mean is not a robust statistic.

**Fix:** K-Medoids (PAM) uses actual data points as cluster centres; more robust but $O(n^2)$.

### 7.4 Curse of Dimensionality

In high dimensions, all pairwise Euclidean distances tend toward the same value. K-Means assignments become nearly random. As $d \to \infty$:

$$ \frac{\max_i |x_i - \mu| - \min_i |x_i - \mu|}{\min_i |x_i - \mu|} \to 0 $$

**Fix:** Dimensionality reduction (PCA, UMAP) before clustering.

### 7.5 Non-Convex Clusters

K-Means cannot discover crescent or ring-shaped clusters because the Voronoi boundaries it creates are always linear (hyperplanes in $\mathbb{R}^d$).

**Fix:** DBSCAN, spectral clustering.

## §8 — K-Means as the E-Step of EM

This is the deepest structural insight: K-Means is a **hard-assignment, degenerate** special case of the EM algorithm for Gaussian Mixture Models.

### The GMM Connection

In a GMM with equal, isotropic covariance $\Sigma_k = \sigma^2 I$ and equal mixing weights $\pi_k = 1/K$:

**E-step (soft):** $$ r_{ik} = \frac{\exp!\left(-\frac{1}{2\sigma^2}|x_i - \mu_k|^2\right)}{\sum_j \exp!\left(-\frac{1}{2\sigma^2}|x_i - \mu_j|^2\right)} $$

**K-Means limit as $\sigma^2 \to 0$:**

As $\sigma \to 0$, the softmax concentrates all probability mass on the nearest centroid:

$$ r_{ik} \to \begin{cases} 1 & k = \arg\min_j |x_i - \mu_j|^2 \ 0 & \text{otherwise} \end{cases} $$

This is exactly the K-Means ASSIGN step. The UPDATE step is the M-step (recompute means).

**Consequence:** K-Means maximises a lower bound on the log-likelihood of a degenerate GMM. GMM generalises K-Means by allowing:

- Soft assignments (uncertainty about cluster membership)
- Non-spherical clusters (full covariance $\Sigma_k$)
- Unequal cluster sizes (mixing weights $\pi_k$)

## §9 — Variants

|Variant|Key Idea|When to Use|
|---|---|---|
|**K-Means++**|Smart initialisation|Always — default|
|**Mini-batch K-Means**|Update on subsets|Large $n$ (millions)|
|**K-Medoids (PAM)**|Centroid must be a data point|Robustness to outliers|
|**Bisecting K-Means**|Split one cluster at a time|Hierarchical structure|
|**Spherical K-Means**|Cosine similarity|Text/document clustering|
|**Fuzzy C-Means**|Soft assignments without full GMM|Overlapping clusters|

**Mini-batch K-Means** is the workhorse for large datasets. Instead of all $n$ points per iteration, sample a mini-batch of size $b$. Convergence is stochastic but fast. Available in sklearn as `MiniBatchKMeans`.

## §10 — Connection to AI / Deep Learning

### 10.1 Vector Quantisation (VQ)

K-Means is the foundation of **vector quantisation**: compress a high-dimensional space by representing each region with its centroid (a "codeword"). The set of centroids is the **codebook**.

This appears in:

- **Image compression** (early work)
- **VQ-VAE** (van den Oord et al., 2017): a VAE variant where the latent space is discrete, quantised by K-Means-like centroids. The encoder output is replaced by its nearest codebook vector. The gradient flows through using the straight-through estimator.

### 10.2 Cluster Assignments as Pseudo-Labels (Self-Supervised Learning)

**DeepCluster** (Caron et al., 2018): Alternate between:

1. Extract features with a CNN, run K-Means on features → get cluster assignments
2. Use cluster assignments as pseudo-labels, train CNN with cross-entropy

The CNN learns to produce features that are cluster-consistent. K-Means is the bridge between an unsupervised representation and a supervised signal.

**SwAV** (2020) extends this idea with online clustering.

### 10.3 Attention and Soft K-Means

Multi-head attention in Transformers can be interpreted as a differentiable, soft K-Means:

- **Queries** $Q$ ↔ data points
- **Keys** $K$ ↔ centroids
- **Attention weights** $\text{softmax}(QK^T/\sqrt{d})$ ↔ soft cluster assignments
- **Values** $V$ ↔ centroid features

This is not a coincidence: both compute weighted averages where the weights are proportional to similarity.

### 10.4 K-Means in Recommendation Systems

Item/user embeddings from matrix factorisation or neural collaborative filtering are clustered with K-Means to:

- Identify user segments
- Build hierarchical item indices for fast approximate nearest-neighbour search (e.g., FAISS uses K-Means-based inverted file indices)

### 10.5 Codebooks in Large Language Models

**Product quantisation** (used in FAISS, Milvus) decomposes the embedding space into subspaces and applies K-Means within each subspace. This enables billion-scale vector search that would be infeasible with exact Euclidean search.

## §11 — Implementation from Scratch

```python
import numpy as np

def kmeans_plus_plus_init(X: np.ndarray, k: int, rng: np.random.Generator) -> np.ndarray:
    """K-Means++ initialisation. Returns (k, d) array of initial centroids."""
    n, d = X.shape
    centroids = [X[rng.integers(n)]]
    
    for _ in range(k - 1):
        # D(x)^2: squared distance to nearest centroid already chosen
        D2 = np.array([min(np.sum((x - c)**2) for c in centroids) for x in X])
        # Sample proportional to D^2
        probs = D2 / D2.sum()
        centroids.append(X[rng.choice(n, p=probs)])
    
    return np.array(centroids)  # (k, d)


def kmeans(
    X: np.ndarray,
    k: int,
    max_iter: int = 300,
    tol: float = 1e-4,
    n_init: int = 10,
    seed: int = 42
) -> tuple[np.ndarray, np.ndarray, float]:
    """
    K-Means with K-Means++ initialisation and multiple restarts.
    
    Returns
    -------
    labels    : (n,) int array — cluster assignment for each point
    centroids : (k, d) float array — final cluster centres
    inertia   : float — final WCSS (J)
    """
    rng = np.random.default_rng(seed)
    best_labels, best_centroids, best_inertia = None, None, np.inf
    
    for _ in range(n_init):
        centroids = kmeans_plus_plus_init(X, k, rng)
        
        for iteration in range(max_iter):
            # ASSIGN step: (n, k) matrix of squared distances
            dists = np.sum((X[:, None, :] - centroids[None, :, :])**2, axis=2)  # (n, k)
            labels = np.argmin(dists, axis=1)  # (n,)
            
            # UPDATE step: recompute centroids as cluster means
            new_centroids = np.array([
                X[labels == j].mean(axis=0) if (labels == j).any() else centroids[j]
                for j in range(k)
            ])
            
            # Check convergence: max centroid shift
            shift = np.max(np.linalg.norm(new_centroids - centroids, axis=1))
            centroids = new_centroids
            if shift < tol:
                break
        
        # Compute inertia
        inertia = sum(
            np.sum((X[labels == j] - centroids[j])**2)
            for j in range(k) if (labels == j).any()
        )
        
        if inertia < best_inertia:
            best_labels, best_centroids, best_inertia = labels, centroids, inertia
    
    return best_labels, best_centroids, best_inertia


def silhouette_score(X: np.ndarray, labels: np.ndarray) -> float:
    """Compute mean silhouette coefficient. O(n^2) — use only for small n."""
    n = len(X)
    s = np.zeros(n)
    unique_labels = np.unique(labels)
    
    for i in range(n):
        same = X[labels == labels[i]]
        a = np.mean(np.linalg.norm(same - X[i], axis=1)) if len(same) > 1 else 0.0
        
        b = np.inf
        for lbl in unique_labels:
            if lbl == labels[i]:
                continue
            other = X[labels == lbl]
            b = min(b, np.mean(np.linalg.norm(other - X[i], axis=1)))
        
        s[i] = (b - a) / max(a, b) if max(a, b) > 0 else 0.0
    
    return float(np.mean(s))
```

## §12 — Complexity Summary

|Operation|Time|Space|
|---|---|---|
|Single K-Means iteration|$O(nkd)$|$O(nk)$|
|Full run ($T$ iters, $r$ restarts)|$O(Trnkd)$|$O(nk)$|
|K-Means++ initialisation|$O(nkd)$|$O(nk)$|
|Silhouette score (exact)|$O(n^2 d)$|$O(n^2)$|
|Mini-batch K-Means|$O(Trbkd)$|$O(bk)$|

where $b$ is mini-batch size. For $n = 10^6$, $k = 100$, $d = 50$: standard K-Means is feasible; silhouette is not.

## §13 — Common Mistakes and Gotchas

1. **Not scaling features.** K-Means uses Euclidean distance. One large-magnitude feature dominates. Always `StandardScaler()` first.
    
2. **Using the elbow method as the final word.** The elbow is often a smear, not a sharp bend. Use it alongside silhouette and domain knowledge.
    
3. **Running K-Means once.** Local minima are real. Always `n_init ≥ 10`.
    
4. **Using K-Means on categorical data.** Euclidean distance on one-hot vectors is meaningful but the centroid (a fractional average) is not a valid point. K-Modes uses mode-based centroids for categorical data.
    
5. **Interpreting cluster indices as stable across runs.** Cluster 0 in run 1 has no correspondence to cluster 0 in run 2. Label permutation is arbitrary.
    
6. **Applying K-Means in high dimensions without PCA.** Distances become uninformative. Reduce first.
