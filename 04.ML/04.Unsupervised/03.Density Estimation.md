
> _The most honest unsupervised learning problem: given samples from an unknown distribution $p(x)$, estimate $p$ itself — not a clustering, not a projection, but the actual probability measure over the input space. Everything generative — VAEs, diffusion models, normalising flows — is a density estimator dressed up in deep learning clothing._

## ✦ What Density Estimation Actually Is

We observe $\mathcal{D} = {x_1, \ldots, x_n} \overset{\text{i.i.d.}}{\sim} p^x$, where $p^x$ is the true but unknown data-generating distribution. The goal: construct an estimate $\hat{p}(x)$ such that $\hat{p} \approx p^*$ in some meaningful sense.

This is harder than it sounds. Three fundamental difficulties:

**1. The curse of dimensionality.** In $d$ dimensions, the volume of space grows as $r^d$. To cover the space with fixed-size bins, you need exponentially many bins. Data becomes exponentially sparse. Every nonparametric method eventually breaks down as $d$ grows — the question is only how gracefully.

**2. The normalisation constraint.** A valid density must satisfy $\int \hat{p}(x), dx = 1$. Enforcing this while fitting data is non-trivial. Some methods (histograms, KDE) build normalisation in by construction; others (energy-based models) require explicit computation of the partition function, which is itself intractable in high dimensions.

**3. The evaluation problem.** How do you measure how well $\hat{p}$ approximates $p^_$? You cannot compute $\int p^_(x) \log \hat{p}(x), dx$ exactly because you don't have $p^*$. You must estimate goodness-of-fit from the same data used for fitting — which requires care to avoid circular reasoning.

## ✦ Parametric Density Estimation

### Maximum Likelihood Estimation (Review)

Given a parametric family ${p_\theta : \theta \in \Theta}$, MLE finds:

$$\hat{\theta} = \arg\max_\theta \sum_{i=1}^n \log p_\theta(x_i)$$

This is equivalent to minimising the KL divergence from the empirical distribution $\hat{p}_n = \frac{1}{n}\sum_i \delta_{x_i}$ to $p_\theta$:

$$D_{KL}(\hat{p}_n | p_\theta) = \mathbb{E}_{\hat{p}_n}[\log \hat{p}_n(x)] - \mathbb{E}_{\hat{p}_n}[\log p_\theta(x)]$$

The first term is constant in $\theta$, so minimising KL is the same as maximising $\sum_i \log p_\theta(x_i)$.

**Implication:** MLE is always doing implicit density estimation — it fits a model $p_\theta$ to minimise KL from the data. This unifies regression, classification, and density estimation under one objective.

**For a Gaussian:** $\hat{\mu} = \bar{x}$, $\hat{\Sigma} = \frac{1}{n}\sum_i (x_i - \bar{x})(x_i - \bar{x})^T$. These are the closed-form MLEs, derivable by setting $\nabla_\mu \log \mathcal{L} = 0$ and $\nabla_\Sigma \log \mathcal{L} = 0$.

**Limitation:** If the true $p^*$ is not in the chosen family, MLE gives the best approximation _within the family_ — which may be very poor. This is **model misspecification**, and it is the central limitation of parametric methods.

## ✦ Histograms

The simplest nonparametric density estimator. Partition $\mathbb{R}^d$ into bins ${B_k}$ of equal width $h$. Estimate:

$$\hat{p}(x) = \frac{#{x_i \in B_k(x)}}{n \cdot h^d}$$

where $B_k(x)$ is the bin containing $x$, and $h^d$ is the bin volume in $d$ dimensions.

### Bias-Variance Analysis

Let $p^_$ be the true density, and suppose $p^_$ is differentiable. Within a bin of width $h$, the density is approximated as constant. The bias is:

$$\text{Bias}[\hat{p}(x)] \approx \frac{h}{2} \cdot \frac{\partial p^*}{\partial x}\bigg|_{x_0}$$

proportional to $h$ — larger bins mean more averaging, more bias.

The variance at a point is (by a Binomial argument, since bin counts are Binomial$(n, p_k)$):

$$\text{Var}[\hat{p}(x)] = \frac{p^*(x)}{n h^d}$$

The mean squared error at a point is then:

$$\text{MSE}[\hat{p}(x)] \approx \frac{h^2}{4}\left(\frac{\partial p^_}{\partial x}\right)^2 + \frac{p^_(x)}{nh^d}$$

Setting $\frac{d}{dh}\text{MSE} = 0$ gives the optimal bandwidth:

$$h^* \propto n^{-1/(d+2)}$$

and the optimal MSE scales as $n^{-2/(d+2)}$. For $d=1$: $n^{-2/3}$. For $d=10$: $n^{-1/6}$. This is the curse of dimensionality made concrete — convergence slows dramatically in high dimensions.

### Problems with Histograms

- **Discontinuities at bin boundaries** — the estimated density is a step function, which is often a poor model.
- **Bin origin sensitivity** — shifting the bin grid by $h/2$ can give a qualitatively different picture.
- **Scaling to $d > 2$** — the number of bins grows as $(1/h)^d$; most bins empty; estimate is zero almost everywhere.

KDE fixes all three.

## ✦ Kernel Density Estimation

### Definition

KDE places a **smooth kernel** centred at each data point:

$$\hat{p}(x) = \frac{1}{n h^d} \sum_{i=1}^n K!\left(\frac{x - x_i}{h}\right)$$

where $K: \mathbb{R}^d \to \mathbb{R}_{\geq 0}$ is a kernel satisfying:

$$\int K(u), du = 1, \quad K(u) = K(-u), \quad \int u K(u), du = 0$$

The bandwidth $h > 0$ controls the smoothness.

**Normalisation is automatic:** since each $K$ integrates to 1, and we have $n$ terms each scaled by $1/n$, $\int \hat{p}(x), dx = 1$ always.

### Intuition

Histograms assign a point's contribution only to the bin it falls in — a hard boundary. KDE spreads each point's contribution smoothly over its neighbourhood. The bandwidth $h$ controls the radius of this neighbourhood.

```
Data points:  ×         ×    ×         ×


Each kernel:  ╭──╮   ╭──╮╭──╮       ╭──╮
              │  │   │  ││  │       │  │
           ───╯  ╰───╯  ╰╯  ╰───────╯  ╰───

KDE (sum):    ╭─╮   ╭────╮          ╭─╮
           ───╯ ╰───╯    ╰──────────╯ ╰────
              h=small: spiky        h=large: smooth
```

### Common Kernels

All kernels $K(u)$ are written in the univariate case. For multivariate, use the product kernel $K(u) = \prod_{j=1}^d K(u_j)$ or a radially symmetric version $K(|u|)$.

| Kernel | $K(u)$, $|u| \leq 1$ | Support | Notes | |---|---|---|---| | **Gaussian** | $\frac{1}{\sqrt{2\pi}}e^{-u^2/2}$ | $\mathbb{R}$ | Smooth, infinite support; most common | | **Epanechnikov** | $\frac{3}{4}(1-u^2)$ | $[-1,1]$ | MSE-optimal; compact support | | **Uniform** | $\frac{1}{2}$ | $[-1,1]$ | Box kernel; same as histogram | | **Triweight** | $\frac{35}{32}(1-u^2)^3$ | $[-1,1]$ | Very smooth, compact | | **Biweight** | $\frac{15}{16}(1-u^2)^2$ | $[-1,1]$ | Common in practice |

**The Epanechnikov kernel is asymptotically optimal** in terms of minimising integrated MSE (MISE). In practice the difference between kernels is small — bandwidth matters far more than kernel choice.

### Bias-Variance Decomposition for KDE

Under smoothness assumptions on $p^_$ (specifically, $p^_$ twice differentiable):

$$\text{Bias}[\hat{p}(x)] \approx \frac{h^2}{2} \sigma_K^2, \nabla^2 p^*(x), \qquad \sigma_K^2 = \int u^2 K(u), du$$

$$\text{Var}[\hat{p}(x)] \approx \frac{p^*(x) |K|_2^2}{n h^d}, \qquad |K|_2^2 = \int K^2(u), du$$

The integrated (over all $x$) mean squared error (MISE) is:

$$\text{MISE}(\hat{p}) \approx \frac{h^4}{4}\sigma_K^4 \int (\nabla^2 p^*)^2, dx + \frac{|K|_2^2}{n h^d}$$

Setting $\frac{d}{dh}\text{MISE} = 0$:

$$h^* = \left(\frac{d |K|_2^2}{\sigma_K^4 n \int (\nabla^2 p^*)^2, dx}\right)^{1/(d+4)}$$

$$h^* \propto n^{-1/(d+4)}$$

and optimal MISE scales as $n^{-4/(d+4)}$. For $d=1$: $n^{-4/5}$. For $d=10$: $n^{-2/7} \approx n^{-0.286}$. Again, the curse — but KDE converges slightly faster than histograms (exponent $\frac{4}{d+4}$ vs $\frac{2}{d+2}$).

### Bandwidth Selection

The optimal $h^_$ depends on $\int (\nabla^2 p^_)^2, dx$, which requires knowing $p^*$ — circular. In practice:

**Silverman's Rule of Thumb** (for Gaussian kernel, 1D, assuming $p^*$ is Gaussian):

$$h_{\text{Silverman}} = 1.06, \hat{\sigma}, n^{-1/5}$$

where $\hat{\sigma} = \min(\text{std dev}, \text{IQR}/1.34)$ — the IQR term makes it robust to outliers.

For multivariate data, generalises to:

$$h_j = \left(\frac{4}{(d+2)n}\right)^{1/(d+4)} \hat{\sigma}_j$$

This is fast but assumes Gaussianity. For multimodal $p^*$, it over-smooths.

**Cross-Validation Bandwidth Selection:**

Minimise the leave-one-out (LOO) log-likelihood:

$$\hat{h} = \arg\max_h \sum_{i=1}^n \log \hat{p}_{-i}(x_i)$$

where $\hat{p}_{-i}$ is the KDE computed without point $x_i$. This is principled but $O(n^2)$ to evaluate for each $h$.

**Likelihood cross-validation** is equivalent to minimising $D_{KL}(p^* | \hat{p})$ — it finds $h$ such that the KDE is closest to the truth in KL sense, given the data.

### KDE as a Convolution

Notice that:

$$\hat{p}(x) = \frac{1}{nh^d}\sum_i K!\left(\frac{x-x_i}{h}\right) = \frac{1}{h^d}(K_h * \hat{p}_n)(x)$$

where $\hat{p}_n = \frac{1}{n}\sum_i \delta_{x_i}$ is the empirical measure and $K_h(u) = K(u/h)$.

KDE is **smoothing the empirical distribution** by convolution with the kernel. In the Fourier domain, convolution becomes pointwise multiplication — the kernel attenuates high-frequency components of the empirical distribution. Larger $h$ → more attenuation → smoother estimate. This is the exact same operation as applying a low-pass filter in signal processing.

## ✦ A Worked Example

Suppose $n = 5$ 1D observations: $x_1 = 0.2,, x_2 = 0.4,, x_3 = 0.5,, x_4 = 1.8,, x_5 = 2.0$.

Using a Gaussian kernel with $h = 0.3$:

$$\hat{p}(x) = \frac{1}{5 \cdot 0.3} \sum_{i=1}^5 \frac{1}{\sqrt{2\pi}} \exp!\left(-\frac{(x-x_i)^2}{2 \cdot 0.09}\right)$$

At $x = 0.35$ (between $x_1$ and $x_2$):

- Contribution from $x_2 = 0.4$: $\exp(-(0.35-0.4)^2/0.18) = \exp(-0.014) \approx 0.986$
- Contribution from $x_1 = 0.2$: $\exp(-(0.35-0.2)^2/0.18) = \exp(-0.125) \approx 0.882$
- Contributions from $x_3, x_4, x_5$: negligible (more than $2h$ away)

The density at $x=0.35$ is notably higher than at $x=1.0$ (between the two clusters). With $h=0.3$, the two clusters at ${0.2, 0.4, 0.5}$ and ${1.8, 2.0}$ remain separated. With $h=1.0$, they blur together into one mode.

```
h = 0.3 (small):                h = 1.0 (large):
    ▄▄▄              ▄▄            ▄▄▄▄▄▄▄▄▄▄▄▄▄▄
   █████            ███          ██████████████████
──────────────────────────    ──────────────────────────
  0.2 0.4 0.5     1.8 2.0      0.2 0.4 0.5   1.8 2.0
  (bimodal visible)             (unimodal — over-smoothed)
```

## ✦ Multivariate KDE and the Bandwidth Matrix

In $d$ dimensions, the full bandwidth generalisation uses a **bandwidth matrix** $H$ (symmetric positive definite, $d \times d$):

$$\hat{p}(x) = \frac{1}{n |H|^{1/2}} \sum_{i=1}^n K!\left(H^{-1/2}(x - x_i)\right)$$

The diagonal case $H = \text{diag}(h_1^2, \ldots, h_d^2)$ uses a separate bandwidth per dimension — appropriate when dimensions have different scales. The full matrix $H$ also captures correlations between dimensions.

In practice: most implementations use a single scalar bandwidth (or one per dimension after standardising). Full bandwidth matrix selection is $O(d^2)$ parameters — expensive, rarely done.

**The practical rule:** always standardise features before applying KDE in $d > 1$.

## ✦ Connection to Other Methods

### KDE and Naive Bayes

In **Gaussian Naive Bayes**, we fit a Gaussian $p(x_j | y=k)$ per feature per class — this is parametric density estimation within each class. **Kernel Naive Bayes** replaces the Gaussian fit with a KDE: $\hat{p}(x_j | y=k) = \text{KDE of class-}k\text{ samples}$. More flexible, no Gaussianity assumption.

### KDE and KNN

The $k$-nearest-neighbour density estimator is closely related:

$$\hat{p}(x) = \frac{k}{n V_k(x)}$$

where $V_k(x)$ is the volume of the ball containing the $k$ nearest neighbours of $x$. This is KDE with a **variable bandwidth** — the bandwidth adapts to local density. In dense regions, the ball is small (narrow bandwidth); in sparse regions, the ball is large (wide bandwidth). This is more adaptive than fixed-bandwidth KDE.

### KDE and Anomaly Detection

An observation $x$ is anomalous if $\hat{p}(x) < \tau$ for some threshold $\tau$. This is the cleanest formulation of density-based anomaly detection. Local Outlier Factor (LOF) is an extension that computes local density ratios rather than absolute densities.

### KDE and Clustering: Mean Shift

The **mean shift algorithm** is gradient ascent on the KDE surface:

$$x \leftarrow \frac{\sum_i x_i K!\left(\frac{x - x_i}{h}\right)}{\sum_i K!\left(\frac{x - x_i}{h}\right)}$$

Each step moves $x$ toward the local mean of nearby points, weighted by the kernel. Iterate to convergence → find local modes of $\hat{p}$. Cluster by assigning points to the mode they converge to. Mean shift discovers clusters as basins of attraction of the density, without requiring the number of clusters to be specified.

## ✦ Evaluating a Density Estimator

### Log-Likelihood on Held-Out Data

The most principled metric: fit $\hat{p}$ on training data, evaluate $\frac{1}{n_{\text{test}}}\sum_i \log \hat{p}(x_i^{\text{test}})$ on a held-out set. Higher is better. This is an unbiased estimator of $\mathbb{E}_{p^_}[\log \hat{p}(x)] = -H(p^_, \hat{p})$ (the negative cross-entropy).

Equivalently, minimising negative log-likelihood on test data is the same as minimising $D_{KL}(p^* | \hat{p})$ (up to a constant $H(p^*)$).

### Integrated Squared Error (ISE)

$$\text{ISE}(\hat{p}) = \int (\hat{p}(x) - p^*(x))^2, dx$$

Theoretically clean, rarely computable in practice without knowing $p^*$. Used for theoretical analysis (MISE = $\mathbb{E}[\text{ISE}]$).

### Visual / Qualitative Assessment

For $d \leq 2$: plot $\hat{p}$ and compare to histograms at multiple resolutions. Check that modes correspond to data clusters. A density that places mass in empty regions or misses visible clusters is poor.

### The Kernel Trick for ISE

An interesting fact: for KDE, $\mathbb{E}[\text{ISE}]$ can be computed analytically up to the unknown term $\int p^{*2}(x), dx$, which is constant in $h$. This enables cross-validation bandwidth selection without numerical integration:

$$\widehat{\text{MISE}}(h) = \int \hat{p}^2(x), dx - \frac{2}{n}\sum_i \hat{p}_{-i}(x_i) + \text{const}$$

The first term has a closed form for Gaussian kernels (it's a sum of Gaussian convolutions), and the second term is the LOO estimate. This is the **least-squares cross-validation** criterion for $h$.

## ✦ Deep Generative Models as Density Estimators (Preview)

Classical nonparametric methods break down in high dimensions. Deep generative models are the modern answer. A brief orientation:

### Normalising Flows

An invertible differentiable map $f_\theta: \mathcal{Z} \to \mathcal{X}$ transforms a simple base distribution $p_Z$ (e.g., $\mathcal{N}(0,I)$) into a complex $p_X$. The change-of-variables formula gives the **exact likelihood**:

$$\log p_X(x) = \log p_Z(f_\theta^{-1}(x)) + \log\left|\det J_{f_\theta^{-1}}(x)\right|$$

Trained by MLE. Exact samples and exact likelihoods. Expensive to compute the Jacobian determinant — modern flows (RealNVP, Glow) use architectures where the Jacobian is triangular, making the determinant $O(d)$.

### VAEs

Approximate the density via the ELBO. No exact likelihood, but scalable and flexible. Covered in [[05.Autoencoders]].

### GANs

Implicit density model — no tractable $\hat{p}(x)$ at all. Train a generator $G_\theta: \mathcal{Z} \to \mathcal{X}$ adversarially. Can sample but cannot evaluate $\hat{p}(x)$.

### Diffusion Models

Model the density implicitly via a denoising process. At inference, reverse a Markov chain from $\mathcal{N}(0,I)$ to $p^_(x)$. Connection to KDE: the forward process is exactly repeated Gaussian convolution — progressive smoothing of $p^_$ toward a Gaussian.

|Method|Exact likelihood|Sampling|Scalable to high-$d$|Flexibility|
|---|---|---|---|---|
|KDE|✅|✅ (trivial)|❌ ($d \leq 3$ practical)|Limited by bandwidth|
|GMM|✅|✅|❌ ($d \leq 50$ practical)|$K$ components|
|Normalising Flows|✅|✅|✅|High|
|VAE|❌ (ELBO)|✅|✅|High|
|GAN|❌|✅|✅|Very high|
|Diffusion|✅ (approx)|✅ (slow)|✅|State-of-the-art|

## ✦ Implementation Sketch

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import gaussian_kde

# --- KDE from scratch (1D Gaussian kernel) ---
def kde_gaussian(X_train, h=None):
    n = len(X_train)
    if h is None:
        # Silverman's rule of thumb
        sigma = min(np.std(X_train), np.percentile(X_train, 75) -
                    np.percentile(X_train, 25)) / 1.34
        h = 1.06 * sigma * n**(-1/5)

    def p_hat(x_eval):
        # x_eval: array of query points
        x_eval = np.atleast_1d(x_eval)
        # Shape: (n_eval, n_train)
        diffs = (x_eval[:, None] - X_train[None, :]) / h
        kernels = np.exp(-0.5 * diffs**2) / np.sqrt(2 * np.pi)
        return kernels.mean(axis=1) / h

    return p_hat, h

# --- Cross-validation bandwidth selection ---
def loo_log_likelihood(X, h):
    """Leave-one-out log-likelihood for bandwidth h."""
    n = len(X)
    log_lik = 0.0
    for i in range(n):
        X_loo = np.delete(X, i)
        diffs = (X[i] - X_loo) / h
        kernels = np.exp(-0.5 * diffs**2) / np.sqrt(2 * np.pi)
        p_i = kernels.mean() / h
        log_lik += np.log(p_i + 1e-300)
    return log_lik

# Usage
rng = np.random.default_rng(42)
X = np.concatenate([rng.normal(-2, 0.5, 100),
                    rng.normal(2, 1.0, 100)])   # bimodal data

p_hat, h_opt = kde_gaussian(X)
x_grid = np.linspace(-6, 6, 300)
density = p_hat(x_grid)

# Verify normalisation (should be ≈ 1)
dx = x_grid[1] - x_grid[0]
print(f"Integral ≈ {np.sum(density) * dx:.4f}")   # → ~1.000

# scipy for production use
kde = gaussian_kde(X, bw_method='silverman')
density_scipy = kde(x_grid)
```
