## ✦ Why Not Just Use t-SNE?

After building t-SNE from scratch in [[03.t-SNE]], the shortcomings are clear:

1. **No out-of-sample projection.** Given a new point $x_*$, t-SNE cannot embed it without re-running the entire algorithm. Useless in a pipeline.
2. **No global structure.** Inter-cluster distances in a t-SNE plot are meaningless. The layout of clusters relative to each other is essentially noise.
3. **Slow.** $O(n \log n)$ with Barnes-Hut, but constants are large.
4. **Not a general-purpose preprocessing step.** t-SNE outputs are not useful as features for downstream models — the optimisation is non-convex and unstable.

UMAP fixes all four. Same local-structure quality as t-SNE, but faster, with more honest global structure, out-of-sample projection, and stable enough to use as a preprocessing step for classifiers.

The cost: the theoretical justification is harder. UMAP's paper invokes Riemannian geometry, fuzzy topology, and category theory. I'll build up to it carefully.

---

## ✦ The Core Theoretical Claim

**Claim:** high-dimensional data lies on (or near) a low-dimensional **Riemannian manifold** embedded in $\mathbb{R}^D$. The task of dimensionality reduction is to find a faithful low-dimensional representation of this manifold.

The key word is _uniform_. UMAP assumes the data is **uniformly distributed on the manifold** with respect to the manifold's intrinsic metric — not with respect to the Euclidean metric of the ambient space. This single assumption does most of the work.

```
Ambient space ℝ^D (Euclidean distances are misleading)
        ·  ·    ·  ·  · ← dense here (but just a curved region)
      ·              ·
    ·    MANIFOLD      ·
      ·              ·
        ·  ·    ·  ·  ·

Intrinsic distances (along manifold) are what UMAP uses.
```

## ✦ Step 1 — Building the Fuzzy Topological Representation

### 1a. The Local Metric

For each point $x_i$, find its $k$ nearest neighbours ${x_{i_1}, \ldots, x_{i_k}}$ and let $\rho_i$ be the distance to the **nearest** neighbour:

$$\rho_i = \min_{j} |x_i - x_{i_j}|$$

This $\rho_i$ is UMAP's analogue of t-SNE's adaptive bandwidth $\sigma_i$. It normalises the local metric so that every point has at least one neighbour at distance zero on the manifold. In dense regions $\rho_i$ is small; in sparse regions $\rho_i$ is large.

Now define, for the $k$-nearest neighbours of $x_i$, a **local similarity**:

$$v_{ij} = \exp!\left(-\frac{d(x_i, x_j) - \rho_i}{\sigma_i}\right)$$

where $\sigma_i$ is a second bandwidth chosen (again by binary search) to achieve a target "effective number of neighbours" — UMAP's analogue of perplexity, called $k$ (the `n_neighbors` parameter). The $-\rho_i$ shift ensures the nearest neighbour always has similarity 1.

For points outside the $k$-neighbourhood, $v_{ij} = 0$.

> **The picture to hold in mind:** $v_{ij}$ is 1 for the nearest neighbour, decays exponentially, and is 0 beyond the $k$-th neighbour. It encodes "how strongly does $x_i$ consider $x_j$ its local neighbour?"

### 1b. Symmetrisation via Fuzzy Union

$v_{ij}$ is again asymmetric: $v_{ij} \neq v_{ji}$. t-SNE symmetrised by averaging. UMAP uses a **fuzzy union** — the probabilistic OR:

$$w_{ij} = v_{ij} + v_{ji} - v_{ij} \cdot v_{ji}$$

This is exactly $P(A \cup B) = P(A) + P(B) - P(A \cap B)$ for independent events. The interpretation: $w_{ij}$ is the probability that the edge $(i,j)$ exists in the "fuzzy simplicial set" — the topological object UMAP constructs on top of the data.

If either $x_i$ considers $x_j$ a neighbour, _or_ $x_j$ considers $x_i$ a neighbour, the edge is retained (with appropriate probability). This is less aggressive than averaging — edges are added, not washed out.

> **t-SNE vs. UMAP symmetrisation:**
> 
> - t-SNE: $p_{ij} = \frac{p_{j|i} + p_{i|j}}{2}$ — arithmetic mean, shrinks both toward each other.
> - UMAP: $w_{ij} = v_{ij} + v_{ji} - v_{ij}v_{ji}$ — fuzzy OR, strictly $\geq \max(v_{ij}, v_{ji})$.
> 
> The OR operation preserves asymmetric local structure better. If $x_i$ strongly pulls toward $x_j$ but not vice versa, t-SNE halves the signal; UMAP keeps it.

The output of Step 1 is a **weighted graph** $G = (V, E, w)$ on the $n$ data points — a discrete approximation of the underlying topological space.

---

## ✦ Step 2 — Building the Low-Dimensional Representation

UMAP initialises the low-dimensional embedding $Y = {y_1, \ldots, y_n} \subset \mathbb{R}^d$ (usually $d = 2$), typically via the spectral embedding of the graph Laplacian of $G$ (much better than random init — it already roughly respects graph connectivity).

It then defines a low-dimensional similarity using a **smooth approximation** to a 1-simplex membership function:

$$\hat{w}_{ij} = \left(1 + a|y_i - y_j|^{2b}\right)^{-1}$$

The parameters $a$ and $b$ are fitted by a nonlinear least-squares procedure to approximate the family of curves:

$$\phi(r) = \begin{cases} 1 & r \leq \text{min_dist} \ \exp(-(r - \text{min_dist})) & r > \text{min_dist} \end{cases}$$

The `min_dist` hyperparameter controls how tightly points are clumped in the embedding. Small `min_dist` → tight clumps. Large `min_dist` → spread-out, preserves more topological spread.

When $b = 1$, $\hat{w}_{ij} = (1 + a|y_i - y_j|^2)^{-1}$, which is exactly the **Student-t kernel** from t-SNE (with $a = 1$). UMAP generalises t-SNE's low-dimensional kernel via the $b$ parameter.

---

## ✦ Step 3 — The Objective: Cross-Entropy of Fuzzy Sets

t-SNE minimises $KL(P | Q)$. UMAP minimises the **binary cross-entropy** between the high-dimensional and low-dimensional fuzzy set memberships:

$$\mathcal{L} = \sum_{(i,j) \in E} \left[ w_{ij} \log \frac{w_{ij}}{\hat{w}_{ij}} + (1 - w_{ij}) \log \frac{1 - w_{ij}}{1 - \hat{w}_{ij}} \right]$$

This is the cross-entropy $H(w, \hat{w})$ over all edges (both present and absent).

Breaking it apart:

$$\mathcal{L} = \underbrace{\sum_{(i,j) \in E} w_{ij} \log \frac{w_{ij}}{\hat{w}_{ij}}}_{\text{attractive term}} + \underbrace{\sum_{(i,j) \in E} (1-w_{ij}) \log \frac{1-w_{ij}}{1-\hat{w}_{ij}}}_{\text{repulsive term}}$$

- **Attractive term:** for edges that exist ($w_{ij}$ large), penalises $\hat{w}_{ij}$ being too small — pulls connected points together.
- **Repulsive term:** for edges that don't exist ($w_{ij} \approx 0$), penalises $\hat{w}_{ij}$ being too large — pushes disconnected points apart.

> **Why cross-entropy instead of KL?** Because the $w_{ij}$ are edge probabilities — binary (0 or 1) in the limit. Cross-entropy is the natural loss for a binary probability. KL divergence only applies to normalised probability distributions; $w_{ij}$ doesn't sum to 1. UMAP's formulation is more honest about what $w_{ij}$ actually represents.

---

## ✦ Step 4 — Stochastic Gradient Descent with Negative Sampling

The repulsive term in $\mathcal{L}$ naïvely requires summing over all $O(n^2)$ non-edges. UMAP approximates this via **negative sampling** — a technique from Word2Vec:

For each positive edge $(i,j)$, sample $m$ negative examples $(i, k)$ where $k$ is a random point. The gradient update becomes:

**Attractive gradient** (pull $y_i$ toward $y_j$):

$$\frac{\partial \mathcal{L}_{\text{attr}}}{\partial y_i} = -\frac{2ab|y_i - y_j|^{2(b-1)}}{1 + a|y_i - y_j|^{2b}} \cdot w_{ij} \cdot (y_i - y_j)$$

**Repulsive gradient** (push $y_i$ away from negative sample $y_k$):

$$\frac{\partial \mathcal{L}_{\text{rep}}}{\partial y_i} = \frac{2b}{(0.001 + |y_i - y_k|^2)(1 + a|y_i - y_k|^{2b})} \cdot (1 - \hat{w}_{ik}) \cdot (y_i - y_k)$$

(The $0.001$ is a numerical stability epsilon.)

This gives $O(n \cdot k \cdot m)$ per epoch, where $k$ is `n_neighbors` and $m$ is the number of negative samples (default 5). In practice, UMAP is significantly faster than t-SNE.

## ✦ Algorithm Summary

```
Input: high-dim data X = {x₁,...,xₙ}, n_neighbors k, n_components d,
       min_dist, n_epochs

1. For each xᵢ: find k-nearest neighbours, compute ρᵢ (dist to nearest)
2. Compute vᵢⱼ = exp(-(d(xᵢ,xⱼ) - ρᵢ) / σᵢ) for each neighbour xⱼ
   (σᵢ found by binary search to achieve effective cardinality = log₂(k))
3. Symmetrise: wᵢⱼ = vᵢⱼ + vⱼᵢ - vᵢⱼ·vⱼᵢ  (fuzzy union)
4. Fit parameters a, b to the min_dist profile via nonlinear least squares
5. Initialise Y using spectral embedding of graph Laplacian of (V, E, w)
6. For each epoch:
     For each edge (i,j) with probability ∝ wᵢⱼ:
       a. Compute attractive gradient; update yᵢ, yⱼ
       b. Sample m negative pairs (i,k); compute repulsive gradient; update yᵢ
7. Return Y
```

## ✦ Worked Example — Two Crescents

Consider the classic "two moons" dataset: 200 points on two interleaved crescents in $\mathbb{R}^2$ (embedded in $\mathbb{R}^{20}$ with noise).

**What PCA does:** projects onto variance axes → the two moons overlap completely. Indistinguishable.

**What t-SNE does:** preserves local neighbourhoods → separates the moons. But the _shape_ of each moon is not preserved — it might look like two blobs.

**What UMAP does:** preserves local neighbourhoods _and_ the topological structure (both moons are 1-dimensional loops). The resulting embedding looks like two actual crescents, with the global arrangement roughly faithful.

```
Original ℝ²⁰            t-SNE output           UMAP output

Points on two          Two separated           Two separated
interleaved            roughly circular        crescent shapes
crescents              blobs                   (topology preserved)

· · · · ·              ● ● ●   ○ ○ ○           ·····    ○○○○○
  ·····                  ● ●   ○ ○                ····    ○○○
○ ○ ○ ○ ○              ● ●●   ○ ○○             ·····      ○○○○
  ○○○○○                                                    ○○○○○
```

The shape is preserved in UMAP because the fuzzy simplicial set construction respects _topology_ — the connectivity structure — not just pairwise distances.

## ✦ Hyperparameters

### `n_neighbors` (default: 15)

Controls the balance between local and global structure.

- **Small** (e.g., 5): only very local structure preserved. Tight local clusters, global layout unreliable.
- **Large** (e.g., 200): more global structure. Clusters may merge; manifold layout more faithful.
- Analogous to perplexity in t-SNE, but with a cleaner interpretation: literally the number of neighbours used to construct the graph.

```
n_neighbors = 5          n_neighbors = 15         n_neighbors = 50
  ·· ·   ·· ·            ···   ···               ··········
  ·· ·   ·· ·            ···   ···               ··········
  (tight clusters)        (balanced)              (global structure)
```

### `min_dist` (default: 0.1)

Controls how tightly points are packed in the embedding.

- **Small** (e.g., 0.001): tight clumps — good for clustering tasks.
- **Large** (e.g., 0.5): spread out — better for seeing the overall manifold topology.

### `n_components` (default: 2)

The output dimensionality. Can be set to 10, 50, etc. for preprocessing — UMAP is not only a 2D visualisation tool.

### `metric` (default: `'euclidean'`)

UMAP supports any metric: `'cosine'`, `'manhattan'`, `'correlation'`, custom. This is powerful for text embeddings (use cosine) or count data (use Hellinger).

## ✦ Out-of-Sample Projection

This is the capability t-SNE lacks entirely. After fitting UMAP on training data $X_{\text{train}}$, new points $x_*$ can be embedded without re-running the full algorithm:

$$y_* = \text{UMAP.transform}(x_*)$$

Mechanically: find $x_*$'s nearest neighbours in $X_{\text{train}}$, compute the fuzzy set memberships, then use a few SGD steps to find the low-dimensional position that best satisfies those memberships given the already-fixed embedding.

This enables:

- Embedding test data using a model fit on training data (no leakage)
- Streaming inference: embed new points as they arrive
- Using UMAP as a feature extractor in an ML pipeline

```python
import umap

reducer = umap.UMAP(n_neighbors=15, min_dist=0.1, n_components=2)
reducer.fit(X_train)

Y_train = reducer.transform(X_train)
Y_test  = reducer.transform(X_test)   # ← t-SNE cannot do this
```

## ✦ UMAP as a Preprocessing Step

Because of out-of-sample projection and relative stability, UMAP can be used as a **dimensionality reduction step before supervised learning** — not just for visualisation.

**Typical pipeline:**

```
Raw features (D dims)
      ↓
   PCA (→ 50 dims, remove noise)
      ↓
   UMAP (→ 10–30 dims, nonlinear structure)
      ↓
   Classifier (Random Forest, SVM, etc.)
```

Studies (particularly in bioinformatics) show UMAP preprocessing can match or exceed PCA preprocessing on downstream classification, especially when the data has nonlinear manifold structure. See McInnes et al. (2018) and Becht et al. (2018, _Nature Biotechnology_).

> **Caution:** UMAP's output is stochastic. If you use it in a pipeline, fix the random seed and document the hyperparameters — both affect the embedding, and therefore downstream model performance.

## ✦ The Riemannian Geometry Behind It

_This section is for understanding why UMAP's theory is well-motivated. The algorithm works without this, but you asked for rigour._

**The uniformity assumption, formalised:**

Assume the data ${x_i}$ lies on a Riemannian manifold $(\mathcal{M}, g)$ and is _uniformly distributed_ with respect to the Riemannian volume form $\text{dvol}_g$. The metric $g$ is unknown.

**The geodesic distance problem:** we observe Euclidean distances in $\mathbb{R}^D$, but we want geodesic distances on $\mathcal{M}$. These can differ dramatically (think: points on opposite sides of a fold that are Euclidean-close but geodesically far).

**UMAP's construction:** estimate the geodesic distance by noting that _under the uniformity assumption_, the distance to the $k$-th nearest neighbour in Euclidean space is proportional to the geodesic ball radius needed to contain $k$ points. The $\rho_i$ normalisation and $\sigma_i$ adaptation are approximating this geodesic ball radius locally.

Concretely: the local connectivity condition (nearest neighbour at distance $\rho_i$ has $v_{ij} = 1$) ensures every point has at least one geodesic neighbour at zero distance — i.e., the manifold is locally connected.

**Fuzzy simplicial sets:** the weighted graph $G$ constructed in Step 1 is interpreted as a **fuzzy simplicial set** — a topological object that captures the manifold's shape (connected components, loops, voids) up to homotopy. The UMAP objective then finds a low-dimensional fuzzy simplicial set with the same topological structure.

The deep theoretical result (Theorem 1 in McInnes et al.) is:

> _Under the uniformity assumption and with sufficient data, the fuzzy simplicial set constructed by UMAP converges (in the sense of the Čech complex) to a topological representation of the underlying manifold._

In practice: UMAP is approximating manifold topology, not just pairwise distances. This is why it preserves structure that t-SNE discards.

## ✦ Connection to AI / ML / Deep Learning

### 1. Feature Space Analysis

Same as t-SNE, but now usable in production. Extract activations from any layer of a neural network, run UMAP, and inspect. The improvement: because UMAP is faster and supports out-of-sample projection, you can do this continuously during training to watch representation learning unfold.

### 2. Preprocessing for Downstream Models

UMAP-reduced features fed into classifiers have shown competitive performance, particularly when:

- The original feature space is very high-dimensional ($D \gg 100$)
- The manifold hypothesis holds (data lives on a lower-dimensional structure)
- The downstream model is sensitive to curse of dimensionality (KNN, SVM with RBF kernel)

### 3. Self-Supervised / Contrastive Learning Evaluation

In contrastive learning (SimCLR, MoCo, DINO), UMAP of the embedding space is the standard diagnostic. Good representations → well-separated, coherent clusters. The key advantage over t-SNE: UMAP embeddings of the training set generalise to the test set, so you can use the same projection to visualise both.

### 4. Large Language Models

Sentence embeddings (from BERT, GPT, etc.) are 768–4096 dimensional. UMAP to 2–3D for visualisation reveals semantic clustering — similar topics cluster together. Used to diagnose dataset balance, discover topic drift, and identify outliers in fine-tuning datasets.

### 5. Anomaly Detection

Because UMAP preserves density structure better than t-SNE, anomalous points (which have few true neighbours) tend to project to isolated positions in the embedding. A simple heuristic: fit UMAP on normal data, embed new points, flag those that fall far from any cluster.

### 6. Generative Models — Latent Space Visualisation

VAE and GAN latent spaces are typically 100–512 dimensional. UMAP to 2D lets you visually inspect:

- Whether the latent space is well-organised
- Whether interpolations between classes pass through sensible intermediate regions
- Whether the encoder is collapsing (representation collapse → single blob)

## ✦ t-SNE vs. UMAP: Full Comparison

|Property|t-SNE|UMAP|
|---|---|---|
|**Global structure preserved**|Poor|Better (not perfect)|
|**Local structure preserved**|Excellent|Excellent|
|**Out-of-sample projection**|No|Yes|
|**Speed**|$O(n \log n)$|$O(n \log n)$, faster constants|
|**Deterministic**|No|No (but more stable)|
|**Cluster sizes meaningful**|No|Somewhat (not fully)|
|**Inter-cluster distances**|No|Partially|
|**Can use as preprocessing**|No|Yes|
|**Theoretical basis**|KL divergence + t-kernel|Riemannian geometry + fuzzy topology|
|**Low-dim kernel**|Student-t ($a=1, b=1$)|Generalised: $(1 + a\|y\|^{2b})^{-1}$|
|**Symmetrisation**|Arithmetic mean|Fuzzy union (probabilistic OR)|
|**Objective**|$KL(P \| Q)$|Binary cross-entropy $H(w, \hat{w})$|
|**Initialisation**|Random (small noise)|Spectral embedding of graph Laplacian|
|**Negative sampling**|No (Barnes-Hut for repulsion)|Yes (Word2Vec-style)|

**The bottom line:** UMAP is t-SNE with better theory, faster optimisation, honest global structure, and out-of-sample projection. Use UMAP unless you have a specific reason for t-SNE (e.g., existing pipeline, legacy code, paper comparison).

## ✦ What UMAP Still Cannot Tell You

Despite the improvements over t-SNE, these caveats remain:

|Claim|Valid?|
|---|---|
|"These two clusters are far apart so they're very different"|⚠️ Partially — UMAP preserves more global structure than t-SNE but inter-cluster distances are still distorted|
|"Cluster A is larger than cluster B so it has more points"|❌ No — `min_dist` controls visual cluster size, not point density|
|"I see 7 clusters so there are 7 classes"|❌ No — UMAP can split or merge clusters depending on hyperparameters|
|"These two points are close so they're similar"|✅ Yes (local structure is preserved)|
|"The topology looks like a loop so the data has a cyclic structure"|✅ Yes (UMAP preserves topology)|

## ✦ Implementation

```python
# Standard usage
import umap
import numpy as np
import matplotlib.pyplot as plt
from sklearn.preprocessing import StandardScaler

# Always scale first — UMAP is distance-sensitive
X_scaled = StandardScaler().fit_transform(X)

# For visualisation
reducer_2d = umap.UMAP(
    n_neighbors=15,      # local/global balance
    min_dist=0.1,        # cluster tightness
    n_components=2,      # output dims
    metric='euclidean',  # or 'cosine' for text embeddings
    random_state=42      # reproducibility
)
Y_2d = reducer_2d.fit_transform(X_scaled)

plt.scatter(Y_2d[:, 0], Y_2d[:, 1], c=labels, cmap='tab10', s=5)
plt.title("UMAP projection")

# For preprocessing (higher-dim)
reducer_prep = umap.UMAP(n_components=20, n_neighbors=30)
reducer_prep.fit(X_train_scaled)
X_train_umap = reducer_prep.transform(X_train_scaled)
X_test_umap  = reducer_prep.transform(X_test_scaled)   # out-of-sample

# Then feed into classifier
from sklearn.ensemble import RandomForestClassifier
clf = RandomForestClassifier()
clf.fit(X_train_umap, y_train)
```

**For neural network activations:**

```python
# Get penultimate layer activations
activations = model.predict(X)  # shape (n, 512) for example

# PCA first to 50 dims (remove noise, speed up UMAP)
from sklearn.decomposition import PCA
pca = PCA(n_components=50)
activations_pca = pca.fit_transform(activations)

# Then UMAP
reducer = umap.UMAP(n_neighbors=15, min_dist=0.05)
Y = reducer.fit_transform(activations_pca)
```

## ✦ A Note on Reproducibility

UMAP has two sources of randomness:

1. Initialisation (spectral decomposition is deterministic, but numpy random state affects tie-breaking)
2. Negative sampling in SGD

Fix `random_state` in the constructor. Even so, floating-point non-determinism across platforms means results may differ slightly between machines. For papers: always report `n_neighbors`, `min_dist`, `metric`, `random_state`, and the UMAP version.
