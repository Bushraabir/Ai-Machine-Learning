## ✦ The Core Idea

An autoencoder is a neural network trained to reproduce its input at the output, subject to a **bottleneck constraint**. The network is split into two parts:

- **Encoder** $f_\phi: \mathcal{X} \to \mathcal{Z}$ — compresses input $x$ to a latent code $z$
- **Decoder** $g_\theta: \mathcal{Z} \to \mathcal{X}$ — reconstructs $\hat{x}$ from $z$

The training objective is simply reconstruction:

$$\mathcal{L}(\phi, \theta) = \frac{1}{n} \sum_{i=1}^n \ell(x_i,, g_\theta(f_\phi(x_i)))$$

where $\ell$ is a reconstruction loss — MSE for continuous data, binary cross-entropy for binary data.

```
Input x ──► [  Encoder f_φ  ] ──► z ──► [  Decoder g_θ  ] ──► x̂ ≈ x
  (high-dim)     (compress)      (low-dim)    (reconstruct)     (high-dim)
                                   ↑
                              bottleneck:
                              dim(z) << dim(x)
```

The bottleneck is the entire mechanism. If $\dim(z) = \dim(x)$, nothing prevents the network from learning the identity map — trivially zero loss, zero useful structure learned. The bottleneck forces the encoder to learn a **compressed representation** that retains the information necessary for reconstruction.

> **Why is this useful?** No labels needed. The supervision signal is the input itself. Given unlabelled data — which is the common case in practice — an autoencoder can learn structure for free.

## ✦ Undercomplete vs. Overcomplete Autoencoders

### Undercomplete

$\dim(z) < \dim(x)$. The bottleneck is the only regulariser. With a **linear** encoder and decoder, and MSE loss, the undercomplete autoencoder recovers exactly the **PCA subspace** — the $k$ principal components. This is a theorem, not an accident.

**Proof sketch:** The MSE objective for a linear autoencoder is $\min_{W_e, W_d} |X - X W_e W_d|_F^2$. The solution is $W_e W_d = V_k V_k^T$, the projection onto the top-$k$ PCA eigenvectors. With nonlinear activations, the autoencoder finds _nonlinear_ generalisations of PCA — more expressive, but no longer has this clean characterisation.

### Overcomplete

$\dim(z) \geq \dim(x)$. Now the bottleneck alone does not prevent the identity. An overcomplete autoencoder **must** be regularised to learn useful features. This leads to sparse autoencoders, denoising autoencoders, and contractual autoencoders.

## ✦ Regularised Autoencoders

### Sparse Autoencoder

Add an L1 penalty on the activations of $z$:

$$\mathcal{L} = |x - \hat{x}|^2 + \lambda |z|_1$$

This forces most hidden units to be inactive for any given input. The network learns **dictionary-like** features: each input is a sparse combination of basis elements. This is the deep learning analogue of sparse coding (Olshausen & Field, 1996).

A common alternative: penalise via KL divergence from a Bernoulli with small mean $\rho$ (e.g. $\rho = 0.05$), interpreted as "5% of neurons should fire on average":

$$\Omega(z) = \sum_j \text{KL}(\rho | \hat{\rho}_j), \qquad \hat{\rho}_j = \frac{1}{n}\sum_i z_j^{(i)}$$

**Connection to ML:** Sparse representations have better sample efficiency for downstream tasks. Each learned feature has a specific, interpretable role. This is the intuition behind the current generation of sparse autoencoders used to interpret large language models (Anthropic's mechanistic interpretability work, 2023–2024).

---

### Denoising Autoencoder (DAE)

Instead of feeding clean $x$, corrupt it first to $\tilde{x}$, then train to reconstruct the original $x$:

$$\mathcal{L} = |x - g_\theta(f_\phi(\tilde{x}))|^2, \qquad \tilde{x} = \text{corrupt}(x)$$

Common corruption: additive Gaussian noise $\tilde{x} = x + \epsilon$, $\epsilon \sim \mathcal{N}(0, \sigma^2 I)$; or masking noise (randomly zero out some dimensions).

```
x ──[corrupt]──► x̃ ──► [Encoder] ──► z ──► [Decoder] ──► x̂
↑                                                          │
└──────────────────── MSE(x, x̂) ◄─────────────────────────┘
     (target is *clean* x, not x̃)
```

**Why this works:** The denoising criterion forces the encoder to learn the **data manifold**. Specifically, the optimal DAE (in the limit of small Gaussian noise) learns to estimate the score function $\nabla_x \log p(x)$ — the gradient of the log data density. This is a deep connection to diffusion models and score matching (Vincent, 2011).

Intuitively: a model that can denoise must understand the structure of the data distribution. Random noise takes you off the manifold; the denoiser learns to project back.

---

### Contractive Autoencoder (CAE)

Penalise the Frobenius norm of the Jacobian of the encoder with respect to the input:

$$\mathcal{L} = |x - \hat{x}|^2 + \lambda |J_{f_\phi}(x)|_F^2, \qquad J_{f_\phi}(x) = \frac{\partial f_\phi(x)}{\partial x}$$

This forces the learned representation to be **insensitive to small perturbations** in input space — the representation doesn't change much when the input changes a little. The encoder learns features that are invariant to nuisance variation.

Formally: DAE penalises sensitivity stochastically (via random corruption); CAE penalises it analytically (via the Jacobian). They are related — both push the representation onto a low-dimensional manifold.

## ✦ Variational Autoencoder (VAE)

The VAE is not merely a regularised autoencoder — it is a **latent variable generative model** trained via approximate Bayesian inference. It earns its own detailed treatment but belongs here conceptually.

### The Generative Model

Assume data is generated by:

1. Sample a latent vector: $z \sim p(z) = \mathcal{N}(0, I)$
2. Sample the observation: $x \sim p_\theta(x|z)$ (the decoder is the likelihood)

The marginal likelihood is:

$$p_\theta(x) = \int p_\theta(x|z), p(z), dz$$

This integral is **intractable** — we cannot compute it for general $p_\theta$.

### The Inference Problem

We want the posterior $p_\theta(z|x) = p_\theta(x|z) p(z) / p_\theta(x)$, but the denominator is the intractable marginal. The VAE's solution: **amortised variational inference** — approximate $p_\theta(z|x)$ with a learned $q_\phi(z|x) = \mathcal{N}(\mu_\phi(x), \text{diag}(\sigma^2_\phi(x)))$.

The encoder now outputs _parameters of a distribution_, not a single vector.

### The ELBO

Maximise the **Evidence Lower BOund** (ELBO) instead of the intractable log-likelihood:

$$\log p_\theta(x) \geq \underbrace{\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]}_{\text{reconstruction term}} - \underbrace{D_{KL}(q_\phi(z|x) ,|, p(z))}_{\text{regularisation term}}$$

**Derivation:** Start from $\log p_\theta(x) = \log \int p_\theta(x|z)p(z) dz$. Introduce $q_\phi$ via importance weighting, apply Jensen's inequality to the concave log:

$$\log p_\theta(x) = \log \mathbb{E}_{q_\phi(z|x)}!\left[\frac{p_\theta(x|z)p(z)}{q_\phi(z|x)}\right] \geq \mathbb{E}_{q_\phi}!\left[\log \frac{p_\theta(x|z)p(z)}{q_\phi(z|x)}\right] = \text{ELBO}$$

Expanding the ELBO:

$$\text{ELBO} = \mathbb{E}_{q_\phi}[\log p_\theta(x|z)] - D_{KL}(q_\phi(z|x) | p(z))$$

The gap between $\log p_\theta(x)$ and the ELBO is exactly $D_{KL}(q_\phi(z|x) | p_\theta(z|x)) \geq 0$.

**For Gaussian $q_\phi$ and Gaussian prior $p(z)$, the KL has a closed form:**

$$D_{KL}(\mathcal{N}(\mu, \sigma^2) | \mathcal{N}(0,1)) = \frac{1}{2}\sum_j \left(\mu_j^2 + \sigma_j^2 - \log\sigma_j^2 - 1\right)$$

### The Reparameterisation Trick

Sampling $z \sim q_\phi(z|x) = \mathcal{N}(\mu_\phi(x), \sigma^2_\phi(x))$ is **non-differentiable** — you cannot backpropagate through a sample. The fix:

$$z = \mu_\phi(x) + \sigma_\phi(x) \odot \epsilon, \qquad \epsilon \sim \mathcal{N}(0, I)$$

Now the randomness is in $\epsilon$, which does not depend on $\phi$. The gradient flows through $\mu_\phi$ and $\sigma_\phi$ cleanly.

```
x ──► Encoder ──► μ_φ(x), σ_φ(x)
                       │
              z = μ + σ⊙ε,  ε ~ 𝒩(0,I)   ← randomness here, not in z
                       │
                    Decoder ──► x̂
                       │
              ℒ = -𝔼[log p(x|z)] + KL
```

### Interpretation of the Two Terms

**Reconstruction term** $\mathbb{E}[\log p_\theta(x|z)]$: forces the decoder to reconstruct $x$ faithfully from $z$. For Gaussian $p_\theta(x|z)$, this is exactly MSE. For Bernoulli, binary cross-entropy.

**KL term** $D_{KL}(q_\phi | p)$: forces the posterior over $z$ to stay close to the prior $\mathcal{N}(0,I)$. This regularises the latent space — it prevents the encoder from mapping each $x_i$ to a delta function at a distinct point (which would give perfect reconstruction but an unusable latent space).

The KL term creates a **smooth, continuous latent space**: nearby points in $z$-space decode to similar outputs. This is what makes VAE useful for generation — sample $z \sim \mathcal{N}(0,I)$, decode, get a plausible $x$.

### VAE vs. Standard Autoencoder

|Property|Standard AE|VAE|
|---|---|---|
|Latent code|Deterministic vector|Distribution $q_\phi(z\|x)$|
|Training objective|Reconstruction only|ELBO = reconstruction − KL|
|Latent space structure|Unstructured|Regularised toward $\mathcal{N}(0,I)$|
|Can generate new samples?|No (no principled sampling)|Yes — sample $z \sim p(z)$, decode|
|Theoretical basis|Ad hoc|Variational Bayes|

## ✦ Masked Autoencoders (MAE)

The modern descendant of denoising autoencoders, scaled to vision transformers.

**Idea:** randomly mask a large fraction (e.g. 75%) of image patches, encode only the visible patches, then reconstruct the full image from the encoded representation.

$$\mathcal{L} = \frac{1}{|\mathcal{M}|} \sum_{i \in \mathcal{M}} |x_i - \hat{x}_i|^2$$

where $\mathcal{M}$ is the set of masked patches.

This is a self-supervised objective — no labels. The model must learn rich semantic features to reconstruct masked regions from context. He et al. (2021) showed this produces representations competitive with supervised pretraining on ImageNet.

**Why 75%?** Unlike text (where masking 15% as in BERT is hard due to semantic density), images have high spatial redundancy — 75% masking is still a learnable task, and it forces the model to reason about global structure, not just local texture.

## ✦ Connection to the Broader ML/DL Landscape

### 1. Dimensionality Reduction

An undercomplete deterministic autoencoder is a **nonlinear generalisation of PCA**. PCA is the special case with linear encoder/decoder. Both minimise reconstruction error; the autoencoder can capture curved manifolds that PCA cannot.

### 2. Generative Models

The VAE is one of three major frameworks for deep generative modelling:

- **VAEs**: latent variable + variational inference
- **GANs**: adversarial training, no explicit likelihood
- **Diffusion models**: iterative denoising, connection to DAEs via score matching

VAEs produce blurry samples (the Gaussian likelihood averages over modes); GANs produce sharp samples but are harder to train; diffusion models currently dominate image quality.

### 3. Representation Learning and Transfer Learning

A pre-trained encoder $f_\phi$ can be used as a **feature extractor**: freeze $\phi$, attach a small classifier on top, train on a small labelled dataset. This is **transfer learning from self-supervised pretraining** — the standard paradigm in NLP (BERT, GPT) and increasingly in vision (MAE, DINO).

### 4. Information-Theoretic View

The ELBO can be rewritten as:

$$\text{ELBO} = -\mathbb{E}[\text{reconstruction error}] - D_{KL}(q_\phi | p)$$

This is exactly the **rate-distortion tradeoff** from information theory:

- Distortion = reconstruction error
- Rate = bits needed to encode $z$, upper bounded by $D_{KL}$

The $\beta$-VAE (Higgins et al., 2017) makes this tradeoff explicit by scaling the KL term: $\mathcal{L} = \text{recon} - \beta \cdot D_{KL}$. High $\beta$ → more compressed, more disentangled latent space; low $\beta$ → better reconstruction.

### 5. Anomaly Detection

A standard autoencoder trained on normal data will have low reconstruction error on normal inputs and high reconstruction error on anomalies (never seen during training, poorly represented). Threshold on reconstruction error → anomaly detector. Widely used in industrial fault detection, fraud detection, cybersecurity.

### 6. Mechanistic Interpretability (2023–present)

Sparse autoencoders are now being applied to the _residual stream_ of large language models. The idea: the superposition hypothesis states that LLMs pack many features into fewer neurons via superposition. A sparse autoencoder trained on LLM activations can decompose these superposed representations into sparse, interpretable features. This is an active research direction at Anthropic and DeepMind.

## ✦ Implementation Sketch (PyTorch)

```python
import torch
import torch.nn as nn

# --- Standard Autoencoder ---
class Autoencoder(nn.Module):
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256), nn.ReLU(),
            nn.Linear(256, latent_dim)
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256), nn.ReLU(),
            nn.Linear(256, input_dim), nn.Sigmoid()
        )

    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z), z

# --- VAE ---
class VAE(nn.Module):
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        self.encoder = nn.Sequential(nn.Linear(input_dim, 256), nn.ReLU())
        self.fc_mu    = nn.Linear(256, latent_dim)
        self.fc_logvar = nn.Linear(256, latent_dim)
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256), nn.ReLU(),
            nn.Linear(256, input_dim), nn.Sigmoid()
        )

    def encode(self, x):
        h = self.encoder(x)
        return self.fc_mu(h), self.fc_logvar(h)

    def reparameterise(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)        # ε ~ N(0, I)
        return mu + eps * std              # z = μ + σ⊙ε

    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterise(mu, logvar)
        x_hat = self.decoder(z)
        return x_hat, mu, logvar

def vae_loss(x, x_hat, mu, logvar):
    recon = nn.functional.binary_cross_entropy(x_hat, x, reduction='sum')
    # KL divergence: -½ Σ(1 + log σ² - μ² - σ²)
    kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + kl
```

> **Implementation notes:**
> 
> - Use `logvar = log σ²` not `σ²` directly — prevents numerical issues (log is unconstrained, σ² must be positive).
> - `reduction='sum'` for BCE — sum over pixels and batch, then divide by batch size outside. Using `'mean'` implicitly scales the KL differently.
> - During inference/generation: pass $z \sim \mathcal{N}(0,I)$ directly to the decoder. No encoder needed.

## ✦ Failure Modes and Diagnostics

**Posterior collapse (VAE):** The encoder ignores the input and outputs $q_\phi(z|x) \approx p(z) = \mathcal{N}(0,I)$ — the KL term goes to zero but reconstruction is poor. The decoder learns to ignore $z$ and models $p(x)$ by itself. Fixes: KL annealing (start with $\beta=0$, slowly increase), free bits (allow a minimum KL per dimension), more expressive decoders.

**Blurry reconstructions (VAE):** The Gaussian reconstruction likelihood averages over modes. A pixel predicted as 0.5 is blurry. Fixes: use a learned variance for $p_\theta(x|z)$, or switch to a discrete (categorical) likelihood, or use a flow-based decoder.

**Trivial identity solution (standard AE):** If the model is expressive enough and unconstrained, it learns identity. Ensure the bottleneck is genuinely narrow, or add explicit regularisation.

**Overfitting the reconstruction (denoising AE):** If corruption is too weak, the model just memorises. If too strong, the task is impossible. Tune noise level — typically validated via downstream task performance or KNN accuracy in $z$-space.
