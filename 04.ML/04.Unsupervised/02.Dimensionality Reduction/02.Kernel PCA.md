## ✦ The Problem with Linear PCA

PCA finds principal axes by eigendecomposing the covariance matrix $\Sigma = \frac{1}{n}X^TX$. It is a linear operation — it can only find structure that lives in a linear subspace of $\mathbb{R}^d$.

Consider two concentric rings:

```
    ● ● ●                  The two rings are linearly inseparable.
  ●       ●                PCA projects onto a line or plane —
 ●  ○ ○ ○  ●               the two classes collapse onto each other.
 ●  ○   ○  ●               No linear projection separates them.
  ●  ○ ○  ●
    ● ● ●
```

Or the Swiss roll: a 2D surface rolled into a 3D ambient space. PCA in $\mathbb{R}^3$ gives you a 2D projection that crushes the roll flat — destroying the intrinsic geometry. The true 2D coordinates (position along the roll, position across the roll) are nonlinearly embedded.

Linear PCA has no recourse. We need to operate in a richer space.

---

## ✦ The Core Idea: PCA in Feature Space

The kernel trick, which you have already seen in SVMs ([[04.Support Vector Machines]]), says: replace $\langle x_i, x_j \rangle$ with $k(x_i, x_j) = \langle \phi(x_i), \phi(x_j) \rangle_\mathcal{H}$ where $\phi : \mathcal{X} \to \mathcal{H}$ maps data into a (possibly infinite-dimensional) Hilbert space $\mathcal{H}$.

The idea for Kernel PCA: **run PCA in $\mathcal{H}$, not in $\mathbb{R}^d$**.

The covariance matrix in $\mathcal{H}$ is: $$\tilde{\Sigma} = \frac{1}{n}\sum_{i=1}^n \phi(x_i)\phi(x_i)^T$$

We want eigenvectors $v \in \mathcal{H}$: $\tilde{\Sigma} v = \lambda v$.

The crucial observation: every eigenvector of $\tilde{\Sigma}$ lies in the span of ${\phi(x_1), \ldots, \phi(x_n)}$.

_Proof._ $\tilde{\Sigma} v = \lambda v$ with $\lambda > 0$ implies $v = \frac{1}{\lambda}\tilde{\Sigma} v = \frac{1}{n\lambda}\sum_{i=1}^n \phi(x_i)\phi(x_i)^T v = \frac{1}{n\lambda}\sum_{i=1}^n \langle \phi(x_i), v\rangle_\mathcal{H} \cdot \phi(x_i)$. This is a linear combination of the $\phi(x_i)$. $\square$

So we can write $v = \sum_{i=1}^n \alpha_i \phi(x_i)$ for some $\alpha \in \mathbb{R}^n$. We never need to know $\phi$ explicitly — we only need inner products $\langle \phi(x_i), \phi(x_j)\rangle_\mathcal{H} = k(x_i, x_j)$.

---

## ✦ Derivation: From Covariance Eigenproblem to Kernel Eigenproblem

### Step 1: The covariance eigenproblem in $\mathcal{H}$

We want $v \in \mathcal{H}$ with $|v| = 1$ satisfying: $$\tilde{\Sigma} v = \lambda v$$

Substituting $\tilde{\Sigma} = \frac{1}{n}\sum_i \phi(x_i)\phi(x_i)^T$ and $v = \sum_j \alpha_j \phi(x_j)$:

$$\frac{1}{n}\sum_{i=1}^n \phi(x_i)\phi(x_i)^T \sum_{j=1}^n \alpha_j \phi(x_j) = \lambda \sum_{j=1}^n \alpha_j \phi(x_j)$$

$$\frac{1}{n}\sum_{i=1}^n \phi(x_i) \sum_{j=1}^n \alpha_j k(x_i, x_j) = \lambda \sum_{j=1}^n \alpha_j \phi(x_j)$$

$$\frac{1}{n}\sum_{i=1}^n \phi(x_i) (K\alpha)_i = \lambda \sum_{j=1}^n \alpha_j \phi(x_j)$$

where $K \in \mathbb{R}^{n \times n}$ is the **kernel (Gram) matrix** with $K_{ij} = k(x_i, x_j)$.

### Step 2: Project onto $\phi(x_l)$ for arbitrary $l$

Take inner product of both sides with $\phi(x_l)$:

$$\frac{1}{n} \sum_{i=1}^n k(x_l, x_i)(K\alpha)_i = \lambda \sum_{j=1}^n \alpha_j k(x_l, x_j)$$

$$\frac{1}{n}(K^2 \alpha)_l = \lambda (K\alpha)_l$$

In matrix form: $\frac{1}{n} K^2 \alpha = \lambda K \alpha$.

### Step 3: Simplify

Assuming $K$ is invertible (or restricting to its range):

$$\frac{1}{n} K \alpha = \lambda \alpha \implies K\alpha = n\lambda \alpha$$

Renaming $n\lambda \to \lambda'$: **we need the eigenvectors of the kernel matrix $K$**.

$$K \alpha = \lambda' \alpha$$

This is the central result. The $n \times n$ kernel matrix $K$ plays the role of the $d \times d$ covariance matrix $\Sigma$. PCA in feature space reduces to eigendecomposing the Gram matrix.

### Step 4: Normalisation constraint

We need $|v|_\mathcal{H} = 1$: $$|v|^2 = \left|\sum_j \alpha_j \phi(x_j)\right|^2 = \sum_{i,j} \alpha_i \alpha_j k(x_i, x_j) = \alpha^T K \alpha = 1$$

Since $K\alpha = \lambda'\alpha$: $\alpha^T K \alpha = \lambda' |\alpha|^2 = 1 \implies |\alpha|^2 = \frac{1}{\lambda'}$.

So after computing eigenvectors $\alpha_l$ of $K$ with eigenvalue $\lambda_l'$, normalise by $\hat{\alpha}_l = \frac{\alpha_l}{\sqrt{\lambda_l'}}$.

---

## ✦ Computing Kernel PCA Projections

The $l$-th kernel PC score for a training point $x_i$ is:

$$z_{il} = \langle v_l, \phi(x_i)\rangle_\mathcal{H} = \sum_{j=1}^n \hat{\alpha}_{jl}, k(x_j, x_i) = \hat{\alpha}_l^T k_i$$

where $k_i = (k(x_1, x_i), \ldots, k(x_n, x_i))^T \in \mathbb{R}^n$.

In matrix form, the full embedding of training data to $p$ kernel PCs:

$$Z = K \hat{A} \quad \in \mathbb{R}^{n \times p}$$

where $\hat{A} = [\hat{\alpha}_1 | \cdots | \hat{\alpha}_p] \in \mathbb{R}^{n \times p}$ (columns are normalised eigenvectors of $K$).

But since $K\hat{\alpha}_l = \lambda_l' \hat{\alpha}_l$:

$$Z = K\hat{A} = \text{diag}(\lambda_1', \ldots, \lambda_p') \hat{A} \cdot \frac{\hat{A}}{\hat{A}} \cdot \hat{A}$$

More directly: $Z_{il} = \lambda_l' \hat{\alpha}_{il}$ — the $l$-th kernel PC score for point $x_i$ is $\lambda_l' \hat{\alpha}_{il}$.

Wait — let us be careful. $K\hat{\alpha}_l = \lambda_l' \hat{\alpha}_l$, so $(K\hat{A})_{il} = \lambda_l' \hat{\alpha}_{il}$. And $\hat{\alpha}_{il} = \frac{\alpha_{il}}{\sqrt{\lambda_l'}}$, so:

$$Z_{il} = \lambda_l' \cdot \frac{\alpha_{il}}{\sqrt{\lambda_l'}} = \sqrt{\lambda_l'}, \alpha_{il}$$

where $\alpha_l$ is the **unit-norm** eigenvector of $K$ (before the $\hat{\alpha}$ normalisation). This is the sklearn convention.

---

## ✦ Centring in Feature Space

The derivation above assumes $\sum_i \phi(x_i) = 0$ — i.e., that the data is centred in feature space $\mathcal{H}$. This is generally not true even if the original data $X$ is centred, because $\phi$ is nonlinear.

We must **centre the kernel matrix**. The centred kernel matrix is:

$$\tilde{K} = K - \mathbf{1}_n K - K \mathbf{1}_n + \mathbf{1}_n K \mathbf{1}_n$$

where $\mathbf{1}_n = \frac{1}{n}\mathbf{1}\mathbf{1}^T \in \mathbb{R}^{n \times n}$ (matrix of all $1/n$).

Entry-wise: $$\tilde{K}_{ij} = k(x_i, x_j) - \frac{1}{n}\sum_{l=1}^n k(x_l, x_j) - \frac{1}{n}\sum_{l=1}^n k(x_i, x_l) + \frac{1}{n^2}\sum_{l,m=1}^n k(x_l, x_m)$$

This is the kernel-space analogue of subtracting the mean before computing PCA. **Always centre the kernel matrix.** Failing to do so is the most common implementation error in Kernel PCA.

Geometric interpretation: $\tilde{K}_{ij} = \langle \phi(x_i) - \mu_\phi,, \phi(x_j) - \mu_\phi\rangle_\mathcal{H}$ where $\mu_\phi = \frac{1}{n}\sum_i \phi(x_i)$ is the mean in feature space. The centred kernel matrix is the covariance inner product matrix in $\mathcal{H}$.

### Centring for out-of-sample points

For a new point $x__$, its kernel vector is $k__ = (k(x_1, x__), \ldots, k(x_n, x__))^T$. The centred version: $$\tilde{k}_* = k_* - \frac{1}{n}K\mathbf{1} - \mathbf{1}_n k_* + \frac{1}{n^2}\mathbf{1}^T K \mathbf{1}$$

where $\mathbf{1} \in \mathbb{R}^n$ is the all-ones vector. In practice, store $\frac{1}{n}K\mathbf{1}$ and $\frac{1}{n^2}\mathbf{1}^T K\mathbf{1}$ from the training fit.

---

## ✦ Out-of-Sample Extension

A fundamental limitation of Kernel PCA (compared to linear PCA): the eigenvectors $v_l$ live in $\mathcal{H}$ and are expressed via the training data. Projecting a **new point** $x_*$ requires:

$$z_{_l} = \langle v_l, \phi(x__)\rangle_\mathcal{H} = \sum_{j=1}^n \hat{\alpha}_{jl}, \tilde{k}(x_j, x_*)$$

where $\tilde{k}(x_j, x__)$ is the centred kernel between training point $x_j$ and the new point $x__$.

This is $O(n)$ per new point per component — and requires storing all $n$ training points. Contrast with linear PCA: out-of-sample projection is $O(d)$ and requires only the $k \times d$ weight matrix.

**Practical consequence:** Kernel PCA does not scale to large $n$. For $n > 10^4$, it is typically replaced by:

- Approximate methods: Nyström approximation (subsample $m \ll n$ "landmark" points, build $K_m$)
- Sparse kernel PCA
- UMAP (which also learns a nonlinear embedding but scales better)

## ✦ The Full Algorithm

```

Output: Z ∈ ℝⁿˣᵖ (kernel PCA scores), stored training data for projection

--- TRAINING ---

1. Compute kernel matrix:
   K ∈ ℝⁿˣⁿ,  K[i,j] = k(xᵢ, xⱼ)

2. Centre the kernel matrix:
   row_means = K.mean(axis=1, keepdims=True)      # (n,1)
```

## ✦ Implementation from Scratch (NumPy)

```python
import numpy as np

def rbf_kernel(X, Y, gamma):
    """K[i,j] = exp(-gamma ||xᵢ - yⱼ||²)"""
    # ||x - y||² = ||x||² + ||y||² - 2 x·y
    X_sq = np.sum(X**2, axis=1, keepdims=True)   # (n,1)
    Y_sq = np.sum(Y**2, axis=1, keepdims=True)   # (m,1)
    cross = X @ Y.T                               # (n,m)
    sq_dists = X_sq + Y_sq.T - 2 * cross
    return np.exp(-gamma * sq_dists)


class KernelPCA:
    def __init__(self, n_components, kernel='rbf', gamma=1.0, degree=3, coef0=1.0):
        self.p = n_components
        self.kernel_name = kernel
        self.gamma = gamma
        self.degree = degree
        self.coef0 = coef0

    def _kernel(self, X, Y):
        if self.kernel_name == 'rbf':
            return rbf_kernel(X, Y, self.gamma)
        elif self.kernel_name == 'poly':
            return (self.gamma * (X @ Y.T) + self.coef0) ** self.degree
        elif self.kernel_name == 'linear':
            return X @ Y.T
        else:
            raise ValueError(f"Unknown kernel: {self.kernel_name}")

    def fit(self, X):
        n = len(X)
        self.X_train_ = X.copy()

        # 1. Compute kernel matrix
        K = self._kernel(X, X)                      # (n,n)

        # 2. Centre: K̃ = K - 1ₙK - K1ₙ + 1ₙK1ₙ
        self.row_means_ = K.mean(axis=1)             # (n,)
        self.col_means_ = K.mean(axis=0)             # (n,)
        self.grand_mean_ = K.mean()

        K_centered = (K
                      - self.row_means_[:, None]
                      - self.col_means_[None, :]
                      + self.grand_mean_)

        # 3. Eigendecompose K̃
        eigenvalues, eigenvectors = np.linalg.eigh(K_centered)
        # eigh returns ascending order; reverse
        idx = np.argsort(eigenvalues)[::-1]
        eigenvalues = eigenvalues[idx]
        eigenvectors = eigenvectors[:, idx]

        # 4. Keep top p, drop near-zero eigenvalues
        eigenvalues = eigenvalues[:self.p]
        eigenvectors = eigenvectors[:, :self.p]

        # Numerical safety: clip tiny negatives (rounding errors)
        eigenvalues = np.clip(eigenvalues, 1e-12, None)

        self.eigenvalues_ = eigenvalues
        # Normalised eigenvectors: α̂_l = α_l / sqrt(λ'_l)
        self.alphas_ = eigenvectors / np.sqrt(eigenvalues)[None, :]  # (n,p)

        return self

    def transform(self, X):
        # Compute kernel between new X and training data
        K = self._kernel(X, self.X_train_)           # (m,n)

        # Centre using stored training statistics
        # k̃[i,j] = k[i,j] - mean_row_train[j] - mean_col_new[i] + grand_mean
        K_centered = (K
                      - self.row_means_[None, :]     # subtract train row means
                      - K.mean(axis=1)[:, None]      # subtract new point col means
                      + self.grand_mean_)

        # Project: z = K̃ · Â  → (m,p)
        return K_centered @ self.alphas_

    def fit_transform(self, X):
        self.fit(X)
        # For training data, use training kernel directly (more stable)
        K = self._kernel(X, X)
        K_centered = (K
                      - self.row_means_[:, None]
                      - self.col_means_[None, :]
                      + self.grand_mean_)
        return K_centered @ self.alphas_


# --- Test on concentric rings ---
from sklearn.datasets import make_circles
import matplotlib.pyplot as plt

X, y = make_circles(n_samples=400, factor=0.3, noise=0.05, random_state=42)

axes[0].scatter(X[:,0], X[:,1], c=y, cmap='bwr', s=10)
axes[0].set_title('Original space')

axes[1].scatter(Z[:,0], Z[:,1], c=y, cmap='bwr', s=10)
axes[1].set_title('Kernel PCA (RBF, γ=10)')
plt.tight_layout()
```

## ✦ Common Kernels and What They Do

### Linear Kernel

$$k(x, y) = x^T y$$

Kernel PCA with the linear kernel is identical to standard PCA. This is the sanity check: if you implement Kernel PCA correctly and run it with a linear kernel, you recover PCA exactly (up to sign and scaling).

_Why?_ The Gram matrix $K_{ij} = x_i^T x_j = (XX^T)_{ij}$. The centred Gram matrix has the same eigenvectors as the centred covariance $X^TX$ (up to the $U$-$V$ duality in the SVD $X = U\Sigma V^T$: $XX^T = U\Sigma^2 U^T$ vs. $X^TX = V\Sigma^2 V^T$).

### Polynomial Kernel

$$k(x, y) = (\gamma x^T y + c_0)^d$$

Corresponds to a feature map $\phi$ containing all monomials of degree $\leq d$. Dimension of $\phi$: $\binom{d+\text{deg}}{\text{deg}}$ — exponential in degree.

For degree $d=2$, $\mathbb{R}^p$ input: $\phi$ has $1 + p + \binom{p}{2}$ features (constant, linear, and quadratic monomials). Kernel PCA in this space can separate data by quadratic boundaries.

### RBF (Gaussian) Kernel

$$k(x, y) = \exp!\left(-\gamma |x - y|^2\right) = \exp!\left(-\frac{|x-y|^2}{2\sigma^2}\right)$$

where $\gamma = \frac{1}{2\sigma^2}$.

Corresponds to an **infinite-dimensional** feature map. The feature space $\mathcal{H}$ is the RKHS of the Gaussian kernel — functions that are smooth in proportion to $\sigma$.

**Proof of infinite dimensionality.** Via Mercer's theorem and the Taylor expansion of the exponential: $$\exp(-\gamma|x-y|^2) = \exp(-\gamma|x|^2)\exp(2\gamma x^Ty)\exp(-\gamma|y|^2)$$ $$= \exp(-\gamma|x|^2)\exp(-\gamma|y|^2)\sum_{n=0}^\infty \frac{(2\gamma)^n}{n!}(x^Ty)^n$$

Each term $(x^Ty)^n$ corresponds to degree-$n$ polynomial features. Summing over all $n = 0, 1, 2, \ldots$ gives countably infinite features. The kernel evaluates all of them implicitly in $O(d)$ via the exponential.

**The $\gamma$ hyperparameter:**

- Large $\gamma$ (small $\sigma$): narrow kernel → points must be very close to have nonzero similarity → captures fine-grained local structure → risk of overfitting / fragmented clusters.
- Small $\gamma$ (large $\sigma$): broad kernel → all points are similar to all others → approaches linear PCA → loses nonlinear structure.

The RBF kernel is the default choice for Kernel PCA when the appropriate feature map is unknown.

### Choosing the Kernel

There is no universal best kernel — this is the No Free Lunch theorem applied to kernels. Guidelines:

- **Linear:** baseline, interpretable, equivalent to PCA.
- **Polynomial:** use when you have reason to believe the relevant structure is polynomial (e.g., interaction terms matter).
- **RBF:** default for smooth, unknown manifolds. Tune $\gamma$ via the explained variance of the kernel PCs or a downstream task.
- **Custom:** domain-specific kernels encode prior knowledge (string kernels for text, graph kernels for molecular structures).

For Kernel PCA specifically, a useful heuristic for $\gamma$ in RBF: set $\sigma = $ median pairwise distance of the training data (the "median heuristic"), i.e., $\gamma = \frac{1}{2 \cdot \text{median}(|x_i - x_j|^2)}$.

## ✦ Mercer's Theorem: When Is $k$ a Valid Kernel?

Not every function $k : \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ corresponds to a valid inner product. We need:

**Mercer's Theorem.** A continuous, symmetric function $k : \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ is a valid kernel (i.e., $\exists$ Hilbert space $\mathcal{H}$ and map $\phi$ s.t. $k(x,y) = \langle\phi(x),\phi(y)\rangle_\mathcal{H}$) if and only if the kernel matrix $K \in \mathbb{R}^{n \times n}$ is **positive semi-definite** for all finite sets of points ${x_1, \ldots, x_n} \subset \mathcal{X}$.

Equivalently: $\sum_{i,j} c_i c_j k(x_i, x_j) \geq 0$ for all $n \in \mathbb{N}$, all ${x_i}$, all $c \in \mathbb{R}^n$.

**Why this matters for Kernel PCA:** If $k$ is not a valid Mercer kernel, the Gram matrix $K$ may have negative eigenvalues. These are not interpretable as variances. Centering can also introduce spurious negative eigenvalues for near-valid kernels. In the implementation, we clip eigenvalues at zero — but if many eigenvalues are substantially negative, the kernel is wrong for the data.

**Constructing valid kernels:** Valid kernels are closed under:

- Positive scaling: $c \cdot k$ for $c > 0$
- Addition: $k_1 + k_2$
- Multiplication: $k_1 \cdot k_2$
- Composition: $k(x,y) = k_0(f(x), f(y))$ for any $f$
- Exponentiation: $\exp(k_0)$

This allows building complex kernels from simple ones: e.g., $k(x,y) = \exp(-\gamma|x-y|^2) \cdot \cos(\omega^T(x-y))$ is valid (product of valid kernels).

## ✦ Geometric Interpretation: Unfolding Manifolds

The RBF kernel assigns high similarity to nearby points and near-zero similarity to distant ones. In the feature space $\mathcal{H}$, all points are mapped to the unit sphere (since $k(x,x) = 1$ for RBF): $|\phi(x)|_\mathcal{H} = 1$ for all $x$.

The Kernel PCA embedding then places similar points near each other on this hypersphere. The first kernel PC direction is the direction of maximum variance on this sphere — which corresponds to the principal direction of the manifold structure.

For the Swiss roll example: the RBF kernel in $\mathbb{R}^3$ assigns high similarity to nearby points along the roll. PCA in this kernel space separates the "position along the roll" direction from the "position across the roll" direction — exactly the intrinsic 2D coordinates. Linear PCA cannot do this because the Swiss roll is not linear.

```
Swiss roll in ℝ³         RBF Kernel PCA embedding

   /────────/             ●●●●●●●●●●
  /  ● ● ● /    →        ●●●●●●●●●●●     (2D unfolding)
 / ●  ●  ● /             ●●●●●●●●●●
/──────────/
```

## ✦ Nyström Approximation: Scaling to Large $n$

The bottleneck of Kernel PCA is $O(n^2)$ to form $K$ and $O(n^3)$ (or $O(n^2 p)$ with partial eigendecomposition) to decompose it. For $n = 10^5$, storing $K$ alone requires 80 GB.

**Nyström approximation** (Williams & Seeger, 2001): subsample $m \ll n$ "landmark" points ${x_{l_1}, \ldots, x_{l_m}}$ and approximate the full kernel matrix:

$$K \approx K_{nm} K_{mm}^{-1} K_{mn}$$

where $K_{nm} \in \mathbb{R}^{n \times m}$: kernel between all $n$ points and the $m$ landmarks, $K_{mm} \in \mathbb{R}^{m \times m}$: kernel among landmarks.

**Approximate Kernel PCA via Nyström:**

1. Choose $m$ landmark points (randomly, or by $k$-means).
2. Compute $K_{nm}$ and $K_{mm}$.
3. Eigendecompose $K_{mm} = U_m \Lambda_m U_m^T$.
4. Approximate eigenvectors of $K$: $\hat{U} = \sqrt{\frac{m}{n}} K_{nm} U_m \Lambda_m^{-1}$.
5. Orthonormalise $\hat{U}$ via QR decomposition.

Cost: $O(m^2 n)$ instead of $O(n^2 p)$. For $m = 1000$, $n = 10^6$: manageable.

The approximation error is controlled by how well $K_{mm}$ "spans" the full kernel matrix — related to the spectral decay of $K$.

## ✦ Kernel PCA vs. Other Nonlinear DR Methods

||Kernel PCA|t-SNE|UMAP|Isomap|
|---|---|---|---|---|
|Objective|Max variance in $\mathcal{H}$|Preserve local prob. distributions|Preserve fuzzy topology|Preserve geodesic distances|
|Global structure|Yes (kernel-dependent)|No|Partially|Yes|
|Out-of-sample|Yes ($O(n)$ per point)|No|Yes (with extra fit)|No|
|Scalability|$O(n^2)$ / Nyström approx|$O(n \log n)$ approx|$O(n \log n)$|$O(n^2)$ naive|
|Interpretability|Kernel eigenvalues|None|None|Geodesic scale|
|Cluster separation|Good (right kernel)|Excellent (visual)|Good|Moderate|
|Noise sensitivity|Depends on kernel|High (perplexity)|Moderate|High (geodesic)|

**When to choose Kernel PCA over t-SNE/UMAP:**

- You need out-of-sample projection (new test points must be embedded).
- You have a domain-informed kernel encoding structural similarity.
- You want a method with a clear objective function (eigenvalue problem) rather than stochastic optimisation.
- You care about explained variance (kernel eigenvalues are interpretable).

**When to prefer UMAP:**

- $n > 10^4$ and Nyström is insufficient.
- You want a good 2D visualisation and don't need out-of-sample.
- The manifold is locally uniform but globally complex.

## ✦ Connection to AI, ML, and Deep Learning

### The Shared Foundation with SVMs

Kernel PCA and SVMs both rest on the same kernel trick and Mercer's theorem. In both cases:

- The algorithm operates entirely through $K_{ij} = k(x_i, x_j)$.
- The solution lives in the span of the training data (representer theorem).
- The feature map $\phi$ is implicit and potentially infinite-dimensional.

The mathematical machinery is identical; the objectives differ (unsupervised variance maximisation vs. supervised margin maximisation).

### The Kernel Trick and Neural Networks

Deep networks can be interpreted as learning the kernel. The **Neural Tangent Kernel** (NTK, Jacot et al., 2018) shows that infinitely wide neural networks trained with gradient descent are equivalent to kernel regression with a specific kernel determined by the architecture. The features learned by a neural network implicitly define a kernel — and the "kernel PCA" of those features is exactly the PCA of the network's learned representation.

This connection provides a theoretical foundation for understanding why deep networks generalise: they are learning good kernels.

### Gaussian Processes and the RKHS

Gaussian Processes ([[03.Gaussian Processes]]) also operate in the RKHS of a kernel $k$. The GP posterior mean is the kernel regression estimate; kernel PCA extracts the principal directions of this space. Both methods share:

- The same $n \times n$ kernel matrix $K$.
- The same $O(n^3)$ computational bottleneck.
- The same Nyström / inducing point approximations for scaling.

The RKHS is the shared mathematical home of SVMs, Kernel PCA, and GPs.

### Kernel Methods and Pre-trained Embeddings

A modern workflow: use a pre-trained neural network (ResNet, BERT) as a fixed feature extractor $\phi$, then run linear PCA on the embeddings $\phi(x_i) \in \mathbb{R}^k$. This is equivalent to Kernel PCA with the kernel $k(x_i, x_j) = \phi(x_i)^T\phi(x_j)$ — the **linear kernel in embedding space**. The pre-trained network has implicitly learned the right $\phi$; linear PCA then extracts the principal structure.

This is a central pattern in modern ML: use deep networks to learn the feature map, then use classical kernel/linear methods on top.

### Spectral Clustering

Spectral clustering (a graph-based clustering method) is closely related to Kernel PCA. Both:

1. Compute a $n \times n$ affinity/kernel matrix.
2. Eigendecompose it.
3. Use the top eigenvectors as a new representation.

The difference: spectral clustering uses the **graph Laplacian** $L = D - K$ (where $D$ is the degree matrix) rather than the centred kernel matrix, and runs K-Means on the eigenvectors rather than using them directly as coordinates. Kernel PCA with subsequent K-Means is essentially spectral clustering with a different normalisation.
