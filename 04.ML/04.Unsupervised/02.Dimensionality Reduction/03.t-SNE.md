## ✦ The Problem PCA Cannot Solve

PCA finds directions of maximum variance. It is a linear projection. That means it can only preserve structure that is linearly separable in the original space.

Consider the Swiss roll: a 2D manifold coiled in 3D. PCA projects it flat and destroys the manifold structure entirely. The distances that matter — distances _along the manifold_ — get collapsed.

```
Swiss roll in 3D          PCA projection          t-SNE projection
     ╭──────╮               ████████               ·  ·  ·
    ╱  ····  ╲              ████████              ·      ·
   ╱  ·    ·  ╲   PCA→     ████████   t-SNE→    ·        ·
  ╱  ·      ·  ╲           ████████              ·      ·
 ╱  ············╲          ████████               ·  ·  ·
     (coiled)             (squashed)              (unrolled)
```

t-SNE's insight: **forget global distances. Preserve local neighbourhoods.** If $x_i$ and $x_j$ are close in high-dimensional space, their images $y_i$ and $y_j$ should be close in 2D. That's the whole game.

---

## ✦ The Algorithm: Full Derivation

### Step 1 — High-Dimensional Similarities

For each pair $(i, j)$, define the **conditional probability** that point $i$ would pick $j$ as its neighbour, where the neighbourhood is defined by a Gaussian centred at $x_i$:

$$p_{j|i} = \frac{\exp!\left(-|x_i - x_j|^2 / 2\sigma_i^2\right)}{\displaystyle\sum_{k \neq i} \exp!\left(-|x_i - x_k|^2 / 2\sigma_i^2\right)}$$

Note this is **asymmetric**: $p_{j|i} \neq p_{i|j}$ in general, because $\sigma_i \neq \sigma_j$.

To get a symmetric joint distribution (which is easier to optimise), symmetrise:

$$p_{ij} = \frac{p_{j|i} + p_{i|j}}{2n}$$

This guarantees $\sum_{i,j} p_{ij} = 1$ and $p_{ij} = p_{ji}$.

> **Why symmetrise?** The original asymmetric SNE (Hinton & Roweis 2002) used $p_{j|i}$ directly and had a separate cost per point. The symmetric version gives a cleaner single KL divergence to minimise, and empirically works at least as well.

---

### Step 2 — The Perplexity Controls $\sigma_i$

The bandwidth $\sigma_i$ is **not a single global parameter** — it is set _per point_, via a binary search that achieves a target **perplexity**:

$$\text{Perp}(P_i) = 2^{H(P_i)}, \qquad H(P_i) = -\sum_j p_{j|i} \log_2 p_{j|i}$$

Intuitively, perplexity ≈ the effective number of neighbours each point considers.

- In **dense regions**, $\sigma_i$ will be small (points are close; a narrow Gaussian suffices).
- In **sparse regions**, $\sigma_i$ will be large (need a wider Gaussian to include enough neighbours).

This adaptive bandwidth is what makes t-SNE robust to non-uniform density.

**Typical values:** $\text{Perp} \in [5, 50]$. Van der Maaten recommends 30 as a default. For very large datasets ($n > 10^4$), larger perplexities (50–100) can be better.

---

### Step 3 — Low-Dimensional Similarities (The t-Distribution)

In the 2D embedding, we _could_ use a Gaussian too. SNE does this. But there's a problem: the **crowding problem**.

**The crowding problem:** In $d$ dimensions, the volume of a thin shell of radius $r$ scales as $r^{d-1}$. When you project to 2D, there simply isn't enough area to faithfully represent moderate distances — points at moderate distance in high-$d$ get crushed against points that are truly close. The 2D Gaussian decays too fast at moderate distances; it can't push moderately-distant points far enough apart.

**The fix:** use a **Student's t-distribution with 1 degree of freedom** (the Cauchy distribution) in the low-dimensional space. Its heavier tails let moderately-distant clusters separate clearly:

$$q_{ij} = \frac{\left(1 + |y_i - y_j|^2\right)^{-1}}{\displaystyle\sum_{k \neq l} \left(1 + |y_k - y_l|^2\right)^{-1}}$$

The $(1 + |y_i - y_j|^2)^{-1}$ is exactly the unnormalised Cauchy kernel. The key property: it has **much heavier tails** than a Gaussian, so unlike the Gaussian, it assigns non-negligible probability to moderately large distances.

```
Probability (unnorm.)
   │
 1 ┤  ██                  ← Gaussian: decays fast
   │  █ █                   t (ν=1): decays slow
   │  █   █        ████████████████████████████
   │  █     ██████
   │  █
   └─────────────────────────────────────────── distance
     0    1    2    3    4    5
```

The **mismatch** is the key: high-dim uses Gaussian ($p_{ij}$), low-dim uses Student-t ($q_{ij}$). This mismatch is intentional — it's what creates the gap between clusters.

---

### Step 4 — The Objective: Minimise KL Divergence

The objective is to find a 2D configuration ${y_i}$ such that $Q$ is as close to $P$ as possible, in the sense of KL divergence:

$$\mathcal{L} = KL(P ,|, Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

**Why KL divergence, not MSE?** Because we're matching probability distributions, not raw distances. KL has an asymmetry that matters:

$$KL(P | Q) = \sum p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

- When $p_{ij}$ is large (nearby in high-dim) and $q_{ij}$ is small (far in low-dim): **huge penalty**. Nearby neighbours _must_ stay nearby.
- When $p_{ij}$ is small (far in high-dim) and $q_{ij}$ is large (close in low-dim): **small penalty**. The algorithm tolerates "false neighbours" more than it tolerates "torn neighbours".

This asymmetry is what gives t-SNE its character: it prioritises preserving tight local structure above everything else.

---

### Step 5 — Gradient and Optimisation

Taking $\partial \mathcal{L} / \partial y_i$:

$$\frac{\partial \mathcal{L}}{\partial y_i} = 4 \sum_j (p_{ij} - q_{ij})(y_i - y_j)\left(1 + |y_i - y_j|^2\right)^{-1}$$

Interpretation: this is a **force equation**. Each pair $(i,j)$ exerts a force on $y_i$:

- If $p_{ij} > q_{ij}$: they're closer in high-dim than in low-dim → **attractive force**, pulls $y_i$ toward $y_j$.
- If $p_{ij} < q_{ij}$: they're closer in low-dim than in high-dim → **repulsive force**, pushes $y_i$ away from $y_j$.

The $(1 + |y_i - y_j|^2)^{-1}$ term is the Cauchy weight — it amplifies attraction for nearby pairs and tapers repulsion at long range.

Optimisation uses gradient descent with momentum. The algorithm also uses **early exaggeration**: in the first ~250 iterations, multiply all $p_{ij}$ by a factor (e.g. 4). This makes clusters form quickly before the repulsive forces spread them.

**Computational complexity:** Naïve: $O(n^2)$ per iteration. Barnes-Hut t-SNE approximation: $O(n \log n)$.

## ✦ Full Algorithm Summary

```
Input: high-dim data X = {x₁,...,xₙ}
       perplexity Perp, iterations T, learning rate η, momentum α

1. Compute pairwise p_{j|i} via Gaussian with σᵢ found by binary search on Perp
2. Symmetrise: p_{ij} = (p_{j|i} + p_{i|j}) / 2n
3. Initialise Y = {y₁,...,yₙ} ~ 𝒩(0, 10⁻⁴ · I)
4. Early exaggeration: multiply p_{ij} × 4 for first 250 iterations
5. For t = 1 to T:
     a. Compute q_{ij} using Student-t kernel on current Y
     b. Compute gradient ∂ℒ/∂yᵢ for all i
     c. Update: Y(t) = Y(t-1) + η · ∇ℒ + α(t)(Y(t-1) - Y(t-2))
6. Return Y
```

## ✦ A Worked Example

Suppose $n = 4$ points in 2D (toy high-dim space), and we want a 1D embedding.

**Points:** $x_1 = 0,\ x_2 = 1,\ x_3 = 5,\ x_4 = 6$

Distances: $(1,2)$ are close. $(3,4)$ are close. $(1,3)$, etc., are far.

After computing $p_{ij}$, the high similarities will be: $p_{12}, p_{34}$ large; $p_{13}, p_{14}, p_{23}, p_{24}$ small.

**What t-SNE will do:** push $y_1, y_2$ close together; push $y_3, y_4$ close together; push the two clusters apart. The gap between clusters will be _amplified_ relative to the gap in the original data — that's the heavy-tailed $q_{ij}$ at work. In 2D MNIST embeddings, you can see this: the 10 digit classes form well-separated blobs, with the within-cluster distances compressed.

```
High-dim:   ·· · · · ·····
            0 1       5 6

t-SNE 1D:   ··             ····
           y₁y₂           y₃y₄
           (gap amplified by heavy tail)
```

## ✦ What t-SNE Preserves vs. Destroys

|Property|Preserved?|Notes|
|---|---|---|
|Local neighbourhood structure|✅ Yes|The whole point|
|Cluster existence|✅ Yes (qualitatively)|Clusters that exist will be visible|
|**Inter-cluster distances**|❌ **No**|Cannot compare distances between clusters|
|**Cluster sizes**|❌ **No**|Cluster area in plot is meaningless|
|Global geometry|❌ No|Far-apart points are unreliable|
|Out-of-sample points|❌ No (standard)|Must re-run entire algorithm|

> **The most common misuse of t-SNE:** "Cluster A is far from cluster B in the plot, so they are very different." This is **wrong**. The inter-cluster distances in a t-SNE plot encode almost nothing. The _existence_ and _cohesion_ of clusters is meaningful; the _layout_ of clusters relative to each other is not.

## ✦ Hyperparameters and Practical Guidance

### Perplexity

- Range: typically 5–50 (some recent work goes up to 100 for $n > 10^5$)
- Think of it as: how many effective neighbours does each point have?
- **Always run multiple perplexities.** Different values reveal different structure.
- If perplexity $\geq n$: every point is a neighbour of every other — you get a blob.

### Number of iterations

- Minimum ~250 for early exaggeration, ~750+ total. For large data, 1000–5000.
- Watch the KL divergence — it should decrease and plateau.

### Learning rate

- Default: 200 (scikit-learn). If the embedding looks like a ball, increase it. If it looks like many tiny clusters, decrease it.
- Recent recommendation (Belkina et al. 2019): $\eta = n/\text{early_exaggeration}$.

### Random seed

- t-SNE is non-convex. **Always try multiple seeds** before trusting a plot.
- Two runs with different seeds can look qualitatively different.

## ✦ Connection to AI / ML / Deep Learning

### 1. Visualising Neural Network Representations

The canonical use: take a trained network (e.g., a CNN trained on MNIST or ImageNet), extract the penultimate layer activations for the test set, run t-SNE, and plot. If the network has learned good representations, the t-SNE plot will show semantically meaningful clusters.

```
Images → CNN → [h₁,...,h₅₁₂] → t-SNE → 2D plot
                  ↑
           high-dim feature space
           (not pixels — learned features)
```

This was how early deep learning papers _demonstrated_ that CNNs learn semantic features rather than just memorising pixels (Krizhevsky et al., 2012; Maaten & Hinton, 2008).

### 2. Self-Supervised and Contrastive Learning

In **SimCLR** and **MoCo**, the quality of learned representations is routinely assessed via t-SNE of the embedding space. A good contrastive model should push same-class examples together and different-class examples apart — exactly what t-SNE will make visible.

### 3. NLP: Word and Sentence Embeddings

Word2Vec, GloVe, and transformer embeddings are typically 300–768 dimensional. t-SNE collapses these to 2D for inspection. The famous word embedding plots (king − man + woman ≈ queen) are often visualised via PCA first, then t-SNE for finer structure.

### 4. Single-Cell Biology / Genomics (ML application)

One of the most successful ML applications of t-SNE outside CV/NLP: single-cell RNA sequencing (scRNA-seq). Each cell is a ~20,000-dimensional vector (gene expression). t-SNE reveals cell-type clusters that are impossible to see otherwise. This is now standard in computational biology.

### 5. Diagnosing Representation Collapse

In representation learning, **representation collapse** = the encoder maps all inputs to nearly the same embedding. t-SNE immediately reveals this: instead of structured clusters, you see a single blob with no structure.

### 6. Limitations in Deep Learning Pipelines

t-SNE's $O(n^2)$ complexity (or $O(n \log n)$ with Barnes-Hut) makes it impractical for full datasets. In practice: subsample $n \leq 10^4$ points before running t-SNE. UMAP is now often preferred at scale (see [[04.Uniform Manifold Approximation and Projection (UMAP)]]).

## ✦ t-SNE vs. PCA vs. UMAP

|Property|PCA|t-SNE|UMAP|
|---|---|---|---|
|Linear?|Yes|No|No|
|Preserves global structure|Best|Worst|Middle|
|Preserves local structure|Poor|Best|Good|
|Out-of-sample projection|Yes|No (standard)|Yes|
|Complexity|$O(nd^2 + d^3)$|$O(n^2)$ or $O(n \log n)$|$O(n \log n)$|
|Deterministic?|Yes|No (random init)|No (random init)|
|General use|Preprocessing, PCA whitening|Visualisation only|Visualisation + preprocessing|
|Theoretical basis|Linear algebra / variance|KL divergence + t-kernel|Riemannian geometry|

Use PCA first to reduce to ~50 dims, then run t-SNE on the PCA output. This is standard practice (van der Maaten's original paper recommends it) — it removes noise while making t-SNE tractable.

## ✦ Implementation from Scratch (NumPy sketch)

```python
import numpy as np

def pairwise_sq_dists(X):
    """||x_i - x_j||^2 for all pairs."""
    sum_sq = np.sum(X**2, axis=1)
    return sum_sq[:,None] + sum_sq[None,:] - 2 * X @ X.T

def compute_p_matrix(X, perplexity=30.0, tol=1e-5):
    n = X.shape[0]
    D = pairwise_sq_dists(X)
    P = np.zeros((n, n))
    log_perp = np.log(perplexity)

    for i in range(n):
        # Binary search for sigma_i
        beta = 1.0  # beta = 1 / (2 * sigma_i^2)
        lo, hi = -np.inf, np.inf
        Di = np.delete(D[i], i)

        for _ in range(50):  # max 50 binary search steps
            Pi = np.exp(-Di * beta)
            Pi /= Pi.sum()
            H = -np.sum(Pi * np.log(Pi + 1e-10))
            Hdiff = H - log_perp
            if np.abs(Hdiff) < tol:
                break
            if Hdiff > 0:
                lo = beta; beta = (beta + hi)/2 if hi != np.inf else beta*2
            else:
                hi = beta; beta = (beta + lo)/2 if lo != -np.inf else beta/2

        Pi_full = np.zeros(n)
        Pi_full[np.arange(n) != i] = Pi
        P[i] = Pi_full

    # Symmetrise
    P = (P + P.T) / (2 * n)
    P = np.maximum(P, 1e-12)
    return P

def tsne(X, n_iter=1000, perplexity=30, lr=200, early_exag=4):
    n = X.shape[0]
    P = compute_p_matrix(X, perplexity)
    P *= early_exag  # early exaggeration

    # Random initialisation
    Y = np.random.randn(n, 2) * 1e-4
    dY = np.zeros_like(Y)
    iY = np.zeros_like(Y)
    gains = np.ones_like(Y)

    for t in range(n_iter):
        if t == 250:
            P /= early_exag  # turn off early exaggeration

        # Compute Q
        D_low = pairwise_sq_dists(Y)
        Q_num = (1 + D_low) ** -1
        np.fill_diagonal(Q_num, 0)
        Q = Q_num / Q_num.sum()
        Q = np.maximum(Q, 1e-12)

        # Gradient
        PQ = P - Q
        grad = np.zeros_like(Y)
        for i in range(n):
            diff = Y[i] - Y  # (n, 2)
            grad[i] = 4 * np.sum((PQ[:,i] * Q_num[:,i])[:,None] * diff, axis=0)

        # Update with momentum
        momentum = 0.5 if t < 250 else 0.8
        gains = (gains + 0.2) * ((grad > 0) != (iY > 0)) + \
                (gains * 0.8) * ((grad > 0) == (iY > 0))
        gains = np.maximum(gains, 0.01)
        iY = momentum * iY - lr * gains * grad
        Y += iY
        Y -= Y.mean(axis=0)

        if t % 100 == 0:
            kl = np.sum(P * np.log(P / Q))
            print(f"Iter {t}: KL = {kl:.4f}")

    return Y
```

> This is for understanding. In practice, use `sklearn.manifold.TSNE` or `openTSNE` (which supports out-of-sample and parallelism).

## ✦ Common Failure Modes and Diagnostics

**Problem: All points collapse to one blob**  
→ Learning rate too low, or too few iterations. Increase `lr` and check KL divergence is decreasing.

**Problem: Many tiny isolated clusters with no structure**  
→ Learning rate too high, or perplexity too low. Reduce `lr` or increase perplexity.

**Problem: Different runs give completely different plots**  
→ This is expected for t-SNE. Average multiple runs or switch to UMAP for reproducibility.

**Problem: "I can see 5 clear clusters, but the true number is 3"**  
→ t-SNE can split true clusters. Each cluster's internal structure can itself be projected as sub-clusters. Never use t-SNE for counting clusters — use silhouette analysis on the original space.
