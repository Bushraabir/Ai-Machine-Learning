
> _A Bayesian network is a marriage of graph theory and probability theory. The graph encodes what is independent of what; the probability tables encode how much. Together they give you a compact, interpretable representation of a joint distribution over dozens or hundreds of variables — something a full joint table could never provide._

## ✦ The Problem Bayesian Networks Solve

Suppose you have $d$ binary random variables $X_1, \ldots, X_d$. A full joint distribution requires specifying $2^d - 1$ numbers. For $d = 30$: over a billion parameters. Storing, estimating, and querying this table is completely intractable.

Bayesian networks exploit **conditional independence** structure to represent the same joint distribution far more compactly. If variable $X_i$ only directly depends on a small set of "parent" variables $\text{Pa}(X_i)$, then specifying $P(X_i \mid \text{Pa}(X_i))$ for each $i$ suffices to reconstruct the full joint — and each conditional table is tiny.

The key insight: **most variables in the world do not directly cause most other variables.** Causal structure is sparse. A Bayesian network makes this sparsity explicit.

---

## ✦ Formal Definition

A **Bayesian network** (also: Bayes net, belief network, directed graphical model) is a pair $(\mathcal{G}, \mathcal{P})$ where:

- $\mathcal{G} = (V, E)$ is a **directed acyclic graph (DAG)** with one node per random variable
- $\mathcal{P} = {P(X_i \mid \text{Pa}_\mathcal{G}(X_i))}_{i=1}^d$ is a set of **conditional probability distributions** (CPDs), one per node

The joint distribution **factorises** as:

$$\boxed{P(X_1, X_2, \ldots, X_d) = \prod_{i=1}^d P(X_i \mid \text{Pa}_\mathcal{G}(X_i))}$$

This is the **chain rule of Bayesian networks** — it follows from the chain rule of probability plus the conditional independence structure encoded in $\mathcal{G}$.

**Notation:** $\text{Pa}_\mathcal{G}(X_i)$ denotes the set of direct parents of $X_i$ in $\mathcal{G}$. If $X_i$ has no parents, $P(X_i \mid \text{Pa}(X_i)) = P(X_i)$, a marginal.

### Why Must $\mathcal{G}$ Be Acyclic?

A cycle $X_i \to X_j \to X_i$ would require $X_i$ to cause $X_j$ and $X_j$ to simultaneously cause $X_i$. This makes the factorisation self-referential and undefined — there is no valid topological ordering in which to apply the chain rule. Acyclicity is not a modelling convenience; it is a logical requirement for the factorisation to define a valid distribution.

## ✦ A Canonical Example: The Student Network

A classic example (Koller & Friedman). Five variables:

- $D$: Difficulty of exam (easy/hard)
- $I$: Intelligence of student (low/high)
- $G$: Grade received (A/B/C)
- $S$: SAT score (high/low)
- $L$: Reference letter quality (strong/weak)

```
    D ──┐     I ──┐
        ▼         ▼
        G         S
        │
        ▼
        L
```

The factorisation:

$$P(D, I, G, S, L) = P(D), P(I), P(G \mid D, I), P(S \mid I), P(L \mid G)$$

**Parameter count:** $D$: 1, $I$: 1, $G \mid D,I$: $2 \times 2 \times 3 - 1 = 11$, $S \mid I$: 2, $L \mid G$: 6. Total: **21 parameters**. The full joint has $2 \times 2 \times 3 \times 2 \times 2 - 1 = 47$ parameters. The network is more than twice as compact — and more interpretable.

For a network with $d$ nodes each with at most $k$ parents over binary variables: at most $d \cdot 2^k$ parameters, vs. $2^d$ for the full joint. Exponential compression.

## ✦ Conditional Independence and D-Separation

### Local Markov Property

The factorisation implies a fundamental independence statement: every variable is **conditionally independent of its non-descendants given its parents**:

$$X_i \perp !!! \perp \text{NonDesc}(X_i) \mid \text{Pa}(X_i)$$

This is the **local Markov property**. It is equivalent to the factorisation above.

### D-Separation

But the Bayesian network encodes more independences than just the local ones. The full set is read off the graph via **d-separation** (directional separation).

**Definition:** A path $\pi$ between nodes $X$ and $Y$ is **blocked** by a set $Z$ if there exists a node $V$ on $\pi$ such that one of:

1. **Chain:** $A \to V \to B$ and $V \in Z$ — conditioning on $V$ blocks the flow of information along the chain.
2. **Fork:** $A \leftarrow V \to B$ and $V \in Z$ — conditioning on the common cause $V$ blocks the spurious correlation it induces.
3. **Collider:** $A \to V \leftarrow B$ and $V \notin Z$ and no descendant of $V$ is in $Z$ — the collider is **not** conditioned on, so it blocks by default.

$X$ and $Y$ are **d-separated** by $Z$ (written $X \perp_\mathcal{G} Y \mid Z$) if every path between $X$ and $Y$ is blocked by $Z$.

**The Soundness Theorem:** If $X \perp_\mathcal{G} Y \mid Z$, then $X \perp!!!\perp Y \mid Z$ in every distribution $P$ consistent with $\mathcal{G}$.

**The Faithfulness/Completeness Theorem:** Conversely (under faithfulness), if $X \not\perp_\mathcal{G} Y \mid Z$, then there exists a distribution $P$ consistent with $\mathcal{G}$ in which $X \not\perp!!!\perp Y \mid Z$. The graph captures all and only the structural independences.

### The Three Fundamental Motifs

These three patterns are the atoms of d-separation. Every complex network decomposes into combinations of them.

#### 1. Chain (Serial Connection)

```
A ──► B ──► C
```

$P(A, B, C) = P(A), P(B \mid A), P(C \mid B)$

$$A \perp!!!\perp C \mid B, \qquad A \not\perp!!!\perp C$$

Intuition: $B$ is the mediator. Conditioning on $B$ blocks the information flow from $A$ to $C$. Example: Season → Sprinkler → Wet grass. Knowing the sprinkler state, the season tells you nothing about the grass.

#### 2. Fork (Common Cause)

```
A ◄── B ──► C
```

$P(A, B, C) = P(B), P(A \mid B), P(C \mid B)$

$$A \perp!!!\perp C \mid B, \qquad A \not\perp!!!\perp C$$

Intuition: $B$ is a common cause that induces a spurious correlation between $A$ and $C$. Conditioning on $B$ removes the spurious dependence. Example: Season → Rain, Season → Sprinkler. Without knowing the season, rain and sprinkler are correlated. Knowing the season, they are independent.

#### 3. Collider (Common Effect / v-Structure)

```
A ──► C ◄── B
```

$P(A, B, C) = P(A), P(B), P(C \mid A, B)$

$$A \perp!!!\perp B, \qquad A \not\perp!!!\perp B \mid C$$

Intuition: The collider $C$ is **blocked by default** — $A$ and $B$ are marginally independent. But conditioning on $C$ (or any descendant of $C$) **opens** the path — it creates a dependence. This is **Berkson's paradox** / **explaining away**.

> **Explaining away:** Suppose $C$ = "car won't start", $A$ = "battery dead", $B$ = "out of fuel". Marginally, battery state and fuel state are independent. But given that the car won't start, learning that the battery is fine makes "out of fuel" much more likely — the two causes compete to explain $C$.

```
Collider example — selection bias:

A = Talent       B = Luck
        \         /
         ▼       ▼
          C = Famous

P(Talent ⊥ Luck) = True     (in the general population)
P(Talent ⊥ Luck | Famous) = False  (among famous people, high luck → lower talent, and vice versa)
```

This is one of the most common sources of spurious correlations in observational studies. Any time you condition on a collider — including by **selecting** a sample based on a collider — you induce spurious dependence between its parents.

## ✦ Worked D-Separation Examples

Using the Student network: $D \to G \leftarrow I \to S$, $G \to L$.

**Query 1:** Is $D \perp!!!\perp I$?  
Path: $D \to G \leftarrow I$. This path has a collider at $G$. $G \notin \emptyset$ and no descendant of $G$ is in $\emptyset$. **Blocked.** No other paths. $\Rightarrow D \perp!!!\perp I$. ✓ (Difficulty and intelligence are marginally independent — sensible.)

**Query 2:** Is $D \perp!!!\perp I \mid G$?  
Path: $D \to G \leftarrow I$. Collider at $G$; now $G \in {G}$. **Not blocked.** $\Rightarrow D \not\perp!!!\perp I \mid G$. ✓ (Given a student's grade, difficulty and intelligence become dependent — knowing the exam was hard makes low grade less informative about intelligence.)

**Query 3:** Is $D \perp!!!\perp S \mid G$?  
Paths from $D$ to $S$: only $D \to G \leftarrow I \to S$.

- At $G$: collider, $G \in {G}$ → **open**.
- At $I$: chain, $I \notin {G}$ → **open**.  
    Path is not blocked. $\Rightarrow D \not\perp!!!\perp S \mid G$.

**Query 4:** Is $D \perp!!!\perp S \mid G, I$?  
Path: $D \to G \leftarrow I \to S$.

- At $G$: collider, $G \in {G, I}$ → open.
- At $I$: chain, $I \in {G, I}$ → **blocked**.  
    All paths blocked. $\Rightarrow D \perp!!!\perp S \mid G, I$. ✓ (Given both grade and intelligence, difficulty tells you nothing about SAT score.)

## ✦ Inference

Given a Bayesian network and some **observed evidence** $\mathbf{e}$ (values of a subset of variables), compute the **posterior** over query variables $\mathbf{Q}$:

$$P(\mathbf{Q} \mid \mathbf{e}) = \frac{P(\mathbf{Q}, \mathbf{e})}{P(\mathbf{e})} = \frac{\sum_{\mathbf{h}} P(\mathbf{Q}, \mathbf{e}, \mathbf{h})}{\sum_{\mathbf{q}, \mathbf{h}} P(\mathbf{q}, \mathbf{e}, \mathbf{h})}$$

where $\mathbf{h}$ are the hidden (unobserved) variables. Naïve computation sums over all configurations of $\mathbf{h}$ — exponential in the number of hidden variables.

### Exact Inference

#### Variable Elimination

The key insight: the factorisation of the joint distribution means many sum-product computations can be reordered and cached.

**Factor:** A factor $\phi(X_{i_1}, \ldots, X_{i_k})$ is a non-negative function over a subset of variables. The joint distribution is a product of factors.

**Variable elimination** sums out one variable at a time in a chosen order, multiplying and marginalising factors:

$$P(Q \mid \mathbf{e}) \propto \sum_{X_1} \sum_{X_2} \cdots \prod_i \phi_i(\text{scope}_i)$$

The order of summation determines the computational cost. Finding the optimal elimination order is NP-hard in general, but good heuristics exist (min-fill, min-degree).

**Example** (using Student network, query $P(L \mid D = d^0)$):

$$P(L \mid d^0) \propto \sum_G P(L \mid G) \sum_I P(G \mid d^0, I), P(I) \sum_S P(S \mid I)$$

The inner sum $\sum_S P(S \mid I) = 1$ (marginalising a complete CPD). This cancels $S$ with zero cost. The remaining sums are over $I$ and $G$ — small tables.

**Complexity:** Exponential in the **treewidth** of the graph — a measure of how "tree-like" the graph is. For trees and polytrees, exact inference is polynomial. For general graphs, it is #P-hard.

#### Belief Propagation (Sum-Product Algorithm)

For **trees** (no undirected cycles), exact inference runs in $O(n \cdot k^2)$ time via message passing.

Each node sends messages to its neighbours. The message $\mu_{X \to Y}(y)$ represents a summary of what $X$ knows about $Y$, based on $X$'s other neighbours:

$$\mu_{X \to Y}(y) = \sum_x \phi(x, y) \prod_{Z \in \text{Ne}(X) \setminus Y} \mu_{Z \to X}(x)$$

The marginal at each node is proportional to the product of all incoming messages:

$$P(X \mid \mathbf{e}) \propto \prod_{Y \in \text{Ne}(X)} \mu_{Y \to X}(x)$$

For directed trees, this decomposes into two passes: a forward (upward, leaves-to-root) pass and a backward (downward, root-to-leaves) pass.

**Loopy belief propagation (LBP):** Run the same message-passing algorithm on a graph with cycles. The algorithm is no longer exact — messages depend on each other cyclically — but it often converges and gives good approximations. LBP underlies modern **LDPC error-correcting codes** and is used in computer vision (stereo matching, image segmentation).

### Approximate Inference

For graphs with high treewidth, exact inference is intractable. The two main approaches:

#### Markov Chain Monte Carlo (MCMC)

Construct a Markov chain whose stationary distribution is $P(\mathbf{X} \mid \mathbf{e})$. Sample from it.

**Gibbs sampling** (most common for Bayesian networks): repeatedly sample each variable from its **Markov blanket** — its parents, children, and children's other parents:

$$X_i^{(t+1)} \sim P(X_i \mid \text{Pa}(X_i), \text{Ch}(X_i), \text{Pa}(\text{Ch}(X_i)))$$

The Markov blanket is exactly the minimal set you need to condition on to make $X_i$ independent of all other variables — a direct consequence of d-separation.

Convergence: guaranteed to the correct distribution asymptotically. Mixing time (how many samples you need) depends on the strength of correlations.

#### Variational Inference

Approximate $P(\mathbf{X} \mid \mathbf{e})$ with a simpler distribution $Q(\mathbf{X}; \lambda)$ by minimising $D_{KL}(Q | P)$. The mean-field approximation assumes $Q$ factorises fully: $Q(\mathbf{X}) = \prod_i Q_i(X_i)$. Leads to a coordinate ascent algorithm — the ELBO you saw in [[05.Autoencoders|VAEs]] is exactly this.

## ✦ Learning Bayesian Networks

Two sub-problems:

### Parameter Learning (Known Structure)

Given a fixed graph $\mathcal{G}$ and i.i.d. data $\mathcal{D} = {x^{(1)}, \ldots, x^{(n)}}$, estimate the CPDs.

With **complete data** (no missing values): MLE decomposes over CPDs. For a discrete variable $X_i$ with parents $\text{Pa}(X_i)$:

$$\hat{P}(X_i = k \mid \text{Pa}(X_i) = \pi) = \frac{#{X_i = k, \text{Pa}(X_i) = \pi}}{#{\text{Pa}(X_i) = \pi}}$$

Just count — each CPD is independent and estimated by its frequency table. MLE is the empirical conditional distribution.

**With missing data:** The log-likelihood no longer decomposes. Must use EM (Expectation-Maximisation):

- **E-step:** Compute expected sufficient statistics using current parameters and BP for inference.
- **M-step:** Update each CPD using the expected counts from the E-step.

**Bayesian parameter estimation:** Place a Dirichlet prior over each CPD (conjugate to the Categorical). Posterior is Dirichlet with counts augmented by pseudo-counts. MAP estimate = $\frac{\text{count} + \alpha - 1}{\text{total count} + K(\alpha-1)}$. This is Laplace smoothing for discrete CPDs.

### Structure Learning (Unknown Structure)

Find the graph $\mathcal{G}$ that best explains the data. Two approaches:

**Score-based:** Search over DAG space, score each candidate graph. Common scores:

- **BIC:** $\text{score}(G, \mathcal{D}) = \log P(\mathcal{D} \mid \hat{\theta}, G) - \frac{\log n}{2} |\theta_G|$ — penalises complexity.
- **BDe score:** Bayesian Dirichlet equivalent — integrates out parameters, proper Bayesian score.

DAG space is super-exponential in $d$ — exact search is impossible. Heuristics: greedy hill-climbing (add/remove/reverse one edge at a time), simulated annealing, genetic algorithms.

**Constraint-based:** Use conditional independence tests ($\chi^2$ tests, mutual information tests) to determine which edges cannot be present. The **PC algorithm** (Peter-Clark) works by:

1. Start with a complete undirected graph.
2. Remove edge $(X, Y)$ if $\exists, Z$ such that $X \perp!!!\perp Y \mid Z$ in the data.
3. Orient remaining edges using v-structure identification.

The PC algorithm recovers the correct skeleton and all v-structures consistently in the large-sample limit, under faithfulness. Output is a **CPDAG** (completed partially directed acyclic graph) — the equivalence class of all DAGs with the same skeleton and v-structures.

---

## ✦ Markov Equivalence

Two DAGs are **Markov equivalent** if they encode the same set of conditional independences (same d-separation statements). Markov equivalent DAGs have the same **skeleton** (same undirected edges) and the same **v-structures** (colliders).

**Implication:** From observational data alone, you cannot distinguish between Markov equivalent DAGs. To identify causal direction within an equivalence class, you need either:

- Interventional data (do-calculus, randomised experiments)
- Additional assumptions (e.g., non-Gaussianity, as in LiNGAM)
- Temporal order constraints

This is why correlation does not imply causation — many causal structures are observationally equivalent.

## ✦ Connection to AI / ML / Deep Learning

### 1. Naive Bayes as a Bayesian Network

The Naive Bayes classifier (see [[06.Naive Bayes]]) is a Bayesian network with a specific structure: one class node $Y$ pointing to all feature nodes $X_1, \ldots, X_d$ with no edges among the $X_i$. The conditional independence assumption $X_i \perp!!!\perp X_j \mid Y$ is exactly the d-separation statement in this graph. Every Naive Bayes model is a Bayesian network; not every Bayesian network is Naive Bayes.

```
Naive Bayes BN:

        Y
      / | \
     ▼  ▼  ▼
    X₁ X₂ X₃   (no edges between Xᵢ)
```

### 2. Hidden Markov Models

An HMM (see [[02.Markov Models]]) is a Bayesian network on a chain:

```
Z₁ ──► Z₂ ──► Z₃ ──► Z₄    (hidden states)
│      │      │      │
▼      ▼      ▼      ▼
X₁     X₂     X₃     X₄    (observations)
```

The Viterbi algorithm is variable elimination on this chain. The forward-backward algorithm is belief propagation on this tree. Everything in sequential modelling — speech recognition, POS tagging, protein folding — is a Bayesian network with temporal structure.

### 3. Variational Autoencoders

The VAE's generative model $z \to x$ is a two-node Bayesian network:

$$P(z), P_\theta(x \mid z) = P(x, z)$$

The inference network $q_\phi(z \mid x)$ approximates the posterior $P(z \mid x)$ — this is variational inference on a Bayesian network. The ELBO is exactly the variational lower bound used in general Bayesian network inference. The VAE is deep learning's answer to intractable Bayesian network inference.

### 4. Causal Inference and Do-Calculus

Judea Pearl's **do-calculus** (Pearl, 2000) extends Bayesian networks to reason about interventions. The **do-operator** $do(X = x)$ removes all incoming edges to $X$ in $\mathcal{G}$, fixing its value. The interventional distribution $P(Y \mid do(X = x))$ is generally different from the observational conditional $P(Y \mid X = x)$ — this is the formal statement of why correlation $\neq$ causation.

Modern causal ML (IRM, causal representation learning) builds on this foundation to train models that generalise across environments — a central problem in robust ML.

### 5. Probabilistic Programming

PyMC, Stan, Pyro, and NumPyro all represent probabilistic models as Bayesian networks (or their continuous generalisations). Inference is automated — the user specifies structure and CPDs; the system runs MCMC or variational inference. The structure-inference separation in Bayesian networks is what makes this abstraction possible.

### 6. Graph Neural Networks

A Bayesian network is a graph with a probability model. GNNs learn representations over graphs — they can be applied to Bayesian networks directly, learning to do amortised inference (predict posteriors for unseen evidence configurations without re-running BP). This is an active research area.

## ✦ Implementation Sketch

```python
import numpy as np
from itertools import product

# --- Exact inference via Variable Elimination (discrete, small BN) ---

class Factor:
    """A factor over a subset of discrete variables."""
    def __init__(self, variables, cardinalities, values):
        self.variables = list(variables)
        self.cards = {v: c for v, c in zip(variables, cardinalities)}
        self.values = np.array(values).reshape(cardinalities)

    def marginalise(self, var):
        """Sum out a variable."""
        axis = self.variables.index(var)
        new_vals = self.values.sum(axis=axis)
        new_vars = [v for v in self.variables if v != var]
        new_cards = [self.cards[v] for v in new_vars]
        return Factor(new_vars, new_cards, new_vals)

    def product(self, other):
        """Multiply two factors, aligning shared variables."""
        all_vars = self.variables + [v for v in other.variables
                                     if v not in self.variables]
        all_cards = [self.cards.get(v, other.cards[v]) for v in all_vars]
        new_vals = np.zeros(all_cards)
        for idx in product(*[range(c) for c in all_cards]):
            assign = dict(zip(all_vars, idx))
            i1 = tuple(assign[v] for v in self.variables)
            i2 = tuple(assign[v] for v in other.variables)
            new_vals[idx] = self.values[i1] * other.values[i2]
        return Factor(all_vars, all_cards, new_vals)

    def condition(self, var, val):
        """Condition on var = val (set all other slices to 0 and fix axis)."""
        axis = self.variables.index(var)
        slices = [slice(None)] * len(self.variables)
        slices[axis] = val
        new_vals = self.values[tuple(slices)]
        new_vars = [v for v in self.variables if v != var]
        new_cards = [self.cards[v] for v in new_vars]
        return Factor(new_vars, new_cards, new_vals)

    def normalise(self):
        self.values = self.values / self.values.sum()
        return self

# --- Student network CPDs ---
# Variables: D (0=easy,1=hard), I (0=low,1=high),
#            G (0=A,1=B,2=C), S (0=low,1=high), L (0=weak,1=strong)

P_D = Factor(['D'], [2], [0.6, 0.4])
P_I = Factor(['I'], [2], [0.7, 0.3])

# P(G | D, I): shape (2, 2, 3) — D, I, G
P_G_DI = Factor(['D','I','G'], [2,2,3],
    [[[0.30, 0.40, 0.30],   # D=easy, I=low
      [0.05, 0.25, 0.70]],  # D=easy, I=high  (note: high I → worse grade is rare)
     [[0.90, 0.08, 0.02],   # D=hard, I=low
      [0.50, 0.30, 0.20]]]) # D=hard, I=high

P_S_I = Factor(['I','S'], [2,2], [[0.95, 0.05], [0.20, 0.80]])
P_L_G = Factor(['G','L'], [3,2], [[0.10, 0.90], [0.40, 0.60], [0.99, 0.01]])

# Query: P(L | D=hard) via variable elimination
# Order: S, I, G (eliminate hidden vars one at a time)
evidence = {'D': 1}  # D = hard

# Condition on evidence
f_D = P_D.condition('D', 1)
f_G = P_G_DI.condition('D', 1)  # Factor(['I','G'], ...)

# Eliminate S: marginalise out S from P_S_I
f_SI = P_S_I  # no evidence on S
f_S_marginalised = f_SI.marginalise('S')  # = Factor(['I'], ...) = [1, 1] (trivial)

# Eliminate I: multiply P(I), P(G|I, D=hard), P(S|I) and sum over I
f_combined = P_I.product(f_G).product(f_S_marginalised)
f_I_out = f_combined.marginalise('I')  # Factor over G

# Eliminate G: multiply P(L|G) and sum over G
f_LG = P_L_G.product(f_I_out)
f_L = f_LG.marginalise('G')
f_L.normalise()
print("P(L | D=hard):", f_L.values)
# → P(L=weak | D=hard), P(L=strong | D=hard)
```
