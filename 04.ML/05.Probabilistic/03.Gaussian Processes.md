
> _"A Gaussian process is a probability distribution over functions, such that any finite collection of function values has a joint Gaussian distribution."_ — Rasmussen & Williams, 2006

---

## ✦ Why This Note Exists

Every model we've built so far has been **parametric**: fix a functional form, learn a finite-dimensional parameter vector $\theta$. A Gaussian process takes a different path entirely — it places a prior directly over the **space of functions**, then conditions on data to obtain a posterior over functions.

This sounds abstract. It is not. The payoff is a model that:

- makes predictions with **calibrated uncertainty** — the posterior variance tells you where it doesn't know
- incorporates **prior knowledge about function structure** through kernel design
- updates **exactly** via Bayes' theorem, no MCMC needed (for regression)
- subsumes kernel ridge regression, spline smoothing, and RBF-kernel SVMs as special cases

GPs sit at the intersection of Bayesian inference, kernel methods, and functional analysis. Understanding them sharpens your intuition for everything that comes after — neural network weight-space vs. function-space perspectives, neural tangent kernels, and Bayesian deep learning.

## ✦ Part 1 — From Parametric to Non-Parametric

### 1.1 The Parametric Bottleneck

In Bayesian linear regression, we place a prior on the **weight vector** $w \in \mathbb{R}^d$: $$ w \sim \mathcal{N}(0, \alpha^{-1} I), \quad y = w^T \phi(x) + \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, \sigma^2) $$

This is expressive — but the function class is fixed by the choice of basis $\phi(x)$. If you pick $d = 10$ basis functions, you've committed to a 10-dimensional function space. You can extend $\phi$ (polynomial, RBF, Fourier features...) but you're always choosing in advance.

**The weight-space view:** place a prior on $w$, integrate out $w$ to get a distribution over function values.

**The function-space view:** place the prior directly on $f : \mathcal{X} \to \mathbb{R}$.

The key insight is that these are equivalent when the prior on $w$ is Gaussian — and the function-space view generalises beyond finite-dimensional $w$.

### 1.2 What Does a Prior Over Functions Mean?

A prior over functions seems infinite-dimensional and unwieldy. The Gaussian process sidesteps this through a consistency condition:

**Definition (Gaussian Process):** A Gaussian process is a collection of random variables ${f(x) : x \in \mathcal{X}}$ such that for any finite set of inputs ${x_1, \ldots, x_n} \subset \mathcal{X}$, the joint distribution: $$ \begin{pmatrix} f(x_1) \ \vdots \ f(x_n) \end{pmatrix} \sim \mathcal{N}(\mathbf{m}, K) $$ where $m_i = m(x_i)$ and $K_{ij} = k(x_i, x_j)$.

We write: $f \sim \mathcal{GP}(m(x),, k(x, x'))$

The GP is completely specified by:

- **Mean function** $m(x) = \mathbb{E}[f(x)]$ — often set to zero a priori
- **Covariance function** (kernel) $k(x, x') = \text{Cov}(f(x), f(x'))$

> **The key move:** you never have to work with an infinite-dimensional object. Any computation with a GP reduces to a finite-dimensional multivariate Gaussian computation on the inputs you actually care about. The Kolmogorov extension theorem guarantees that a consistent family of finite-dimensional Gaussians uniquely defines a valid stochastic process.

## ✦ Part 2 — The Kernel: Encoding Prior Beliefs

### 2.1 The Kernel as a Similarity Measure

The kernel $k(x, x')$ encodes our prior belief about how correlated $f(x)$ and $f(x')$ are. If $k(x, x')$ is large, we believe $f(x)$ and $f(x')$ should take similar values.

This is precisely the kernel trick from SVMs — but now it has a direct probabilistic interpretation: $k$ is the prior covariance between function values.

**Validity condition (Mercer's theorem):** $k$ is a valid kernel iff the Gram matrix $K$ with $K_{ij} = k(x_i, x_j)$ is positive semi-definite for all finite input sets. Equivalently, $k(x, x') = \langle \phi(x), \phi(x') \rangle_\mathcal{H}$ for some (possibly infinite-dimensional) feature map $\phi$ into a Hilbert space $\mathcal{H}$.

### 2.2 Standard Kernels

Each kernel encodes a different structural assumption about $f$:

#### Squared Exponential (RBF / Gaussian) Kernel

$$ k_{\text{SE}}(x, x') = \sigma_f^2 \exp!\left(-\frac{|x - x'|^2}{2\ell^2}\right) $$

- **Hyperparameters:** $\sigma_f^2$ (output scale), $\ell$ (length-scale)
- **Encodes:** infinitely differentiable, very smooth functions
- **Length-scale $\ell$:** controls how quickly the function varies. Small $\ell$ → wiggly functions; large $\ell$ → smooth, slowly varying functions
- **Feature map:** corresponds to an infinite-dimensional feature space (Fourier features expansion)
- **Caution:** too smooth for many real phenomena — geological, financial, biological data often has rougher structure

#### Matérn Kernel

$$ k_{\nu}(x, x') = \frac{2^{1-\nu}}{\Gamma(\nu)} \left(\frac{\sqrt{2\nu} |x-x'|}{\ell}\right)^\nu K_\nu!\left(\frac{\sqrt{2\nu}|x-x'|}{\ell}\right) $$ where $K_\nu$ is a modified Bessel function.

- **Parameter $\nu$:** controls differentiability. $\nu = 1/2$ → once-differentiable (Ornstein-Uhlenbeck); $\nu = 3/2$ → twice; $\nu = 5/2$ → three times; $\nu \to \infty$ → SE kernel
- **Why it matters:** $\nu = 5/2$ is the default recommendation in practice — smoother than OU, less pathologically smooth than SE
- **Special cases:**
    - $\nu = 1/2$: $k(r) = \sigma^2 e^{-r/\ell}$ — exponential kernel, continuous but not differentiable
    - $\nu = 3/2$: $k(r) = \sigma^2(1 + \frac{\sqrt{3}r}{\ell})\exp(-\frac{\sqrt{3}r}{\ell})$

#### Periodic Kernel

$$ k_{\text{per}}(x, x') = \sigma_f^2 \exp!\left(-\frac{2\sin^2(\pi|x-x'|/p)}{\ell^2}\right) $$ Encodes exactly periodic functions with period $p$. Combine with SE kernel (multiply) to get functions that are approximately periodic with decaying coherence — standard for seasonal time series.

#### Linear Kernel

$$ k_{\text{lin}}(x, x') = \sigma_b^2 + \sigma_v^2 (x - c)(x' - c) $$ GP with this kernel is equivalent to Bayesian linear regression through the point $c$. Useful as a component in additive kernel decompositions.

#### Rational Quadratic Kernel

$$ k_{\text{RQ}}(x, x') = \sigma_f^2\left(1 + \frac{|x-x'|^2}{2\alpha\ell^2}\right)^{-\alpha} $$ A scale mixture of SE kernels with different length-scales. As $\alpha \to \infty$, recovers SE. Useful when the signal varies at multiple scales.

### 2.3 Kernel Engineering

Kernels compose:

- **Sum:** $k = k_1 + k_2$ → function is a sum of two independent GPs (additive structure)
- **Product:** $k = k_1 \cdot k_2$ → modulation; e.g., periodic $\times$ SE = locally periodic
- **Composition:** $k_{\text{SE}}(g(x), g(x'))$ for learned feature maps $g$ → deep kernels, connection to deep learning

> **Notebook margin — the kernel as a scientist's voice:** When you choose a kernel, you are making a scientific claim about the structure of $f$. SE says "I believe $f$ is very smooth." Matérn-5/2 says "I believe $f$ is three-times differentiable but not more." Periodic says "I believe there is a repeating structure with period $p$." The posterior will update these beliefs given data — but the kernel determines what it's even possible to learn.

## ✦ Part 3 — GP Regression (Exact Inference)

### 3.1 The Setup

**Observed data:** $\mathcal{D} = {(x_i, y_i)}_{i=1}^n$ where $y_i = f(x_i) + \varepsilon_i$, $\varepsilon_i \overset{\text{i.i.d.}}{\sim} \mathcal{N}(0, \sigma_n^2)$

**Prior:** $f \sim \mathcal{GP}(0, k(x, x'))$ (zero mean WLOG)

**Goal:** compute the posterior $p(f \mid \mathcal{D})$ and the predictive distribution $p(f_* \mid x__, \mathcal{D})$ at new test points $x__$.

### 3.2 The Joint Prior

Let $\mathbf{f} = (f(x_1), \ldots, f(x_n))^T$ and $f_* = f(x__)$. By the GP definition: $$ \begin{pmatrix} \mathbf{y} \ f__ \end{pmatrix} \sim \mathcal{N}!\left(\mathbf{0},, \begin{pmatrix} K(X, X) + \sigma_n^2 I & K(X, x__) \ K(x__, X) & k(x__, x__) \end{pmatrix}\right) $$

where:

- $K(X, X) \in \mathbb{R}^{n \times n}$: Gram matrix, $K_{ij} = k(x_i, x_j)$
- $K(X, x__) \in \mathbb{R}^{n \times 1}$: covariance between training inputs and test point, $[K(X, x__)]_i = k(x_i, x_*)$
- $k(x__, x__)$: prior variance at test point

### 3.3 The Posterior Predictive Distribution

Using the **Gaussian conditioning formula** (the fundamental operation of all Gaussian inference):

> **Lemma (Conditional Gaussian):** If $\begin{pmatrix} \mathbf{a} \ \mathbf{b} \end{pmatrix} \sim \mathcal{N}!\left(\begin{pmatrix} \mu_a \ \mu_b \end{pmatrix}, \begin{pmatrix} \Sigma_{aa} & \Sigma_{ab} \ \Sigma_{ba} & \Sigma_{bb} \end{pmatrix}\right)$ then: $$\mathbf{a} \mid \mathbf{b} \sim \mathcal{N}!\left(\mu_a + \Sigma_{ab}\Sigma_{bb}^{-1}(\mathbf{b} - \mu_b),\ \Sigma_{aa} - \Sigma_{ab}\Sigma_{bb}^{-1}\Sigma_{ba}\right)$$

Applying this with $\mathbf{a} = f_*$, $\mathbf{b} = \mathbf{y}$:

$$ \boxed{f_* \mid x__, X, \mathbf{y} \sim \mathcal{N}(\mu__, \sigma_*^2)} $$

$$ \mu_* = K(x_*, X),[K(X,X) + \sigma_n^2 I]^{-1},\mathbf{y} $$

$$ \sigma__^2 = k(x__, x__) - K(x__, X),[K(X,X) + \sigma_n^2 I]^{-1},K(X, x_*) $$

This is the complete solution. No approximation. Exact Bayesian inference in closed form.

### 3.4 Interpretation — What These Equations Say

**The mean $\mu_*$:** $$ \mu_* = K(x_*, X) \underbrace{[K(X,X) + \sigma_n^2 I]^{-1} \mathbf{y}}_{\alpha} $$

This is a **linear combination of training outputs**, weighted by the covariance between $x__$ and each training input. Nearby training points (large $k(x__, x_i)$) contribute more. This is kernel regression — the GP mean is identical to kernel ridge regression with $\lambda = \sigma_n^2 / \sigma_f^2$.

**The variance $\sigma_*^2$:** $$ \sigma__^2 = \underbrace{k(x__, x__)}_{\text{prior variance}} - \underbrace{K(x__, X)[K + \sigma_n^2 I]^{-1}K(X, x_*)}_{\text{variance reduction from data}} $$

The posterior variance is the **prior variance minus what we learned from the data**. The second term is always non-negative (positive semi-definiteness of $K$), so data never increases uncertainty. Crucially:

- At a training input $x_i$: $\sigma_*^2 \to 0$ as $\sigma_n^2 \to 0$ (exact fit)
- Far from all training data: $\sigma__^2 \approx k(x__, x_*)$ (reverts to prior)

> **Worked example — 1D regression:**  
> Prior: $f \sim \mathcal{GP}(0, k_{\text{SE}})$ with $\ell = 1$, $\sigma_f^2 = 1$, $\sigma_n^2 = 0.1$  
> Training data: 5 points sampled from $\sin(x) + \mathcal{N}(0, 0.1)$
> 
> The posterior mean threads through the data. The posterior variance (shaded region) is:
> 
> - narrow near training points (we've observed $f$ there)
> - wide in regions with no data (we genuinely don't know)
> - wider than the noise level even at training points (because we know $y = f + \varepsilon$, not $f$ directly)
> 
> This is **calibrated uncertainty** — the 95% confidence band actually contains 95% of future observations.

### 3.5 Predicting at Multiple Test Points

For a vector of test points $X_* = (x_{_1}, \ldots, x_{_m})$: $$ \mathbf{f}__ \mid X__, X, \mathbf{y} \sim \mathcal{N}(\boldsymbol{\mu}_*, \Sigma__) $$ $$ \boldsymbol{\mu}__ = K(X__, X)[K(X,X) + \sigma_n^2 I]^{-1}\mathbf{y} $$ $$ \Sigma__ = K(X__, X__) - K(X__, X)[K(X,X) + \sigma_n^2 I]^{-1}K(X, X__) $$

The full posterior covariance $\Sigma_*$ captures correlation between predictions at different test points — this is something no other standard regression model directly provides.

## ✦ Part 4 — Computational Complexity and the $O(n^3)$ Wall

### 4.1 The Bottleneck

The posterior requires computing $[K(X,X) + \sigma_n^2 I]^{-1}$. For $n$ training points:

- **Matrix inversion:** $O(n^3)$ via Cholesky decomposition
- **Storage:** $O(n^2)$ for the kernel matrix
- **Prediction:** $O(n)$ per test point (after precomputing $\alpha$), $O(n^2)$ for variance

For $n = 10^3$: trivial. For $n = 10^4$: feasible. For $n = 10^5$: painful. For $n = 10^6$: intractable.

**In practice:** use the Cholesky decomposition $L L^T = K + \sigma_n^2 I$ (numerically stable, also gives $O(n^3)$ but with a smaller constant than naive inversion).

### 4.2 Approximate GP Methods

|Method|Idea|Complexity|
|---|---|---|
|**Sparse GPs (Inducing Points)**|Summarise $n$ points with $m \ll n$ inducing points $Z$|$O(nm^2)$|
|**FITC / VFE**|Variational lower bound on the marginal likelihood|$O(nm^2)$|
|**Random Fourier Features**|Approximate $k(x, x') \approx \phi(x)^T\phi(x')$, finite $D$ features|$O(nD)$|
|**State-space GPs**|Convert GP to a linear dynamical system; Kalman filter|$O(n)$ for 1D inputs|
|**Structured kernel interpolation (SKI)**|Interpolate onto a grid; FFT-based matrix-vector products|$O(n \log n)$|

The inducing point / variational approach (Titsias 2009, Hensman et al. 2013) is the modern standard. The $m$ inducing points $Z$ are **learnable** — optimise them jointly with the hyperparameters.

> **Notebook margin — why Cholesky and not direct inversion?** Numerically, $K$ is often nearly singular (especially with the SE kernel and nearby points). Cholesky decomposition is numerically stable, reveals singularity, gives the log-determinant cheaply (sum of log diagonal), and allows efficient solves. Always use Cholesky for GP computations. Never call `np.linalg.inv` on a kernel matrix.

## ✦ Part 5 — Learning Hyperparameters: Marginal Likelihood

### 5.1 The Problem

The GP prior $k(x, x'; \theta)$ has hyperparameters $\theta$ — e.g., $\theta = (\ell, \sigma_f^2, \sigma_n^2)$ for the SE kernel. How do we choose them?

**Not cross-validation:** too expensive (each CV fold requires an $O(n^3)$ solve).

**Bayesian answer:** marginalise over the latent function $f$ (already done in the posterior), then maximise the marginal likelihood $p(\mathbf{y} \mid X, \theta)$ over $\theta$.

### 5.2 The Log Marginal Likelihood

$$ \log p(\mathbf{y} \mid X, \theta) = -\frac{1}{2}\mathbf{y}^T (K_\theta + \sigma_n^2 I)^{-1}\mathbf{y} - \frac{1}{2}\log|K_\theta + \sigma_n^2 I| - \frac{n}{2}\log 2\pi $$

Three terms:

1. $-\frac{1}{2}\mathbf{y}^T (K_\theta + \sigma_n^2 I)^{-1}\mathbf{y}$ — **data fit**: how well the model fits the observations
2. $-\frac{1}{2}\log|K_\theta + \sigma_n^2 I|$ — **complexity penalty** (Occam factor): penalises models that can explain too many datasets
3. $-\frac{n}{2}\log 2\pi$ — normalisation constant

The marginal likelihood automatically balances fit and complexity — models that overfit have high fit but are penalised by the log-determinant term. This is **automatic Occam's razor**.

### 5.3 Optimisation

Differentiate with respect to $\theta$ and use gradient-based optimisation (L-BFGS or Adam): $$ \frac{\partial}{\partial \theta_j} \log p(\mathbf{y} \mid X, \theta) = \frac{1}{2} \text{tr}!\left[(K^{-1}\mathbf{y}\mathbf{y}^T K^{-1} - K^{-1})\frac{\partial K}{\partial \theta_j}\right] $$

where $K = K_\theta + \sigma_n^2 I$.

**Gradient cost:** $O(n^3)$ per evaluation — same as the posterior. Run from multiple random initialisations (marginal likelihood is non-convex in $\theta$).

> **What the marginal likelihood does to hyperparameters:**  
> If $\ell$ is too small: the model is wiggly, many functions fit the data, large $|K|$, large complexity penalty.  
> If $\ell$ is too large: the model is too smooth, fits the data poorly, large data-fit term.  
> The marginal likelihood finds the length-scale where these forces balance — the one that best explains the data without overfitting. This is Bayesian model selection in action.

## ✦ Part 6 — GP Classification

For binary classification, $y \in {0, 1}$ and we want: $$ p(y_* = 1 \mid x__, \mathcal{D}) = \int \sigma(f__), p(f_* \mid x__, \mathcal{D}), df__ $$

The GP prior is the same — but now the likelihood is Bernoulli, not Gaussian. The posterior $p(f \mid X, \mathbf{y})$ is **no longer Gaussian**. Exact inference is intractable.

**Approximate inference methods:**

|Method|Idea|Quality|
|---|---|---|
|**Laplace approximation**|Fit a Gaussian at the posterior mode|Fast, good for unimodal posteriors|
|**Expectation Propagation (EP)**|Approximate each factor in the likelihood with a Gaussian|More accurate than Laplace|
|**Variational inference (SVGP)**|Variational lower bound, scalable with inducing points|Modern standard for large-scale GP classification|

**Multi-class:** one GP per class (one-vs-rest) with softmax likelihood, or multi-output GP.

## ✦ Part 7 — Connection to Other Models

### 7.1 GP = Kernel Ridge Regression (in the noise-free limit)

The GP posterior mean: $$ \mu_* = K(x_*, X)[K(X,X) + \sigma_n^2 I]^{-1}\mathbf{y} $$ is exactly the kernel ridge regression predictor with regularisation parameter $\lambda = \sigma_n^2 / \sigma_f^2$. GPs give kernel ridge regression a Bayesian interpretation — and add uncertainty quantification for free.

### 7.2 GP = Infinite-Width Neural Networks

Neal (1994) showed: a single-hidden-layer neural network with i.i.d. priors on weights converges in distribution to a GP as the number of hidden units $\to \infty$. The kernel is determined by the activation function.

More recently (Lee et al. 2018, Jacot et al. 2018): deep networks trained with gradient descent correspond (in the infinite-width limit) to kernel regression with the **Neural Tangent Kernel (NTK)**. This is a GP in function space. The NTK connection is an active research area linking GPs to deep learning theory.

### 7.3 GP Prior on the Parameters of a Deep Network

In **deep kernel learning**, replace the hand-crafted feature map with a neural network: $$ k(x, x') = k_{\text{base}}(g_\theta(x), g_\theta(x')) $$ where $g_\theta$ is a neural network. Jointly optimise $\theta$ and GP hyperparameters via marginal likelihood. This combines the representational power of deep networks with GP uncertainty quantification — a practically important model class.

### 7.4 GP and SVMs

The SVM with RBF kernel and GP regression with SE kernel share the same kernel function. The difference:

- **SVM:** finds a maximum-margin classifier; no probabilistic output; sparse (support vectors)
- **GP:** full posterior distribution over $f$; uncertainty quantification; not sparse

For classification: GP gives calibrated class probabilities; SVM gives a hard boundary. The GP is preferred when uncertainty matters (medical diagnosis, active learning, safety-critical systems).

### 7.5 GP in Bayesian Optimisation

This is the primary industrial application of GPs today — used inside every major AutoML system and neural architecture search tool.

**Setup:** You want to minimise $f(x)$ where $f$ is expensive to evaluate (e.g., training a neural network with hyperparameters $x$). You have a budget of $T$ evaluations.

**Algorithm:**

1. Place a GP prior on $f$
2. Observe $f(x_1), \ldots, f(x_t)$ — update GP posterior
3. Choose next query $x_{t+1}$ by maximising an **acquisition function** $\alpha(x)$
4. Repeat

**Acquisition functions:**

|Function|Formula|Behaviour|
|---|---|---|
|**Expected Improvement (EI)**|$\mathbb{E}[\max(f^* - f(x), 0)]$|Balances exploration/exploitation; most popular|
|**Upper Confidence Bound (UCB)**|$\mu(x) + \kappa \sigma(x)$|Explicit exploration parameter $\kappa$|
|**Probability of Improvement**|$P(f(x) < f^* - \xi)$|Pure exploitation focus|
|**Thompson Sampling**|Sample $\tilde{f} \sim p(f \mid \mathcal{D})$, maximise $\tilde{f}$|Simple, strong theoretical guarantees|

**Why GPs specifically?** The acquisition function requires both a prediction $\mu(x)$ and an uncertainty $\sigma(x)$ at every point in the input space — not just at training points. GPs provide exactly this, in closed form, for any $x$.

> **Connection to hyperparameter tuning (Section 2.5 of the index):** every time you use Optuna, Ray Tune, or a Gaussian Process optimiser to tune neural network hyperparameters, there is a GP computing posterior means and variances under the hood.

## ✦ Part 8 — Sampling from a GP

How do you actually draw a sample function $f \sim \mathcal{GP}(0, k)$?

1. Choose a grid of points $X = (x_1, \ldots, x_n)$
2. Compute the Gram matrix $K_{ij} = k(x_i, x_j)$
3. Add a small jitter for numerical stability: $K \leftarrow K + 10^{-6} I$
4. Compute Cholesky: $L = \text{chol}(K)$
5. Sample $z \sim \mathcal{N}(0, I_n)$
6. Return $f = Lz$

This gives a draw from $\mathcal{N}(0, K)$ — a finite-dimensional realisation of the GP prior.

```python
import numpy as np

def sample_gp_prior(x_grid, kernel_fn, n_samples=5, jitter=1e-6):
    """
    Sample functions from a GP prior.
    x_grid: (n,) array of input points
    kernel_fn: callable, k(x, x') -> scalar
    """
    n = len(x_grid)
    K = np.array([[kernel_fn(x_grid[i], x_grid[j])
                   for j in range(n)] for i in range(n)])
    K += jitter * np.eye(n)  # numerical stability
    L = np.linalg.cholesky(K)
    z = np.random.randn(n, n_samples)
    return L @ z  # (n, n_samples): each column is a function sample

def se_kernel(x, xp, l=1.0, sf2=1.0):
    return sf2 * np.exp(-0.5 * (x - xp)**2 / l**2)

x = np.linspace(-5, 5, 200)
samples = sample_gp_prior(x, se_kernel, n_samples=5)
# samples[:, i] is the i-th function sample
```

Try this with different kernels (SE with varying $\ell$, Matérn, periodic) — you will immediately develop intuition for what each kernel encodes.

## ✦ Part 9 — Full GP Regression Implementation

```python
import numpy as np

class GaussianProcessRegressor:
    """
    Exact GP regression with SE kernel.
    Hyperparameters fixed (no marginal likelihood optimisation).
    """
    def __init__(self, length_scale=1.0, output_scale=1.0, noise_var=0.1):
        self.l   = length_scale
        self.sf2 = output_scale
        self.sn2 = noise_var
        self.L   = None  # Cholesky of (K + sn2*I)
        self.alpha = None  # (K + sn2*I)^{-1} y

    def _kernel(self, X1, X2):
        """SE kernel: (n1, d) x (n2, d) -> (n1, n2)"""
        diff = X1[:, None, :] - X2[None, :, :]   # (n1, n2, d)
        sq_dist = np.sum(diff**2, axis=-1)         # (n1, n2)
        return self.sf2 * np.exp(-0.5 * sq_dist / self.l**2)

    def fit(self, X_train, y_train):
        self.X_train = X_train
        n = len(X_train)
        K = self._kernel(X_train, X_train) + self.sn2 * np.eye(n)
        self.L = np.linalg.cholesky(K)                    # O(n^3)
        self.alpha = np.linalg.solve(
            self.L.T, np.linalg.solve(self.L, y_train))   # L L^T alpha = y

    def predict(self, X_test):
        K_star = self._kernel(X_test, self.X_train)        # (m, n)
        mu = K_star @ self.alpha                            # (m,)

        v = np.linalg.solve(self.L, K_star.T)              # (n, m)
        k_ss = self._kernel(X_test, X_test)                 # (m, m)
        cov = k_ss - v.T @ v                                # posterior covariance

        return mu, np.diag(cov)                             # mean, variance

    def log_marginal_likelihood(self, y_train):
        """Equation from Part 5. Useful for hyperparameter optimisation."""
        n = len(y_train)
        log_lik = (
            -0.5 * y_train @ self.alpha
            - np.sum(np.log(np.diag(self.L)))
            - 0.5 * n * np.log(2 * np.pi)
        )
        return log_lik
```

> **Usage:** after `gp.fit(X_train, y_train)`, call `mu, var = gp.predict(X_test)`. The 95% confidence interval is $\mu \pm 1.96\sqrt{\text{var}}$.

## ✦ Part 10 — Deep Learning Connections (Extended)

### 10.1 The Weight-Space / Function-Space Duality

|Perspective|Prior lives on|Inference|Predictions|
|---|---|---|---|
|**Bayesian NN (weight-space)**|$p(w)$|Posterior $p(w \mid \mathcal{D})$ — intractable|$\int p(y \mid x, w) p(w \mid \mathcal{D}) dw$ — approximate|
|**GP (function-space)**|$p(f)$ — GP prior|Posterior $p(f \mid \mathcal{D})$ — exact Gaussian|Exact predictive distribution|

For finite-width networks, the weight-space prior induces a non-Gaussian process over functions. In the infinite-width limit, the central limit theorem makes it Gaussian — a GP. The kernel is determined by the network architecture and activation function.

### 10.2 Why This Matters for Modern DL

- **Uncertainty quantification:** GPs give exact posteriors. BNNs don't. The deep learning community's various approaches to UQ (MC Dropout, deep ensembles, SWAG) can often be understood as approximate GP inference.
- **Generalisation theory:** NTK analysis (a GP analysis) is currently one of the main tools for understanding why overparameterised networks generalise — a central open question in deep learning theory.
- **Structured prediction:** GP multi-output models, GP latent variable models, and deep GPs are active research directions that directly compete with (and complement) deep learning approaches.

### 10.3 Where GPs Are Preferred Over Neural Networks

|Scenario|Reason|
|---|---|
|Small data ($n < 10^3$)|GPs are non-parametric; NNs overfit|
|Calibrated uncertainty needed|Exact GP posteriors vs. approximate NN uncertainty|
|Scientific inference|Kernel design encodes domain knowledge interpretably|
|Optimisation (Bayesian opt.)|Closed-form acquisition; cheap to evaluate|
|Interpretability required|Kernel hyperparameters have direct physical meaning|

```
   mean function threads through observations
```

This picture — posterior mean threading through data, variance collapsing at observations and widening in unobserved regions — is the visual essence of a Gaussian Process. Any model that behaves this way is doing Bayesian inference correctly.
