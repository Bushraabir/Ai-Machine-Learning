## ✦ Why This Note Exists

Before we can talk about speech recognition, part-of-speech tagging, biological sequence alignment, or the underpinnings of modern reinforcement learning, we need a principled way to model **sequences**. The Markov model is that foundation.

This note builds the full machinery — from the basic chain to Hidden Markov Models and their three canonical algorithms — with every equation derived, every assumption named, and every connection to the broader ML map made explicit.

## ✦ The Core Problem: Sequential Data

Most of classical ML assumes i.i.d. data: each $(x_i, y_i)$ is drawn independently from the same distribution. That assumption collapses the moment you have:

- A sentence (each word depends on preceding words)
- A time series (stock prices, sensor readings)
- A DNA strand (nucleotide probabilities shift with context)
- A trajectory in an RL environment

You need a model that respects **temporal or positional structure**. Markov models are the principled answer.

## ✦ Part 1 — The Markov Chain

### 1.1 The Markov Property

Let $X_1, X_2, \ldots, X_T$ be a sequence of random variables taking values in a finite state space $\mathcal{S} = {s_1, s_2, \ldots, s_N}$.

**Definition (Markov Property, first-order):** $$ P(X_t \mid X_{t-1}, X_{t-2}, \ldots, X_1) = P(X_t \mid X_{t-1}) $$

The present state $X_{t-1}$ is a _sufficient statistic_ for predicting the future. The entire history is irrelevant — all relevant information has already been "compressed" into the current state.

> **Notebook margin:** The first-order assumption is a modelling choice, not a law of nature. You can always extend to $k$-th order: $P(X_t \mid X_{t-k}, \ldots, X_{t-1})$. A $k$-th order chain over $|\mathcal{S}|$ states is equivalent to a first-order chain over $|\mathcal{S}|^k$ states, so we lose no generality by studying first-order chains.

### 1.2 The Transition Matrix

Assuming **time homogeneity** (transition probabilities don't change with $t$), we define: $$ A_{ij} = P(X_t = s_j \mid X_{t-1} = s_i), \quad A \in \mathbb{R}^{N \times N} $$

Properties:

- $A_{ij} \geq 0$ for all $i, j$
- $\sum_j A_{ij} = 1$ for all $i$ — each row is a probability distribution
- $A$ is called a **stochastic matrix** (or row-stochastic matrix)

The **initial distribution** $\pi_0 \in \mathbb{R}^N$, where $(\pi_0)_i = P(X_1 = s_i)$, completes the model specification.

### 1.3 Computing Sequence Probabilities

The joint probability of a sequence $x_1, x_2, \ldots, x_T$ follows directly from the chain rule + Markov property: $$ P(X_1 = x_1, \ldots, X_T = x_T) = P(X_1 = x_1) \prod_{t=2}^T P(X_t = x_t \mid X_{t-1} = x_{t-1}) $$ $$ = (\pi_0)_{x_1} \prod_{t=2}^T A_{x_{t-1}, x_t} $$

This is a product of matrix entries — beautifully simple.

### 1.4 The Stationary Distribution

After many steps, does the chain "settle" into a stable distribution?

**Definition:** A distribution $\pi^_$ is **stationary** (invariant) if: $$ \pi^_ = \pi^* A \quad \Longleftrightarrow \quad (\pi^_)^T = A^T (\pi^_)^T $$ $\pi^*$ is the **left eigenvector** of $A$ corresponding to eigenvalue 1 (which always exists for stochastic matrices by the Perron-Frobenius theorem).

**Convergence:** If the chain is **irreducible** (every state reachable from every state) and **aperiodic** (no cycles), then for any initial distribution $\pi_0$: $$ \lim_{t \to \infty} \pi_0 A^t = \pi^* $$

> **Example — A simple 2-state weather model:**  
> States: ${$Sunny, Rainy$}$, transition matrix: $$A = \begin{pmatrix} 0.9 & 0.1 \ 0.5 & 0.5 \end{pmatrix}$$ Row 1: if sunny today → 90% sunny tomorrow, 10% rainy.  
> Stationary distribution: solve $\pi^* A = \pi^_$, $\pi^__S + \pi^__R = 1$: $$\pi^__S = 0.833, \quad \pi^*_R = 0.167$$ On a long enough timescale, ~83% of days are sunny regardless of today's weather.

## ✦ Part 2 — Hidden Markov Models

### 2.1 Motivation: When the State Is Not Observed

In a plain Markov chain, the state $X_t$ is directly observed. But in many real problems:

- In **speech recognition**, the underlying phoneme is hidden; we observe an acoustic signal.
- In **POS tagging**, the underlying grammar tag (noun, verb, ...) is hidden; we observe words.
- In **finance**, "market regime" (bull/bear) is latent; we observe returns.

The **Hidden Markov Model (HMM)** separates the latent state sequence from the observed sequence.

### 2.2 Formal Definition

An HMM is a tuple $\lambda = (A, B, \pi)$:

|Symbol|Name|Definition|
|---|---|---|
|$\mathcal{S} = {s_1, \ldots, s_N}$|Hidden state space|Latent states|
|$\mathcal{V} = {v_1, \ldots, v_M}$|Observation vocabulary|Observable symbols|
|$A \in \mathbb{R}^{N \times N}$|Transition matrix|$A_{ij} = P(z_t = s_j \mid z_{t-1} = s_i)$|
|$B \in \mathbb{R}^{N \times M}$|Emission matrix|$B_{ik} = P(x_t = v_k \mid z_t = s_i)$|
|$\pi \in \mathbb{R}^N$|Initial distribution|$\pi_i = P(z_1 = s_i)$|

The generative process:

1. Sample initial state $z_1 \sim \pi$
2. Emit observation $x_t \sim B_{z_t}$ (the $z_t$-th row of $B$)
3. Transition to $z_{t+1} \sim A_{z_t}$ (the $z_t$-th row of $A$)
4. Repeat from step 2

The joint distribution: $$ P(x_{1:T}, z_{1:T}) = \pi_{z_1} \prod_{t=1}^T B_{z_t, x_t} \prod_{t=2}^T A_{z_{t-1}, z_t} $$

```
Graphical structure:

  z_1  -->  z_2  -->  z_3  --> ... -->  z_T
   |          |          |                |
   v          v          v                v
  x_1        x_2        x_3             x_T

Hidden states form a Markov chain (top).
Each state independently emits an observation (bottom).
```

### 2.3 The Three Problems of HMMs

Rabiner (1989) identified three canonical problems that must be solved for HMMs to be useful in practice. Every algorithm below solves exactly one of them.

|#|Problem|Question|Algorithm|
|---|---|---|---|
|1|**Evaluation**|Given $\lambda$ and observations $x_{1:T}$, what is $P(x_{1:T} \mid \lambda)$?|Forward Algorithm|
|2|**Decoding**|Given $\lambda$ and $x_{1:T}$, what is the most likely state sequence $z_{1:T}^*$?|Viterbi Algorithm|
|3|**Learning**|Given $x_{1:T}$, find $\lambda^* = \arg\max_\lambda P(x_{1:T} \mid \lambda)$|Baum-Welch (EM)|

## ✦ Part 3 — Problem 1: Evaluation (The Forward Algorithm)

### 3.1 Why Naïve Computation Fails

The observation likelihood is: $$ P(x_{1:T} \mid \lambda) = \sum_{z_{1:T}} P(x_{1:T}, z_{1:T} \mid \lambda) $$

Summing over all $N^T$ state sequences is $O(N^T)$ — exponential in sequence length. For $N = 10$, $T = 100$: $10^{100}$ terms. Completely intractable.

### 3.2 The Forward Variable

Define the **forward variable**: $$ \alpha_t(i) := P(x_1, x_2, \ldots, x_t, z_t = s_i \mid \lambda) $$ "The probability of seeing the partial observation sequence $x_1, \ldots, x_t$ **and** being in state $s_i$ at time $t$."

### 3.3 The Forward Recursion

**Initialisation** ($t = 1$): $$ \alpha_1(i) = \pi_i \cdot B_{i, x_1} $$

**Recursion** ($t = 2, \ldots, T$): $$ \boxed{\alpha_t(j) = \left[ \sum_{i=1}^N \alpha_{t-1}(i) \cdot A_{ij} \right] \cdot B_{j, x_t}} $$

The bracket sums over all ways to arrive in state $j$ at time $t$, then multiplies by the emission probability of $x_t$ from state $j$.

**Termination:** $$ P(x_{1:T} \mid \lambda) = \sum_{i=1}^N \alpha_T(i) $$

**Complexity:** $O(N^2 T)$ — polynomial. We've converted exponential summation into dynamic programming.

> **Derivation sketch:**  
> $$\alpha_t(j) = P(x_{1:t}, z_t = j) = \sum_i P(x_{1:t}, z_{t-1} = i, z_t = j)$$ $$= \sum_i P(x_t \mid z_t = j) \cdot P(z_t = j \mid z_{t-1} = i) \cdot P(x_{1:t-1}, z_{t-1} = i)$$ $$= B_{j,x_t} \sum_i A_{ij} \cdot \alpha_{t-1}(i) \qquad \square$$

### 3.4 Numerical Issues: Log-Space Computation

Products of small probabilities underflow to zero rapidly. Standard fix: work in log-space using the **log-sum-exp** trick: $$ \log \sum_i e^{a_i} = a^* + \log \sum_i e^{a_i - a^_}, \quad a^_ = \max_i a_i $$

> **Notebook margin:** This same trick appears in logistic regression (softmax), language model decoding, and variational inference. It's one of the half-dozen truly universal numerical tools in ML.

## ✦ Part 4 — Problem 2: Decoding (The Viterbi Algorithm)

### 4.1 What We Want

Find the single most probable state sequence given the observations: $$ z^*_{1:T} = \arg\max_{z_{1:T}} P(z_{1:T} \mid x_{1:T}, \lambda) = \arg\max_{z_{1:T}} P(z_{1:T}, x_{1:T} \mid \lambda) $$

Note: This is the **joint MAP** over the entire sequence — it is _not_ the sequence of individually most probable states $\arg\max_{z_t} P(z_t \mid x_{1:T})$ at each time step (those are obtained by the backward algorithm and do not form a coherent sequence).

### 4.2 The Viterbi Variable

Define: $$ \delta_t(i) := \max_{z_{1:t-1}} P(z_1, \ldots, z_{t-1}, z_t = s_i, x_1, \ldots, x_t \mid \lambda) $$ "The probability of the **most likely path** ending in state $s_i$ at time $t$, given observations so far."

To recover the actual path, store the backpointer: $$ \psi_t(j) := \arg\max_i \left[ \delta_{t-1}(i) \cdot A_{ij} \right] $$

### 4.3 The Viterbi Recursion

**Initialisation:** $$ \delta_1(i) = \pi_i \cdot B_{i, x_1}, \quad \psi_1(i) = 0 $$

**Recursion** ($t = 2, \ldots, T$): $$ \boxed{\delta_t(j) = \max_i \left[ \delta_{t-1}(i) \cdot A_{ij} \right] \cdot B_{j, x_t}} $$ $$ \psi_t(j) = \arg\max_i \left[ \delta_{t-1}(i) \cdot A_{ij} \right] $$

**Termination:** $$ P^* = \max_i \delta_T(i), \quad z_T^* = \arg\max_i \delta_T(i) $$

**Backtracking** ($t = T-1, \ldots, 1$): $$ z_t^* = \psi_{t+1}(z_{t+1}^*) $$

**Complexity:** $O(N^2 T)$ — identical to the forward algorithm, because we replaced $\sum$ with $\max$.

> **Insight:** Viterbi is just the forward algorithm with $\sum \to \max$. This swap is not cosmetic — it changes the semantics entirely from marginalisation to optimisation. The same substitution appears in belief propagation (sum-product vs. max-product) and in dynamic programming generally (shortest path vs. counting paths).

> **Example — POS Tagging:**  
> Sentence: "flies like arrows"  
> States: ${$NOUN, VERB, PREP$}$  
> The word "flies" can be NOUN ("the flies") or VERB ("he flies"). "like" can be VERB or PREP. Viterbi finds the globally most coherent assignment — e.g., VERB-PREP-NOUN — using transition probabilities between POS tags (which encode grammar) and emission probabilities (which encode lexical likelihood).

## ✦ Part 5 — Problem 3: Learning (Baum-Welch / EM)

### 5.1 The Problem Setup

Given: Observation sequences $x_{1:T}$ (the state sequence $z_{1:T}$ is latent).  
Find: $\lambda^* = (A^_, B^_, \pi^*) = \arg\max_\lambda P(x_{1:T} \mid \lambda)$.

Direct MLE is intractable because the state sequence is hidden. This is exactly the structure EM was designed for.

Recall from [[03.Probability Review]]: EM maximises the ELBO (Evidence Lower Bound): $$ \log P(x \mid \lambda) \geq \mathbb{E}_{q(z)}[\log P(x, z \mid \lambda)] - \mathbb{E}_{q(z)}[\log q(z)] $$

In the E-step, set $q(z) = P(z \mid x, \lambda_{\text{old}})$. In the M-step, maximise the expected complete-data log-likelihood.

### 5.2 The Backward Variable

To compute the posterior $P(z_{1:T} \mid x_{1:T}, \lambda)$, we need both a forward and a backward pass.

Define the **backward variable**: $$ \beta_t(i) := P(x_{t+1}, x_{t+2}, \ldots, x_T \mid z_t = s_i, \lambda) $$ "The probability of the **future** observations $x_{t+1:T}$, given that we're in state $s_i$ at time $t$."

**Initialisation:** $$ \beta_T(i) = 1 \quad \text{for all } i $$

**Recursion** ($t = T-1, \ldots, 1$): $$ \boxed{\beta_t(i) = \sum_j A_{ij} \cdot B_{j, x_{t+1}} \cdot \beta_{t+1}(j)} $$

### 5.3 The Posterior Statistics

Using both $\alpha$ and $\beta$, we can compute the two key posterior quantities:

**State occupancy** — probability of being in state $i$ at time $t$: $$ \gamma_t(i) := P(z_t = s_i \mid x_{1:T}, \lambda) = \frac{\alpha_t(i) \cdot \beta_t(i)}{\sum_k \alpha_t(k) \cdot \beta_t(k)} $$

**Transition occupancy** — probability of transitioning $i \to j$ at time $t$: $$ \xi_t(i,j) := P(z_t = s_i, z_{t+1} = s_j \mid x_{1:T}, \lambda) = \frac{\alpha_t(i) \cdot A_{ij} \cdot B_{j, x_{t+1}} \cdot \beta_{t+1}(j)}{\sum_{i'}\sum_{j'} \alpha_t(i') \cdot A_{i'j'} \cdot B_{j', x_{t+1}} \cdot \beta_{t+1}(j')} $$

### 5.4 The M-Step: Parameter Re-estimation

The Baum-Welch re-estimation equations are the M-step closed-form solutions (derivable by Lagrange multipliers subject to the stochastic constraints):

$$ \hat{\pi}_i = \gamma_1(i) $$

$$ \hat{A}_{ij} = \frac{\sum_{t=1}^{T-1} \xi_t(i,j)}{\sum_{t=1}^{T-1} \gamma_t(i)} \quad = \frac{\text{expected number of } i \to j \text{ transitions}}{\text{expected time in state } i} $$

$$ \hat{B}_{ik} = \frac{\sum_{t=1}^T \gamma_t(i) \cdot \mathbf{1}[x_t = v_k]}{\sum_{t=1}^T \gamma_t(i)} \quad = \frac{\text{expected time in state } i \text{ emitting } v_k}{\text{expected time in state } i} $$

### 5.5 Convergence Properties

- **Guaranteed:** Each EM iteration increases $P(x_{1:T} \mid \lambda)$ (or leaves it unchanged).
- **Not guaranteed:** Convergence to the global maximum. HMM likelihood surfaces are non-convex with multiple local optima.
- **Practical fix:** Multiple random initialisations; choose the run with highest final likelihood.

> **The EM lens:** Baum-Welch is EM with:
> 
> - **Latent variable:** state sequence $z_{1:T}$
> - **E-step:** run Forward-Backward to compute $\gamma_t, \xi_t$ (soft assignments)
> - **M-step:** plug into re-estimation formulas (weighted MLE with soft counts) Compare to K-Means (hard assignments) vs. GMM (soft assignments). Baum-Welch is the sequential analogue of GMM's EM.

## ✦ Part 6 — Continuous Observations

So far, $x_t$ has been discrete (a vocabulary symbol). For real-valued observations (e.g., acoustic feature vectors in speech), replace the discrete emission matrix $B$ with a continuous density:

$$ b_i(x_t) = \mathcal{N}(x_t \mid \mu_i, \Sigma_i) \quad \text{or} \quad b_i(x_t) = \sum_m c_{im} \mathcal{N}(x_t \mid \mu_{im}, \Sigma_{im}) $$

The Gaussian Mixture emission is standard in speech recognition. The M-step updates for $\mu_i, \Sigma_i$ are just weighted MLE updates — same structure as GMM.

## ✦ Part 7 — Connection to AI and Deep Learning

### 7.1 Classical Applications

|Application|Hidden States|Observations|
|---|---|---|
|Speech recognition|Phoneme sequence|Acoustic feature vectors (MFCCs)|
|POS tagging|Grammar tag sequence|Word tokens|
|Biological sequences|Functional region (exon/intron)|Nucleotides|
|Gesture recognition|Gesture phase|Accelerometer readings|
|Financial regime detection|Market regime (bull/bear/volatile)|Log-returns|

### 7.2 HMMs as a Precursor to Deep Sequence Models

HMMs encode a specific inductive bias:

- **Markov assumption:** only one step of memory
- **Parametric emissions:** fixed functional form (Gaussian, multinomial)
- **Fixed state space:** $N$ states defined ahead of time

Modern sequence models relax all three assumptions:

|Limitation|How DL addressed it|
|---|---|
|1-step memory|**RNNs/LSTMs:** learn arbitrary-length memory via gated hidden state $h_t$|
|Parametric emission|**Neural language models:** learn emission distribution from data|
|Fixed discrete states|**Continuous latent spaces:** VAEs, RNNs with dense hidden state $h_t \in \mathbb{R}^d$|
|Markov property|**Transformers:** full attention over all positions — explicitly non-Markovian|

Yet HMMs remain the conceptual foundation. Understanding the three HMM problems — evaluation, decoding, learning — gives you the right vocabulary for understanding attention (soft assignment over positions), CTC decoding (sequence alignment), and the ELBO in VAEs (same EM structure, continuous latent variables).

### 7.3 HMMs in the Modern ML Stack

```
HMM (1960s–1990s)
  │
  ├── Recurrent Neural Networks (1986, Elman)
  │     └── LSTM (1997, Hochreiter & Schmidhuber)
  │           └── GRU (2014) → seq2seq → Attention → Transformer (2017)
  │
  ├── CRF (2001) → BiLSTM-CRF (still used in NER)
  │
  └── Structured State Spaces (S4, Mamba, 2021–2023)
        — HMM dynamics as a continuous-time model
```

> Mamba (2023) explicitly revisits selective state-space models, a direct generalisation of HMMs, and achieves Transformer-level performance with linear complexity. HMMs are not historical curiosities — they are the theoretical backbone of an active research direction.

### 7.4 RL Connection

The HMM generative model $(A, B)$ is structurally identical to a **Partially Observable MDP (POMDP)**:

- Transition matrix $A$ ↔ transition function $P(s' \mid s, a)$
- Emission matrix $B$ ↔ observation function $P(o \mid s)$

The Viterbi algorithm for decoding corresponds to finding the optimal policy in a degenerate POMDP. This connection deepens when we study RL with partial observability — the belief state maintained by the forward algorithm becomes the "sufficient statistic for planning."

## ✦ Part 8 — Illustrated Summary

### The Forward-Backward Lattice

```
Time:      t=1          t=2          t=3        ...   t=T
           ┌──────┐    ┌──────┐    ┌──────┐         ┌──────┐
State s_1  │α₁(1) │───▶│α₂(1) │───▶│α₃(1) │ ───▶   │αT(1) │
           └──────┘    └──────┘    └──────┘         └──────┘
                \    /        \    /
           ┌──────┐    ┌──────┐    ┌──────┐         ┌──────┐
State s_2  │α₁(2) │───▶│α₂(2) │───▶│α₃(2) │ ───▶   │αT(2) │
           └──────┘    └──────┘    └──────┘         └──────┘
                 ▲           ▲           ▲                 ▲
Observations:  x_1          x_2          x_3              x_T

Forward pass: → propagates probability left-to-right
Backward pass: ← propagates probability right-to-left
Both passes together: give full posterior γ_t(i) at every node
```

### The Three Algorithms at a Glance

```
           Forward            Viterbi            Baum-Welch
           ───────            ───────            ──────────
Goal:    P(x_{1:T}|λ)      best z*_{1:T}       best λ*

How:     sum over paths    max over paths       EM: E=Forward-Backward
                                                    M=re-estimation

Replace: sum → sum         sum → max            iterate until convergence

Output:  scalar            sequence             new parameters (A,B,π)

Cost:    O(N²T)            O(N²T)              O(N²T) per iteration
```

## ✦ Complexity and Practical Notes

|Operation|Complexity|Notes|
|---|---|---|
|Forward algorithm|$O(N^2 T)$|Compute $P(x_{1:T})$|
|Backward algorithm|$O(N^2 T)$|Needed for Baum-Welch E-step|
|Viterbi decoding|$O(N^2 T)$|Path backtracking adds $O(NT)$ memory|
|Baum-Welch (per iteration)|$O(N^2 T)$|Often needs 50–200 iterations|
|State space size $N$|$N^2$ in complexity|$N$ is a key hyperparameter; typically 5–100|

**Implementation notes for Python:**

- Work in log-space throughout; use `scipy.special.logsumexp`
- Multiple restarts: train 10 random models, keep the best
- `hmmlearn` is the standard Python library; understand it well enough to reimplement it
- For large-scale applications (speech), deep learning replaced HMMs; but the decoding algorithms survive as postprocessing steps
