## ✦ Why Feature Selection?

You have engineered $p$ features. Not all of them help. Some are redundant (correlated with other features — they add no new information). Some are irrelevant (independent of the target — they add only noise). Some are harmful (they carry enough noise that the model overfits to them).

The motivations for removing features are distinct and worth separating:

**1. Variance reduction.** Every feature is a degree of freedom. More degrees of freedom → more ways to overfit. Removing irrelevant features reduces the effective model complexity without sacrificing expressiveness. This is the bias-variance argument for feature selection.

**2. Computational efficiency.** Training cost scales with $p$ (or $p^2$, or worse, depending on the algorithm). Fewer features → faster training and inference.

**3. Interpretability.** A model with 5 features is understandable. A model with 500 features is not. Clinical and legal settings often require sparse, interpretable models.

**4. Avoiding the curse of dimensionality.** Distance-based algorithms (KNN, SVM with RBF kernel, K-Means) degrade as $p$ grows because all points become equidistant in high dimensions. Feature selection directly combats this.

**5. Avoiding Multicollinearity.** In linear models, if two features are nearly perfectly correlated, $(X^TX)$ is nearly singular, numerical issues arise, and coefficient estimates become unstable.

## ✦ The Three Families

There are three fundamentally different approaches to feature selection. They differ in _what information they use_ to score features.

```
Feature Selection Methods
│
├── Filter Methods
│   Score features using statistics of the data alone.
│   Model-agnostic, fast, computed once.
│   Weakness: ignores feature interactions.
│
├── Wrapper Methods
│   Score feature subsets by training and evaluating a model.
│   Model-specific, expensive, but captures interactions.
│   Weakness: O(2^p) subsets in the worst case.
│
└── Embedded Methods
    Feature selection happens during model training.
    Model-specific, efficient (no extra training loops).
    Examples: L1 regularisation, tree feature importance.
```

## ✦ 1. Filter Methods

Filter methods score each feature independently of the model, using only the relationship between the feature and the target (or between features themselves). They are computed _before_ any model is trained.

### 1.1 Variance Threshold

Remove features with variance below a threshold $\tau$:

$$\text{Var}(x_j) = \frac{1}{n} \sum_{i=1}^n (x_{ij} - \bar{x}_j)^2 < \tau \implies \text{remove feature } j$$

**Rationale:** A feature with near-zero variance is nearly constant — it carries almost no information. A constant feature contributes nothing to any model.

**Caveat:** Variance depends on scale. If you have not standardised, a feature with small values (e.g., a probability in $[0,1]$) will always have low variance compared to one in $[0, 10^6]$. Apply after standardisation, or use a relative threshold.

```python
import numpy as np

def variance_threshold(X, threshold=0.01):
    """Remove features with variance < threshold. Fit on train only."""
    variances = X.var(axis=0)
    mask = variances >= threshold
    return X[:, mask], mask

# Usage — fit on train, apply same mask to test
X_train_filtered, mask = variance_threshold(X_train, threshold=0.01)
X_test_filtered = X_test[:, mask]
```

### 1.2 Correlation-Based Selection

Remove features that are highly correlated with each other (redundancy) or weakly correlated with the target (irrelevance).

**Pearson correlation** between feature $j$ and target $y$:

$$r(x_j, y) = \frac{\sum_i (x_{ij} - \bar{x}_j)(y_i - \bar{y})}{\sqrt{\sum_i (x_{ij} - \bar{x}_j)^2} \sqrt{\sum_i (y_i - \bar{y})^2}}$$

$r \in [-1, 1]$. Remove features with $|r| < \tau_{\text{relevance}}$.

**Redundancy:** For each pair $(j, k)$ with $|r(x_j, x_k)| > \tau_{\text{redundancy}}$, keep the one with higher $|r(x_j, y)|$ and remove the other.

**Critical limitation:** Pearson correlation only measures _linear_ association. A feature with a strong nonlinear relationship to the target (e.g., $y = x^2$) will have $r(x, y) \approx 0$ if $x$ is symmetric around zero. Correlation-based selection will incorrectly discard it.

```python
def correlation_selection(X, y, threshold=0.1):
    """Keep features with |correlation to target| >= threshold."""
    correlations = np.array([
        np.corrcoef(X[:, j], y)[0, 1] for j in range(X.shape[1])
    ])
    mask = np.abs(correlations) >= threshold
    return X[:, mask], mask, correlations
```

### 1.3 Mutual Information

Mutual information between feature $X_j$ and target $Y$ measures the reduction in uncertainty about $Y$ from knowing $X_j$:

$$I(X_j; Y) = \sum_{x, y} p(x, y) \log \frac{p(x, y)}{p(x), p(y)}$$

For continuous variables, replace sums with integrals. In practice, estimated using $k$-nearest-neighbour density estimators (Kraskov et al., 2004).

**Why it is superior to correlation:** $I(X_j; Y) = 0$ if and only if $X_j$ and $Y$ are _statistically independent_ — not just linearly uncorrelated. It captures any form of dependence, including nonlinear and non-monotone.

**Properties:**

- $I(X_j; Y) \geq 0$ always.
- $I(X_j; Y) = 0 \iff X_j \perp Y$.
- $I(X_j; Y) = H(Y) - H(Y | X_j)$ — the information gain about $Y$ from observing $X_j$.
- Symmetric: $I(X; Y) = I(Y; X)$.

**Relationship to entropy:**

$$I(X; Y) = H(X) + H(Y) - H(X, Y) = H(X) - H(X|Y)$$

where $H(X) = -\sum_x p(x) \log p(x)$ is the Shannon entropy.

```python
from sklearn.feature_selection import mutual_info_classif, mutual_info_regression

# Classification target
mi_scores = mutual_info_classif(X_train, y_train, random_state=42)

# Regression target
mi_scores = mutual_info_regression(X_train, y_train, random_state=42)

# Select top-k features
k = 20
top_k_mask = np.argsort(mi_scores)[::-1][:k]
X_train_selected = X_train[:, top_k_mask]
X_test_selected  = X_test[:,  top_k_mask]
```

### 1.4 Statistical Tests

For classification tasks, classical statistical tests can assess the independence of a feature from the class label.

**Chi-squared test** (for non-negative discrete features):

$$\chi^2 = \sum_{j} \frac{(O_j - E_j)^2}{E_j}$$

where $O_j$ are observed counts and $E_j = (\text{row sum}) \times (\text{col sum}) / n$ are expected counts under independence. High $\chi^2$ → reject independence → feature is informative.

**ANOVA F-test** (for continuous features, categorical target):

$$F = \frac{\text{variance between class means}}{\text{variance within classes}}$$

High $F$ → means differ across classes → feature is discriminative.

**Limitation of all filter methods:** They score features _individually_. A feature that is individually uninformative but becomes informative _in combination_ with another feature will be discarded. This is the fundamental weakness of filter methods.

### 1.5 Filter Method Summary

|Method|Type of Target|Detects Nonlinearity|Handles Interaction|Speed|
|---|---|---|---|---|
|Variance threshold|Any|N/A|No|$O(np)$|
|Pearson correlation|Continuous|No|No|$O(np)$|
|Mutual information|Any|Yes|No|$O(np \log n)$|
|Chi-squared|Categorical|No|No|$O(np)$|
|ANOVA F-test|Categorical class|No|No|$O(np)$|

## ✦ 2. Wrapper Methods

Wrapper methods treat the feature subset as a combinatorial search problem. They evaluate subsets by training a model on that subset and measuring cross-validated performance. This is model-specific and captures feature interactions — but it is expensive.

### 2.1 The Combinatorial Problem

For $p$ features, there are $2^p$ possible subsets. For $p = 50$: $2^{50} \approx 10^{15}$ subsets — exhaustive search is impossible. All practical wrapper methods are heuristics that search this space efficiently.

### 2.2 Recursive Feature Elimination (RFE)

**Algorithm:**

```
1. Train model on all p features.
2. Score each feature by its importance (model-specific).
3. Remove the least important feature (or bottom k%).
4. Repeat until the desired number of features remains.
```

The importance score is model-specific:

- Linear models: $|w_j|$ (the magnitude of the coefficient after standardisation).
- Tree-based models: mean decrease in impurity, or permutation importance.

```python
from sklearn.feature_selection import RFE
from sklearn.linear_model import LogisticRegression

estimator = LogisticRegression(C=1.0, max_iter=1000)
selector  = RFE(estimator, n_features_to_select=10, step=1)
selector.fit(X_train, y_train)

X_train_selected = selector.transform(X_train)
X_test_selected  = selector.transform(X_test)

print(selector.ranking_)  # 1 = selected, k = eliminated at step k
```

**With cross-validation — RFECV:** Rather than specifying `n_features_to_select`, use cross-validation to select the optimal number of features:

```python
from sklearn.feature_selection import RFECV
from sklearn.model_selection import StratifiedKFold

cv = StratifiedKFold(n_splits=5)
selector = RFECV(estimator, step=1, cv=cv, scoring='f1', min_features_to_select=1)
selector.fit(X_train, y_train)
print(f"Optimal features: {selector.n_features_}")
```

**Cost:** $O(p)$ model fits in the naive case (one per elimination step). With `step=k`, $O(p/k)$ fits.

---

### 2.3 Forward Selection and Backward Elimination

**Forward selection:**

```
Start with empty set S = {}.
At each step: add the feature j ∉ S that maximises CV performance when added to S.
Stop when performance stops improving (or budget is exhausted).
```

**Backward elimination:**

```
Start with full set S = {1, ..., p}.
At each step: remove the feature j ∈ S whose removal causes the least CV performance drop.
Stop when performance starts dropping significantly.
```

**Bidirectional (stepwise):** Combines both — add or remove at each step, whichever improves performance most.

These are hill-climbing heuristics. They can get stuck at local optima. Forward selection is $O(p^2)$ model fits in the worst case (each of $p$ steps considers up to $p$ candidates). This is feasible for moderate $p$ ($\lesssim 100$) but not for $p \sim 10^4$.

---

### 2.4 The Leakage Trap in Wrapper Methods

This is the most common and most consequential mistake in feature selection.

**Wrong:**

```python
# BUG: Feature selection uses all data including test
selector.fit(X, y)           # fitted on ALL data
X_selected = selector.transform(X)
X_train_s, X_test_s = train_test_split(X_selected, ...)
# Test set used to select features → optimistic evaluation
```

**Correct:**

```python
# CORRECT: Feature selection inside the pipeline, inside CV
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score

pipe = Pipeline([
    ('select', RFE(LogisticRegression(), n_features_to_select=10)),
    ('model',  LogisticRegression()),
])
# cross_val_score fits the entire pipeline on each training fold
# The RFE never sees the validation fold during its fit
scores = cross_val_score(pipe, X_train, y_train, cv=5, scoring='f1')
```

**Why this matters:** If you select features using the full dataset, the selected features are the ones most correlated with $y$ in the _entire_ dataset — including the test set. The test set has leaked into the feature selection step. This inflates CV scores and produces overly optimistic estimates.

**Rule:** Feature selection is a step of model _fitting_. It must be done inside the training fold, not before the split.

## ✦ 3. Embedded Methods

Embedded methods perform feature selection as part of the model training process. They are more efficient than wrappers (no extra training loop) and more model-aware than filters (they consider the feature's contribution within the model).

### 3.1 L1 Regularisation (Lasso)

**The Lasso objective:**

$$\hat{w} = \arg\min_{w} \left[ \frac{1}{n} |Xw - y|^2 + \lambda |w|_1 \right]$$

where $|w|_1 = \sum_j |w_j|$.

**Why L1 produces sparsity (geometric derivation):**

The Lasso can be written as a constrained optimisation:

$$\hat{w} = \arg\min_{w} \frac{1}{n} |Xw - y|^2 \quad \text{s.t.} \quad |w|_1 \leq t$$

The constraint set ${|w|_1 \leq t}$ is an $\ell^1$ ball — a diamond in 2D, a cross-polytope in higher dimensions. Its extreme points (corners) are on the coordinate axes, where exactly one component of $w$ is nonzero and all others are zero.

The unconstrained minimum (the OLS solution) is typically not on the constraint boundary. As we shrink $t$ (increase $\lambda$), the constraint boundary moves inward. The first time the OLS ellipsoidal contours touch the $\ell^1$ ball, they almost always touch at a corner — a sparse solution.

**Contrast with L2 (Ridge):** The $\ell^2$ ball ${|w|_2 \leq t}$ is a sphere. It has no corners. The constrained optimum can be anywhere on the sphere's surface, and generically all components of $w$ will be nonzero. L2 shrinks all coefficients toward zero but never sets them exactly to zero.

```
L2 ball (circle):          L1 ball (diamond):
        *                         *
      *   *                      /|\
     *     *                    / | \
     *  ●  *    OLS min → ●    /  |  \
     *     *                  / ● |   \   ← OLS solution likely
      *   *                  /   |    \    hits a corner → sparse
        *                   *----+----*
```

**Connection to MAP estimation:** As shown in [[03.Probability Review]], Lasso is equivalent to MAP estimation with a Laplace prior on $w$:

$$p(w_j) = \frac{\lambda}{2} e^{-\lambda |w_j|}$$

The Laplace prior has a sharp peak at zero and heavy tails — it strongly encourages sparsity while allowing a few large weights.

```python
from sklearn.linear_model import Lasso, LassoCV

# LassoCV selects lambda via cross-validation
lasso_cv = LassoCV(cv=5, random_state=42, max_iter=10000)
lasso_cv.fit(X_train, y_train)

print(f"Optimal lambda: {lasso_cv.alpha_:.4f}")
print(f"Non-zero features: {(lasso_cv.coef_ != 0).sum()} / {X_train.shape[1]}")

# Selected feature mask
mask = lasso_cv.coef_ != 0
X_train_selected = X_train[:, mask]
X_test_selected  = X_test[:,  mask]
```

**Elastic Net:** Convex combination of L1 and L2:

$$\hat{w} = \arg\min_w \frac{1}{n}|Xw-y|^2 + \lambda_1 |w|_1 + \lambda_2 |w|_2^2$$

Adds the grouping effect: when features are correlated, Lasso arbitrarily picks one. Elastic Net tends to select correlated features together. Controlled by the mixing parameter $\rho = \lambda_1 / (\lambda_1 + \lambda_2)$.

### 3.2 Tree-Based Feature Importance

Decision trees make splits by choosing the feature and threshold that maximally reduces impurity at each node. The total reduction in impurity attributed to feature $j$ across all splits in all trees is its importance.

**Mean Decrease in Impurity (MDI) — also called Gini importance:**

$$\text{Importance}(j) = \sum_{t \in \mathcal{T}: \text{split on } j} p(t) \cdot \Delta I(t)$$

where $p(t) = n_t / n$ is the fraction of samples reaching node $t$, and $\Delta I(t)$ is the impurity reduction at that split.

For Random Forests, average over all trees:

$$\overline{\text{Importance}}(j) = \frac{1}{B} \sum_{b=1}^B \text{Importance}_b(j)$$

**Known bias of MDI:** High-cardinality features (many unique values) tend to score higher, even if they have no real predictive power. A random feature with 1000 unique values will have higher MDI than an informative binary feature, all else equal, because there are more possible splits. This is a well-documented deficiency.

```python
from sklearn.ensemble import RandomForestClassifier
import numpy as np

rf = RandomForestClassifier(n_estimators=200, random_state=42)
rf.fit(X_train, y_train)

importances = rf.feature_importances_  # MDI, normalised to sum to 1
ranked = np.argsort(importances)[::-1]
for i in ranked[:10]:
    print(f"Feature {i}: {importances[i]:.4f}")
```

---

### 3.3 Permutation Importance

**Algorithm:**

```
1. Train model on (X_train, y_train). Record baseline score S₀.
2. For each feature j:
   a. Shuffle column j in X_val (break its association with y).
   b. Score the model on the shuffled X_val → S_j.
   c. Importance(j) = S₀ - S_j.
3. Features with large importance drop are important.
   Features with near-zero or negative drop are uninformative.
```

**Why permutation importance is superior to MDI:**

- **Model-agnostic:** Works for any trained model, not just trees.
- **No cardinality bias:** Shuffling a high-cardinality feature does not give it a false advantage.
- **Measures actual predictive contribution:** It asks "how much does the model's performance suffer if this feature is unavailable?" — which is precisely what we care about.
- **Evaluated on validation data:** MDI is computed on training data (biased toward training fit); permutation importance is computed on held-out data.

**Caveat:** For correlated features, permutation importance underestimates importance. If features $j$ and $k$ are strongly correlated, shuffling $j$ has little effect because $k$ still carries the same information. The model compensates. Both features may appear unimportant individually despite being jointly essential.

```python
from sklearn.inspection import permutation_importance

result = permutation_importance(
    rf, X_val, y_val,
    n_repeats=10,        # shuffle each feature 10 times, take mean
    random_state=42,
    scoring='f1'
)

perm_importances = result.importances_mean
perm_std         = result.importances_std

# Select features with importance mean > 2 standard deviations above zero
mask = perm_importances > 2 * perm_std
```

---

### 3.4 Comparison: MDI vs. Permutation Importance

|Property|MDI (Gini Importance)|Permutation Importance|
|---|---|---|
|Computed on|Training data|Validation / test data|
|Cardinality bias|Yes — high-card features preferred|No|
|Model-agnostic|No (tree only)|Yes|
|Handles correlated features|Poorly|Poorly (different reason)|
|Speed|Free (computed during training)|$O(p \cdot n_{\text{repeats}})$ model evaluations|
|Preferred for|Quick, rough ranking|Production feature selection|

## ✦ 4. Stability of Feature Selection

A feature selection method is _stable_ if small perturbations to the training set produce similar selected feature sets. Unstable selection is a sign of overfitting at the feature level.

**Measuring stability:** Train the selector on $B$ bootstrap samples of the training data. For each feature $j$, compute its selection frequency $f_j \in [0, 1]$. Features with $f_j \approx 1$ are stably selected; features with $f_j \approx 0$ are stably rejected; features with $f_j \approx 0.5$ are unstable.

$$\text{Stability}(j) = f_j = \frac{1}{B} \sum_{b=1}^B \mathbf{1}[j \in S_b]$$

**Stability selection** (Meinshausen & Bühlmann, 2010): Select only features with $f_j > \pi_{\text{thr}}$ (e.g., 0.8). This provides finite-sample control of the false discovery rate for feature selection.

```python
from sklearn.utils import resample

def stability_selection(X, y, selector, n_bootstrap=50, threshold=0.8):
    n, p = X.shape
    selection_counts = np.zeros(p)
    
    for _ in range(n_bootstrap):
        X_boot, y_boot = resample(X, y, random_state=None)
        selector.fit(X_boot, y_boot)
        # assumes selector has a boolean mask or support_ attribute
        selection_counts += selector.support_.astype(float)
    
    frequencies = selection_counts / n_bootstrap
    stable_mask  = frequencies >= threshold
    return stable_mask, frequencies
```

## ✦ 5. The Feature Selection Leakage Trap (Revisited, Precisely)

This deserves its own section because it is so commonly wrong in practice — including in published papers.

### The Theorem

Let $\hat{f}_{S}$ be a model trained on feature subset $S$. Let $\hat{S}$ be the feature subset selected using the full dataset $(X, y)$. Then:

$$\mathbb{E}[\text{CV error of } \hat{f}_{\hat{S}}] < \mathbb{E}[\text{true generalisation error of } \hat{f}_{\hat{S}}]$$

i.e., the cross-validated error _underestimates_ the true error. The selection step has already "peeked" at the validation folds.

### The Correct Procedure: Nested CV

For simultaneous feature selection and model evaluation:

```
Outer loop (evaluation):    Split data into K outer folds
  For each outer fold:
    Inner loop (selection): Use remaining K-1 folds for selection
      Train selector on inner training data
      Select features S_inner
    Train model on outer training data using S_inner
    Evaluate on outer test fold

Report: mean ± std of outer fold scores
```

In sklearn:

```python
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score, KFold
from sklearn.feature_selection import SelectKBest, mutual_info_classif
from sklearn.linear_model import LogisticRegression

# Everything inside a single Pipeline — leakage-free by construction
pipe = Pipeline([
    ('select', SelectKBest(mutual_info_classif, k=10)),
    ('model',  LogisticRegression(C=1.0, max_iter=1000)),
])

outer_cv = KFold(n_splits=5, shuffle=True, random_state=42)
scores   = cross_val_score(pipe, X, y, cv=outer_cv, scoring='f1')
print(f"Unbiased F1: {scores.mean():.3f} ± {scores.std():.3f}")
```

The `Pipeline` guarantees that `SelectKBest.fit()` is called only on the training fold of each outer split. The mutual information scores are recomputed fresh for every fold.

## ✦ 6. Connection to AI, Machine Learning, and Deep Learning

### 6.1 Feature Selection as Regularisation

Feature selection is regularisation with a different norm. L0 regularisation:

$$\min_w \frac{1}{n}|Xw - y|^2 + \lambda |w|_0$$

where $|w|_0 = |{j : w_j \neq 0}|$ is the number of nonzero coefficients. This directly penalises the number of features used. It is NP-hard to optimise exactly. Lasso (L1) is its _convex relaxation_ — the tightest convex approximation to the L0 ball.

The path from L0 → L1 → L2 corresponds to increasingly relaxed sparsity constraints:

- L0: exactly sparse (combinatorial)
- L1: tends to be sparse (convex, tractable)
- L2: never exactly sparse (smooth)

### 6.2 In Deep Learning: Implicit Feature Selection

Deep learning does not use explicit feature selection in the classical sense. Instead, several mechanisms implicitly select features:

**Dropout:** Randomly zeroes activations during training. This prevents the network from relying on any single neuron (or feature), equivalent to learning a model that is robust to feature removal. It is closest in spirit to stability selection.

**Attention mechanisms (Transformers):** Attention weights $\alpha_{ij}$ measure how much position $j$ is attended to when producing the representation for position $i$. High attention weight ≈ important feature. This is a _soft_, differentiable form of feature selection, learned end-to-end.

**Sparse autoencoders:** L1 penalty on the latent activations produces sparse representations — only a few neurons activate for each input. The active neurons are the "selected features" for that input. This is the basis of mechanistic interpretability research on language models (Anthropic's "Towards Monosemanticity" and "Scaling Monosemanticity" papers).

**Feature importance in XGBoost/LightGBM:** These models compute MDI and permutation importance natively. The standard practice in tabular ML competitions (Kaggle) is to train a tree ensemble, extract feature importances, remove the bottom fraction, retrain, and repeat. This is iterative embedded feature selection.

### 6.3 The Irrelevance of Feature Selection to Overparameterised Models

Modern neural networks are _massively_ overparameterised — far more parameters than training examples. Classical learning theory says this should overfit catastrophically. It does not, due to the double descent phenomenon ([[04.Bias Variance Tradeoff]]) and implicit regularisation from gradient descent (SGD finds flat minima).

In this regime, explicit feature selection is less critical: the model can in principle learn to ignore irrelevant features by setting their weights to near-zero. Whether it reliably does so is an active research question. In practice, for tabular data with modest $n$, feature selection still provides measurable benefits even for neural networks.

## ✦ 7. Decision Guide: Which Method?

```
How many features p?
│
├── p < 20
│     → Exhaustive search or Wrapper (forward/backward) is feasible
│       Use RFECV with cross-validation
│
├── 20 ≤ p ≤ 500
│     → Embedded methods: Lasso / LassoCV, or RFECV with step > 1
│       Validate with permutation importance on held-out data
│
├── 500 < p ≤ 10,000
│     → Filter first (MI or variance threshold) to get to p ~ 200
│       Then embedded (Lasso or tree importance)
│       Use Pipeline to prevent leakage
│
└── p > 10,000 (text, genomics, wide tabular)
      → Variance threshold + hashing (text)
        Lasso with coordinate descent (scales to p ~ 10^6)
        Filter: chi-squared or MI for quick pruning
```

**Are features correlated?**

- Lasso arbitrarily picks one from correlated groups → use Elastic Net instead
- MDI is unreliable → use permutation importance

**Is interpretability required?**

- Yes → Lasso (sparse linear model), RFECV with logistic regression
- No → Permutation importance + any model

## ✦ 8. Worked Example: Full Pipeline

```python
import numpy as np
from sklearn.datasets import make_classification
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.impute import SimpleImputer
from sklearn.feature_selection import SelectFromModel, RFECV
from sklearn.linear_model import LassoCV, LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.inspection import permutation_importance
from sklearn.model_selection import (
    train_test_split, StratifiedKFold, cross_val_score
)
from sklearn.metrics import classification_report

# ── 1. Simulate data: 200 samples, 50 features, 10 informative ───────────────
X, y = make_classification(
    n_samples=200, n_features=50, n_informative=10,
    n_redundant=10, n_repeated=5, random_state=42
)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# ── 2. Strategy A: Lasso embedded selection ───────────────────────────────────
lasso_pipe = Pipeline([
    ('impute',  SimpleImputer(strategy='median')),
    ('scale',   StandardScaler()),
    ('select',  SelectFromModel(
                    LassoCV(cv=5, random_state=42, max_iter=5000),
                    threshold='mean'
                )),
    ('model',   LogisticRegression(C=1.0, max_iter=1000)),
])

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores_lasso = cross_val_score(lasso_pipe, X_train, y_train, cv=cv, scoring='f1')
print(f"Lasso pipeline CV F1: {scores_lasso.mean():.3f} ± {scores_lasso.std():.3f}")

# ── 3. Strategy B: Tree importance + permutation validation ──────────────────
rf = RandomForestClassifier(n_estimators=200, random_state=42)
rf.fit(X_train, y_train)

# Permutation importance on held-out validation split
X_tr, X_val, y_tr, y_val = train_test_split(
    X_train, y_train, test_size=0.2, stratify=y_train, random_state=0
)
rf.fit(X_tr, y_tr)
perm = permutation_importance(rf, X_val, y_val, n_repeats=20,
                               scoring='f1', random_state=42)

# Keep features where importance mean > 1 std above zero
perm_mask = perm.importances_mean > perm.importances_std

print(f"\nPermutation-selected features: {perm_mask.sum()} / {X_train.shape[1]}")
print(f"Expected ~10 informative features.")

# ── 4. Evaluate on test set (spend once) ─────────────────────────────────────
lasso_pipe.fit(X_train, y_train)
y_pred = lasso_pipe.predict(X_test)
print("\n── Lasso Pipeline Test Performance ──")
print(classification_report(y_test, y_pred))
```

## ✦ Key Facts and Theorems

**Fact 1 (L0-L1 relaxation):** The L1 ball is the convex hull of the L0 "ball" (union of coordinate axes). Minimising over the L1 constraint is the tightest convex relaxation of minimising over the L0 constraint. This is why Lasso is the principled convex surrogate for subset selection.

**Fact 2 (Mutual information and sufficiency):** $I(X_j; Y) = 0 \iff X_j \perp Y$. But $I(X_j; Y | X_{-j}) = 0$ (conditional independence given all other features) is the correct criterion for irrelevance in a multivariate model. Filter methods compute marginal MI; the conditional version is computationally intractable for large $p$.

**Fact 3 (Lasso consistency):** Under appropriate conditions (the "irrepresentable condition" on the design matrix $X$), the Lasso with $\lambda \propto \sqrt{\log p / n}$ recovers the true support (set of nonzero coefficients) with probability → 1 as $n \to \infty$. This is the Lasso's model selection consistency theorem (Zhao & Yu, 2006).

**Fact 4 (No free lunch for feature selection):** There is no universally best feature selection method. The optimal method depends on the true data-generating process — specifically, on the structure of the relationship between features and target. This is the No Free Lunch theorem applied to feature selection.
