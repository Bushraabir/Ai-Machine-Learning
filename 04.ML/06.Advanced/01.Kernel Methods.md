## The Central Problem

Most machine learning algorithms assume a fixed, finite-dimensional feature representation. You hand the model a vector $x \in \mathbb{R}^d$, and it does something with it. But what if the data is not linearly separable in $\mathbb{R}^d$? The naive solution: **map to a higher-dimensional space** where it is.

$$\phi: \mathbb{R}^d \to \mathbb{R}^D, \quad D \gg d \quad (\text{possibly } D = \infty)$$

The brutal fact: computing in $\mathbb{R}^D$ is expensive. If $D = d^2$ (all pairwise products), you're already in trouble for modest $d$. If $D = \infty$ — you literally cannot compute $\phi(x)$ explicitly.

The kernel trick resolves this: **you never need $\phi(x)$ explicitly**. You only ever need inner products $\langle \phi(x_i), \phi(x_j) \rangle$, and a kernel function computes this cheaply.

$$k(x_i, x_j) \stackrel{\text{def}}{=} \langle \phi(x_i), \phi(x_j) \rangle_\mathcal{H}$$

## 1. Feature Maps and the Kernel Function

### Setup

Let $\mathcal{X}$ be the input space (e.g., $\mathbb{R}^d$). A **feature map** is $\phi: \mathcal{X} \to \mathcal{H}$ where $\mathcal{H}$ is a Hilbert space (a complete inner product space — possibly infinite dimensional).

A **kernel function** is any function $k: \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ such that:

$$k(x, x') = \langle \phi(x), \phi(x') \rangle_\mathcal{H}$$

for some feature map $\phi$. The point: we define $k$ first, then the feature map $\phi$ is _implied_ by $k$ — we may never need to construct it.

### Canonical Example: The Polynomial Kernel

Let $x, x' \in \mathbb{R}^2$. Define:

$$k(x, x') = (x^\top x' + c)^2, \quad c \geq 0$$

Expand for $x = (x_1, x_2)^\top$, $x' = (x_1', x_2')^\top$, $c = 1$:

$$k(x, x') = (x_1 x_1' + x_2 x_2' + 1)^2$$ $$= x_1^2 x_1'^2 + x_2^2 x_2'^2 + 1 + 2x_1 x_1' x_2 x_2' + 2x_1 x_1' + 2x_2 x_2'$$

This is precisely $\langle \phi(x), \phi(x') \rangle$ where:

$$\phi(x) = (x_1^2,\ x_2^2,\ 1,\ \sqrt{2}x_1 x_2,\ \sqrt{2}x_1,\ \sqrt{2}x_2)^\top \in \mathbb{R}^6$$

**Key observation:** computing $k(x, x')$ costs $O(d)$ — one inner product and a squaring. Computing $\langle \phi(x), \phi(x') \rangle$ directly costs $O(D)$ where $D = \binom{d+2}{2}$ grows quadratically in $d$. The kernel saves us this cost.

### The RBF (Gaussian) Kernel — An Infinite-Dimensional Feature Map

$$k(x, x') = \exp!\left(-\frac{|x - x'|^2}{2\sigma^2}\right)$$

**Claim:** This corresponds to an _infinite-dimensional_ feature map.

**Proof sketch:** Expand via the Taylor series of $\exp$:

$$\exp!\left(-\frac{|x|^2 + |x'|^2 - 2x^\top x'}{2\sigma^2}\right) = e^{-|x|^2/2\sigma^2} \cdot e^{-|x'|^2/2\sigma^2} \cdot e^{x^\top x'/\sigma^2}$$

The last factor: $e^{x^\top x'/\sigma^2} = \sum_{n=0}^\infty \frac{(x^\top x')^n}{n! \sigma^{2n}}$. Each term $(x^\top x')^n$ corresponds to a degree-$n$ polynomial kernel, which has its own feature map of dimension $\binom{d+n}{n}$. Summing infinitely many such maps gives an infinite-dimensional $\phi$. Yet $k(x, x')$ is a single exponential — $O(d)$ to evaluate. $\square$

This is the fundamental power: the feature space can be infinite-dimensional while the kernel evaluation remains cheap.

## 2. Mercer's Theorem — When is $k$ a Valid Kernel?

Not every function $k: \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ corresponds to a valid inner product. We need a characterisation.

### Definition: Positive Semi-Definite Kernel

A symmetric function $k$ is a **positive semi-definite (PSD) kernel** (also called a **Mercer kernel**) if for any finite set ${x_1, \ldots, x_n} \subset \mathcal{X}$ and any $c \in \mathbb{R}^n$:

$$\sum_{i=1}^n \sum_{j=1}^n c_i c_j k(x_i, x_j) \geq 0$$

Equivalently, the **Gram matrix** $K \in \mathbb{R}^{n \times n}$ with $K_{ij} = k(x_i, x_j)$ is positive semi-definite for all choices of ${x_1, \ldots, x_n}$.

### Mercer's Theorem (Statement)

Let $k: \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ be continuous and symmetric. Then $k$ is a valid kernel (i.e., $k(x, x') = \langle \phi(x), \phi(x') \rangle_\mathcal{H}$ for some Hilbert space $\mathcal{H}$ and feature map $\phi$) **if and only if** $k$ is positive semi-definite.

> **Proof direction (necessity):** If $k(x,x') = \langle \phi(x), \phi(x') \rangle$, then $\sum_{i,j} c_i c_j k(x_i, x_j) = \left|\sum_i c_i \phi(x_i)\right|^2 \geq 0$. $\square$
> 
> The sufficiency direction is harder — it constructs the RKHS explicitly from $k$. See Section 4.

### How to Check Validity

1. **Directly verify PSD** on the Gram matrix.
2. **Use closure properties** (easier in practice):
    - If $k_1, k_2$ are valid kernels, so are $k_1 + k_2$, $\alpha k_1$ ($\alpha > 0$), $k_1 \cdot k_2$
    - $k(x, x') = f(x) k_1(x, x') f(x')$ for any function $f$
    - $k(x, x') = \phi(x)^\top A \phi(x')$ where $A \succeq 0$

**Common valid kernels:**

|Kernel|Formula|Hyperparameters|
|---|---|---|
|Linear|$x^\top x'$|—|
|Polynomial|$(x^\top x' + c)^d$|$c \geq 0$, $d \in \mathbb{N}$|
|RBF (Gaussian)|$\exp(-\|x-x'\|^2 / 2\sigma^2)$|$\sigma > 0$|
|Matérn|(smoothness-controlled)|$\nu, \ell > 0$|
|Laplace|$\exp(-\|x-x'\| / \sigma)$|$\sigma > 0$|
|Sigmoid|$\tanh(\alpha x^\top x' + c)$|only PSD for some $\alpha, c$ — careful!|

## 3. The Representer Theorem

This is the theorem that _justifies_ using kernels for learning. It says: **when you minimise a regularised loss over a function space, the optimal solution lives in the span of the training data's feature maps.**

### Theorem (Representer Theorem — Schölkopf, Herbrich, Smola, 2001)

Let $\mathcal{H}$ be an RKHS with kernel $k$. Consider the optimisation problem:

$$\hat{f} = \arg\min_{f \in \mathcal{H}} \underbrace{\sum_{i=1}^n L(y_i, f(x_i))}_{\text{empirical loss}} + \underbrace{\Omega(|f|_\mathcal{H})}_{\text{regularisation}}$$

where $L$ is any loss function and $\Omega: [0, \infty) \to \mathbb{R}$ is any strictly increasing function. Then the optimal $\hat{f}$ has the form:

$$\hat{f}(\cdot) = \sum_{i=1}^n \alpha_i k(\cdot, x_i)$$

for some coefficients $\alpha_1, \ldots, \alpha_n \in \mathbb{R}$.

### Proof

Decompose $f \in \mathcal{H}$ into a component in the span of ${k(\cdot, x_i)}$ and its orthogonal complement:

$$f = \underbrace{\sum_{i=1}^n \alpha_i k(\cdot, x_i)}_{f_\parallel} + f_\perp$$

By the **reproducing property** of the RKHS (see Section 4):

$$f(x_j) = \langle f, k(\cdot, x_j) \rangle_\mathcal{H} = \langle f_\parallel, k(\cdot, x_j) \rangle_\mathcal{H} + \underbrace{\langle f_\perp, k(\cdot, x_j) \rangle_\mathcal{H}}_{= 0} = f_\parallel(x_j)$$

So adding $f_\perp$ does not change any training prediction, but $|f|^2_\mathcal{H} = |f_\parallel|^2 + |f_\perp|^2 \geq |f_\parallel|^2$, which means $f_\perp$ only _hurts_ the regularisation term. Therefore the optimal solution has $f_\perp = 0$. $\square$

**What this buys you:** instead of optimising over an infinite-dimensional function space, you optimise over $n$ real numbers $\alpha_1, \ldots, \alpha_n$. The learning problem reduces to finite-dimensional optimisation.

## 4. The Reproducing Kernel Hilbert Space (RKHS)

This is the natural home for kernel methods. Understanding the RKHS is understanding _why_ the whole framework is coherent.

### Construction

Given a PSD kernel $k$, we construct the RKHS $\mathcal{H}_k$ as follows:

**Step 1.** Start with the set of all "feature functions": for each $x \in \mathcal{X}$, define $k_x := k(\cdot, x)$ as a function $\mathcal{X} \to \mathbb{R}$.

**Step 2.** Consider the linear span: $\mathcal{F} = \left{ \sum_{i=1}^m \alpha_i k_{x_i} \mid m \in \mathbb{N},\ \alpha_i \in \mathbb{R},\ x_i \in \mathcal{X} \right}$

**Step 3.** Define an inner product on $\mathcal{F}$:

$$\left\langle \sum_i \alpha_i k_{x_i},\ \sum_j \beta_j k_{x_j'} \right\rangle_\mathcal{H} := \sum_i \sum_j \alpha_i \beta_j k(x_i, x_j')$$

This is well-defined because $k$ is PSD.

**Step 4.** Complete $\mathcal{F}$ under the induced norm $|f|_\mathcal{H} = \sqrt{\langle f, f \rangle_\mathcal{H}}$ to get the Hilbert space $\mathcal{H}_k$.

### The Reproducing Property

The defining property of the RKHS, which gives it its name:

$$f(x) = \langle f, k(\cdot, x) \rangle_\mathcal{H} \quad \forall f \in \mathcal{H}_k,\ x \in \mathcal{X}$$

In other words: **evaluating a function $f$ at a point $x$ is an inner product with the kernel function $k(\cdot, x)$**. This is the reason the representer theorem proof works.

**Sanity check:** Set $f = k(\cdot, x')$. Then:

$$\langle k(\cdot, x'), k(\cdot, x) \rangle_\mathcal{H} = k(x', x) \checkmark$$

The RKHS norm $|f|_\mathcal{H}$ has a concrete meaning: **it measures the smoothness of $f$ with respect to the kernel**. The smoother the function, the smaller the norm. Regularisation with $|f|_\mathcal{H}^2$ is literally a smoothness penalty.

## 5. Kernelised Algorithms

The representer theorem + the kernel trick allows us to kernelise any algorithm that depends on the data only through inner products. We replace $x_i^\top x_j \mapsto k(x_i, x_j)$ everywhere.

### Kernel Ridge Regression

**Primal:** $\min_{w \in \mathcal{H}} |Xw - y|^2 + \lambda |w|^2$

**Primal solution:** $\hat{w} = (X^\top X + \lambda I)^{-1} X^\top y$ — works in $\mathbb{R}^d$, breaks for infinite-dim $\phi$.

**Dual:** By the representer theorem, $\hat{f} = \sum_i \alpha_i k(\cdot, x_i)$. Substituting:

$$\hat{\alpha} = (K + \lambda I)^{-1} y$$

where $K_{ij} = k(x_i, x_j)$ is the $n \times n$ Gram matrix. Prediction:

$$\hat{f}(x^_) = \sum_i \hat{\alpha}_i k(x^_, x_i) = k(x^*)^\top (K + \lambda I)^{-1} y$$

where $k(x^_)_i = k(x^_, x_i)$.

**Cost:** $O(n^3)$ to invert $(K + \lambda I)$ vs. $O(d^3)$ for ridge regression. Kernel methods trade $d$ for $n$ in complexity — they scale with the _number of samples_, not the feature dimensionality.

**Connection to Gaussian Processes:** The kernel ridge regression predictor is identical to the GP posterior mean under an RBF prior. This is not a coincidence — they are the same model derived from different perspectives.

### Kernel SVM

Recall the SVM dual:

$$\max_\alpha \sum_i \alpha_i - \frac{1}{2} \sum_{i,j} \alpha_i \alpha_j y_i y_j x_i^\top x_j \quad \text{s.t. } \sum_i \alpha_i y_i = 0,\ 0 \leq \alpha_i \leq C$$

Replace $x_i^\top x_j \mapsto k(x_i, x_j)$:

$$\max_\alpha \sum_i \alpha_i - \frac{1}{2} \sum_{i,j} \alpha_i \alpha_j y_i y_j k(x_i, x_j) \quad \text{s.t. } \sum_i \alpha_i y_i = 0,\ 0 \leq \alpha_i \leq C$$

Decision function:

$$\hat{y}(x) = \text{sign}!\left(\sum_{i \in \text{SV}} \alpha_i y_i k(x, x_i) + b\right)$$

We never need $\phi$ explicitly. We never need to work in the feature space. Everything is mediated by $k$.

### Kernel Logistic Regression

Replace the weight vector $w$ with $\sum_i \alpha_i k(\cdot, x_i)$ and optimise over $\alpha$. The log-likelihood becomes:

$$\ell(\alpha) = \sum_i \log \sigma!\left(y_i \sum_j \alpha_j k(x_i, x_j)\right) - \lambda \alpha^\top K \alpha$$

The regularisation term $\alpha^\top K \alpha = |f|^2_\mathcal{H}$ — this is the RKHS norm.

### Kernel PCA

Standard PCA: find eigenvectors of the covariance matrix $C = \frac{1}{n} X^\top X$.

Kernel PCA: find principal components in the RKHS by computing the eigendecomposition of the _centred_ Gram matrix $\tilde{K}$:

$$\tilde{K} = K - \mathbf{1}_n K - K \mathbf{1}_n + \mathbf{1}_n K \mathbf{1}_n$$

where $(\mathbf{1}_n)_{ij} = 1/n$. This centres the data in feature space without constructing $\phi(x)$ explicitly.

The projection of a new point $x^*$ onto the $j$-th principal component:

$$z_j(x^_) = \sum_i \alpha_j^{(i)} k(x^_, x_i)$$

## 6. Kernel Design — Encoding Prior Knowledge

The choice of kernel is a choice of **inductive bias**. When you pick the RBF kernel, you're saying: "I believe the function is smooth, and similar inputs should have similar outputs, where similarity decays exponentially with Euclidean distance." Different kernels encode different beliefs.

### Stationary Kernels

$k(x, x') = k(x - x')$ — depends only on the _difference_ between inputs. Implies translation invariance (stationarity).

- **RBF:** $k(r) = \exp(-r^2/2\sigma^2)$ — infinitely differentiable. Use when the target function is very smooth.
- **Laplace:** $k(r) = \exp(-|r|/\sigma)$ — only once-differentiable. Better for non-smooth functions.
- **Matérn-$\nu$:** interpolates between Laplace ($\nu = 1/2$) and RBF ($\nu \to \infty$). The parameter $\nu$ controls differentiability.

### Non-stationary Kernels

- **Linear:** $k(x, x') = x^\top x'$ — equivalent to linear models. No nonlinearity.
- **Polynomial:** encodes feature interactions up to degree $d$.
- **Periodic:** $k(x, x') = \exp(-2\sin^2(\pi|x-x'|/p) / \ell^2)$ — for periodic functions (e.g., seasonal data).

### Kernels for Structured Data

The true power of kernels: **they extend machine learning to non-Euclidean inputs** by defining similarity directly.

- **String kernels:** $k(s, s') = $ number of common substrings of length $p$. Used in text classification and bioinformatics (protein sequences).
- **Graph kernels:** $k(G, G') = $ a function of shared subgraph patterns. Used in chemoinformatics (molecular property prediction).
- **Tree kernels:** for parse trees in NLP.

The definition of valid kernel is preserved: as long as the Gram matrix is PSD, any similarity function is a valid kernel — no matter how exotic the input space.

## 7. Computational Considerations

### The $O(n^3)$ Problem

Kernel methods require computing and storing the $n \times n$ Gram matrix $K$. For large $n$:

- Storage: $O(n^2)$ — infeasible at $n = 10^6$
- Inversion (kernel ridge): $O(n^3)$ — infeasible much earlier
- This is why kernel methods fell out of favour when deep learning arrived

### Approximate Kernel Methods

**Nyström approximation:** approximate $K$ using a low-rank factorisation:

$$K \approx K_{nm} K_{mm}^{-1} K_{nm}^\top$$

where $m \ll n$ are randomly selected "landmark" points. Reduces to $O(nm^2)$ computation.

**Random Fourier Features (Rahimi & Recht, 2007):** for shift-invariant kernels, Bochner's theorem guarantees:

$$k(x, x') = \mathbb{E}_\omega!\left[e^{i\omega^\top(x-x')}\right]$$

Sample $D$ random frequencies $\omega_1, \ldots, \omega_D \sim p(\omega)$ and construct:

$$z(x) = \sqrt{\frac{2}{D}}!\left(\cos(\omega_1^\top x + b_1), \ldots, \cos(\omega_D^\top x + b_D)\right)^\top$$

Then $k(x, x') \approx z(x)^\top z(x')$. This reduces to a standard linear method in $\mathbb{R}^D$, costing $O(nD)$.

## 8. Connection to AI, Deep Learning, and Neural Networks

Kernel methods and deep learning are more deeply connected than they might appear. Understanding this connection is important for a modern practitioner.

### Neural Tangent Kernel (NTK)

**The central result (Jacot, Gabriel, Hongler, 2018):** An infinitely wide neural network trained by gradient descent is equivalent to kernel regression with the **Neural Tangent Kernel**:

$$k_\text{NTK}(x, x') = \mathbb{E}_{\theta \sim \text{init}}!\left[\nabla_\theta f(x;\theta)^\top \nabla_\theta f(x';\theta)\right]$$

At infinite width:

- The NTK stays constant throughout training (it doesn't change as weights update)
- The network converges to a global minimum
- The final predictor is a kernel regression predictor with kernel $k_\text{NTK}$

**What this means practically:** For very wide networks, the NTK regime predicts behaviour well. For finite-width networks (real practice), the NTK drifts during training — this is called **feature learning** and is what makes practical deep learning powerful beyond kernel methods. The kernel regime cannot learn new features; it is fixed by the initialisation.

### Deep Kernels

One response to the expressive power gap: **learn** the kernel. Define:

$$k(x, x') = k_\text{base}(\phi_\theta(x), \phi_\theta(x'))$$

where $\phi_\theta$ is a neural network and $k_\text{base}$ is a standard kernel (e.g., RBF). Train $\theta$ jointly with the kernel machine. This is the basis of **Deep Kernel Learning** (Wilson et al., 2016) and gives a principled Bayesian uncertainty estimate from a deep model.

### SVMs vs. Neural Networks — Historical Perspective

In the 1990s, SVMs were the dominant large-margin classifier. The shift to neural networks was driven by:

1. **Scalability:** Deep learning trains in $O(n)$ per epoch via SGD; kernel SVMs scale as $O(n^2)$–$O(n^3)$.
2. **Representation learning:** CNNs learn the right features for images; SVMs with RBF kernels cannot exploit spatial structure as efficiently.
3. **Compositionality:** Deep networks compose simple functions hierarchically; kernels are a single layer of feature mapping.

Yet SVMs with carefully designed kernels still dominate in domains with small $n$ and structured data (e.g., genomics, cheminformatics). The choice is not doctrinal; it's empirical.

### Kernel Machines in Modern ML

- **Gaussian Processes** (used in hyperparameter optimisation via Bayesian optimisation, as uncertainty-aware surrogates in RL and AutoML) are fully defined by their kernel.
- **Graph Neural Networks** can be understood as learnable graph kernels.
- **Attention in Transformers:** the scaled dot-product attention $\text{softmax}(QK^\top / \sqrt{d_k})V$ is a _data-dependent_ kernel. $Q^\top K / \sqrt{d_k}$ computes kernel similarities between query and key representations. This is the connection that has been formalised in **kernel attention** and **Performer** architectures.
- **Support Vector Regression** (SVR) is still used as a strong baseline in time series and scientific ML.

## 9. Worked Example — Kernel SVM vs. Linear SVM

Consider the XOR problem: 4 points in $\mathbb{R}^2$:

$$x_1 = (1,1),\ y_1 = +1 \quad x_2 = (-1,-1),\ y_2 = +1$$ $$x_3 = (1,-1),\ y_3 = -1 \quad x_4 = (-1,1),\ y_4 = -1$$

**Linear SVM:** not separable in $\mathbb{R}^2$ — no hyperplane separates the classes.

**Polynomial kernel of degree 2:** $k(x, x') = (x^\top x')^2$. The feature map is (for $c=0$):

$$\phi(x_1, x_2) = (x_1^2, x_2^2, \sqrt{2} x_1 x_2)$$

In this space:

- $\phi(1,1) = (1, 1, \sqrt{2})$, class $+1$
- $\phi(-1,-1) = (1, 1, \sqrt{2})$, class $+1$
- $\phi(1,-1) = (1, 1, -\sqrt{2})$, class $-1$
- $\phi(-1,1) = (1, 1, -\sqrt{2})$, class $-1$

Now the classes separate perfectly along the third coordinate $\sqrt{2}x_1 x_2$: positive class has $x_1 x_2 > 0$, negative has $x_1 x_2 < 0$. The hyperplane is $w_3 z_3 = 0$, i.e., the product $x_1 x_2 = 0$.

## 10. Implementation from Scratch

```python
import numpy as np

# ── Kernel functions ──────────────────────────────────────────────────────────
def rbf_kernel(X1, X2, sigma=1.0):
    """
    Compute the RBF Gram matrix between rows of X1 and rows of X2.
    K[i,j] = exp(-||X1[i] - X2[j]||^2 / (2 * sigma^2))
    """
    # ||a - b||^2 = ||a||^2 + ||b||^2 - 2 a.b
    sq_dist = (np.sum(X1**2, axis=1, keepdims=True)
               + np.sum(X2**2, axis=1)
               - 2 * X1 @ X2.T)
    return np.exp(-sq_dist / (2 * sigma**2))

def polynomial_kernel(X1, X2, degree=2, c=1.0):
    return (X1 @ X2.T + c) ** degree

def linear_kernel(X1, X2):
    return X1 @ X2.T

# ── Kernel Ridge Regression ───────────────────────────────────────────────────
class KernelRidgeRegression:
    """
    Dual form: alpha = (K + lambda * I)^{-1} y
    Prediction: f(x*) = k(x*)^T alpha
    """
    def __init__(self, kernel=rbf_kernel, lam=1.0, **kernel_kwargs):
        self.kernel = kernel
        self.lam = lam
        self.kw = kernel_kwargs

    def fit(self, X, y):
        self.X_train = X
        n = X.shape[0]
        K = self.kernel(X, X, **self.kw)
        self.alpha = np.linalg.solve(K + self.lam * np.eye(n), y)
        return self

    def predict(self, X_test):
        K_star = self.kernel(X_test, self.X_train, **self.kw)
        return K_star @ self.alpha

# ── Kernel PCA ────────────────────────────────────────────────────────────────
class KernelPCA:
    """
    1. Compute Gram matrix K
    2. Centre in feature space: K_tilde = K - 1_n K - K 1_n + 1_n K 1_n
    3. Eigendecompose K_tilde
    4. Project: Z = E_k Λ_k^{1/2}  (keep top k eigenvectors)
    """
    def __init__(self, n_components=2, kernel=rbf_kernel, **kernel_kwargs):
        self.n_components = n_components
        self.kernel = kernel
        self.kw = kernel_kwargs

    def fit_transform(self, X):
        self.X_train = X
        n = X.shape[0]
        K = self.kernel(X, X, **self.kw)

        # Centre the Gram matrix
        one_n = np.ones((n, n)) / n
        self.K_mean_rows = K.mean(axis=0)
        self.K_mean = K.mean()
        K_tilde = K - one_n @ K - K @ one_n + one_n @ K @ one_n

        # Eigendecomposition (K_tilde is symmetric PSD)
        eigvals, eigvecs = np.linalg.eigh(K_tilde)
        # Sort descending
        idx = np.argsort(eigvals)[::-1]
        eigvals, eigvecs = eigvals[idx], eigvecs[:, idx]

        self.eigvals = eigvals[:self.n_components]
        self.eigvecs = eigvecs[:, :self.n_components]

        return self.eigvecs * np.sqrt(np.maximum(self.eigvals, 0))

# ── Usage example ─────────────────────────────────────────────────────────────
if __name__ == "__main__":
    rng = np.random.default_rng(42)
    n = 100
    # Two concentric circles — not linearly separable
    theta = rng.uniform(0, 2*np.pi, n)
    r = rng.choice([1.0, 2.5], n)
    X = np.column_stack([r * np.cos(theta), r * np.sin(theta)])
    y = (r > 1.5).astype(float) * 2 - 1   # +1 outer, -1 inner

    # Kernel ridge regression with RBF kernel
    model = KernelRidgeRegression(kernel=rbf_kernel, lam=0.1, sigma=1.0)
    model.fit(X, y)
    preds = np.sign(model.predict(X))
    print(f"Accuracy: {(preds == y).mean():.2%}")

    # Kernel PCA for visualisation
    kpca = KernelPCA(n_components=2, kernel=rbf_kernel, sigma=1.0)
    Z = kpca.fit_transform(X)
    print(f"Projected shape: {Z.shape}")
```

## 11. Visualisation — The Kernel Trick Geometrically

```
Original space ℝ²                   Feature space ℝ^D (or ∞-dim)
(not linearly separable)             (linearly separable)

    ●  ○                             ○  ○  ○
  ○   ●   ○          φ: x → φ(x)    ──────────  ← separating hyperplane
    ●  ○                             ●  ●  ●

The kernel trick: we find this hyperplane WITHOUT ever moving to the
feature space. We work entirely with k(xᵢ, xⱼ) in the original space.

Decision boundary in ℝ² (from SVM dual):
  f(x) = Σᵢ αᵢ yᵢ k(x, xᵢ) + b = 0

This is a *nonlinear* curve in ℝ², induced by the kernel.
With RBF kernel, σ → 0: memorises every point (overfit)
With RBF kernel, σ → ∞: approaches linear classifier (underfit)
```

**The bias-variance view of $\sigma$ in the RBF kernel:**

|$\sigma$ small|$\sigma$ large|
|---|---|
|Narrow Gaussians|Wide Gaussians|
|High variance (overfitting)|High bias (underfitting)|
|Complex decision boundary|Simple (near-linear) boundary|
|Few effective support vectors|Many support vectors|

## 12. Summary and Exit Criteria

**The logical chain:**

$$\text{PSD kernel} \xrightarrow{\text{Mercer}} \text{valid inner product} \xrightarrow{\text{RKHS construction}} \text{Hilbert space } \mathcal{H}_k \xrightarrow{\text{Representer theorem}} \text{finite-dim optimisation over } \alpha \in \mathbb{R}^n$$

**The kernel trick reduces any algorithm that uses data only through inner products to kernel evaluations.** Kernelise the inner products, and the algorithm works in an implicit infinite-dimensional space at the cost of $O(n^2)$ storage and $O(n^3)$ computation.
