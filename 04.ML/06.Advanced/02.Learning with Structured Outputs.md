
> _"Most of what matters in the world has structure. A sentence is not a bag of words. A protein is not a bag of amino acids. The challenge is to make the model know this."_

## ✦ Why This Note Exists in the Map

Standard supervised learning assumes $\mathcal{Y}$ is simple: a scalar (regression) or a finite label (classification). But the real world rarely cooperates. Consider:

- **Sequence labelling:** given a sentence, tag each word as `NOUN`, `VERB`, `ADJ`, ... — the output is a sequence of the same length as the input.
- **Parsing:** given a sentence, produce a parse tree — the output is a variable-size tree.
- **Image segmentation:** given an image, label each pixel — the output has $H \times W$ interdependent labels.
- **Translation:** given an English sentence, produce a German sentence of different length.

In all of these, **the output components are not independent given the input.** The label of word $i$ depends on the label of word $i-1$. Predicting each output independently (the "flat" approach) discards this structure and produces incoherent outputs.

This note builds the machinery to handle structured outputs **properly**, with rigour. Two major frameworks:

1. **Linear-Chain Conditional Random Fields (CRFs)** — for sequence labelling.
2. **Structured SVMs** — generalising the SVM margin to arbitrary output spaces.

## ✦ 1. The Structured Prediction Problem — Setting Up Formally

**Setup.** Let:

- $\mathcal{X}$ = input space (e.g., sequences of words)
- $\mathcal{Y}$ = _structured_ output space (e.g., sequences of POS tags, trees, segmentation masks)
- $(x_i, y_i)_{i=1}^n \sim \mathcal{D}$ — training data

The goal is to learn $f: \mathcal{X} \to \mathcal{Y}$ that minimises some task-specific loss $\Delta(y, \hat{y})$ (e.g., Hamming loss on sequences, tree edit distance on parse trees).

**The key difficulty.** $|\mathcal{Y}|$ is exponentially large. For a sequence of length $T$ with $K$ possible labels per position, $|\mathcal{Y}| = K^T$. We cannot enumerate all outputs. Any tractable algorithm must exploit the _structure_ of $\mathcal{Y}$.

**The key insight.** We will define a _scoring function_ $s(x, y; \theta) \in \mathbb{R}$ and predict: $$\hat{y} = \arg\max_{y \in \mathcal{Y}} s(x, y; \theta)$$

If $s$ _decomposes_ over the structure of $y$, then this maximisation — called _inference_ or _decoding_ — can be solved efficiently by dynamic programming.

## ✦ 2. Sequence Labelling — The Problem in Concrete Form

> **Running example throughout this note:** Part-of-speech (POS) tagging.
> 
> Input: $x = [\text{`The''}, \text{`cat''}, \text{``sat''}]$ Output: $y = [\text{DET}, \text{NOUN}, \text{VERB}]$
> 
> The correct label of "sat" is `VERB`, and this is _influenced_ by the fact that the previous tag is `NOUN` — the model should know `NOUN → VERB` is a common transition.

**Why not just use a classifier at each position?** We could train a multi-class classifier $P(y_t | x)$ and apply it independently at each timestep. This ignores two critical signals:

1. **Transition patterns:** `DET → NOUN` is far more common than `DET → VERB`.
2. **Long-range context:** the label of position $t$ may depend on positions far away.

A _sequence model_ encodes both.

## ✦ 3. Conditional Random Fields (CRFs)

### 3.1 From Logistic Regression to CRF

Logistic regression for a single label: $$P(y | x) = \frac{\exp(w^\top \phi(x, y))}{\sum_{y'} \exp(w^\top \phi(x, y'))}$$

where $\phi(x, y)$ is a feature vector. This is a **log-linear model** (also called a maximum entropy model).

The linear-chain CRF is exactly this — but the output $y$ is now a _sequence_, and $\phi(x, y)$ is defined to decompose over the sequence. Define:

$$P(y | x; w) = \frac{1}{Z(x, w)} \exp!\left(\sum_{t=1}^T w^\top \phi(x, y_t, y_{t-1}, t)\right)$$

where: $$Z(x, w) = \sum_{y' \in \mathcal{Y}} \exp!\left(\sum_{t=1}^T w^\top \phi(x, y'_t, y'_{t-1}, t)\right)$$

is the _partition function_ (normalisation constant), and $\phi(x, y_t, y_{t-1}, t)$ is a feature vector that can look at the _entire input_ $x$ at position $t$, the current label $y_t$, and the previous label $y_{t-1}$.

> **Why "conditional"?** A CRF models $P(y | x)$ directly — it is _discriminative_. Compare to a Hidden Markov Model (HMM), which models $P(x, y)$ jointly — it is _generative_. The CRF does not need to model the distribution of inputs; it only models the output given input.

### 3.2 The Feature Function — What Goes In $\phi$

The power of a CRF lies in the **richness of the feature function** $\phi(x, y_t, y_{t-1}, t)$. Typical features:

|Feature type|Example feature|
|---|---|
|Emission|$\mathbf{1}[\text{word}_t = \text{"sat"} \wedge y_t = \text{VERB}]$|
|Transition|$\mathbf{1}[y_{t-1} = \text{NOUN} \wedge y_t = \text{VERB}]$|
|Prefix/suffix|$\mathbf{1}[\text{word}_t \text{ ends in "-ing"} \wedge y_t = \text{VERB}]$|
|Context window|$\mathbf{1}[\text{word}_{t+1} = \text{"."} \wedge y_t = \text{NOUN}]$|
|Capitalisation|$\mathbf{1}[\text{word}_t \text{ is capitalised} \wedge y_t = \text{NOUN}]$|

Each feature is an indicator function (0 or 1). The weight vector $w$ has one entry per feature. Training learns which features are informative.

**Contrast with HMMs:** An HMM must model $P(\text{word}_t | y_t)$ as a probability distribution over all words — it cannot naturally use arbitrary features of the input. A CRF can.

### 3.3 Training — Maximum Conditional Log-Likelihood

Given training data ${(x_i, y_i)}_{i=1}^n$, we maximise: $$\mathcal{L}(w) = \sum_{i=1}^n \log P(y_i | x_i; w) = \sum_{i=1}^n \left[\sum_{t=1}^{T_i} w^\top \phi(x_i, y_{it}, y_{i,t-1}, t) - \log Z(x_i, w)\right]$$

**Computing the gradient.** Taking $\nabla_w \mathcal{L}$: $$\nabla_w \mathcal{L}(w) = \sum_{i=1}^n \left[\underbrace{\sum_t \phi(x_i, y_{it}, y_{i,t-1}, t)}_{\text{observed feature counts}} - \underbrace{\mathbb{E}_{P(y|x_i;w)}!\left[\sum_t \phi(x_i, y_t, y_{t-1}, t)\right]}_{\text{expected feature counts}}\right]$$

The gradient is: **observed counts minus expected counts under the model**. This is the standard result for log-linear models — the same structure appears in logistic regression.

**The hard part:** computing the expectation — i.e., the partition function $Z(x, w)$ and the marginals $P(y_t, y_{t-1} | x; w)$ — over all $K^T$ possible sequences. We need dynamic programming.

### 3.4 The Forward-Backward Algorithm

Define the _score_ of a partial sequence ending with label $k$ at position $t$: $$\alpha_t(k) = \sum_{y_{1:t}: y_t = k} \exp!\left(\sum_{s=1}^t w^\top \phi(x, y_s, y_{s-1}, s)\right)$$

**Forward recursion:** $$\alpha_t(k) = \sum_{j=1}^K \alpha_{t-1}(j) \cdot \exp!\left(w^\top \phi(x, k, j, t)\right)$$

with base case $\alpha_1(k) = \exp(w^\top \phi(x, k, \text{START}, 1))$.

The partition function: $Z(x, w) = \sum_k \alpha_T(k)$.

**Backward recursion:** similarly define $\beta_t(k)$ summing over all extensions from position $t$ to $T$: $$\beta_t(k) = \sum_{j=1}^K \exp!\left(w^\top \phi(x, j, k, t+1)\right) \cdot \beta_{t+1}(j)$$

**Marginals:** $$P(y_t = k, y_{t-1} = j | x; w) = \frac{\alpha_{t-1}(j) \cdot \exp(w^\top \phi(x, k, j, t)) \cdot \beta_t(k)}{Z(x, w)}$$

Both forward and backward passes are $O(K^2 T)$ — polynomial in sequence length, not exponential.

**Complexity summary:**

|Operation|Complexity|
|---|---|
|Naïve (enumerate all $y$)|$O(K^T)$|
|Forward-backward (train)|$O(K^2 T)$|
|Viterbi (decode)|$O(K^2 T)$|

### 3.5 Decoding — The Viterbi Algorithm

At test time, we want: $$\hat{y} = \arg\max_{y \in \mathcal{Y}} \sum_t w^\top \phi(x, y_t, y_{t-1}, t)$$

This is the maximum weight path in a trellis — solved by the **Viterbi algorithm**, which replaces the $\sum$ in the forward pass with $\max$:

$$\delta_t(k) = \max_{y_{1:t-1}} \sum_{s=1}^t w^\top \phi(x, y_s, y_{s-1}, s) \Big|_{y_t = k}$$

**Viterbi recursion:** $$\delta_t(k) = \max_j \left[\delta_{t-1}(j) + w^\top \phi(x, k, j, t)\right]$$

Store the argmax at each step for backtracing.

```
Trellis diagram for POS tagging (T=3, K=3 tags: DET, NOUN, VERB):

t=1 (The)    t=2 (cat)    t=3 (sat)
  DET -------> DET -------> DET
    \         /|\          /|\
     \       / | \        / | \
  NOUN -----> NOUN -----> NOUN
     /       \ | /        \ | /
    /         \|/          \|/
  VERB -------> VERB -------> VERB

Each arrow = transition score + emission score.
Viterbi finds the maximum-weight path through this graph.
Optimal path here: DET → NOUN → VERB.
```

### 3.6 CRF vs. HMM — A Precise Comparison

|Property|HMM|Linear-Chain CRF|
|---|---|---|
|Model type|Generative: $P(x, y)$|Discriminative: $P(y \| x)$|
|Emission model|$P(x_t \| y_t)$ — one word given tag|$\phi(x, y_t, t)$ — arbitrary features of entire $x$|
|Transition|$P(y_t \| y_{t-1})$ — Markov|$\phi(y_t, y_{t-1})$ — same, but learned discriminatively|
|Training|EM (Baum-Welch) or MLE on counts|Gradient ascent on conditional log-likelihood|
|Decoding|Viterbi|Viterbi (identical structure)|
|Performance on NLP|Good baseline|Typically stronger (richer features)|

**The key advantage of CRFs:** you can add any features you want without worrying about whether they form a valid probability model. You cannot do this in an HMM.

## ✦ 4. Structured SVMs — Margin for Structured Outputs

### 4.1 Motivation

The CRF is a probabilistic model. The structured SVM is the _large-margin_ alternative — it extends the binary SVM to arbitrary output spaces. This is sometimes called **Structured Output Learning** or the **Max-Margin Markov Network** (M³N).

**Recall the binary SVM.** We want: $$w^\top \phi(x, y) - w^\top \phi(x, y') \geq 1 \quad \forall y' \neq y$$

The left side is the _score margin_ between the correct output and the best competitor. We want this margin to be at least 1.

### 4.2 The Structured SVM Objective

For structured outputs, we want the correct output $y_i$ to score higher than every other output $y' \in \mathcal{Y}$ by at least a _task-specific margin_ $\Delta(y_i, y')$:

$$w^\top \phi(x_i, y_i) \geq w^\top \phi(x_i, y') + \Delta(y_i, y') - \xi_i \quad \forall y' \neq y_i$$

where $\xi_i \geq 0$ is a slack variable and $\Delta(y_i, y')$ is the task loss (e.g., Hamming distance between two label sequences).

This gives the optimisation problem: $$\min_{w, \xi} \frac{1}{2}|w|^2 + C\sum_i \xi_i$$ $$\text{s.t.} \quad w^\top [\phi(x_i, y_i) - \phi(x_i, y')] \geq \Delta(y_i, y') - \xi_i \quad \forall i, \forall y' \in \mathcal{Y}$$

**The problem:** there are exponentially many constraints (one per output $y'$ per training example). We cannot enumerate them.

**The solution:** Cutting-plane algorithms maintain a _working set_ of active constraints and iterate:

1. Solve the QP with the current working set.
2. Find the _most violated constraint_ for each training example — this is the key step.
3. Add it to the working set.
4. Repeat until convergence.

### 4.3 The Loss-Augmented Inference Step

Finding the most violated constraint for example $i$ reduces to: $$y_i^* = \arg\max_{y' \in \mathcal{Y}} \left[\Delta(y_i, y') + w^\top \phi(x_i, y') - w^\top \phi(x_i, y_i)\right]$$

Dropping the constant $-w^\top \phi(x_i, y_i)$: $$y_i^* = \arg\max_{y' \in \mathcal{Y}} \left[\Delta(y_i, y') + w^\top \phi(x_i, y')\right]$$

This is called **loss-augmented inference** — it is inference (finding the best output) but with the task loss $\Delta$ added to the score. For Hamming loss on sequences, this decomposes position-by-position and can be solved with a modified Viterbi in $O(K^2 T)$ — the same complexity as standard decoding.

### 4.4 The Structured Hinge Loss

We can write the Structured SVM objective as minimising a regularised _structured hinge loss_: $$\mathcal{L}_{\text{struct}}(w) = \frac{1}{2}|w|^2 + C\sum_i \underbrace{\max_{y'} \left[\Delta(y_i, y') + w^\top \phi(x_i, y')\right] - w^\top \phi(x_i, y_i)}_{\text{structured hinge loss for example } i}$$

The structured hinge loss is zero when the correct output scores higher than all others _by the loss-weighted margin_, and positive otherwise.

**Comparison with binary SVM:**

||Binary SVM|Structured SVM|
|---|---|---|
|Output space|${-1, +1}$|Arbitrary $\mathcal{Y}$|
|Margin|1|$\Delta(y, y')$ (task-dependent)|
|Inference at test time|$\text{sign}(w^\top x)$|$\arg\max_{y} w^\top \phi(x, y)$|
|Training-time computation|Solve a QP|Solve a QP + loss-augmented inference|
|Key algorithm|SMO|Cutting-plane + Viterbi (for sequences)|

### 4.5 Connection: Structured SVM ↔ CRF

The structured SVM and the CRF are two approaches to the same problem. Their relationship:

- **CRF:** log-linear model, trained with maximum likelihood. Produces probability estimates.
- **Structured SVM:** margin-based model, trained with structured hinge loss. Does not produce probabilities.
- **Unifying perspective:** both learn a scoring function $s(x, y; w)$ of the form $w^\top \phi(x, y)$. They differ only in the loss function used to train $w$.

In practice, CRFs became the dominant approach in NLP because:

1. Probabilistic outputs are useful downstream.
2. Maximum likelihood has cleaner behaviour in the large-data regime.
3. Deep learning made it natural to parameterise $\phi(x, y_t, y_{t-1}, t)$ with a neural network — the **BiLSTM-CRF** and **Transformer-CRF** architectures.

## ✦ 5. Application: Named Entity Recognition (NER)

> **NER:** tag each word in a sentence with its entity type — `PER` (person), `ORG` (organisation), `LOC` (location), `O` (other).

Input: `["Barack", "Obama", "visited", "Paris"]` Output: `[B-PER, I-PER, O, B-LOC]`

(BIO tagging scheme: `B-` = beginning of entity, `I-` = inside, `O` = outside.)

**Why a CRF?** `B-PER` followed by `I-ORG` is illegal — the CRF's transition features learn to assign $-\infty$ weight to such transitions. A flat classifier would not know this.

**Features a linear-chain CRF might use:**

- Is the word capitalised? → strong signal for `PER` or `ORG`
- Does the word appear in a gazetteer of known cities? → signal for `LOC`
- Previous word is a title ("President", "Dr.")? → strong signal for `B-PER`
- Transition: `B-PER → I-PER` vs. `B-PER → O` — the CRF learns this

**Modern approach:** Replace the hand-crafted $\phi$ with a BiLSTM or Transformer to compute a contextual embedding at each position. The CRF layer sits on top:

```
Input:  [Barack,   Obama,    visited,  Paris  ]
         ↓         ↓         ↓         ↓
BiLSTM: [h_1,     h_2,      h_3,      h_4    ]  ← contextual embeddings
         ↓         ↓         ↓         ↓
Emit:   [e(h_1),  e(h_2),   e(h_3),   e(h_4) ]  ← emission scores (K-dim)
         ↓         ↓         ↓         ↓
CRF:    [B-PER,   I-PER,    O,        B-LOC  ]  ← globally optimal sequence
```

The CRF layer adds $K \times K$ transition parameters on top of the neural emission scores. Viterbi decoding is applied at the output.

## ✦ 6. Other Structured Output Spaces

### 6.1 Sequence-to-Sequence (different input/output length)

For translation, summarisation, and similar tasks, the output sequence length differs from the input. The linear-chain CRF cannot handle this — the output structure no longer aligns position-by-position with the input.

Solutions:

- **Encoder-decoder with attention** (the dominant approach): encode the input into a context vector, decode autoregressively. Not a CRF — instead uses a left-to-right language model.
- **CTC (Connectionist Temporal Classification):** for speech recognition, where output phonemes don't align to input frames.

### 6.2 Structured Outputs Over Trees

For constituency parsing, the output is a parse tree. The structured SVM generalises — you need:

- A feature function $\phi(x, \text{tree})$ that decomposes over tree nodes.
- An efficient inference algorithm — in this case, the CKY (Cocke-Kasami-Younger) algorithm for context-free grammars.

### 6.3 Image Segmentation

Each pixel receives a label. The output space is a grid. A **Markov Random Field (MRF)** or **CRF on a grid** is the natural model — neighbouring pixels tend to have the same label. Inference on grids is harder than on chains (NP-hard in general) and requires approximate methods (graph cuts, mean field).

## ✦ 7. Connection to AI / ML / Deep Learning

### 7.1 The BiLSTM-CRF — State of the Art Before Transformers

The model of Lample et al. (2016) set the standard for NER:

- BiLSTM encodes the sentence bidirectionally → contextual embeddings.
- CRF decodes the label sequence respecting transition constraints.

This architecture _is_ a structured prediction model where $\phi(x, y_t, y_{t-1}, t)$ is parameterised by a neural network. The Viterbi and forward-backward algorithms run exactly as above — the only difference is that emission scores come from a BiLSTM rather than hand-crafted features.

### 7.2 Transformers and Structured Outputs

Modern approaches (BERT-based NER, T5, etc.) often drop the CRF layer entirely, relying on the Transformer's attention mechanism to model inter-label dependencies implicitly. But CRF layers are still used when:

- Hard constraints matter (e.g., no `I-PER` without a preceding `B-PER`).
- The training set is small (CRF transition constraints act as regularisation).
- Calibrated probabilities are needed.

### 7.3 Structured Outputs in Reinforcement Learning

The policy $\pi(a|s)$ in RL is a structured output when the action space has structure. Sequence-to-sequence models trained with RLHF (Reinforcement Learning from Human Feedback) generate tokens autoregressively — this is structured prediction with an RL objective.

### 7.4 The Argmax Bottleneck and Differentiable Structure

A core challenge: the $\arg\max$ in structured prediction is not differentiable, which complicates end-to-end training with neural networks. Active research directions:

|Method|Idea|
|---|---|
|**STE (Straight-Through Estimator)**|Treat argmax as identity in backward pass|
|**Gumbel-Softmax**|Continuous relaxation of categorical sampling|
|**CRF with marginal inference**|Replace $\arg\max$ with expected feature counts — differentiable|
|**SPIGOT**|Project gradient through argmax using polytope geometry|

These are active research areas connecting structured prediction to deep learning.

## ✦ 8. Worked Example — CRF Forward Pass by Hand

**Setup.** 2 positions ($T=2$), 2 labels ($K=2$: `N`=NOUN, `V`=VERB).

Define emission scores (already computed from $w^\top \phi$):

||$t=1$ (cat)|$t=2$ (sat)|
|---|---|---|
|N|1.5|0.2|
|V|0.3|2.0|

Transition scores (learned):

||→ N|→ V|
|---|---|---|
|START|0.8|0.1|
|N →|0.6|1.2|
|V →|0.4|0.9|

**Forward pass ($\alpha_t(k)$ in log-space $= \log \alpha$, actually in score space here for clarity):**

$t=1$: $$\alpha_1(\text{N}) = \exp(0.8 + 1.5) = e^{2.3} \approx 9.97$$ $$\alpha_1(\text{V}) = \exp(0.1 + 0.3) = e^{0.4} \approx 1.49$$

$t=2$: $$\alpha_2(\text{N}) = \alpha_1(\text{N}) \cdot e^{0.6 + 0.2} + \alpha_1(\text{V}) \cdot e^{0.4 + 0.2}$$ $$= 9.97 \cdot e^{0.8} + 1.49 \cdot e^{0.6} \approx 9.97 \cdot 2.23 + 1.49 \cdot 1.82 \approx 22.23 + 2.71 = 24.94$$

$$\alpha_2(\text{V}) = \alpha_1(\text{N}) \cdot e^{1.2 + 2.0} + \alpha_1(\text{V}) \cdot e^{0.9 + 2.0}$$ $$= 9.97 \cdot e^{3.2} + 1.49 \cdot e^{2.9} \approx 9.97 \cdot 24.53 + 1.49 \cdot 18.17 \approx 244.6 + 27.1 = 271.7$$

**Partition function:** $$Z = \alpha_2(\text{N}) + \alpha_2(\text{V}) \approx 24.94 + 271.7 = 296.64$$

**Posterior of `N → V` (the true sequence "cat sat" = NOUN VERB):** $$P(\text{N}, \text{V} | x) = \frac{\alpha_1(\text{N}) \cdot e^{1.2 + 2.0}}{Z} = \frac{9.97 \cdot 24.53}{296.64} \approx \frac{244.6}{296.64} \approx 0.824$$

The model assigns 82.4% probability to the correct sequence — before any training.

**Viterbi decoding** would trace back the path through the trellis:

- At $t=2$: `V` wins ($\alpha_2(\text{V}) = 271.7 \gg \alpha_2(\text{N}) = 24.94$)
- Backtrack: best predecessor of `V` at $t=2$ is `N` (score $9.97 \cdot e^{3.2} = 244.6$ vs $1.49 \cdot e^{2.9} = 27.1$)
- Prediction: `[N, V]` = `[NOUN, VERB]` ✓

## ✦ 9. Implementation Sketch

### 9.1 CRF Forward-Backward in NumPy

```python
import numpy as np

def log_sum_exp(x):
    """Numerically stable log-sum-exp."""
    m = x.max()
    return m + np.log(np.sum(np.exp(x - m)))

def crf_forward(emission_scores, transition_scores):
    """
    emission_scores: (T, K) — score of each label at each position
    transition_scores: (K, K) — transition[i, j] = score of going from i to j
    
    Returns: log Z (partition function)
    """
    T, K = emission_scores.shape
    
    # alpha[k] = log sum of scores of all paths ending in label k at current t
    alpha = emission_scores[0]  # at t=0, no transition
    
    for t in range(1, T):
        # alpha_new[k] = log_sum_exp over j of (alpha[j] + trans[j,k] + emit[t,k])
        alpha_new = np.zeros(K)
        for k in range(K):
            scores = alpha + transition_scores[:, k] + emission_scores[t, k]
            alpha_new[k] = log_sum_exp(scores)
        alpha = alpha_new
    
    return log_sum_exp(alpha)  # log Z

def viterbi(emission_scores, transition_scores):
    """
    Returns: best label sequence (T,)
    """
    T, K = emission_scores.shape
    delta = emission_scores[0].copy()
    psi = np.zeros((T, K), dtype=int)  # backpointers
    
    for t in range(1, T):
        for k in range(K):
            scores = delta + transition_scores[:, k] + emission_scores[t, k]
            psi[t, k] = np.argmax(scores)
            delta[k] = scores[psi[t, k]]
    
    # Backtrace
    path = np.zeros(T, dtype=int)
    path[T-1] = np.argmax(delta)
    for t in range(T-2, -1, -1):
        path[t] = psi[t+1, path[t+1]]
    
    return path

# --- Example ---
T, K = 3, 3  # 3 positions, 3 labels
emission = np.random.randn(T, K)
transition = np.random.randn(K, K)

log_Z = crf_forward(emission, transition)
best_path = viterbi(emission, transition)
print(f"log Z = {log_Z:.3f}, best path = {best_path}")
```
