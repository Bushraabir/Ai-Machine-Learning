
> _"The SVM is one of the most intellectually satisfying algorithms in machine learning. It turns a geometric intuition — find the widest possible street between two classes — into a convex quadratic programme, solves it via Lagrangian duality, and then, almost miraculously, discovers that the solution never needed the data points themselves — only their pairwise inner products. That last observation is the kernel trick, and it opens the door to infinite-dimensional feature spaces computed in finite time."_

## The Geometric Intuition

Given linearly separable data with $y^{(i)} \in {-1, +1}$, many hyperplanes separate the classes. Which one should we pick?

```
Class +1: ●         Class -1: ○

    ○  ○                ○  ○
  ○      ○            ○      ○
      /                   |
  ●  / ●               ●  | ●
    /                      |
  ●/    ●               ●  |  ●

  Arbitrary separator    Maximum margin separator
  (unstable, poor gen.)  (SVMs pick this one)
```

The **maximum margin** separator is the most robust: it is as far as possible from both classes. Small perturbations in the data or new test points near the boundary are less likely to be misclassified.

---

## Setting Up the Geometry

A hyperplane in $\mathbb{R}^d$ is ${x : w^T x + b = 0}$. The signed distance from a point $x^{(i)}$ to this hyperplane is:

$$\text{dist}(x^{(i)}) = \frac{w^T x^{(i)} + b}{|w|}$$

We want all positives on one side and all negatives on the other:

$$y^{(i)} = +1 \implies w^T x^{(i)} + b \geq 0$$ $$y^{(i)} = -1 \implies w^T x^{(i)} + b \leq 0$$

Combined: $y^{(i)}(w^T x^{(i)} + b) \geq 0$ for all $i$.

### Canonical Scale Convention

The margin is the distance from the hyperplane to the closest point on either side. We have a free scaling: if $(w, b)$ defines a separator, so does $(cw, cb)$ for any $c > 0$. We use this freedom to **fix the scale** so that the closest points satisfy:

$$y^{(i)}(w^T x^{(i)} + b) = 1$$

These closest points are the **support vectors**. Under this convention:

- Points with $y^{(i)} = +1$ satisfy $w^T x^{(i)} + b \geq 1$
- Points with $y^{(i)} = -1$ satisfy $w^T x^{(i)} + b \leq -1$

The two margin hyperplanes are $w^Tx + b = +1$ and $w^Tx + b = -1$.

### Width of the Margin

The distance between the two margin planes is the width of the **street** we are trying to maximise:

$$\text{margin} = \frac{2}{|w|}$$

_Derivation:_ Take two points $x_+$ on the $+1$ margin and $x_-$ on the $-1$ margin. Then $w^T(x_+ - x_-) = 2$. The distance between the planes in the direction of $w/|w|$ is $\frac{w^T(x_+ - x_-))}{|w|} = \frac{2}{|w|}$.

```
        w^Tx + b = +1
             |     ← margin = 1/‖w‖
w^Tx + b = 0 |
             |     ← margin = 1/‖w‖
        w^Tx + b = -1

Total street width = 2/‖w‖
Support vectors lie on the margin planes
```

## Hard-Margin SVM

### Primal Problem

Maximising $\frac{2}{|w|}$ is equivalent to minimising $\frac{1}{2}|w|^2$:

$$\min_{w, b} \frac{1}{2}|w|^2 \quad \text{subject to} \quad y^{(i)}(w^Tx^{(i)} + b) \geq 1 \quad \forall i$$

This is a **convex quadratic programme** (QP): convex objective, linear constraints. Unique global minimum.

> **Why $\frac{1}{2}$?** Purely notational convenience — the $\frac{1}{2}$ cancels the factor of 2 that appears when differentiating $|w|^2$. The solution is identical.

### Limitations of Hard-Margin

Hard-margin SVM fails when:

1. Data is **not linearly separable** — no feasible solution exists.
2. Data is **nearly separable** but with a few outliers — a single mislabelled point forces a tiny margin.

## Soft-Margin SVM

### The Relaxation

Introduce **slack variables** $\xi_i \geq 0$ that allow points to violate the margin:

$$\min_{w, b, \xi} \frac{1}{2}|w|^2 + C\sum_{i=1}^n \xi_i$$

$$\text{subject to} \quad y^{(i)}(w^Tx^{(i)} + b) \geq 1 - \xi_i \quad \forall i, \quad \xi_i \geq 0 \quad \forall i$$

Three cases for each point:

- $\xi_i = 0$: correctly classified, outside or on margin. No penalty.
- $0 < \xi_i < 1$: inside the margin but correct side. Small penalty.
- $\xi_i \geq 1$: misclassified. Penalty $\geq 1$.

```
                  w^Tx+b=1    w^Tx+b=0    w^Tx+b=-1
                      |           |            |
class +1:  ●          |  ●(ξ=0)   |            |
           ●(ξ=0.3)───│───────────│            |
                      |           |  ●(ξ=1.5)  |     ← misclassified
class -1:             |           |            | ○
                      |           |            | ○  ○
                      |      ○(ξ=0.7)          |
```

### The Hyperparameter $C$

$C$ controls the **tradeoff between margin width and training error**:

- Large $C$: heavily penalises slack → small margin, few violations → risk of overfitting.
- Small $C$: tolerates violations → wide margin → risk of underfitting.

```
Small C (wide margin,          Large C (narrow margin,
 tolerates violations)          almost no violations)

  ○  ○    ●  ●                  ○ ○   ●  ●
   ○  |  |  ●                   ○|   | ●
      |  |                       |   |
   ○  |  |  ●                   ○|   | ●
                                 |   |
Wide street, outliers OK        Narrow street, near-perfect fit
```

$C$ is a hyperparameter tuned by cross-validation.

## Lagrangian Duality — The Dual Problem

This is the theoretical core of SVMs. We derive the dual formulation, which:

1. Reveals that the solution depends only on **inner products** between training points.
2. Identifies **support vectors** as the only points that matter.
3. Opens the door to the kernel trick.

### The Lagrangian

For the soft-margin SVM, form the Lagrangian by introducing multipliers $\alpha_i \geq 0$ for the margin constraints and $\mu_i \geq 0$ for $\xi_i \geq 0$:

$$\mathcal{L}(w, b, \xi, \alpha, \mu) = \frac{1}{2}|w|^2 + C\sum_i \xi_i - \sum_i \alpha_i \left[y^{(i)}(w^Tx^{(i)} + b) - 1 + \xi_i\right] - \sum_i \mu_i \xi_i$$

### Primal → Dual: Minimise over Primal Variables

**$\nabla_w \mathcal{L} = 0$:**

$$w - \sum_i \alpha_i y^{(i)} x^{(i)} = 0 \implies \boxed{w = \sum_i \alpha_i y^{(i)} x^{(i)}}$$

**$\frac{\partial \mathcal{L}}{\partial b} = 0$:**

$$\sum_i \alpha_i y^{(i)} = 0$$

**$\frac{\partial \mathcal{L}}{\partial \xi_i} = 0$:**

$$C - \alpha_i - \mu_i = 0 \implies \alpha_i = C - \mu_i$$

Since $\mu_i \geq 0$: $0 \leq \alpha_i \leq C$.

### Substituting Back

Substitute $w = \sum_i \alpha_i y^{(i)} x^{(i)}$ into $\mathcal{L}$:

$$\mathcal{L} = \frac{1}{2}\left|\sum_i \alpha_i y^{(i)} x^{(i)}\right|^2 + \ldots$$

After substitution and simplification (expand the norm, cancel terms using $\sum_i \alpha_i y^{(i)} = 0$):

$$\boxed{\max_\alpha \sum_i \alpha_i - \frac{1}{2}\sum_{i,j} \alpha_i \alpha_j y^{(i)} y^{(j)} (x^{(i)})^T x^{(j)}}$$

$$\text{subject to} \quad 0 \leq \alpha_i \leq C \quad \forall i, \quad \sum_i \alpha_i y^{(i)} = 0$$

This is the **dual problem**. It is also a convex QP, but in the $n$-dimensional dual variable $\alpha$ rather than the $(d+1)$-dimensional primal variable $(w, b)$.

> **When is the dual better?** When $n \ll d$ — many features, few examples — solve the dual. When $d \ll n$, solve the primal. For kernel SVMs, the dual is always used (since $d$ is infinite).

### KKT Conditions

The KKT conditions are necessary and sufficient for optimality (since the problem is convex). The **complementary slackness** conditions are particularly important:

$$\alpha_i \left[y^{(i)}(w^Tx^{(i)} + b) - 1 + \xi_i\right] = 0 \quad \forall i$$

$$\mu_i \xi_i = 0 \implies (C - \alpha_i)\xi_i = 0 \quad \forall i$$

From the first condition:

- If $\alpha_i > 0$: the constraint is **active** — $y^{(i)}(w^Tx^{(i)} + b) = 1 - \xi_i$.
- If $\alpha_i = 0$: the point is outside the margin, contributes nothing to $w$.

**Definition:** A **support vector** is any training point with $\alpha_i > 0$. These are the only points that determine $w$.

Three types of support vectors:

```
α_i = 0           0 < α_i < C        α_i = C
────────────       ───────────────    ───────────────
Not a support      On the margin      Inside margin or
vector. Ignored.   ξ_i = 0            misclassified
in prediction.     (from KKT)         ξ_i > 0
```

### Prediction at Test Time

$$\hat{y}(x) = \text{sign}\left(\sum_i \alpha_i y^{(i)} (x^{(i)})^T x + b\right)$$

Again, only support vectors ($\alpha_i > 0$) contribute. If the data has $m \ll n$ support vectors, prediction costs $O(md)$ not $O(nd)$.

## The Kernel Trick

### The Key Observation

The dual objective and the prediction function depend on the data **only through inner products** $x^{(i)T} x^{(j)}$.

This means: if we map $x \to \phi(x)$ for some feature map $\phi: \mathbb{R}^d \to \mathcal{F}$, the inner products become $\phi(x^{(i)})^T \phi(x^{(j)})$. And we never need to compute $\phi(x)$ explicitly — only the inner product.

**Define:** A **kernel function** $k: \mathbb{R}^d \times \mathbb{R}^d \to \mathbb{R}$ is any function of the form:

$$k(x, x') = \phi(x)^T \phi(x')$$

for some feature map $\phi$. Replace every $x^{(i)T}x^{(j)}$ in the dual with $k(x^{(i)}, x^{(j)})$.

### Why This is Remarkable

The feature map $\phi$ can map to **infinite-dimensional spaces**. Yet if we can compute $k(x, x')$ efficiently, we get the power of infinite features at finite computational cost.

The **kernel matrix** (Gram matrix) $K \in \mathbb{R}^{n \times n}$ with $K_{ij} = k(x^{(i)}, x^{(j)})$ is all we need.

### Common Kernels

**Linear kernel:**

$$k(x, x') = x^T x'$$

Equivalent to no feature mapping. Hard-margin SVM with linear kernel = standard hard-margin SVM.

**Polynomial kernel:**

$$k(x, x') = (x^T x' + c)^d$$

Corresponds to a feature map containing all monomials up to degree $d$. For $d=2$, $c=1$, $x \in \mathbb{R}^2$:

$$k(x, x') = (x_1 x_1' + x_2 x_2' + 1)^2 = x_1^2 x_1'^2 + x_2^2 x_2'^2 + 1 + 2x_1 x_2 x_1' x_2' + 2x_1 x_1' + 2x_2 x_2'$$

This corresponds to $\phi(x) = (x_1^2, x_2^2, 1, \sqrt{2}x_1 x_2, \sqrt{2}x_1, \sqrt{2}x_2)$ — a 6-dimensional feature map computed in $O(d)$ via the kernel, not $O(d^2)$ explicitly.

**RBF (Gaussian) kernel:**

$$k(x, x') = \exp!\left(-\frac{|x - x'|^2}{2\sigma^2}\right)$$

This corresponds to an **infinite-dimensional** feature map. Let's prove it.

_Proof that RBF corresponds to an infinite feature map:_

Using the Taylor expansion of $e^z = \sum_{n=0}^\infty z^n / n!$:

$$\exp!\left(-\frac{|x - x'|^2}{2\sigma^2}\right) = \exp!\left(-\frac{|x|^2}{2\sigma^2}\right) \exp!\left(\frac{x^Tx'}{\sigma^2}\right) \exp!\left(-\frac{|x'|^2}{2\sigma^2}\right)$$

$$= \exp!\left(-\frac{|x|^2}{2\sigma^2}\right) \sum_{n=0}^\infty \frac{(x^Tx')^n}{\sigma^{2n} n!} \exp!\left(-\frac{|x'|^2}{2\sigma^2}\right)$$

Each term $(x^Tx')^n / n!$ corresponds to a polynomial feature map. The infinite sum corresponds to an infinite-dimensional feature map. $\blacksquare$

The **bandwidth** $\sigma$ controls the locality:

- Small $\sigma$: kernel decays quickly → model fits locally, high variance, risk of overfitting.
- Large $\sigma$: kernel decays slowly → model is smoother, higher bias.

```
Small σ (peaked kernel)      Large σ (flat kernel)
  k(x,x')                      k(x,x')
  1 ┤ ╭╮                       1 ┤ ╭──────╮
    │ │ │                         │╭╯      ╰╮
  0 ┤─╯ ╰─────  ‖x-x'‖         0 ┤╯        ╰──  ‖x-x'‖
   Local influence only          Wide influence
   → Complex boundary            → Smooth boundary
```

### Mercer's Theorem

When is a function $k$ a valid kernel (i.e., corresponds to some inner product in some feature space)?

**Mercer's Theorem:** A symmetric function $k: \mathcal{X} \times \mathcal{X} \to \mathbb{R}$ is a valid kernel if and only if the kernel matrix $K$ is **positive semi-definite** for every finite set of inputs ${x^{(1)}, \ldots, x^{(n)}}$:

$$\sum_{i,j} c_i c_j k(x^{(i)}, x^{(j)}) \geq 0 \quad \forall c \in \mathbb{R}^n$$

Equivalently: all eigenvalues of $K$ are $\geq 0$.

**Closure properties** (useful for building new kernels):

|Operation|Result|
|---|---|
|$k_1 + k_2$|Valid kernel|
|$c \cdot k_1$ for $c > 0$|Valid kernel|
|$k_1 \cdot k_2$|Valid kernel|
|$\exp(k_1)$|Valid kernel|
|$f(x) k_1(x,x') f(x')$ for any $f$|Valid kernel|

This lets you design kernels for structured inputs (strings, graphs, trees) where Euclidean distance is not meaningful.

## SVM as Hinge Loss Minimisation

The soft-margin SVM primal can be rewritten in **unconstrained form**. Substituting the optimal slack variables:

$$\xi_i = \max(0, 1 - y^{(i)}(w^Tx^{(i)} + b))$$

The primal becomes:

$$\min_{w,b} \frac{\lambda}{2}|w|^2 + \frac{1}{n}\sum_{i=1}^n \max!\left(0, 1 - y^{(i)}(w^Tx^{(i)} + b)\right)$$

where $\lambda = 1/(nC)$. The second term is the **hinge loss**:

$$\ell_{\text{hinge}}(y, \hat{y}) = \max(0, 1 - y\hat{y})$$

```
Hinge loss vs. Logistic loss vs. 0-1 loss

  Loss
  3 ┤
    │  hinge ╲
  2 ┤         ╲
    │  logistic ╲___
  1 ┤            ╲  ─────
    │  0-1 loss   ╲________
  0 ┤──────────────╲────────────
   -2   -1    0    1    2    yŷ

                   ↑
              margin = 1
```

This is the **loss perspective** on SVMs:

- Hinge loss is zero when $y\hat{y} \geq 1$ (correct and confident). No gradient from well-classified points.
- Linear penalty for $y\hat{y} \in [0,1]$ (correct but within margin).
- Linear penalty for $y\hat{y} < 0$ (misclassified).

Compare with logistic loss (smooth everywhere) and 0-1 loss (non-convex, combinatorial). Hinge loss is the **tightest convex surrogate** for 0-1 loss.

## SMO Algorithm (Conceptual)

The dual QP has $n$ variables with a linear equality constraint. Sequential Minimal Optimisation (SMO) solves it efficiently by:

1. Select the **two most violating** $\alpha_i, \alpha_j$ (by the KKT conditions).
2. Optimise over $\alpha_i, \alpha_j$ analytically while keeping all others fixed.
3. The 2-variable subproblem with one linear constraint has a closed-form solution.
4. Repeat until KKT conditions are satisfied to tolerance.

SMO is $O(n^2)$ in the worst case but much faster in practice due to caching of kernel evaluations.

## Implementation: Linear SVM with SGD (Pegasos)

For large-scale problems, primal SGD (Pegasos algorithm) is faster than solving the dual QP:

```python
import numpy as np

def pegasos_svm(X, y, lam=0.01, n_epochs=100):
    """
    Pegasos: Primal Estimated sub-GrAdient SOlver for SVM.
    y must be in {-1, +1}.
    Loss: (λ/2)‖w‖² + (1/n)Σ max(0, 1 - y_i w^T x_i)
    """
    n, d = X.shape
    w = np.zeros(d)
    t = 1  # global step counter
    
    for epoch in range(n_epochs):
        # Shuffle data each epoch
        idx = np.random.permutation(n)
        for i in idx:
            t += 1
            lr = 1 / (lam * t)  # decreasing step size
            
            margin = y[i] * (w @ X[i])
            if margin < 1:  # hinge loss is active
                w = (1 - lr * lam) * w + lr * y[i] * X[i]
            else:            # point outside margin, only regularise
                w = (1 - lr * lam) * w
    
    return w

def predict(X, w):
    return np.sign(X @ w)
```

For kernel SVMs, use `sklearn.svm.SVC` which wraps `libsvm` (dual SMO solver):

```python
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler

# CRITICAL: Always scale before SVM
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

clf = SVC(kernel='rbf', C=1.0, gamma='scale')
clf.fit(X_scaled, y_train)

# Inspect support vectors
print(f"Number of support vectors: {clf.support_vectors_.shape[0]}")
print(f"Support vector indices: {clf.support_[:10]}")
```

## Practical Considerations

### Scaling is Mandatory

SVMs use Euclidean distances. If feature $j$ has range $[0, 10000]$ and feature $k$ has range $[0, 1]$, the distance is dominated by feature $j$. **Always standardise** before fitting an SVM.

### Choosing the Kernel

```
Data characteristics            Kernel to try
──────────────────────────────────────────────
Many features, sparse           Linear (fast)
Moderate features, nonlinear    RBF (default)
Known polynomial structure      Polynomial
Text/sequences                  String kernel
Graphs                          Graph kernel
```

### Hyperparameter Grid

For RBF SVM, always tune both $C$ and $\gamma$ together:

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'C': [0.01, 0.1, 1, 10, 100],
    'gamma': ['scale', 'auto', 0.001, 0.01, 0.1]
}
grid = GridSearchCV(SVC(kernel='rbf'), param_grid, cv=5, n_jobs=-1)
grid.fit(X_train_scaled, y_train)
```

### Computational Complexity

|Task|Complexity|
|---|---|
|Training (SMO, kernel SVM)|$O(n^2)$ to $O(n^3)$|
|Training (Pegasos, linear)|$O(nd / \lambda\epsilon)$|
|Prediction (kernel SVM)|$O(n_{\text{sv}} \cdot d)$|
|Memory|$O(n^2)$ for kernel matrix|

SVMs do not scale well beyond $n \approx 10^5$. For large datasets, use linear SVM (Pegasos, `LinearSVC`) or approximate kernel methods (Nyström approximation, random Fourier features).

## Connection to AI, Machine Learning, and Deep Learning

### SVMs vs. Neural Networks: A Historical Note

Through the 1990s and early 2000s, SVMs dominated ML benchmarks. The kernel trick gave them nonlinear power, the max-margin principle gave them strong generalisation, and convexity gave them reliable training. Neural networks were widely dismissed as unreliable and hard to train.

The 2012 ImageNet result (AlexNet) reversed this. Deep networks outperformed kernel SVMs on vision tasks by a large margin — not because the theory of SVMs was wrong, but because:

1. SVMs require hand-engineered features; deep networks learn them.
2. The kernel matrix is $O(n^2)$ memory; deep networks scale to $n = 10^8$.
3. Deep networks compose representations; SVMs compute a single kernel.

### SVMs Survive in Deep Learning

Despite this history, SVM ideas are alive in modern deep learning:

**Hinge loss** appears directly in:

- Multi-class SVM loss (Weston-Watkins, Crammer-Singer): used in some image classifiers.
- Margin-based losses in metric learning (triplet loss, contrastive loss — used in FaceNet, SimCLR).
- `torch.nn.MarginRankingLoss`, `torch.nn.MultiMarginLoss`.

**Max-margin thinking** underpins:

- The success of large-margin classifiers in NLP (structured SVMs for sequence labelling).
- Implicit regularisation in deep networks — recent theoretical work shows SGD on deep networks with cross-entropy loss has an **implicit max-margin bias** on linearly separable data.

**The kernel perspective** is central to:

- Neural Tangent Kernel (NTK) theory: infinitely wide neural networks are equivalent to kernel methods with a specific kernel determined by the architecture.
- Gaussian Processes ([[03.Gaussian Processes]]): Bayesian kernel machines, dual to SVMs.
- Attention mechanisms in Transformers can be interpreted as kernel-weighted averaging: $$\text{Attention}(Q, K, V) = \text{softmax}!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$ The term $QK^T$ is a matrix of inner products — exactly the kernel matrix.

**Support vectors and sparsity:**

- The SVM solution is sparse in the dual — most $\alpha_i = 0$.
- This is analogous to L1 regularisation (Lasso) inducing sparsity in the primal.
- Sparse solutions generalise better and are cheaper at inference.

### The Representer Theorem

The SVM solution satisfies:

$$w^* = \sum_i \alpha_i y^{(i)} \phi(x^{(i)})$$

This says: the optimal weight vector in feature space is a **linear combination of the training data feature maps**. This is an instance of the **Representer Theorem** ([[01.Kernel Methods]]):

> For any regularised ERM problem with a differentiable, strictly increasing regulariser $\Omega(|f|_\mathcal{H})$, the optimal solution has the form $f^*(x) = \sum_i \alpha_i k(x^{(i)}, x)$.

Neural networks can be seen as learning a data-dependent kernel — and the final classifier is still a linear combination over training data in the learned representation space.
