## Why Linear Regression Fails for Classification

Suppose $y \in {0, 1}$ (binary classification). Naively applying linear regression gives $\hat{y} = w^Tx$. Three problems:

1. **Range:** $w^Tx \in (-\infty, +\infty)$, but probabilities must live in $[0, 1]$.
2. **Sensitivity to outliers:** A single far-away point can drag the decision boundary entirely.
3. **Wrong loss:** MSE penalises confident correct predictions — if $y = 1$ and $\hat{y} = 10$, OLS says the model is wrong by $81$. That is backwards.

```
Linear regression on binary labels     Logistic regression
  y                                       P(y=1|x)
  1 •  •  •  ____/-----                  1 ┤         ╭────────
  0 •  •  •-/                              │       ╭─╯
              ↑                          0.5┤     ╭─╯
           Prediction > 1                  │   ╭─╯
           for far points                0 ┤╭─╯
                                           └──────────────────
                                              x
```

The fix: **model the probability directly** using a function that maps $\mathbb{R} \to (0,1)$.

---

## The Sigmoid Function

Define:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Properties you must know cold:

- Range: $(0, 1)$ — strictly, never 0 or 1.
- $\sigma(0) = 0.5$
- $\sigma(-z) = 1 - \sigma(z)$ — antisymmetric about $(0, 0.5)$
- $\lim_{z \to +\infty} \sigma(z) = 1$, $\lim_{z \to -\infty} \sigma(z) = 0$

**Derivative** (derive this once, memorise the form):

$$\sigma'(z) = \sigma(z)(1 - \sigma(z))$$

_Proof:_

$$\frac{d}{dz}\frac{1}{1+e^{-z}} = \frac{e^{-z}}{(1+e^{-z})^2} = \frac{1}{1+e^{-z}} \cdot \frac{e^{-z}}{1+e^{-z}} = \sigma(z)(1 - \sigma(z)) \quad \blacksquare$$

This elegant form is why sigmoid is analytically convenient — and why it appears as the activation function in the output layer of binary classification networks.

> **Professor's note:** The derivative achieves its maximum of $1/4$ at $z = 0$ and approaches $0$ at both tails. This is the root cause of the **vanishing gradient problem** in deep networks — sigmoid saturates, gradients die. In hidden layers, ReLU has replaced sigmoid precisely because of this. But in output layers for binary classification, sigmoid is still correct.

```
σ(z)                        σ'(z) = σ(z)(1 - σ(z))
  1 ┤        ─────────          0.25┤       ╭─╮
    │      ╭─╯                      │     ╭─╯ ╰─╮
 0.5┤    ──╯                      0 ┤───╯─╯     ╰─╯───
    │  ╭─╯                          └──────────────────
  0 ┤──╯                                   z
    └──────────────── z          Max 1/4 at z=0
```

## The Model

$$P(y = 1 \mid x; w) = \sigma(w^T x) = \frac{1}{1 + e^{-w^Tx}}$$

$$P(y = 0 \mid x; w) = 1 - \sigma(w^T x)$$

Combined into one expression:

$$P(y \mid x; w) = \sigma(w^Tx)^y \cdot (1 - \sigma(w^Tx))^{1-y}$$

This is the Bernoulli likelihood. The model is a **discriminative classifier**: it models $P(y|x)$ directly, without modelling $P(x)$. Compare with Naive Bayes ([[06.Naive Bayes]]) which is generative.

## The Loss Function — Binary Cross-Entropy

### Derivation from MLE

Given $n$ i.i.d. training examples $(x^{(i)}, y^{(i)})$, write the log-likelihood:

$$\log \mathcal{L}(w) = \sum_{i=1}^n \log P(y^{(i)} \mid x^{(i)}; w)$$

$$= \sum_{i=1}^n \left[ y^{(i)} \log \sigma(w^Tx^{(i)}) + (1 - y^{(i)}) \log(1 - \sigma(w^Tx^{(i)})) \right]$$

Maximising log-likelihood = minimising the **negative log-likelihood** = minimising **binary cross-entropy**:

$$\boxed{\mathcal{L}(w) = -\frac{1}{n}\sum_{i=1}^n \left[ y^{(i)} \log \hat{p}^{(i)} + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) \right]}$$

where $\hat{p}^{(i)} = \sigma(w^T x^{(i)})$.

### Why Not MSE?

For a probabilistic model, MSE is the wrong loss. Formally:

- MSE assumes Gaussian noise (see [[01.Linear Regression]]).
- Binary outcomes are Bernoulli — Bernoulli NLL is cross-entropy.
- MSE loss produces a **non-convex** objective for logistic regression; cross-entropy is **convex**.

Convexity means gradient descent finds the global minimum. This is not guaranteed with MSE + sigmoid.

### Intuition

|Prediction $\hat{p}$|True label $y$|$-\log \hat{p}$ (loss)|
|---|---|---|
|0.99|1|$-\log(0.99) \approx 0.01$ — correct, tiny loss|
|0.5|1|$-\log(0.5) \approx 0.69$ — uncertain, moderate loss|
|0.01|1|$-\log(0.01) \approx 4.6$ — confidently wrong, large loss|

The logarithm **penalises confident mistakes severely** — much more than MSE would.

```
Loss = -log(p) for y=1
  5 ┤•
  4 ┤ •
  3 ┤  •
  2 ┤    •
  1 ┤       •
  0 ┤           ────•
    └────────────────── p
    0   0.2  0.5   1.0

As p → 0 with y=1: loss → ∞
```

## The Gradient — The Beautiful Residual Form

### Derivation

$$\mathcal{L}(w) = -\frac{1}{n}\sum_{i=1}^n \left[ y^{(i)} \log \sigma(z^{(i)}) + (1-y^{(i)}) \log(1 - \sigma(z^{(i)})) \right]$$

where $z^{(i)} = w^T x^{(i)}$.

Differentiating with respect to $w_j$, using $\frac{\partial z^{(i)}}{\partial w_j} = x_j^{(i)}$ and $\frac{d}{dz}\log\sigma(z) = 1 - \sigma(z)$:

$$\frac{\partial \mathcal{L}}{\partial w_j} = -\frac{1}{n}\sum_i \left[ y^{(i)}(1 - \hat{p}^{(i)}) - (1-y^{(i)})\hat{p}^{(i)} \right] x_j^{(i)}$$

Simplifying the bracket:

$$y^{(i)}(1 - \hat{p}^{(i)}) - (1 - y^{(i)})\hat{p}^{(i)} = y^{(i)} - \hat{p}^{(i)}$$

Therefore:

$$\boxed{\nabla_w \mathcal{L} = \frac{1}{n} X^T (\hat{p} - y)}$$

where $\hat{p} = \sigma(Xw) \in \mathbb{R}^n$ is the vector of predicted probabilities.

**This is structurally identical to the OLS gradient** $\frac{2}{n}X^T(Xw - y)$. This is not a coincidence — it follows from the GLM framework ([[03.Generalised Linear Models]]): for any model in the exponential family with its canonical link, the gradient of the NLL always has the form $X^T(\hat{y} - y)$.

### The Update Rule

$$w \leftarrow w - \alpha \cdot \frac{1}{n} X^T(\hat{p} - y)$$

The $\sigma'$ term **cancels exactly** in the gradient computation. This is why cross-entropy + sigmoid is a natural pair: the "difficult" derivative of the sigmoid disappears, leaving a clean residual.

## The Decision Boundary

The model predicts class 1 when $\hat{p} > 0.5$, i.e., when $\sigma(w^Tx) > 0.5$, i.e., when $w^Tx > 0$.

The **decision boundary** is the hyperplane ${x : w^Tx = 0}$.

In 2D: a line. In 3D: a plane. In $d$ dimensions: a $(d-1)$-dimensional hyperplane.

```
     x_2
      |     * (class 1)
   *  |  * *
  * * |    *
──────────────── x_1
     *|* *
      | *  *
      |  (class 0)
      
w^Tx = 0 is the boundary
```

> **Key insight:** Logistic regression can only learn **linearly separable** boundaries. For nonlinear boundaries, the options are: (1) engineer nonlinear features (polynomial, interaction terms), (2) use kernel logistic regression, (3) use a neural network to learn the features.

## Numerical Stability

### The Log-Sum-Exp Issue

When computing $\log(1 + e^{-z})$ for large positive $z$, or $\log(1 + e^z)$ for large negative $z$, floating point underflow/overflow occurs.

**Numerically stable BCE loss:**

$$\text{BCE}(z, y) = \max(z, 0) - zy + \log(1 + e^{-|z|})$$

This avoids evaluating $e^z$ for large $z$. PyTorch's `BCEWithLogitsLoss` implements this.

### Why Work with Logits?

The raw model output $z = w^Tx$ is called the **logit**. It is often better to work with logits directly and apply sigmoid inside the loss, rather than computing $\hat{p} = \sigma(z)$ and passing it to a log-loss. This avoids $\log(\sigma(z))$ which suffers precision loss near 0 and 1.

---

## Regularisation

The cross-entropy loss is convex but **not strongly convex** when classes are linearly separable: the weights diverge to $\pm\infty$ (the model becomes infinitely confident). L2 regularisation fixes this.

**Regularised objective:**

$$\mathcal{L}(w) = -\frac{1}{n}\sum_i \left[ y^{(i)}\log\hat{p}^{(i)} + (1-y^{(i)})\log(1-\hat{p}^{(i)}) \right] + \frac{\lambda}{2}|w|^2$$

In sklearn, the regularisation parameter is written as $C = 1/\lambda$. Default is `C=1.0` (L2). Larger $C$ = less regularisation.

```python
from sklearn.linear_model import LogisticRegression
# C is inverse of λ: large C = less regularisation
clf = LogisticRegression(C=1.0, penalty='l2', solver='lbfgs')
```

## Multi-class: Softmax Regression

When $y \in {1, 2, \ldots, K}$, we generalise sigmoid to **softmax**.

### The Softmax Function

For a vector $z \in \mathbb{R}^K$:

$$\text{softmax}(z)_k = \frac{e^{z_k}}{\sum_{j=1}^K e^{z_j}}$$

Properties:

- Outputs sum to 1: $\sum_k \text{softmax}(z)_k = 1$.
- All outputs in $(0,1)$.
- Reduces to sigmoid when $K = 2$ (with the redundancy that one weight vector can be set to 0).

### The Model

We learn $K$ weight vectors $w_1, \ldots, w_K \in \mathbb{R}^d$ (or equivalently a weight matrix $W \in \mathbb{R}^{K \times d}$):

$$P(y = k \mid x; W) = \frac{e^{w_k^T x}}{\sum_{j=1}^K e^{w_j^T x}}$$

### Categorical Cross-Entropy Loss

Let $\hat{p}_k^{(i)} = P(y = k \mid x^{(i)})$. Using one-hot encoding $\mathbf{1}[y^{(i)} = k]$:

$$\mathcal{L}(W) = -\frac{1}{n}\sum_{i=1}^n \sum_{k=1}^K \mathbf{1}[y^{(i)} = k] \log \hat{p}_k^{(i)}$$

Since exactly one term is non-zero per example:

$$\mathcal{L}(W) = -\frac{1}{n}\sum_{i=1}^n \log \hat{p}_{y^{(i)}}^{(i)}$$

Minimise the negative log-probability assigned to the correct class.

### The Log-Sum-Exp Trick

Computing $\log \text{softmax}(z)_k = z_k - \log\sum_j e^{z_j}$ directly overflows for large $z_j$.

**Trick:** subtract the maximum before exponentiating:

$$\log\sum_j e^{z_j} = \log\sum_j e^{z_j - z_{\max}} + z_{\max}$$

The subtracted constant cancels in the softmax but keeps all exponents $\leq 1$:

```python
def stable_log_softmax(z):
    z_max = np.max(z)
    return z - z_max - np.log(np.sum(np.exp(z - z_max)))
```

### Redundancy in Softmax

The softmax model has a redundancy: adding a constant vector $c$ to all $w_k$ doesn't change the output. We can always set $w_K = 0$ and learn only $K-1$ weight vectors. This is called the **reference category**. sklearn's multi-class logistic regression handles this implicitly.

## Implementation from Scratch

```python
import numpy as np

class LogisticRegression:
    def __init__(self, lr=0.1, n_iter=1000, lam=0.0):
        self.lr = lr
        self.n_iter = n_iter
        self.lam = lam  # L2 regularisation

    def fit(self, X, y):
        n, d = X.shape
        # Augment with bias
        X_aug = np.hstack([np.ones((n, 1)), X])
        self.w = np.zeros(X_aug.shape[1])
        
        self.losses = []
        for _ in range(self.n_iter):
            z = X_aug @ self.w
            p_hat = 1 / (1 + np.exp(-z))                    # sigmoid
            
            # BCE loss + L2
            loss = -np.mean(y*np.log(p_hat+1e-15) + 
                           (1-y)*np.log(1-p_hat+1e-15))
            loss += (self.lam / 2) * np.sum(self.w[1:]**2)  # skip bias
            self.losses.append(loss)
            
            # Gradient (residual form) + L2 term
            grad = (1/n) * X_aug.T @ (p_hat - y)
            grad[1:] += self.lam * self.w[1:]
            
            self.w -= self.lr * grad
        return self

    def predict_proba(self, X):
        X_aug = np.hstack([np.ones((len(X), 1)), X])
        return 1 / (1 + np.exp(-X_aug @ self.w))

    def predict(self, X, threshold=0.5):
        return (self.predict_proba(X) >= threshold).astype(int)
```

## Logistic Regression as a Linear Classifier

Define the **log-odds** (or **logit**):

$$\log \frac{P(y=1|x)}{P(y=0|x)} = \log \frac{\sigma(w^Tx)}{1-\sigma(w^Tx)} = w^Tx$$

Logistic regression models the log-odds as a **linear function of $x$**. This is the "regression" in its name — we're doing regression on the log-odds scale.

```
P(y=1|x)                 log-odds
  1 ┤        ─────        3 ┤         /
    │      ╭─╯            2 ┤        /
 0.5┤    ──╯              1 ┤       /
    │  ╭─╯                0 ┼──────/────────
  0 ┤──╯                 -1 ┤     /
    └─────────── x        -2 ┤    /
                             └─────────── x

     Sigmoid (nonlinear)         Linear in log-odds space
```

## Connection to AI, Machine Learning, and Deep Learning

### The Output Layer of Every Classification Network

In deep learning, a binary classification network is:

$$\hat{p} = \sigma(w^T \phi_\theta(x))$$

where $\phi_\theta(x)$ is the representation learned by the hidden layers. The final layer is logistic regression on top of learned features. Training the whole network end-to-end with binary cross-entropy is just logistic regression with gradient descent — the gradient of the loss w.r.t. the final layer weights is **exactly** the residual form derived above.

For multi-class ($K$ classes), the output layer is softmax regression:

$$\hat{p} = \text{softmax}(W\phi_\theta(x))$$

with categorical cross-entropy loss. This is the **universal structure** of classification in deep learning: learned representation + linear classifier + softmax/sigmoid.

### Cross-Entropy as the Universal Classification Loss

Binary cross-entropy and categorical cross-entropy are the default losses for all classification tasks in:

- `torch.nn.BCEWithLogitsLoss` — binary
- `torch.nn.CrossEntropyLoss` — multi-class (applies log-softmax + NLL internally)
- `tf.keras.losses.BinaryCrossentropy`, `CategoricalCrossentropy`

Understanding the derivation from MLE tells you exactly why these are the right choices and when you might deviate (e.g., focal loss for severe class imbalance).

### Logistic Regression as a Shallow Neural Network

```
Binary classification network (depth 1):

  x_1 ──w_1─┐
  x_2 ──w_2─┤──→ z = w^Tx ──→ σ(z) = p̂ ──→ BCE(p̂, y)
   ⋮        ⋮
  x_d ──w_d─┘

Gradient flow (backprop through one layer):
  ∂L/∂w = x · (p̂ - y)    [the residual, computed above]
```

Every concept from this note — sigmoid activation, cross-entropy loss, gradient descent, the residual form — scales **unchanged** to networks with 100 layers. The only addition is the chain rule.

### Softmax Temperature

In large language models and generative models, the softmax is often modified with a **temperature parameter** $T$:

$$\text{softmax}(z/T)_k = \frac{e^{z_k/T}}{\sum_j e^{z_j/T}}$$

- $T \to 0$: distribution concentrates on the argmax (greedy decoding).
- $T \to \infty$: distribution becomes uniform (maximum entropy, maximum randomness).
- $T = 1$: standard softmax.

This is why you see `temperature` as a parameter in GPT/Claude APIs — it's the $T$ in front of the logits before the final softmax.

### Logistic Regression in Practice (tabular ML)

Despite being a simple model, logistic regression with good feature engineering is surprisingly competitive:

- Interpretable: $\exp(w_j)$ is the **odds ratio** for feature $j$.
- Fast: $O(nd)$ per gradient step, scales to millions of examples.
- Baseline: always fit logistic regression before trying XGBoost or neural networks. If logistic regression achieves 90% of the performance of a complex model, the complex model may not be worth the interpretability cost.
