
> _"Once you see the GLM framework, you cannot unsee it. Linear regression and logistic regression stop being two separate algorithms and become two instantiations of a single idea. Poisson regression, gamma regression, and multinomial regression are three more. The framework tells you exactly how to construct a new regression model for any response distribution in the exponential family — and guarantees that the gradient will always have the same clean residual form."_


## The Unifying Observation

Look at the gradients derived in the two previous notes:

|Model|Loss|Gradient $\nabla_w \mathcal{L}$|
|---|---|---|
|Linear Regression|MSE (= Gaussian NLL)|$\frac{1}{n}X^T(\hat{y} - y)$|
|Logistic Regression|Binary cross-entropy (= Bernoulli NLL)|$\frac{1}{n}X^T(\hat{p} - y)$|

**The gradient has the same form in both cases:** $X^T(\text{prediction} - \text{target})$.

This is not a coincidence. Both models are **Generalised Linear Models (GLMs)**. The GLM framework explains _why_ this happens, and constructs the full family of models that share this property.

## The Exponential Family

The exponential family is the class of probability distributions that can be written as:

$$p(y ; \eta) = h(y) \exp!\left(\eta^T T(y) - A(\eta)\right)$$

where:

- $\eta$ is the **natural parameter** (also called canonical parameter)
- $T(y)$ is the **sufficient statistic** — all information about $\eta$ in the data is contained in $T(y)$
- $A(\eta)$ is the **log-partition function** (log-normaliser): ensures the distribution integrates to 1
- $h(y)$ is the **base measure**

For most distributions we care about, $T(y) = y$ (the identity sufficient statistic). We focus on this case.

### Why "Sufficient Statistic"?

$T(y)$ is sufficient for $\eta$ if $p(y | T(y); \eta) = p(y | T(y))$ — the data $y$ contains no additional information about $\eta$ beyond what $T(y)$ already tells us. For the Gaussian, the sample mean and variance are sufficient statistics for $\mu$ and $\sigma^2$. The exponential family is precisely the class of distributions admitting finite-dimensional sufficient statistics (by the Pitman–Koopman–Darmois theorem).

### Key Members

#### Gaussian (for regression)

$$p(y; \mu) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp!\left(-\frac{(y-\mu)^2}{2\sigma^2}\right)$$

Rewrite in exponential family form (treating $\sigma^2$ as fixed):

$$= \frac{1}{\sqrt{2\pi\sigma^2}} \exp!\left(\frac{\mu}{\sigma^2} y - \frac{\mu^2}{2\sigma^2}\right) \cdot \exp!\left(-\frac{y^2}{2\sigma^2}\right)$$

Matching terms:

$$\eta = \frac{\mu}{\sigma^2}, \quad T(y) = y, \quad A(\eta) = \frac{\mu^2}{2\sigma^2} = \frac{\eta^2 \sigma^2}{2}, \quad h(y) = \frac{1}{\sqrt{2\pi\sigma^2}}e^{-y^2/2\sigma^2}$$

The natural parameter is $\eta = \mu/\sigma^2$, so $\mu = \eta\sigma^2$. The **mean function** is $\mu(\eta) = \eta\sigma^2$ — linear in $\eta$.

#### Bernoulli (for classification)

$$p(y; \phi) = \phi^y (1-\phi)^{1-y}, \quad y \in {0,1}$$

$$= \exp!\left(y \log\frac{\phi}{1-\phi} + \log(1-\phi)\right)$$

Matching terms:

$$\eta = \log\frac{\phi}{1-\phi} \quad \text{(the log-odds!)}, \quad T(y) = y, \quad A(\eta) = \log(1 + e^\eta), \quad h(y) = 1$$

Inverting: $\phi = \frac{e^\eta}{1+e^\eta} = \sigma(\eta)$. The **mean function** is $\mu(\eta) = \sigma(\eta)$ — the sigmoid.

#### Poisson (for count data)

$$p(y; \lambda) = \frac{\lambda^y e^{-\lambda}}{y!}, \quad y \in {0, 1, 2, \ldots}$$

$$= \frac{1}{y!}\exp!\left(y \log\lambda - \lambda\right)$$

Matching terms:

$$\eta = \log\lambda, \quad T(y) = y, \quad A(\eta) = e^\eta = \lambda, \quad h(y) = \frac{1}{y!}$$

Inverting: $\lambda = e^\eta$. The **mean function** is $\mu(\eta) = e^\eta$ — the exponential.

### The Log-Partition Function Encodes Everything

This is a key theorem: the derivatives of $A(\eta)$ give the cumulants.

$$\frac{dA}{d\eta} = \mathbb{E}[T(y)] = \mathbb{E}[y] = \mu(\eta)$$

$$\frac{d^2A}{d\eta^2} = \text{Var}(y)$$

_Proof sketch:_ Differentiate $\int p(y;\eta) dy = 1$ with respect to $\eta$ twice. The first derivative gives $\mathbb{E}[T(y)] = A'(\eta)$; the second gives $\text{Var}(T(y)) = A''(\eta) \geq 0$, which also proves $A$ is **convex**.

```
Distribution    A(η)           A'(η) = μ       A''(η) = Var(y)
─────────────────────────────────────────────────────────────
Gaussian        η²σ²/2         ησ² = μ         σ² (constant)
Bernoulli       log(1+eη)      σ(η) = p        p(1−p)
Poisson         eη             eη = λ          λ (mean=variance!)
```

The Poisson identity $\text{Var}(y) = \mathbb{E}[y] = \lambda$ falls out automatically from $A'' = A'$ when $A(\eta) = e^\eta$.

## Three Components of a GLM

A GLM specifies three things:

### 1. Random Component

The response $y^{(i)}$ is drawn from an exponential family distribution with natural parameter $\eta^{(i)}$: $$y^{(i)} \sim \text{ExpFam}(\eta^{(i)})$$

### 2. Systematic Component (Linear Predictor)

The natural parameter is a **linear function** of the input: $$\eta^{(i)} = w^T x^{(i)}$$

### 3. Link Function

The link function $g$ connects the mean $\mu = \mathbb{E}[y]$ to the linear predictor: $$g(\mu^{(i)}) = \eta^{(i)} = w^T x^{(i)}$$

Equivalently, the **mean function** (inverse link) is $\mu = g^{-1}(\eta)$.

When $g$ is the **canonical link** — the natural mapping from the mean to the natural parameter — the model has the cleanest properties (see gradient derivation below).

```
Input x ──→ Linear predictor η = w^Tx ──→ Mean μ = g⁻¹(η) ──→ Response y ~ ExpFam(μ)

        [systematic component]      [random component]
              (always linear)         (chosen by modeller)
```

### The Three Canonical GLMs

|Distribution|Canonical Link $g(\mu)$|Mean Function $g^{-1}(\eta)$|Typical Use|
|---|---|---|---|
|Gaussian|Identity: $g(\mu) = \mu$|$\mu = \eta$|Continuous response|
|Bernoulli|Logit: $g(\mu) = \log\frac{\mu}{1-\mu}$|$\mu = \sigma(\eta)$|Binary classification|
|Poisson|Log: $g(\mu) = \log\mu$|$\mu = e^\eta$|Count data|
|Gamma|Inverse: $g(\mu) = 1/\mu$|$\mu = 1/\eta$|Positive continuous, heavy tail|
|Inverse Gaussian|$g(\mu) = 1/\mu^2$|$\mu = \eta^{-1/2}$|Positive, right-skewed|

## The Universal Gradient Result

This is the theorem that explains why linear regression and logistic regression have the same gradient structure.

**Theorem.** For any GLM with canonical link, the gradient of the negative log-likelihood with respect to $w$ is:

$$\nabla_w \mathcal{L}(w) = \frac{1}{n} X^T (\hat{\mu} - y)$$

where $\hat{\mu}^{(i)} = g^{-1}(w^T x^{(i)})$ is the predicted mean.

### Proof

The log-likelihood for a single example (with $T(y) = y$, $\eta = w^Tx$):

$$\log p(y^{(i)}; \eta^{(i)}) = \eta^{(i)} y^{(i)} - A(\eta^{(i)}) + \log h(y^{(i)})$$

Negative log-likelihood over $n$ examples:

$$\mathcal{L}(w) = -\frac{1}{n}\sum_{i=1}^n \left[\eta^{(i)} y^{(i)} - A(\eta^{(i)})\right] + \text{const}$$

Since $\eta^{(i)} = w^T x^{(i)}$, differentiate with respect to $w_j$:

$$\frac{\partial \mathcal{L}}{\partial w_j} = -\frac{1}{n}\sum_{i=1}^n \left[y^{(i)} x_j^{(i)} - A'(\eta^{(i)}) x_j^{(i)}\right]$$

Now use the key identity $A'(\eta) = \mu(\eta) = \hat{\mu}^{(i)}$:

$$\frac{\partial \mathcal{L}}{\partial w_j} = \frac{1}{n}\sum_{i=1}^n \left[\hat{\mu}^{(i)} - y^{(i)}\right] x_j^{(i)}$$

In matrix form:

$$\boxed{\nabla_w \mathcal{L} = \frac{1}{n} X^T(\hat{\mu} - y)} \quad \blacksquare$$

The canonical link choice is what makes $A'(\eta) = \mu$ appear so cleanly. A non-canonical link introduces an extra Jacobian term.

### Why This Matters

One gradient formula covers every GLM. To implement gradient descent for:

- **Linear regression:** $\hat{\mu} = \eta = w^Tx$. Same formula.
- **Logistic regression:** $\hat{\mu} = \sigma(w^Tx)$. Same formula.
- **Poisson regression:** $\hat{\mu} = e^{w^Tx}$. Same formula.

The training loop is identical. Only the mean function changes.

```python
# Universal GLM training loop
def train_glm(X, y, mean_fn, lr=0.01, n_iter=1000):
    """
    mean_fn: the inverse link g^{-1}
      - Linear regression: mean_fn = lambda z: z
      - Logistic regression: mean_fn = sigmoid
      - Poisson regression: mean_fn = np.exp
    """
    n, d = X.shape
    X_aug = np.hstack([np.ones((n,1)), X])
    w = np.zeros(X_aug.shape[1])
    
    for _ in range(n_iter):
        eta = X_aug @ w                # linear predictor
        mu_hat = mean_fn(eta)          # predicted mean (apply inverse link)
        grad = (1/n) * X_aug.T @ (mu_hat - y)  # universal residual form
        w -= lr * grad
    
    return w
```

## Poisson Regression in Detail

Poisson regression is the GLM for **count data**: number of events, website visits, defects per unit, etc.

### Model

$$\lambda^{(i)} = e^{w^T x^{(i)}}, \quad y^{(i)} \sim \text{Poisson}(\lambda^{(i)})$$

The log link ensures $\lambda > 0$ (counts cannot be negative). The model is:

$$\log \mathbb{E}[y | x] = w^T x$$

So a unit increase in $x_j$ multiplies the expected count by $e^{w_j}$.

### Loss (Poisson NLL)

$$\mathcal{L}(w) = -\frac{1}{n}\sum_i \left[y^{(i)} \log\lambda^{(i)} - \lambda^{(i)}\right] = -\frac{1}{n}\sum_i \left[y^{(i)} w^T x^{(i)} - e^{w^Tx^{(i)}}\right]$$

### Gradient

By the universal result:

$$\nabla_w \mathcal{L} = \frac{1}{n}X^T(e^{Xw} - y)$$

where the exponential is applied elementwise.

### When to Use

- Response is a non-negative integer (count).
- Events are rare relative to the exposure (otherwise Gaussian approximation works).
- Overdispersion ($\text{Var}(y) > \mathbb{E}[y]$): use Negative Binomial regression instead.

```
Use case             Response type        GLM to use
───────────────────────────────────────────────────────
House price          Continuous           Gaussian (OLS)
Spam or not          Binary               Logistic (Bernoulli)
# of website visits  Count (Poisson)      Poisson regression
Insurance claim amt  Positive continuous  Gamma regression
# of accidents/year  Count, overdispersed Negative Binomial
```

## The Inductive Bias of GLMs

Every GLM makes the same core assumption:

> **The mean of the response is related to a linear function of the inputs.**

Specifically: $g(\mathbb{E}[y|x]) = w^Tx$. The relationship between the _mean_ (not the raw output) and the inputs is linear. For logistic regression, the log-odds is linear. For Poisson regression, the log-rate is linear.

This is the GLM's **inductive bias** — the prior assumption baked into the model family. It fails when the true relationship (on the link scale) is genuinely nonlinear. Remedies: (1) feature engineering, (2) kernel methods, (3) neural networks.

## MLE for GLMs: Convexity

For any GLM with canonical link, the negative log-likelihood is **convex** in $w$.

_Proof sketch:_ The Hessian is:

$$H = \frac{1}{n} X^T W X$$

where $W = \text{diag}(A''(\eta^{(i)}))$ is a diagonal matrix of variances. Since $A(\eta)$ is convex, $A''(\eta) \geq 0$, so $W \succeq 0$, so $H \succeq 0$. The objective is convex. $\blacksquare$

When $W \succ 0$ (strictly positive variances) and $X$ has full column rank, the problem is **strictly convex** — unique global minimum, gradient descent converges.

This Hessian is also the key to **Newton's method** (aka IRLS — Iteratively Reweighted Least Squares), the standard solver for GLMs in statistics:

$$w \leftarrow w - H^{-1} \nabla_w \mathcal{L}$$

IRLS converges in far fewer iterations than gradient descent (quadratic vs. linear convergence), but each step costs $O(d^3)$ for the matrix inverse.

## The GLM as a Probabilistic Model

GLMs are **discriminative probabilistic models**: they model $p(y|x; w)$ and give calibrated probability estimates (unlike, e.g., SVMs). This matters for:

- **Uncertainty quantification:** the model outputs a full distribution, not just a point prediction.
- **Decision theory:** if the loss is not symmetric (e.g., false negatives cost more than false positives), you threshold at something other than 0.5 using the probability output.
- **Bayesian extension:** place a prior on $w$, compute the MAP estimate (equivalent to regularised GLM) or the full posterior.

## Connection to AI, Machine Learning, and Deep Learning

### GLMs as Single-Layer Networks

Every GLM is a **neural network with zero hidden layers**:

```
Input layer             Output
  x_1 ──w_1─┐
  x_2 ──w_2─┤──→ η = w^Tx ──→ μ = g⁻¹(η) ──→ y ~ ExpFam(μ)
   ⋮        ⋮
  x_d ──w_d─┘

        Linear layer       Activation        Distribution
       (always linear)   (= inverse link)   (= loss function)
```

The **activation function** in deep learning is exactly the **inverse link function** of the corresponding GLM:

- Sigmoid output + BCE loss ← Bernoulli GLM
- Linear output + MSE loss ← Gaussian GLM
- Softmax output + categorical CE loss ← Multinomial GLM
- Exponential output + Poisson NLL ← Poisson GLM

### Generalised Linear Models → Generalised Neural Networks

Deep learning generalises GLMs by inserting hidden layers between input and the final GLM output layer:

$$\hat{\mu} = g^{-1}(W_L \cdot \text{ReLU}(W_{L-1} \cdots \text{ReLU}(W_1 x)))$$

The final layer is always a GLM. The hidden layers learn the features $\phi_\theta(x)$ on which the GLM operates. The loss function is always the NLL of the appropriate exponential family distribution.

This is why knowing the GLM framework matters for deep learning: **it tells you exactly which loss function and output activation to pair**, and why those specific combinations are principled.

### The Canonical Pairings in Deep Learning

|Task|Output activation|Loss function|GLM family|
|---|---|---|---|
|Regression|Linear (identity)|MSE|Gaussian|
|Binary classification|Sigmoid|BCE|Bernoulli|
|Multi-class classification|Softmax|Categorical CE|Multinomial|
|Count prediction|Exponential (or softplus)|Poisson NLL|Poisson|
|Positive-valued regression|Softplus / exp|Gamma NLL|Gamma|

When you see a deep learning practitioner choose `nn.BCEWithLogitsLoss` for binary classification or `nn.CrossEntropyLoss` for multi-class, they are — whether they know it or not — choosing the canonical GLM for the Bernoulli or Multinomial family.

### Natural Gradient Descent

The Fisher information matrix of a GLM is:

$$F = \mathbb{E}\left[\nabla_w \log p(y|x;w) \cdot \nabla_w \log p(y|x;w)^T\right] = \frac{1}{n}X^T W X$$

This is the same as the Hessian. **Natural gradient descent** uses $F^{-1}$ instead of the identity as the metric:

$$w \leftarrow w - \alpha F^{-1} \nabla_w \mathcal{L}$$

This is equivalent to IRLS for GLMs, and is the motivation for **K-FAC** and other second-order optimisers used in large neural network training. The GLM is where this theory is cleanest and most tractable.
