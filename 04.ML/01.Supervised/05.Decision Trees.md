## What a Decision Tree Is

A decision tree partitions the input space $\mathcal{X} = \mathbb{R}^d$ into **axis-aligned rectangular regions**, then predicts a constant in each region.

```
Training data (2 features)       Learned tree

  x_2                              Is x_1 < 2.5?
  4 в”Ө  в—Ӣ  в—Ӣ  в”Ӯ  в—Ҹ  в—Ҹ             в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”җ

  2 в”Өв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ          (в—Ӣ)     Is x_2 < 2?
  1 в”Ө  в—Ҹ  в—Ҹ  в”Ӯ  в—Ҹ  в—Ҹ                  в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”җ
    в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ x_1            Yes        No
         2.5                         (в—Ҹ)        (в—Ҹ)

Four rectangles, each predicts one class
```

Every **internal node** applies a threshold test on a single feature: $x_j \leq t$. Every **leaf node** contains a prediction: a class label (classification) or a real value (regression).

This is a **non-parametric** model вҖ” the number of parameters grows with the data, unlike linear models where $|w| = d$ is fixed.

## The CART Algorithm

**CART** (Classification And Regression Trees, Breiman et al. 1984) is the standard algorithm for growing decision trees. It is **greedy**: at each node, it picks the single best split without lookahead.

### The Splitting Procedure

At each node containing a subset $\mathcal{S}$ of training examples:

1. For every feature $j \in {1, \ldots, d}$ and every candidate threshold $t$:
    - Split $\mathcal{S}$ into $\mathcal{S}_L = {(x,y) \in \mathcal{S} : x_j \leq t}$ and $\mathcal{S}_R = {(x,y) \in \mathcal{S} : x_j > t}$
    - Compute the **impurity reduction** (gain) of this split
2. Choose the $(j^_, t^_)$ that maximises the gain.
3. Recurse on $\mathcal{S}_L$ and $\mathcal{S}_R$ until a stopping criterion is met.
4. At each leaf, predict the majority class (classification) or mean $y$ (regression).

**Candidate thresholds:** For a feature with values ${v_1 < v_2 < \cdots < v_m}$, the candidates are the $m-1$ midpoints ${(v_k + v_{k+1})/2}$. Total work per node: $O(nd \log n)$ if features are pre-sorted.

---

## Splitting Criteria

### For Regression: Variance Reduction

Predict the mean $\bar{y}_R$ in each region $R$. The loss is:

$$\mathcal{L}(R) = \sum_{(x,y) \in R} (y - \bar{y}_R)^2$$

The gain of a split into $\mathcal{S}_L, \mathcal{S}_R$ is:

$$\Delta = \mathcal{L}(\mathcal{S}) - \mathcal{L}(\mathcal{S}_L) - \mathcal{L}(\mathcal{S}_R)$$

Greedily maximise $\Delta$. This is equivalent to finding the split that minimises the weighted sum of child variances.

### For Classification: Gini Impurity

Let $p_k$ be the fraction of class-$k$ examples in node $\mathcal{S}$. The **Gini impurity** is:

$$G(\mathcal{S}) = \sum_{k=1}^K p_k(1 - p_k) = 1 - \sum_{k=1}^K p_k^2$$

Interpretation: if you randomly pick two examples from $\mathcal{S}$ and randomly assign labels according to the class distribution, $G$ is the probability they disagree.

- Pure node (all one class): $G = 0$.
- Maximum impurity ($K$ equal classes): $G = 1 - 1/K$.

The **Gini gain** of a split:

$$\Delta G = G(\mathcal{S}) - \frac{|\mathcal{S}_L|}{|\mathcal{S}|} G(\mathcal{S}_L) - \frac{|\mathcal{S}_R|}{|\mathcal{S}|} G(\mathcal{S}_R)$$

### For Classification: Information Gain (Entropy)

The **entropy** of a node:

$$H(\mathcal{S}) = -\sum_{k=1}^K p_k \log_2 p_k$$

- Pure node: $H = 0$.
- Maximum impurity ($K$ equal classes): $H = \log_2 K$.

The **information gain** of a split:

$$\text{IG} = H(\mathcal{S}) - \frac{|\mathcal{S}_L|}{|\mathcal{S}|} H(\mathcal{S}_L) - \frac{|\mathcal{S}_R|}{|\mathcal{S}|} H(\mathcal{S}_R)$$

This is the **mutual information** between the split and the class label вҖ” the reduction in uncertainty about $y$ after observing which side of the split $x$ falls on.

### The Bias of Information Gain: High-Cardinality Features

Information gain **favours features with many distinct values**. An ID column that is unique per example would always be chosen вҖ” it perfectly partitions the data вҖ” but generalises to zero.

**Fix: Gain Ratio** (C4.5 algorithm):

$$\text{GainRatio} = \frac{\text{IG}}{\text{SplitInfo}}, \quad \text{SplitInfo} = -\sum_v \frac{|\mathcal{S}_v|}{|\mathcal{S}|}\log_2\frac{|\mathcal{S}_v|}{|\mathcal{S}|}$$

SplitInfo penalises splits that produce many small partitions. CART avoids this problem entirely by only considering binary splits.

## Stopping Criteria and Pruning

### Stopping Criteria (Pre-Pruning)

Stop splitting when:

- `max_depth` is reached.
- `min_samples_split`: node has fewer than $m$ examples.
- `min_samples_leaf`: a proposed split would create a leaf with fewer than $m$ examples.
- `min_impurity_decrease`: gain $\Delta < \epsilon$.

Pre-pruning is fast but may stop too early (a split that looks bad locally may be the root of a good subtree).

### Cost-Complexity Pruning (Post-Pruning, CART)

Grow the full tree $T_0$ to maximum depth. Then define the **cost-complexity** of a subtree $T$:

$$R_\alpha(T) = R(T) + \alpha |T|$$

where $R(T) = \sum_{\text{leaves}} \frac{|\mathcal{S}_\ell|}{n} \cdot \text{impurity}(\mathcal{S}_\ell)$ is the training error and $|T|$ is the number of leaves.

For each $\alpha \geq 0$, there is a unique **minimal subtree** $T(\alpha) \subseteq T_0$ that minimises $R_\alpha$. As $\alpha$ increases, $T(\alpha)$ shrinks (more leaves are collapsed into their parent). This produces a sequence of nested subtrees:

$$T_0 \supset T_1 \supset T_2 \supset \cdots \supset {\text{root}}$$

Select $\alpha$ (and thus $T(\alpha)$) by cross-validation.

## Decision Trees as Function Approximators

A regression tree with $M$ leaves approximates $f(x)$ as:

$$\hat{f}(x) = \sum_{m=1}^M c_m \cdot \mathbf{1}[x \in R_m]$$

where $R_1, \ldots, R_M$ are the rectangular regions and $c_m = \bar{y}_{R_m}$ is the mean response in region $m$.

This is a **piecewise constant function** вҖ” a step function in each dimension. It can approximate any continuous function arbitrarily well given enough leaves (universal approximation on compact domains), but requires exponentially many leaves for smooth functions.


Compare with linear regression (globally smooth, low variance, high bias for nonlinear $f$) and polynomial regression (smooth, can overfit). Trees are in a third camp: adaptive but blocky.

## The Variance Problem

A decision tree grown to full depth **memorises the training data**. Every training example can be isolated in its own leaf with zero training loss.

**Why high variance?** The greedy splitting procedure is highly sensitive to the training set. A small change вҖ” one mislabelled point, one duplicated example вҖ” can cause an entirely different split to be chosen at the root, which cascades into a completely different tree structure.

This is not a failure of the algorithm; it is a fundamental property of the hypothesis class. Deep trees have very high **VC dimension** (exponential in depth).

```
Dataset 1                Dataset 2 (one point moved)
     Is x_1 < 2.5?            Is x_2 < 3.1?
    в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”җ               в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”җ
   Yes        No             Yes        No
  (в—Ӣ)    Is x_2<2?          ...     completely
         в”Ңв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”җ                different
        Yes        No               subtree
        (в—Ҹ)        (в—Ӣ)
```

**The fix:** ensembles. [[01.Bagging and Random Forests]] reduces variance by averaging many trees. [[02.Boosting]] reduces bias by building trees sequentially.

## Feature Importance

A trained CART tree gives a natural feature importance score: **mean decrease in impurity (MDI)**.

For each feature $j$, sum the impurity reduction across all nodes that split on $j$, weighted by the fraction of samples reaching that node:

$$\text{importance}(j) = \sum_{\text{nodes } t : \text{split on } j} \frac{|\mathcal{S}_t|}{n} \cdot \Delta\text{impurity}(t)$$

Normalise to sum to 1. sklearn's `feature_importances_` attribute implements this.

**Caveat:** MDI is biased towards high-cardinality numerical features. Use **permutation importance** (see [[03.Model Interpretability]]) for a less biased estimate.

## Full Implementation from Scratch

```python
import numpy as np
from collections import Counter

class Node:
    def __init__(self, feature=None, threshold=None,
                 left=None, right=None, value=None):
        self.feature   = feature    # split feature index
        self.threshold = threshold  # split threshold
        self.left      = left       # left child (x_j <= t)
        self.right     = right      # right child (x_j > t)
        self.value     = value      # leaf prediction (if leaf)

    def is_leaf(self):
        return self.value is not None


class DecisionTreeClassifier:
    def __init__(self, max_depth=None, min_samples_split=2,
                 criterion='gini'):
        self.max_depth        = max_depth
        self.min_samples_split = min_samples_split
        self.criterion        = criterion
        self.root             = None

    # в”Җв”Җ Impurity functions в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    def _gini(self, y):
        n = len(y)
        if n == 0:
            return 0.0
        counts = np.bincount(y)
        probs  = counts / n
        return 1.0 - np.sum(probs ** 2)

    def _entropy(self, y):
        n = len(y)
        if n == 0:
            return 0.0
        counts = np.bincount(y)
        probs  = counts[counts > 0] / n
        return -np.sum(probs * np.log2(probs))

    def _impurity(self, y):
        return self._gini(y) if self.criterion == 'gini' \
               else self._entropy(y)

    # в”Җв”Җ Best split search в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    def _best_split(self, X, y):
        n, d = X.shape
        best_gain = -np.inf
        best_feat, best_thresh = None, None
        parent_impurity = self._impurity(y)

        for j in range(d):
            thresholds = np.unique(X[:, j])
            for t in (thresholds[:-1] + thresholds[1:]) / 2:
                left_mask  = X[:, j] <= t
                right_mask = ~left_mask
                if left_mask.sum() == 0 or right_mask.sum() == 0:
                    continue

                gain = parent_impurity \
                     - (left_mask.sum()/n)  * self._impurity(y[left_mask]) \
                     - (right_mask.sum()/n) * self._impurity(y[right_mask])

                if gain > best_gain:
                    best_gain   = gain
                    best_feat   = j
                    best_thresh = t

        return best_feat, best_thresh, best_gain

    # в”Җв”Җ Recursive tree building в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    def _build(self, X, y, depth):
        n_samples, _ = X.shape
        # Stopping conditions
        if (self.max_depth is not None and depth >= self.max_depth) \
                or n_samples < self.min_samples_split \
                or len(np.unique(y)) == 1:
            return Node(value=Counter(y).most_common(1)[0][0])

        feat, thresh, gain = self._best_split(X, y)
        if feat is None or gain <= 0:
            return Node(value=Counter(y).most_common(1)[0][0])

        left_mask = X[:, feat] <= thresh
        left  = self._build(X[left_mask],  y[left_mask],  depth + 1)
        right = self._build(X[~left_mask], y[~left_mask], depth + 1)
        return Node(feature=feat, threshold=thresh,
                    left=left, right=right)

    def fit(self, X, y):
        self.root = self._build(X, y, depth=0)
        return self

    # в”Җв”Җ Prediction в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    def _predict_one(self, x, node):
        if node.is_leaf():
            return node.value
        if x[node.feature] <= node.threshold:
            return self._predict_one(x, node.left)
        return self._predict_one(x, node.right)

    def predict(self, X):
        return np.array([self._predict_one(x, self.root) for x in X])
```

## Connection to AI, Machine Learning, and Deep Learning

### Trees as the Foundation of the Best Tabular Models

Decision trees are not used alone in modern ML вҖ” but they are the **base learner** in the two most powerful tabular ML algorithms:

- **Random Forests** ([[01.Bagging and Random Forests]]): average $B$ independently trained trees. Reduces variance while keeping bias low.
- **Gradient Boosted Trees** ([[02.Boosting]]): fit trees sequentially to pseudo-residuals of the loss. **XGBoost**, **LightGBM**, **CatBoost** вҖ” the algorithms that dominate Kaggle competitions and production tabular ML вҖ” are all gradient boosted trees.

Understanding CART deeply is a prerequisite for understanding why these ensembles work and what their hyperparameters mean.

### Why Trees Dominate Tabular ML (and Neural Networks Don't)

On tabular data, tree ensembles consistently outperform deep neural networks. Several reasons, rooted in the tree's structure:

1. **Axis-aligned splits handle mixed feature types naturally.** Categorical, ordinal, and continuous features can coexist without normalisation or embedding.
2. **Piecewise constant functions are robust to outliers.** A single point at $x_j = 10^6$ does not distort the global model вҖ” it just determines one split threshold.
3. **Trees are invariant to monotone transformations of features.** $\log x$, $x^2$, $\sqrt{x}$ all produce identical trees (since thresholds adjust). Neural networks must learn these invariances.
4. **Tabular data is often not smooth.** The inductive bias of deep networks вҖ” smooth, compositional representations вҖ” matches image/text/audio well. Tabular data with heterogeneous features often has discontinuities that trees handle directly.

### Oblique Trees and Neural Networks

Standard CART splits on one feature at a time: $x_j \leq t$. This produces axis-aligned boundaries. A neural network (even with one hidden layer) can produce **oblique** boundaries: $w^Tx \leq t$, a linear combination of features.

**Oblique trees** generalise CART by allowing splits of the form $w^Tx \leq t$ вҖ” each internal node is a mini linear classifier. They are more powerful but harder to optimise (the split search is no longer over a 1D threshold).

Recent work (2020s) on **Neural Oblivious Decision Trees (NODE)**, **TabNet**, and **GRANDE** attempts to bring differentiable tree-like structures into deep learning for tabular data, enabling end-to-end gradient training.

### Trees and Explainability in Production ML

In regulated industries (finance, healthcare, lending), model decisions must often be **explained to regulators or customers**. Decision trees remain the go-to for this:

- A shallow tree (depth 3вҖ“5) produces human-readable rules.
- **SHAP TreeExplainer** ([[03.Model Interpretability]]) computes exact Shapley values for tree ensembles in $O(TLD^2)$ time (where $T$ = number of trees, $L$ = leaves, $D$ = depth) вҖ” far faster than model-agnostic SHAP.
- Regulatory frameworks (EU AI Act, US fair lending) often require that models be auditable вҖ” trees and tree-based SHAP values are the standard tool.

### The Information-Theoretic View

The information gain criterion is directly the **mutual information** $I(X_j; Y | \mathcal{S})$ between feature $j$ and the label in the current node. This connects decision trees to:

- **Feature selection** (maximum information feature is always split on first).
- **Decision theory** (optimal strategy under a sequence of binary tests вҖ” ID3 is a greedy solution to this).
- **Huffman coding** (optimal binary trees for source coding also maximise information per bit).
