# 06. Naive Bayes

> _"Naive Bayes is the first generative classifier we encounter, and the word 'naive' deserves unpacking. The assumption — that features are conditionally independent given the class — is almost never true. And yet the classifier works. Sometimes embarrassingly well. The reason is subtle: for classification, we only need to get the argmax of the posterior right, not the posterior itself. A model can be badly miscalibrated in its probability estimates while still making correct decisions at the decision boundary. Understanding exactly why the naive assumption survives despite being wrong is more instructive than dismissing the model as a toy."_

## Generative vs. Discriminative: The Fork in the Road

Every supervised classifier learns some mapping from $x$ to $y$. But there are two fundamentally different strategies:

**Discriminative classifiers** (logistic regression, SVM, neural networks) model $P(y \mid x)$ directly. They learn the boundary between classes without asking how the data was generated.

**Generative classifiers** (Naive Bayes, LDA, QDA) model the **joint distribution** $P(x, y) = P(x \mid y) P(y)$. They learn how each class _generates_ data, then use Bayes' theorem to invert: given $x$, which class most likely generated it?

```
Discriminative                   Generative
─────────────────────────────────────────────────────────
Model P(y|x) directly            Model P(x|y) and P(y)
Learn the boundary               Learn each class's shape
Can't generate new x             Can sample new x ~ P(x|y)
Often higher accuracy            Often needs fewer data
No probabilistic story for x     Full probability model

     ○  ○   │  ●  ●                 ○  ○       ●  ●
     ○  ○   │  ●  ●               (○ cloud)  (● cloud)
            │                      Bayes: which cloud
       boundary                    does this x come from?
```

Naive Bayes is the simplest generative classifier. It models $P(x \mid y)$ with a strong independence assumption that makes the model tractable even in high dimensions.

---

## Bayes' Theorem for Classification

Given a new point $x$, we want the **posterior**:

$$P(y = k \mid x) = \frac{P(x \mid y = k), P(y = k)}{P(x)}$$

The denominator $P(x) = \sum_{k} P(x \mid y=k) P(y=k)$ is constant across classes. So the decision rule is:

$$\hat{y} = \arg\max_k P(y = k \mid x) = \arg\max_k P(x \mid y = k), P(y = k)$$

Three quantities needed:

- $P(y = k)$: the **prior** — how common is class $k$?
- $P(x \mid y = k)$: the **class-conditional likelihood** — how likely is $x$ given class $k$?
- $P(x)$: the **evidence** — ignored for classification (constant across classes).

The challenge is $P(x \mid y = k)$. If $x \in \mathbb{R}^d$, estimating a full $d$-dimensional density from data requires exponentially many samples. We need a simplifying assumption.

---

## The Naive Assumption

**Assumption:** Given the class label $y$, the features $x_1, x_2, \ldots, x_d$ are **conditionally independent**:

$$P(x \mid y = k) = \prod_{j=1}^d P(x_j \mid y = k)$$

This factorises the joint density into a product of $d$ one-dimensional densities. The problem becomes tractable: estimate each $P(x_j \mid y = k)$ separately from data.

### When Is This Wrong?

Almost always. In a spam classifier, the words "cheap" and "price" are correlated — seeing one makes the other more likely. The assumption says they are independent given spam/not-spam. That is false.

### Why Does It Still Work?

The decision rule requires only:

$$\arg\max_k P(y=k \mid x) = \arg\max_k P(y=k)\prod_j P(x_j \mid y=k)$$

For a **correct classification**, we only need the right class to have the highest posterior — not for the posteriors to be accurate. The argmax can be correct even when the individual probabilities are wildly off.

Formally: if the true decision boundary is $P(y=1|x) = P(y=0|x)$, the Naive Bayes decision boundary is in the right place when the errors in $P(x_j|y)$ cancel out symmetrically. This happens more often than one might expect.

A second reason: conditional independence reduces variance. Fewer parameters to estimate means less overfitting, which can outweigh the bias introduced by the wrong assumption — especially when $n$ is small relative to $d$.

## Estimating the Parameters

### Prior: $P(y = k)$

From the training data, use the empirical class frequency:

$$\hat{P}(y = k) = \frac{n_k}{n}$$

where $n_k$ is the number of training examples with label $k$.

### Class-Conditional Likelihood: $P(x_j \mid y = k)$

The choice here defines which _variant_ of Naive Bayes we use.

---

## Variant 1 — Gaussian Naive Bayes

**When to use:** continuous features.

**Assumption:** Each feature $x_j$ given class $k$ follows a Gaussian:

$$P(x_j \mid y = k) = \mathcal{N}(x_j; \mu_{jk}, \sigma_{jk}^2) = \frac{1}{\sqrt{2\pi\sigma_{jk}^2}}\exp!\left(-\frac{(x_j - \mu_{jk})^2}{2\sigma_{jk}^2}\right)$$

**Parameter estimation (MLE):**

$$\hat{\mu}_{jk} = \frac{1}{n_k}\sum_{i: y^{(i)}=k} x_j^{(i)}, \qquad \hat{\sigma}_{jk}^2 = \frac{1}{n_k}\sum_{i: y^{(i)}=k}(x_j^{(i)} - \hat{\mu}_{jk})^2$$

That is: compute the mean and variance of feature $j$ **within each class** separately.

**Classification:**

$$\hat{y} = \arg\max_k \left[\log P(y=k) + \sum_{j=1}^d \log \mathcal{N}(x_j; \hat{\mu}_{jk}, \hat{\sigma}_{jk}^2)\right]$$

Working in log-space avoids numerical underflow from multiplying many small probabilities.

**Relationship to LDA/QDA:**

- **LDA** (Linear Discriminant Analysis): Gaussian NB with shared covariance $\Sigma_k = \Sigma$ for all $k$, but without the independence assumption — uses the full covariance matrix. Produces a **linear** decision boundary.
- **QDA** (Quadratic Discriminant Analysis): class-specific full covariance matrices $\Sigma_k$. Produces a **quadratic** boundary.
- **Gaussian NB**: class-specific diagonal covariance matrices (independence assumption = zero off-diagonals). Decision boundary depends on $\sigma_{jk}$; can be quadratic.

```
LDA (shared Σ)       QDA (class Σ_k)      Gaussian NB (diag Σ_k)
  ○  ○   │  ●  ●     ○  ○  )  ●  ●          ○  ○  )  ●  ●
  ○  ○   │  ●  ●     ○  ○ )    ●  ●          ○  ○  )  ●  ●
         │            ○  )       ●                (
  Linear boundary     Quadratic boundary     Axis-aligned quadratic
```

## Variant 2 — Multinomial Naive Bayes

**When to use:** count features — most commonly, **bag-of-words text classification**.

Represent a document as a vector of word counts: $x_j = $ count of word $j$ in the document.

**Assumption:** The words in a document are drawn independently from a multinomial distribution parameterised by $\theta_k = (\theta_{1k}, \ldots, \theta_{|V|k})$ where $\theta_{jk} = P(\text{word } j \mid \text{class } k)$ and $\sum_j \theta_{jk} = 1$.

The likelihood of document $x$ given class $k$:

$$P(x \mid y = k) = \frac{(\sum_j x_j)!}{\prod_j x_j!} \prod_j \theta_{jk}^{x_j}$$

The factorial term is constant across classes, so the decision rule becomes:

$$\hat{y} = \arg\max_k \left[\log P(y=k) + \sum_j x_j \log \theta_{jk}\right]$$

**MLE for $\theta_{jk}$:** count how often word $j$ appears in class-$k$ documents, divided by total words in class-$k$ documents:

$$\hat{\theta}_{jk} = \frac{\sum_{i: y^{(i)}=k} x_j^{(i)}}{\sum_{j'} \sum_{i: y^{(i)}=k} x_{j'}^{(i)}}$$

### Laplace Smoothing — The Zero-Count Problem

If word $j$ never appears in class-$k$ training documents, then $\hat{\theta}_{jk} = 0$ and $\log \hat{\theta}_{jk} = -\infty$. A single unseen word **annihilates the entire posterior** for class $k$.

This is **catastrophic** — a test document containing one word not in the training vocabulary for class $k$ will never be assigned to class $k$, regardless of all other evidence.

**Fix: Laplace (add-$\alpha$) smoothing.** Add a pseudo-count $\alpha > 0$ to every word count:

$$\hat{\theta}_{jk}^{\text{smooth}} = \frac{\left(\sum_{i: y^{(i)}=k} x_j^{(i)}\right) + \alpha}{\left(\sum_{j'}\sum_{i: y^{(i)}=k} x_{j'}^{(i)}\right) + \alpha|V|}$$

where $|V|$ is the vocabulary size.

- $\alpha = 1$: **Laplace smoothing** (add-one smoothing). Standard default.
- $\alpha \to 0$: recovers MLE.
- $\alpha \to \infty$: all words become equally likely (uniform prior over vocabulary).

**Bayesian interpretation:** Laplace smoothing is MAP estimation with a **Dirichlet prior** $\text{Dir}(\alpha, \alpha, \ldots, \alpha)$ over $\theta_k$. The prior adds $\alpha$ pseudo-observations of each word to every class.

```
Without smoothing:                With Laplace smoothing (α=1):

Word "antidisestablishmentarianism"    Same word, unseen in class "sport"
  → θ̂ = 0/500 = 0                    → θ̂ = (0+1)/(500+|V|) ≈ tiny but > 0
  → log(0) = -∞                       → log(tiny) = large negative, but finite
  → P(sport | doc) = 0                → P(sport | doc) > 0, other words still count
```

## Variant 3 — Bernoulli Naive Bayes

**When to use:** binary features — typically, **word presence/absence** in a document (not counts).

Each feature $x_j \in {0, 1}$:

$$P(x_j \mid y = k) = \theta_{jk}^{x_j}(1 - \theta_{jk})^{1-x_j}$$

where $\theta_{jk} = P(x_j = 1 \mid y = k)$.

**MLE:** $\hat{\theta}_{jk} = \frac{\text{(# documents in class } k \text{ containing word } j)}{n_k}$.

**Key difference from Multinomial NB:** Bernoulli NB explicitly models the **absence** of a word. A word not in the document contributes $\log(1 - \theta_{jk})$ to the log-likelihood. Multinomial NB ignores absent words.

For short documents, Bernoulli NB often works better. For long documents (many word counts), Multinomial NB is preferred.

---

## The Full Generative Story

Naive Bayes defines a complete generative model:

$$P(x, y) = P(y) \prod_{j=1}^d P(x_j \mid y)$$

To **sample** a new document from class $k$:

1. Draw $y = k$ with probability $P(y=k)$.
2. For each feature $j$, independently draw $x_j \sim P(x_j \mid y=k)$.

This is a proper probabilistic generative process — unlike discriminative models, which have no story for how $x$ is generated.

---

## Decision Boundary of Naive Bayes

For **binary classification** ($K = 2$), the decision rule is $P(y=1|x) \geq P(y=0|x)$, i.e.:

$$\log\frac{P(y=1|x)}{P(y=0|x)} \geq 0$$

Expanding using Bayes' theorem and the naive assumption:

$$\log\frac{P(y=1)}{P(y=0)} + \sum_{j=1}^d \log\frac{P(x_j|y=1)}{P(x_j|y=0)} \geq 0$$

For **Gaussian NB with equal variances** ($\sigma_{j0} = \sigma_{j1} = \sigma_j$), each log-ratio term is:

$$\log\frac{\mathcal{N}(x_j;\mu_{j1},\sigma_j^2)}{\mathcal{N}(x_j;\mu_{j0},\sigma_j^2)} = \frac{(\mu_{j1}-\mu_{j0})}{\sigma_j^2}x_j - \frac{\mu_{j1}^2 - \mu_{j0}^2}{2\sigma_j^2}$$

This is **linear in $x_j$**, so the sum over $j$ is linear in $x$. The decision boundary is a **hyperplane** — identical in form to logistic regression.

> **Theoretical connection:** Gaussian NB with equal class variances and logistic regression both produce linear decision boundaries. They differ in _how_ they estimate the parameters: Gaussian NB uses generative MLE (fits each class separately), logistic regression uses discriminative MLE (fits the boundary directly). On finite data, logistic regression typically has lower asymptotic error; Gaussian NB converges faster with fewer samples. This is the classic **generative-discriminative tradeoff** (Ng & Jordan, 2002).

For **unequal class variances**, the boundary is quadratic (one quadratic term per feature from the ratio of two Gaussians with different $\sigma^2$).

---

## Naive Bayes as a Linear Classifier

For Multinomial NB, the decision function is:

$$f_k(x) = \log P(y=k) + \sum_j x_j \log \theta_{jk}$$

This is a **linear function of the feature vector $x$**. In matrix form: $f(x) = Wx + b$ where:

- $W_{kj} = \log \theta_{jk}$ (log word probabilities)
- $b_k = \log P(y=k)$ (log prior)

Multinomial Naive Bayes is a **linear classifier** — the same hypothesis class as logistic regression. The difference is only in how the weights are estimated: generatively (MLE of class-conditional densities) vs. discriminatively (MLE of conditional likelihood).

## Implementation from Scratch

```python
import numpy as np

class GaussianNaiveBayes:
    def fit(self, X, y):
        self.classes_ = np.unique(y)
        self.priors_  = {}
        self.means_   = {}
        self.vars_    = {}
        n = len(y)
        for k in self.classes_:
            X_k = X[y == k]
            self.priors_[k] = len(X_k) / n
            self.means_[k]  = X_k.mean(axis=0)       # shape (d,)
            self.vars_[k]   = X_k.var(axis=0) + 1e-9 # shape (d,) — add ε for stability
        return self

    def _log_likelihood(self, x, k):
        mu, var = self.means_[k], self.vars_[k]
        # sum of log N(x_j; mu_jk, var_jk)
        return -0.5 * np.sum(np.log(2 * np.pi * var) + (x - mu)**2 / var)

    def predict(self, X):
        preds = []
        for x in X:
            log_posts = {
                k: np.log(self.priors_[k]) + self._log_likelihood(x, k)
                for k in self.classes_
            }
            preds.append(max(log_posts, key=log_posts.get))
        return np.array(preds)

    def predict_proba(self, X):
        # Numerically stable: subtract max log-posterior before exponentiating
        proba = []
        for x in X:
            log_posts = np.array([
                np.log(self.priors_[k]) + self._log_likelihood(x, k)
                for k in self.classes_
            ])
            log_posts -= log_posts.max()      # log-sum-exp trick
            posts = np.exp(log_posts)
            proba.append(posts / posts.sum())
        return np.array(proba)


class MultinomialNaiveBayes:
    def __init__(self, alpha=1.0):
        self.alpha = alpha  # Laplace smoothing

    def fit(self, X, y):
        """X: count matrix (n_docs × vocab_size), y: integer class labels."""
        self.classes_ = np.unique(y)
        n, V = X.shape
        self.log_priors_  = {}
        self.log_thetas_  = {}  # log P(word j | class k)
        for k in self.classes_:
            X_k = X[y == k]
            self.log_priors_[k] = np.log(len(X_k) / n)
            word_counts = X_k.sum(axis=0) + self.alpha      # Laplace smoothed
            self.log_thetas_[k] = np.log(word_counts / word_counts.sum())
        return self

    def predict(self, X):
        log_posts = np.array([
            self.log_priors_[k] + X @ self.log_thetas_[k]
            for k in self.classes_
        ]).T                                  # shape (n_docs, n_classes)
        return self.classes_[log_posts.argmax(axis=1)]
```

## Spam Filter: End-to-End Example

```python
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report

# Toy corpus
docs = [
    "buy cheap viagra now",      "win money prize lottery",
    "click here free offer",     "get rich quick scheme",
    "meeting at 3pm tomorrow",   "project report attached",
    "lunch tomorrow?",           "team standup 9am",
]
labels = [1, 1, 1, 1, 0, 0, 0, 0]   # 1 = spam, 0 = ham

# Vectorise to word counts
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(docs)

# Train
X_train, X_test, y_train, y_test = train_test_split(
    X, labels, test_size=0.25, random_state=42)

clf = MultinomialNB(alpha=1.0)
clf.fit(X_train, y_train)

# Inspect learned log-probabilities
vocab = vectorizer.get_feature_names_out()
# For class 1 (spam), highest log-prob words:
spam_word_probs = clf.feature_log_prob_[1]
top_spam_words  = vocab[spam_word_probs.argsort()[-5:][::-1]]
print("Top spam words:", top_spam_words)
```

## Practical Notes

### When Naive Bayes Is a Good Choice

- **Very high-dimensional, sparse features:** text, DNA sequences, click logs. The independence assumption becomes less damaging as $d$ grows (central limit theorem intuition: the errors in each factor tend to cancel).
- **Tiny training sets:** with few examples, the variance of a discriminative model (which must estimate the boundary from few points) can exceed the bias of the independence assumption.
- **Real-time inference at scale:** training is a single pass over the data ($O(nd)$), prediction is $O(Kd)$. No matrix inversions, no gradient descent.
- **Streaming / online updates:** new data can be incorporated by updating sufficient statistics (counts, means, variances) without retraining.

### When Naive Bayes Fails

- **Strong feature correlations:** document classification where context matters (the word "not" preceding a sentiment word is ignored by bag-of-words NB).
- **Calibrated probabilities needed:** the probability outputs of Naive Bayes are notoriously unreliable — they tend to be too extreme (too close to 0 or 1). The independence assumption causes the model to be overconfident. Use Platt scaling or isotonic regression to recalibrate.
- **Numerical features with complex distributions:** if the true $P(x_j|y)$ is bimodal or heavy-tailed, the Gaussian assumption will hurt.

## Connection to AI, Machine Learning, and Deep Learning

### Naive Bayes and Text Classification in NLP

Before deep learning, Multinomial Naive Bayes was the standard text classifier. It still appears in:

- **Spam filtering:** GMail's spam filter has historically used NB variants.
- **Sentiment analysis (baseline):** Always compare deep NLP models against a Naive Bayes baseline. If BERT only beats NB by 2%, question whether BERT is worth the cost.
- **Document classification:** fast, interpretable, works well on short documents.

Modern NLP has replaced NB with neural language models (BERT, GPT), but NB is the **conceptual ancestor**: the bag-of-words model, the log-linear scoring function, and the idea of word probabilities per class all survive in modern language models in more sophisticated form.

### The Naive Bayes–Logistic Regression Duality

Ng & Jordan (2002) formally proved:

- **Naive Bayes converges** to its optimal error faster (needs fewer samples) — $O(\log d)$ samples for $d$ features under the independence assumption.
- **Logistic regression converges** to a lower asymptotic error (its long-run performance is better when the independence assumption is violated).

In practice: use Naive Bayes when $n$ is small or you need speed; use logistic regression (or neural nets) when $n$ is large and accuracy matters.

This generalises: generative models beat discriminative models in the low-data regime; discriminative models win in the large-data regime. This principle recurs throughout ML — e.g., GPT-style pre-training (generative) + fine-tuning (discriminative) exploits both regimes.

### Naive Bayes and the Exponential Family

The Multinomial NB scoring function $\log P(y=k) + \sum_j x_j \log\theta_{jk}$ is a linear classifier. Recall from [[03.Generalised Linear Models]] that logistic regression is also linear. Both are instances of a linear model in the exponential family. The key difference:

||Naive Bayes|Logistic Regression|
|---|---|---|
|Estimates|$P(x|y)$ and $P(y)$|
|Parameters|Class-conditional + prior|One global weight vector|
|Optimisation|MLE per class (closed form)|MLE of conditional (gradient descent)|
|Expressiveness|Constrained by generative model|More flexible boundary|

### Naive Bayes as a Baseline in Production ML

In production ML systems, Naive Bayes serves three roles:

1. **Speed baseline:** train in milliseconds, predict in microseconds. If a complex model isn't significantly better, use NB.
2. **Cold-start model:** when a new product launches with no data, NB can be trained on synthetic or transferred priors immediately.
3. **Sanity check:** if your deep model doesn't beat NB, something is wrong with your data pipeline or label quality.
