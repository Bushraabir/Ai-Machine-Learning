## The Setup

We have $n$ training examples ${(x^{(i)}, y^{(i)})}_{i=1}^n$ where $x^{(i)} \in \mathbb{R}^d$ and $y^{(i)} \in \mathbb{R}$. We want to learn a function $f: \mathbb{R}^d \to \mathbb{R}$.

Linear regression says: **assume $f$ is affine.**

$$\hat{y} = w^T x + b = w_1 x_1 + w_2 x_2 + \cdots + w_d x_d + b$$

**Absorbing the bias.** Append a constant feature $x_0 = 1$ to every input vector. Then:

$$\hat{y} = w^T x, \quad w \in \mathbb{R}^{d+1}, \quad x = \begin{bmatrix} 1 \ x_1 \ \vdots \ x_d \end{bmatrix}$$

In matrix form, stack training examples as rows:

$$X = \begin{bmatrix} 1 & x_1^{(1)} & \cdots & x_d^{(1)} \ \vdots & & & \vdots \ 1 & x_1^{(n)} & \cdots & x_d^{(n)} \end{bmatrix} \in \mathbb{R}^{n \times (d+1)}, \quad y = \begin{bmatrix} y^{(1)} \ \vdots \ y^{(n)} \end{bmatrix}$$

Predictions: $\hat{y} = Xw$.

---

## The Loss Function

We minimise the **Mean Squared Error (MSE)**:

$$\mathcal{L}(w) = \frac{1}{n}|Xw - y|^2 = \frac{1}{n}(Xw - y)^T(Xw - y)$$

> **Why squared error?** Three answers, increasing in depth:
> 
> 1. _Convenient:_ differentiable everywhere, unique minimum (if $X$ has full column rank).
> 2. _Geometric:_ minimising MSE = finding the projection of $y$ onto $\text{col}(X)$.
> 3. _Probabilistic:_ MSE = MLE under a Gaussian noise model (derived below).

---

## Solution 1 — The Normal Equation (Closed Form)

### Derivation

Expand the loss:

$$\mathcal{L}(w) = \frac{1}{n}(w^T X^T X w - 2 w^T X^T y + y^T y)$$

Take the gradient with respect to $w$ and set to zero. Using the matrix calculus identities $\nabla_w (w^T A w) = 2Aw$ (for symmetric $A$) and $\nabla_w (b^T w) = b$:

$$\nabla_w \mathcal{L} = \frac{2}{n}(X^T X w - X^T y) = 0$$

$$\boxed{\hat{w} = (X^T X)^{-1} X^T y}$$

This is the **Normal Equation**.

### Geometric Interpretation

The residual $r = y - X\hat{w}$ must be orthogonal to every column of $X$:

$$X^T r = X^T(y - X\hat{w}) = 0 \implies X^T y = X^T X \hat{w}$$

These are exactly the _normal equations_. The fitted values $\hat{y} = X\hat{w}$ are the **orthogonal projection** of $y$ onto $\text{col}(X)$.

```
         y
         |  \
         |    \  residual r = y - ŷ
         |      \
col(X) ──────────ŷ = Xw_hat
```

The projection matrix (hat matrix) is:

$$H = X(X^T X)^{-1} X^T, \quad \hat{y} = Hy$$

Properties: $H^2 = H$ (idempotent), $H^T = H$ (symmetric), eigenvalues are 0 or 1.

### When Does $(X^T X)$ Fail to Be Invertible?

$(X^T X)$ is singular when the columns of $X$ are **linearly dependent** — i.e., **multicollinearity**.

Causes:

- Exact duplicates of a feature.
- One feature is a linear combination of others (e.g., temperature in Celsius and Fahrenheit both included).
- $d > n$: more features than samples — infinitely many solutions exist.

Fix: use Ridge regression (adds $\lambda I$ to make $(X^T X + \lambda I)$ always invertible) or feature selection.

### Computational Cost

Forming $X^T X$: $O(nd^2)$. Inverting it: $O(d^3)$. For large $d$ (e.g., $d > 10^4$), prefer gradient descent.

---

## Solution 2 — Gradient Descent

For large $n$ or $d$, the Normal Equation is impractical. We iteratively move in the direction of steepest descent.

### Gradient

$$\nabla_w \mathcal{L} = \frac{2}{n} X^T (Xw - y)$$

Intuition: $Xw - y$ is the vector of **residuals**. We're summing up the influence of each training example, weighted by how wrong we are.

### Update Rule

$$w \leftarrow w - \alpha \nabla_w \mathcal{L} = w - \frac{2\alpha}{n} X^T(Xw - y)$$

where $\alpha > 0$ is the **learning rate** (step size).

### Variants

|Variant|Gradient computed on|Update frequency|Notes|
|---|---|---|---|
|**Batch GD**|Full dataset|Once per epoch|Stable, exact gradient, slow for large $n$|
|**Stochastic GD (SGD)**|Single example $i$|Every example|Noisy, can escape local minima, fast|
|**Mini-batch GD**|Batch of size $B$|Every $B$ examples|Best of both; standard in practice|

The mini-batch gradient estimator is unbiased: $\mathbb{E}[\nabla_w \hat{\mathcal{L}}_B] = \nabla_w \mathcal{L}$.

### Convergence

For batch GD, convergence is guaranteed if $\alpha < \frac{1}{L}$ where $L$ is the Lipschitz constant of the gradient (largest eigenvalue of $\frac{2}{n}X^T X$).

**Learning rate schedules:**

- **Constant:** $\alpha_t = \alpha$. May oscillate near minimum.
- **Step decay:** $\alpha_t = \alpha_0 \cdot \gamma^{\lfloor t/k \rfloor}$.
- **Cosine annealing:** $\alpha_t = \alpha_{\min} + \frac{1}{2}(\alpha_{\max} - \alpha_{\min})(1 + \cos(\pi t/T))$.
- **1-cycle:** warm up, then decay — widely used in deep learning.

### Implementation (NumPy)

```python
import numpy as np

def linear_regression_gd(X, y, alpha=0.01, n_epochs=1000):
    n, d = X.shape
    # Add bias column
    X_aug = np.hstack([np.ones((n, 1)), X])
    w = np.zeros(X_aug.shape[1])
    
    losses = []
    for epoch in range(n_epochs):
        residuals = X_aug @ w - y           # shape (n,)
        grad = (2/n) * X_aug.T @ residuals  # shape (d+1,)
        w -= alpha * grad
        losses.append(np.mean(residuals**2))
    
    return w, losses

def normal_equation(X, y):
    X_aug = np.hstack([np.ones((len(X), 1)), X])
    return np.linalg.solve(X_aug.T @ X_aug, X_aug.T @ y)
```

## Probabilistic View — MLE Recovers OLS

### The Noise Model

Assume each observation is generated as:

$$y^{(i)} = w^T x^{(i)} + \epsilon^{(i)}, \quad \epsilon^{(i)} \sim \mathcal{N}(0, \sigma^2) \text{ i.i.d.}$$

Then $y^{(i)} | x^{(i)}; w \sim \mathcal{N}(w^T x^{(i)}, \sigma^2)$.

### Likelihood

$$\mathcal{L}(w) = \prod_{i=1}^n p(y^{(i)} | x^{(i)}; w) = \prod_{i=1}^n \frac{1}{\sqrt{2\pi\sigma^2}} \exp!\left(-\frac{(y^{(i)} - w^T x^{(i)})^2}{2\sigma^2}\right)$$

### Log-Likelihood

$$\log \mathcal{L}(w) = -\frac{n}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2} \sum_{i=1}^n (y^{(i)} - w^T x^{(i)})^2$$

**Maximising this is equivalent to minimising $|y - Xw|^2$.**

So: **OLS = MLE under the Gaussian noise assumption.** This is not a coincidence — it tells us exactly when OLS is appropriate (when noise is symmetric and light-tailed).

## Regularisation

Regularisation adds a penalty on $w$ to the loss, discouraging complex models. This is the standard solution to overfitting.

### Ridge Regression (L2)

$$\mathcal{L}_{\text{Ridge}}(w) = \frac{1}{n}|Xw - y|^2 + \lambda|w|^2$$

**Closed form:** $\hat{w} = (X^T X + \lambda I)^{-1} X^T y$

Note: $(X^T X + \lambda I)$ is **always** invertible for $\lambda > 0$. This is another reason to use Ridge.

**Effect:** shrinks all coefficients toward zero, but never exactly to zero.

**Probabilistic view:** Ridge = MAP estimation with a Gaussian prior $w \sim \mathcal{N}(0, \frac{\sigma^2}{\lambda} I)$.

$$\underbrace{-\log P(w|\mathcal{D})}_{\text{MAP objective}} \propto \underbrace{|Xw - y|^2}_{\text{NLL}} + \underbrace{\lambda|w|^2}_{\text{log-prior}}$$

### Lasso Regression (L1)

$$\mathcal{L}_{\text{Lasso}}(w) = \frac{1}{n}|Xw - y|^2 + \lambda|w|_1$$

No closed form (L1 is not differentiable at 0). Solved with coordinate descent or subgradient methods.

**Effect:** produces **sparse solutions** — many coefficients are exactly zero. Performs implicit feature selection.

**Why sparsity?** Geometrically, the L1 ball (a diamond in 2D) has corners on the axes. The loss contours will likely touch the constraint at a corner where some $w_j = 0$.

```
L2 ball (circle)          L1 ball (diamond)
      ___                      /\
    /     \                   /  \
   |       |                 /    \
   |       |    ←→          /      \
   |       |               \      /
    \     /                 \    /
      ---                    \  /
                              \/

Loss ellipse touches          Loss ellipse likely
at interior point             touches at a corner
(no sparsity)                 (sparsity: w_j = 0)
```

**Probabilistic view:** Lasso = MAP with Laplace prior $w_j \sim \text{Laplace}(0, 1/\lambda)$.

### Elastic Net

$$\mathcal{L}_{\text{EN}}(w) = \frac{1}{n}|Xw - y|^2 + \lambda_1|w|_1 + \lambda_2|w|^2$$

Combines L1 (sparsity) and L2 (handles correlated features gracefully). Preferred over pure Lasso when features are correlated.

### Summary Table

|Method|Penalty|Solution|Effect|Use When|
|---|---|---|---|---|
|OLS|None|$(X^TX)^{-1}X^Ty$|Unbiased, high variance if multicollinear|Clean, low-dim data|
|Ridge|$\lambda\|w\|^2$|$(X^TX+\lambda I)^{-1}X^Ty$|Shrinks toward 0, never zero|Correlated features|
|Lasso|$\lambda\|w\|_1$|No closed form|Sparse, feature selection|High-dim, sparse signal|
|Elastic Net|$\lambda_1\|w\|_1 + \lambda_2\|w\|^2$|No closed form|Sparse + stable|Correlated + sparse|

## Polynomial Regression as Feature Engineering

This deserves its own callout: **polynomial regression is still linear regression**.

Given $x \in \mathbb{R}$, define new features:

$$\phi(x) = \begin{bmatrix} 1 \ x \ x^2 \ x^3 \end{bmatrix}$$

Then fit $y = w^T \phi(x)$. The model is **linear in $w$**, not linear in $x$. Everything above — Normal Equation, gradient descent, regularisation — applies without modification.

More generally: **feature engineering = mapping $x \to \phi(x)$, then doing linear regression in $\phi$-space.** This is also the foundation of the kernel trick (see [[04.Support Vector Machines]]).

```
Raw data (nonlinear)     Feature space (linear)
         *                    *
    *         *          * *   * *
  *     |       *   →  *           *
    *         *          * *   * *
         *                    *

Cannot fit with line         Fits with hyperplane
```

## Assumptions of Linear Regression

Linear regression is optimal **when these hold:**

1. **Linearity:** $\mathbb{E}[y|x] = w^T x$. The true relationship is linear (or close to it).
2. **Independence:** Examples $(x^{(i)}, y^{(i)})$ are i.i.d.
3. **Homoscedasticity:** $\text{Var}(\epsilon^{(i)}) = \sigma^2$ constant — variance doesn't change with $x$.
4. **No multicollinearity:** Columns of $X$ are linearly independent.
5. **Normality of residuals:** For inference (confidence intervals, hypothesis tests), not for prediction.

> **Professor's note:** In practice, linear regression is surprisingly robust to violations of 3 and 5. What kills it is violation of 1 (genuine nonlinearity) and severe violation of 4 (near-singular $X^TX$). Always plot your residuals — they tell you more than any single metric.

## Diagnosing and Evaluating

### Metrics

- **MSE:** $\frac{1}{n}|y - \hat{y}|^2$ — in squared units, penalises large errors heavily.
- **RMSE:** $\sqrt{\text{MSE}}$ — in original units, interpretable.
- **MAE:** $\frac{1}{n}|y - \hat{y}|_1$ — robust to outliers; doesn't penalise large errors as heavily.
- **$R^2$:** $1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}} = 1 - \frac{|y - \hat{y}|^2}{|y - \bar{y}|^2}$. Fraction of variance explained. $R^2 = 1$ → perfect fit; $R^2 = 0$ → model no better than predicting the mean.

> $R^2$ can be negative on test data (your model is worse than the mean — a bad sign).

### Residual Plots

Always plot $\hat{y}$ vs. $\epsilon = y - \hat{y}$:

## Connection to AI, Machine Learning, and Deep Learning

### Linear Regression as the Simplest Neural Network

A linear regression model with a single output is a **neural network with no hidden layers and a linear activation**:

```
Input layer          Output
  x_1 ──── w_1 ───┐
  x_2 ──── w_2 ───┤──→ ŷ = w^Tx + b
   ⋮              ⋮
  x_d ──── w_d ───┘
```

Every concept here reappears in deep learning:

- **Gradient descent** is the core training algorithm for all neural networks.
- **Regularisation** (L1, L2, dropout, weight decay) fights overfitting in all of them.
- **The MSE loss** is the default loss for regression heads in deep networks.
- **The Normal Equation** is the closed-form solution for the final linear layer in least-squares regression (used in transfer learning when fine-tuning just the head).

### Feature Learning

In classical ML, we hand-engineer $\phi(x)$. Deep learning learns $\phi$ from data. The final layer of a neural network is still a linear model:

$$\hat{y} = W_{\text{out}} \cdot \underbrace{\phi_\theta(x)}_{\text{learned representation}} + b$$

Training the whole network end-to-end jointly optimises the feature extractor $\phi_\theta$ and the linear head $W_{\text{out}}$. This is why linear regression is not a toy model — it is the **building block of every output layer in deep learning**.

### Gradient Descent at Scale

Mini-batch SGD on the MSE loss for linear regression has the exact same update rule as a forward pass + backward pass in a single-layer neural network:

$$w \leftarrow w - \frac{2\alpha}{B} \sum_{i \in \mathcal{B}} (w^T x^{(i)} - y^{(i)}) x^{(i)}$$

This is backpropagation with one layer. Learning this first makes the chain rule for deep networks much less mysterious.

### Regularisation and Weight Decay

In deep learning, L2 regularisation on all weights is called **weight decay** and is standard (e.g., `weight_decay=1e-4` in PyTorch optimisers). It corresponds exactly to a Gaussian prior on every weight. The Bayesian interpretation of Ridge carries over unchanged.

### Ordinary Least Squares in Statistics vs. ML

|Perspective|OLS is...|Goal|
|---|---|---|
|Classical Statistics|An estimator with known distribution (Gauss-Markov)|Inference: confidence intervals, hypothesis tests|
|Machine Learning|A model with a training loss|Prediction: minimise test error|
|Deep Learning|The final linear layer|Representation learning + prediction|

## Example: Fitting a Curve to Data

```python
import numpy as np
import matplotlib.pyplot as plt

# Generate noisy quadratic data
rng = np.random.default_rng(42)
x = np.linspace(-3, 3, 100)
y = 0.5 * x**2 - x + 2 + rng.normal(0, 0.5, size=100)

# Feature matrices
def make_poly_features(x, degree):
    return np.column_stack([x**k for k in range(degree + 1)])

X1 = make_poly_features(x, 1)   # Linear features [1, x]
X2 = make_poly_features(x, 2)   # Quadratic features [1, x, x^2]
X9 = make_poly_features(x, 9)   # Degree-9 polynomial

# Normal equation
def fit(X, y):
    return np.linalg.solve(X.T @ X, X.T @ y)

w1 = fit(X1, y)
w2 = fit(X2, y)
w9 = fit(X9, y)

# degree=1 underfits, degree=2 is right, degree=9 overfits
```

## The Gauss-Markov Theorem (Why OLS is Special)

Among all **linear unbiased estimators** of $w$, OLS has the **smallest variance**. This is BLUE: Best Linear Unbiased Estimator.

Formal statement: if $\tilde{w}$ is any other linear estimator satisfying $\mathbb{E}[\tilde{w}] = w$, then for any vector $c$:

$$\text{Var}(c^T \hat{w}_{\text{OLS}}) \leq \text{Var}(c^T \tilde{w})$$

**Conditions:** linearity, independence, zero-mean errors, homoscedasticity (no normality required).

> **Note:** "best among linear unbiased" is a limited guarantee. Ridge is biased but can have lower MSE than OLS. The Gauss-Markov theorem says Ridge is worse _within the linear unbiased class_, but Ridge wins when we're willing to accept bias for reduced variance. This is the bias-variance tradeoff in action.
