
> _"Bagging asks: can many independent weak learners agree? Boosting asks: can each new learner correct what the last one got wrong?"_

---

## ✦ The Meta-Idea

Bagging reduces variance by **averaging independent learners**. The learners don't communicate — each one sees a different bootstrap sample and contributes an independent vote.

Boosting is fundamentally different. The learners are trained **sequentially**, and each new learner is explicitly shaped by the errors of all previous learners. The ensemble is built additively:

$$F_M(x) = \sum_{m=1}^M \alpha_m h_m(x)$$

where $h_m$ are **weak learners** (typically shallow trees) and $\alpha_m$ are their weights.

A **weak learner** is one that does only slightly better than random guessing — formally, its error rate $\epsilon < 1/2$ for binary classification. The celebrated result of boosting theory (Schapire, 1990) is that **any weak learner can be boosted into an arbitrarily accurate strong learner**, provided the data is not noisy beyond a certain threshold.

## ✦ Part 1 — AdaBoost

### Motivation: Reweighting the Hard Examples

After the first classifier $h_1$ makes predictions, some examples are correctly classified, others are not. The intuition: train $h_2$ to focus on what $h_1$ got wrong. Do this by **upweighting misclassified examples** and **downweighting correctly classified ones**.

The question AdaBoost answers rigorously: _how much_ to reweight, and _how much weight_ to give each classifier?

### The Algorithm

**Setting:** Binary classification, $y_i \in {-1, +1}$, $n$ training examples.

```
Initialise: w_i^(1) = 1/n for i = 1, ..., n

for m = 1, 2, ..., M:

    1. Train weak learner h_m on weighted dataset (X, y, w^(m))
       → h_m: X → {-1, +1}

    2. Compute weighted error:
       ε_m = Σᵢ wᵢ^(m) · 𝟙[h_m(xᵢ) ≠ yᵢ]

    3. Compute classifier weight:
       α_m = (1/2) ln((1 - ε_m) / ε_m)

    4. Update sample weights:
       w_i^(m+1) = w_i^(m) · exp(-α_m · y_i · h_m(x_i))
       Normalise: w_i^(m+1) ← w_i^(m+1) / Σⱼ wⱼ^(m+1)

Final classifier: F(x) = sign(Σ_m α_m h_m(x))
```

### Derivation of $\alpha_m$: Where Does the Formula Come From?

We want $\alpha_m$ such that the new ensemble $F_m = F_{m-1} + \alpha_m h_m$ minimises the **exponential loss**:

$$\mathcal{L}_{\exp} = \sum_{i=1}^n \exp(-y_i F_m(x_i))$$

Substituting $F_m = F_{m-1} + \alpha_m h_m$:

$$\mathcal{L}_{\exp} = \sum_i \exp(-y_i F_{m-1}(x_i)) \cdot \exp(-y_i \alpha_m h_m(x_i))$$

Let $\tilde{w}_i = \exp(-y_i F_{m-1}(x_i))$ (the "unnormalised weight" — it's the current sample difficulty). Split the sum over correctly and incorrectly classified examples:

- Correct: $y_i h_m(x_i) = +1$, so $\exp(-\alpha_m)$ factor
- Incorrect: $y_i h_m(x_i) = -1$, so $\exp(+\alpha_m)$ factor

$$\mathcal{L}_{\exp} = e^{-\alpha_m} \sum_{y_i = h_m(x_i)} \tilde{w}_i + e^{\alpha_m} \sum_{y_i \neq h_m(x_i)} \tilde{w}_i$$

Let $W_c = \sum_{y_i = h_m(x_i)} \tilde{w}_i$ (correct weight) and $W_e = \sum_{y_i \neq h_m(x_i)} \tilde{w}_i$ (error weight), so $W_c + W_e = W_{\text{total}}$.

$$\mathcal{L}_{\exp} = e^{-\alpha_m} W_c + e^{\alpha_m} W_e$$

Differentiate with respect to $\alpha_m$ and set to zero:

$$\frac{\partial \mathcal{L}_{\exp}}{\partial \alpha_m} = -e^{-\alpha_m} W_c + e^{\alpha_m} W_e = 0$$

$$e^{2\alpha_m} = \frac{W_c}{W_e} \implies \alpha_m = \frac{1}{2}\ln\frac{W_c}{W_e} = \frac{1}{2}\ln\frac{1 - \epsilon_m}{\epsilon_m}$$

where $\epsilon_m = W_e / (W_c + W_e)$ is the normalised weighted error. $\blacksquare$

**Interpretation of $\alpha_m$:**

|$\epsilon_m$|$\alpha_m$|Meaning|
|---|---|---|
|$0$|$+\infty$|Perfect — dominates the ensemble|
|$0.5$|$0$|Random — contributes nothing|
|$> 0.5$|$< 0$|Worse than random — negated|

The weight update is then derived by substituting $\alpha_m$ back and tracing how the unnormalised weights evolve:

$$\tilde{w}_i^{(m+1)} = \tilde{w}_i^{(m)} \cdot e^{-\alpha_m y_i h_m(x_i)}$$

which after normalisation gives the update rule in the algorithm above.

### Derivation: AdaBoost as Forward Stagewise Additive Modelling

The deeper result: AdaBoost is **Forward Stagewise Additive Modelling (FSAM)** with the exponential loss.

FSAM builds $F_M = \sum_m \alpha_m h_m$ by solving at each step:

$$(\alpha_m, h_m) = \arg\min_{\alpha, h} \sum_i \mathcal{L}(y_i, F_{m-1}(x_i) + \alpha h(x_i))$$

For the **exponential loss** $\mathcal{L}(y, F) = e^{-yF}$, solving this exactly gives the AdaBoost update. This is the Friedman-Hastie-Tibshirani (2000) characterisation — it reveals _why_ exponential loss: it's the unique loss for which FSAM reduces to the elegant reweighting scheme.

**Why exponential loss can be a problem:** It is extremely sensitive to outliers and noisy labels. An example with $y_i = +1$ that keeps being misclassified gets weight $\propto e^{F_{m-1}(x_i)}$, which grows without bound. Gradient Boosting replaces exponential loss with something more robust.

### AdaBoost in Code

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier

class AdaBoost:
    def __init__(self, n_estimators=50, learning_rate=1.0):
        self.M = n_estimators
        self.lr = learning_rate
        self.alphas = []
        self.clfs = []

    def fit(self, X, y):
        # y must be {-1, +1}
        n = len(y)
        w = np.ones(n) / n

        for m in range(self.M):
            # Train stump (depth-1 tree)
            clf = DecisionTreeClassifier(max_depth=1)
            clf.fit(X, y, sample_weight=w)
            h = clf.predict(X)

            # Weighted error
            err = w @ (h != y)
            err = np.clip(err, 1e-10, 1 - 1e-10)  # numerical safety

            # Classifier weight
            alpha = 0.5 * np.log((1 - err) / err) * self.lr

            # Update weights
            w = w * np.exp(-alpha * y * h)
            w /= w.sum()

            self.alphas.append(alpha)
            self.clfs.append(clf)

        return self

    def predict(self, X):
        F = sum(a * clf.predict(X) for a, clf in zip(self.alphas, self.clfs))
        return np.sign(F)

    def score(self, X, y):
        return (self.predict(X) == y).mean()


# Usage
from sklearn.datasets import make_classification
X, y = make_classification(n_samples=1000, n_features=20, random_state=42)
y = 2 * y - 1   # {0,1} → {-1, +1}

ada = AdaBoost(n_estimators=200)
ada.fit(X_train, y_train)
print(f"AdaBoost accuracy: {ada.score(X_test, y_test):.4f}")
```

## ✦ Part 2 — Gradient Boosting

### The Key Generalisation

AdaBoost is tied to exponential loss. What if we want to use:

- **Logistic loss** (more robust classification)
- **Huber loss** (robust regression)
- **Quantile loss** (predicting percentiles)
- **Any differentiable loss**

Friedman (2001) observed: the AdaBoost reweighting scheme can be interpreted as **gradient descent in function space**. This generalises to arbitrary differentiable losses.

### Gradient Descent in Function Space

Ordinary gradient descent optimises parameters $\theta$:

$$\theta \leftarrow \theta - \alpha \nabla_\theta \mathcal{L}(\theta)$$

Suppose instead we treat the prediction function $F$ itself as the parameter. At each training point $(x_i, y_i)$, we want to move $F(x_i)$ in the direction that most reduces the loss:

$$F(x_i) \leftarrow F(x_i) - \alpha \frac{\partial \mathcal{L}(y_i, F(x_i))}{\partial F(x_i)}$$

Define the **pseudo-residual** at step $m$:

$$r_i^{(m)} = -\frac{\partial \mathcal{L}(y_i, F(x_i))}{\partial F(x_i)}\Bigg|_{F = F_{m-1}}$$

This is the negative gradient of the loss with respect to the current prediction at $x_i$. If we could update $F$ directly at the training points, we'd just set $F_m(x_i) = F_{m-1}(x_i) + \nu r_i^{(m)}$.

But we need $F_m$ to **generalise** to new points. The trick: fit a weak learner $h_m$ to the pseudo-residuals, then add it to the ensemble. $h_m$ learns to approximate the negative gradient everywhere, not just at training points.

### The Gradient Boosting Algorithm

```
Input: training data {(xᵢ, yᵢ)}, loss L, M rounds, learning rate ν, weak learner ℒ
────────────────────────────────────────────────────────────────────────────────────
1. Initialise: F₀(x) = argmin_γ Σᵢ L(yᵢ, γ)
               (the constant prediction that minimises average loss)

for m = 1, 2, ..., M:

    2. Compute pseudo-residuals:
       rᵢ^(m) = -∂L(yᵢ, F(xᵢ))/∂F(xᵢ) |_{F = F_{m-1}}

    3. Fit a regression tree h_m to {(xᵢ, rᵢ^(m))}
       (always regression, even for classification — fitting the gradient values)

    4. Find optimal leaf weights by line search:
       For each leaf j of h_m:
           γⱼ^(m) = argmin_γ Σᵢ∈leaf_j L(yᵢ, F_{m-1}(xᵢ) + γ)

    5. Update:
       F_m(x) = F_{m-1}(x) + ν · Σⱼ γⱼ^(m) · 𝟙[x ∈ leaf_j]

Return F_M
```

**Step 1:** For MSE loss, $F_0 = \bar{y}$. For log-loss, $F_0 = \log(\bar{p}/(1-\bar{p}))$.

**Step 4:** The line search adjusts each leaf's contribution to best reduce the loss. For MSE, this is trivially the mean of residuals in the leaf. For other losses it requires a 1D optimisation per leaf.

**Step 5:** $\nu \in (0, 1]$ is the **learning rate** (also called shrinkage). Small $\nu$ requires more trees but typically generalises better.

### Pseudo-Residuals for Common Losses

**MSE Regression** ($\mathcal{L} = \frac{1}{2}(y - F)^2$):

$$r_i = -\frac{\partial}{\partial F}\frac{1}{2}(y_i - F)^2 = y_i - F_{m-1}(x_i)$$

The pseudo-residual is the **actual residual**. Gradient Boosting for MSE is exactly iterative residual fitting. Each tree fits the residuals left by the previous ensemble.

**Binary Classification** ($\mathcal{L} = \log(1 + e^{-y_i F(x_i)})$, $y_i \in {-1,+1}$):

$$r_i = \frac{2y_i}{1 + e^{y_i F_{m-1}(x_i)}}$$

**Binary Cross-Entropy** ($\mathcal{L} = -[y_i \log p_i + (1-y_i)\log(1-p_i)]$, $F = \log\text{-odds}$, $p_i = \sigma(F)$):

$$r_i = y_i - p_i^{(m-1)}$$

The pseudo-residual is again the **probability residual** — how far the current predicted probability is from the true label. Intuitive: fit the next tree to how wrong our probabilities are.

**MAE / $L_1$ Regression** ($\mathcal{L} = |y - F|$):

$$r_i = \text{sign}(y_i - F_{m-1}(x_i))$$

Each tree fits the **signs** of the residuals (not their magnitude). This makes gradient boosting with MAE robust to outliers.

### The Three Knobs

Gradient Boosting has three primary hyperparameters that govern the bias-variance tradeoff:

**1. $n_estimators$ (M):** Number of trees. More trees → lower bias. But unlike Random Forests, GB _can_ overfit as $M$ increases — the model keeps fitting the training data harder. Use early stopping.

**2. $learning_rate$ ($\nu$):** Shrinkage per step. Smaller $\nu$ → each tree contributes less → need more trees for the same fit → better generalisation (with sufficient $M$). The **Friedman shrinkage rule**: decrease $\nu$, increase $M$ proportionally. Common range: $\nu \in [0.01, 0.3]$.

**3. $max_depth$:** Depth of individual trees. Depth-1 trees (stumps) = additive model (no interactions). Depth-2 = captures pairwise interactions. Depth 3–5 is typical. Unlike bagging, deep trees in boosting lead to overfitting.

Interaction between $\nu$ and $M$: they are not independent. A common pattern:

```
ν = 0.1, M = 100   →   same performance as   ν = 0.01, M = 1000
```

The smaller $\nu$, the more trees you need. Tune $\nu$ first (or use a fixed small $\nu$), then find $M$ with early stopping.

### Gradient Boosting From Scratch (MSE)

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor

class GradientBoostingMSE:
    """Gradient Boosting for regression with MSE loss."""

    def __init__(self, n_estimators=100, learning_rate=0.1, max_depth=3):
        self.M = n_estimators
        self.nu = learning_rate
        self.max_depth = max_depth
        self.trees = []
        self.F0 = None

    def fit(self, X, y):
        # Step 1: Initialise with mean prediction
        self.F0 = y.mean()
        F = np.full(len(y), self.F0)

        for m in range(self.M):
            # Step 2: Pseudo-residuals (= actual residuals for MSE)
            r = y - F

            # Step 3: Fit regression tree to residuals
            tree = DecisionTreeRegressor(max_depth=self.max_depth)
            tree.fit(X, r)

            # Step 4: Line search (trivial for MSE: mean of residuals per leaf)
            # DecisionTreeRegressor already does this internally
            update = tree.predict(X)

            # Step 5: Update ensemble
            F += self.nu * update
            self.trees.append(tree)

        return self

    def predict(self, X):
        F = np.full(len(X), self.F0)
        for tree in self.trees:
            F += self.nu * tree.predict(X)
        return F

    def mse(self, X, y):
        return np.mean((y - self.predict(X))**2)


# Demonstration: residuals shrink monotonically
from sklearn.datasets import make_regression
X, y = make_regression(n_samples=500, n_features=10, noise=10, random_state=42)

gb = GradientBoostingMSE(n_estimators=200, learning_rate=0.1, max_depth=3)
gb.fit(X_train, y_train)
print(f"Train MSE: {gb.mse(X_train, y_train):.2f}")
print(f"Test  MSE: {gb.mse(X_test, y_test):.2f}")
```

### Early Stopping

Since GB can overfit, use staged predictions to find the optimal $M$:

## ✦ Part 3 — Stochastic Gradient Boosting

Friedman (1999) observed that subsampling the training data at each boosting step (without replacement) both **speeds up training** and **improves generalisation** by adding variance to individual trees, reducing inter-tree correlation — the same intuition as Random Forests.

At each step $m$, randomly sample a fraction $0 < s \leq 1$ of training examples (without replacement). Fit $h_m$ on this subsample only.

Effects:

- Reduces overfitting (analogous to learning rate shrinkage)
- Enables approximate computation on large datasets
- `subsample=0.5`–`0.8` is a common range; lower values = more regularisation

Combined with column subsampling (`max_features`), this is essentially a boosted random forest.

## ✦ Part 4 — XGBoost

XGBoost (Chen & Guestrin, 2016) re-derives gradient boosting from first principles, adding three critical improvements: a regularised objective, second-order Taylor expansion, and algorithmic optimisations for scale.

### The Regularised Objective

Standard gradient boosting minimises only the loss. XGBoost adds explicit regularisation on the tree structure:

$$\mathcal{O}^{(m)} = \sum_{i=1}^n \mathcal{L}(y_i, F_{m-1}(x_i) + h_m(x_i)) + \Omega(h_m)$$

where the regularisation term is:

$$\Omega(h) = \gamma T + \frac{1}{2}\lambda \sum_{j=1}^T w_j^2$$

- $T$: number of leaves (penalises complexity)
- $w_j$: weight (prediction) in leaf $j$
- $\gamma$: minimum loss reduction required to make a split (structural regularisation)
- $\lambda$: L2 regularisation on leaf weights

This directly controls overfitting at the tree structure level.

### Second-Order Taylor Expansion

Approximate the loss using a **second-order Taylor expansion** around $F_{m-1}(x_i)$:

$$\mathcal{L}(y_i, F_{m-1}(x_i) + h_m(x_i)) \approx \mathcal{L}(y_i, F_{m-1}(x_i)) + g_i h_m(x_i) + \frac{1}{2} h_i h_m(x_i)^2$$

where: $$g_i = \frac{\partial \mathcal{L}(y_i, \hat{y}_i)}{\partial \hat{y}_i}\Bigg|_{\hat{y}_i = F_{m-1}(x_i)}, \quad h_i = \frac{\partial^2 \mathcal{L}(y_i, \hat{y}_i)}{\partial \hat{y}_i^2}\Bigg|_{\hat{y}_i = F_{m-1}(x_i)}$$

$g_i$ is the **gradient** (same as the pseudo-residual, up to sign), $h_i$ is the **Hessian** — the curvature of the loss at the current prediction.

Dropping constants and grouping by leaves, the objective for a fixed tree structure becomes:

$$\mathcal{O}^{(m)} \approx \sum_{j=1}^T \left[ \left(\sum_{i \in \text{leaf}_j} g_i\right) w_j + \frac{1}{2}\left(\sum_{i \in \text{leaf}_j} h_i + \lambda\right) w_j^2 \right] + \gamma T$$

This is a quadratic in $w_j$ for each leaf. The optimal leaf weight is:

$$w_j^* = -\frac{G_j}{H_j + \lambda}, \quad G_j = \sum_{i \in \text{leaf}_j} g_i, \quad H_j = \sum_{i \in \text{leaf}_j} h_i$$

Substituting back gives the **minimum objective value for a given tree structure**:

$$\mathcal{O}^* = -\frac{1}{2}\sum_{j=1}^T \frac{G_j^2}{H_j + \lambda} + \gamma T$$

The **gain from a split** (used to greedily build the tree) is:

$$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L + G_R)^2}{H_L + H_R + \lambda}\right] - \gamma$$

This has an elegant interpretation: it measures how much the regularised objective improves by splitting a leaf into left and right children. If $\text{Gain} < 0$ for all possible splits, the leaf is not split (the $\gamma$ term enforces a minimum gain threshold).

**Why does the Hessian matter?** The Hessian $h_i$ tells us the local curvature of the loss. Points with high $h_i$ (near the decision boundary in classification, where the sigmoid is steep) receive more emphasis in the split search. This is a **Newton step**, not just a gradient step. Newton methods converge faster near the optimum.

For MSE: $g_i = \hat{y}_i - y_i$, $h_i = 1$. The Hessian is constant — Newton and gradient methods are identical. The benefit of second-order information is largest for asymmetric losses (logistic, Poisson, etc.).

### XGBoost in Practice

```python
import xgboost as xgb

model = xgb.XGBClassifier(
    n_estimators=1000,
    learning_rate=0.05,
    max_depth=6,
    subsample=0.8,            # row subsampling

```

## ✦ Part 5 — LightGBM and CatBoost

### LightGBM

LightGBM (Ke et al., 2017) addresses XGBoost's two main bottlenecks: finding optimal splits (requires sorting all features for all data points) and memory for large datasets.

**Key innovations:**

**1. Histogram-based splits.** Instead of sorting continuous feature values to find all possible split points, bin each feature into $B$ buckets (typically $B = 256$). Compute gradient/Hessian sums per bucket. Split search is $O(Bd)$ instead of $O(nd)$. Memory is $O(Bd)$ instead of $O(nd)$.

**2. Leaf-wise tree growth.** Standard gradient boosting grows trees **level-wise** (all leaves at the same depth). LightGBM grows **leaf-wise**: always split the leaf with the highest gain, regardless of depth. This grows asymmetric trees that can model complex patterns with fewer splits. Risk: can overfit on small datasets — control with `min_data_in_leaf` and `max_depth`.

**3. GOSS (Gradient-based One-Side Sampling).** Keep all instances with large gradients (high loss — these are the hard examples). Randomly subsample instances with small gradients. This maintains informativeness while reducing computation.

**4. EFB (Exclusive Feature Bundling).** Many features in high-dimensional data are mutually exclusive (rarely both nonzero). Bundle them into a single feature, reducing effective dimensionality.

```python
import lightgbm as lgb

params = {
    'objective': 'binary',
    'metric': 'binary_logloss',
    'learning_rate': 0.05,
    'num_leaves': 63,           # LightGBM's primary complexity control (≤ 2^max_depth - 1)
    'max_depth': -1,            # -1 = no limit; use num_leaves instead
    'min_data_in_leaf': 20,     # prevents overfitting in leaf-wise growth
    'feature_fraction': 0.8,
    'bagging_fraction': 0.8,
    'bagging_freq': 5,
    'lambda_l1': 0.1,
    'lambda_l2': 0.1,
    'verbose': -1,
    'n_jobs': -1,
    'seed': 42
}

train_data = lgb.Dataset(X_train, label=y_train)
val_data   = lgb.Dataset(X_val,   label=y_val, reference=train_data)

callbacks = [lgb.early_stopping(50), lgb.log_evaluation(100)]
model = lgb.train(
    params,
    train_data,
    num_boost_round=2000,
    valid_sets=[val_data],
    callbacks=callbacks
)
```

### CatBoost

CatBoost (Prokhorenkova et al., 2018) focuses on **categorical features** and **target leakage**.

**The target leakage problem in standard target encoding:** When computing the target mean for a categorical value, you include the current sample's label — introducing leakage that overfits to the training distribution.

CatBoost fixes this with **ordered target statistics**: for sample $i$, the target encoding for category $c$ uses only samples $j < i$ (earlier in a random permutation). This prevents leakage by construction.

**Ordered boosting:** Similarly, CatBoost builds each tree on a random permutation of the training data, using only preceding examples to compute gradients. This reduces prediction shift bias that arises in standard gradient boosting.

## ✦ Comparison: Bagging vs. Boosting

|Property|Bagging / Random Forest|Boosting|
|---|---|---|
|Trees trained|In parallel (independent)|Sequentially (dependent)|
|Primary effect|Reduces variance|Reduces bias (and variance)|
|Weak learner|Deep trees (low bias)|Shallow trees (stumps or depth 3–6)|
|Overfitting|Doesn't overfit as $B \to \infty$|Can overfit; use early stopping|
|Noisy labels|Robust|Sensitive (especially AdaBoost)|
|Training cost|Parallelisable|Sequential (less parallelisable)|
|Interpretability|Feature importances|Feature importances|
|Typical performance|Strong baseline|Usually better on tabular data|

The bias-variance lens:

- Random Forests target the **variance term** — many decorrelated trees average out noise
- Boosting targets the **bias term** — sequential correction drives residuals toward zero
- Both benefit from regularisation to control the tradeoff

## ✦ Why Does Boosting Generalise?

The early theory suggested boosting should overfit badly — it's explicitly fitting the training data harder at each step. Empirically, it often didn't. Schapire et al. (1998) explained this through **margin theory**:

Define the **margin** of example $(x_i, y_i)$ as:

$$m(x_i, y_i) = y_i \cdot \frac{\sum_m \alpha_m h_m(x_i)}{\sum_m \alpha_m}$$

The margin is positive if correctly classified, and measures confidence. AdaBoost has been shown to **increase margins even after training error reaches zero**, which explains the observed generalisation. The ensemble keeps pushing confidently classified examples away from the decision boundary.

The modern understanding through **statistical learning theory**: boosting with a small learning rate is closely related to $L_1$-regularised regression in a very large feature space (one feature per weak learner). The implicit regularisation from many small steps + early stopping prevents overfitting.

## ✦ Connection to AI / ML / Deep Learning

**Gradient Boosting ≡ Gradient Descent on Trees.** The pseudo-residual derivation reveals that boosting is literally gradient descent — just in function space rather than parameter space. Every time you run `model.fit()` on a neural network, you're doing gradient descent in parameter space. Gradient boosting does the same thing, but represents the function as a sum of trees rather than as a neural network. The conceptual unification is: **both are minimising a loss via iterative refinement; they differ only in their function representation class.**

**XGBoost and Newton's method in deep learning.** The second-order update in XGBoost is the same idea behind optimisers like Adagrad, Adam, and K-FAC in deep learning — using curvature information to take better steps. Adam maintains a running estimate of the second moment of gradients (a diagonal approximation to the Hessian). XGBoost uses the exact Hessian per sample for trees.

**Residual connections in neural networks.** ResNets (He et al., 2016) train each layer to learn the **residual** $F(x) - x$ rather than the full mapping $F(x)$. This is the same philosophy as gradient boosting: each new component corrects the error of the previous one. Huang et al. (2017) made this connection explicit in _Multi-Scale Dense Networks_ and "Snapshot Ensembles".

**Tabular data dominance.** On structured tabular data — the regime where most applied ML happens (fraud detection, recommendation systems, medical records, financial modelling) — gradient-boosted trees consistently outperform neural networks. The 2022 NeurIPS paper "Why do tree-based models still outperform deep learning on tabular data?" (Grinsztajn et al.) identified three reasons: robustness to uninformative features, preserving the orientation of the data (trees don't rotate the feature space), and ability to learn sharp step-function boundaries.

**LLM training dynamics as boosting.** The "loss spike → recovery" pattern observed in large language model training has been interpreted through a boosting lens: the model temporarily increases loss on some distribution of examples while correcting others, then a subsequent phase recovers — a noisy version of the sequential correction in boosting.

## ✦ From Scratch: Gradient Boosting with Log-Loss

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor

class GradientBoostingLogistic:
    """
    Gradient Boosting for binary classification with log-loss.
    F(x) is the log-odds. Prediction: p = sigmoid(F(x))
    Loss: L(y, F) = log(1 + exp(-y*F)), y ∈ {-1, +1}
    Gradient: r_i = y_i / (1 + exp(y_i * F(x_i)))
    """
    def __init__(self, n_estimators=100, learning_rate=0.1, max_depth=3):
        self.M = n_estimators
        self.nu = learning_rate
        self.max_depth = max_depth
        self.trees = []
        self.F0 = None

    @staticmethod
    def sigmoid(x):
        return 1 / (1 + np.exp(-x))

    def fit(self, X, y):
        # y must be {-1, +1}
        # Initialise F₀: optimal constant = log-odds of positive class
        p_bar = (y == 1).mean()
        p_bar = np.clip(p_bar, 1e-6, 1 - 1e-6)
        self.F0 = np.log(p_bar / (1 - p_bar))
        F = np.full(len(y), self.F0)

        for m in range(self.M):
            # Gradient of log-loss w.r.t. F: ∂L/∂F = -y/(1+exp(yF))
            # Pseudo-residual = negative gradient
            r = y / (1 + np.exp(y * F))

            # Fit regression tree to pseudo-residuals
            tree = DecisionTreeRegressor(max_depth=self.max_depth)
            tree.fit(X, r)

            # Line search per leaf (Newton step: sum(r) / sum(|r|(2-|r|)))
            # This is the optimal step for log-loss; standard GB uses mean of r
            leaves = tree.apply(X)
            update = np.zeros(len(y))
            for leaf_id in np.unique(leaves):
                mask = leaves == leaf_id
                r_leaf = r[mask]
                # Newton step for log-loss in this leaf
                num = r_leaf.sum()
                denom = np.abs(r_leaf) * (2 - np.abs(r_leaf))
                step = num / (denom.sum() + 1e-10)
                update[mask] = step

            F += self.nu * update
            self.trees.append((tree, leaves, update))   # store for prediction

        return self

    def predict_proba(self, X):
        F = np.full(len(X), self.F0)
        for (tree, _, _) in self.trees:
            F += self.nu * tree.predict(X)
        return self.sigmoid(F)

    def predict(self, X):
        return (self.predict_proba(X) >= 0.5).astype(int) * 2 - 1

    def score(self, X, y):
        return (self.predict(X) == y).mean()
```

## ✦ Summary

|Algorithm|Loss|Update Rule|Notes|
|---|---|---|---|
|**AdaBoost**|Exponential|Reweight samples|FSAM + exp loss; sensitive to noise|
|**Gradient Boosting**|Any $\mathcal{L}$|Fit tree to $-\nabla\mathcal{L}$|General, sklearn implementation|
|**Stochastic GB**|Any $\mathcal{L}$|As above + row subsampling|Regularises and speeds up|
|**XGBoost**|Any $\mathcal{L}$|Newton step, regularised tree|Best for most tabular tasks|
|**LightGBM**|Any $\mathcal{L}$|Histograms, leaf-wise growth|Faster, better on large data|
|**CatBoost**|Any $\mathcal{L}$|Ordered target stats|Best native categorical support|
