
> _"One tree is a guess. A thousand trees, each wrong in a different way, average out to something close to truth."_

## ✦ The Problem: Why Trees Alone Fail

From [[05.Decision Trees|Decision Trees]], recall the fundamental tension: a fully grown tree has **low bias** (it can fit any training set perfectly) but **high variance** (small perturbations in the training data produce radically different trees). The tree is an unstable learner — it has memorised the data, not learned from it.

More precisely: decision trees are **high-variance, low-bias** estimators. The bias-variance decomposition tells us that expected test error decomposes as:

$$\mathbb{E}[(y - \hat{f}(x))^2] = \underbrace{\text{Bias}^2[\hat{f}(x)]}_{\text{stays low}} + \underbrace{\text{Var}[\hat{f}(x)]}_{\text{we need to kill this}} + \sigma^2$$

The question is: can we reduce variance _without_ increasing bias? Yes. The answer is **aggregation**.

---

## ✦ Part 1 — Bagging

### The Core Idea

**Bagging** = **B**ootstrap **Agg**regat**ing**. Proposed by Leo Breiman (1996).

The variance of a random variable $Z$ with variance $\sigma^2$ satisfies:

$$\text{Var}!\left[\frac{1}{B}\sum_{b=1}^B Z_b\right] = \frac{\sigma^2}{B} \quad \text{if the } Z_b \text{ are i.i.d.}$$

If we could train $B$ independent models $\hat{f}_1, \ldots, \hat{f}_B$ on $B$ independent datasets and average their predictions, variance would shrink by a factor of $B$. Bias is unchanged because averaging unbiased estimators is unbiased.

We don't have $B$ independent datasets. We have one. But we can **simulate** independent datasets by **resampling with replacement**.

### Bootstrap Sampling

A **bootstrap sample** $\mathcal{D}^*_b$ is constructed by drawing $n$ samples **with replacement** from $\mathcal{D} = {(x_i, y_i)}_{i=1}^n$.

Key facts about bootstrap samples:

- Each $\mathcal{D}^*_b$ has $n$ points, same as $\mathcal{D}$
- Each original point appears in $\mathcal{D}^*_b$ with probability $1 - (1 - 1/n)^n \approx 1 - e^{-1} \approx 0.632$
- Roughly **36.8%** of original points are absent from each bootstrap sample — these are the **out-of-bag (OOB)** points

### The Bagging Algorithm

```
Input: training data 𝒟, base learner ℒ, number of bags B
───────────────────────────────────────────────────────────
for b = 1 to B:
    𝒟*_b ← bootstrap sample from 𝒟
    f̂_b ← ℒ(𝒟*_b)    ← train a fully grown tree (no pruning)

Aggregate:
    Regression:      f̂_bag(x) = (1/B) Σ_b f̂_b(x)
    Classification:  f̂_bag(x) = majority_vote{ f̂_b(x) }
                  or f̂_bag(x) = argmax_k (1/B) Σ_b P̂_b(y=k|x)  [soft voting]
```

### Variance Reduction: The Full Derivation

Suppose the $B$ base learners each have variance $\sigma^2$ and **pairwise correlation $\rho$** (they're not independent — they're all trained on draws from the same $\mathcal{D}$).

The variance of the average of $B$ identically distributed random variables with pairwise correlation $\rho$:

$$\text{Var}!\left[\frac{1}{B}\sum_{b=1}^B \hat{f}_b(x)\right] = \rho \sigma^2 + \frac{1-\rho}{B} \sigma^2$$

**Proof.** Let $Z_b = \hat{f}_b(x)$, $\text{Var}[Z_b] = \sigma^2$, $\text{Cov}[Z_b, Z_{b'}] = \rho\sigma^2$ for $b \neq b'$.

$$\text{Var}!\left[\frac{1}{B}\sum_b Z_b\right] = \frac{1}{B^2}\left(\sum_b \text{Var}[Z_b] + \sum_{b \neq b'}\text{Cov}[Z_b, Z_{b'}]\right)$$ $$= \frac{1}{B^2}\left(B\sigma^2 + B(B-1)\rho\sigma^2\right) = \frac{\sigma^2}{B} + \frac{(B-1)}{B}\rho\sigma^2$$ $$= \rho\sigma^2 + \frac{1-\rho}{B}\sigma^2$$

**Interpretation:**

$$\underbrace{\rho\sigma^2}_{\substack{\text{irreducible floor} \ \text{(correlation)}\vphantom{B}}} + \underbrace{\frac{1-\rho}{B}\sigma^2}_{\substack{\text{vanishes as} \ B \to \infty}}$$

As $B \to \infty$, the variance approaches $\rho\sigma^2$, **not zero**. The correlation between trees is the bottleneck. **The key to reducing ensemble variance is reducing $\rho$, not increasing $B$.**

This is the central insight that motivates Random Forests.

### What Bagging Does to Bias

Averaging is a linear operation. If each $\hat{f}_b$ is an unbiased estimator of $f$, then the average is also unbiased. If each $\hat{f}_b$ has bias $b$ (e.g., from underfitting), the average also has bias $b$.

**Bagging reduces variance. It does not reduce bias.**

This is why we use _fully grown_ trees as base learners — they have low bias. Pruned trees would have higher bias that bagging cannot fix.

### Out-of-Bag (OOB) Evaluation

For each training point $(x_i, y_i)$, roughly $B/e \approx 0.368B$ bootstrap samples **did not include** $(x_i, y_i)$. Call these the OOB samples for $i$. The **OOB prediction** for $x_i$ is:

$$\hat{f}_{\text{OOB}}(x_i) = \frac{1}{|{b : i \notin \mathcal{D}^__b}|} \sum_{b: i \notin \mathcal{D}^__b} \hat{f}_b(x_i)$$

The **OOB error** is the average loss of OOB predictions over the training set:

$$\text{OOB error} = \frac{1}{n} \sum_{i=1}^n \mathcal{L}(y_i, \hat{f}_{\text{OOB}}(x_i))$$

**This is a nearly unbiased estimate of the test error** — each prediction is made by models that never saw that point, exactly as in cross-validation, but **at no extra computational cost**. OOB evaluation comes for free.

```python
from sklearn.ensemble import BaggingClassifier
from sklearn.tree import DecisionTreeClassifier

bag = BaggingClassifier(
    estimator=DecisionTreeClassifier(),  # fully grown tree
    n_estimators=500,
    max_samples=1.0,           # bootstrap sample size = n
    bootstrap=True,
    oob_score=True,            # compute OOB score
    n_jobs=-1,
    random_state=42
)
bag.fit(X_train, y_train)

print(f"OOB Score:  {bag.oob_score_:.4f}")    # free validation
print(f"Test Score: {bag.score(X_test, y_test):.4f}")
# They should be close
```

## ✦ Part 2 — Random Forests

### Reducing Correlation Between Trees

Bagging reduces variance, but trees grown on bootstrap samples of the same data are still **highly correlated**: if there's one very strong predictor $x_j$, almost every tree will split on $x_j$ near the root, making all trees look similar → $\rho$ remains high → variance floor $\rho\sigma^2$ remains high.

**The Random Forest fix:** At each node, before finding the best split, **randomly sample $m$ features** and restrict the search to only those $m$ features.

This **injects randomness** into each split decision:

- Trees grown on different bootstrap samples will now make different choices at each node
- The correlation $\rho$ between any two trees decreases
- The variance floor $\rho\sigma^2$ decreases

The price: each individual tree is slightly worse (we might miss the best feature at some nodes). The gain: much lower inter-tree correlation. In practice the gain dominates.

### The Random Forest Algorithm

```
Input: training data 𝒟, number of trees B, feature subsample size m
───────────────────────────────────────────────────────────────────────
for b = 1 to B:
    𝒟*_b ← bootstrap sample from 𝒟
    T_b ← grow_tree(𝒟*_b):
        At each node:
            Sample m features uniformly without replacement from all d features
            Find best split among only those m features
            Split node → recurse until stopping criterion
        Return fully grown tree T_b

Predict:
    Regression:      f̂_RF(x) = (1/B) Σ_b T_b(x)
    Classification:  f̂_RF(x) = majority_vote{ T_b(x) }
```

### Choosing $m$: The Key Hyperparameter

Default recommendations (Breiman, 2001):

- Classification: $m = \lfloor\sqrt{d}\rfloor$
- Regression: $m = \lfloor d/3 \rfloor$

**Intuition for why these work:**

- $m = d$: standard bagging (all features at each split) — maximum individual tree quality, maximum correlation
- $m = 1$: random trees (one random feature per split) — minimum correlation, very poor individual trees
- $m = \sqrt{d}$: the empirical sweet spot — trees are decorrelated enough, quality is retained

As $m$ decreases: $\rho \downarrow$, $\sigma^2 \uparrow$ (individual trees get worse). The tradeoff is nonlinear and the defaults work well in practice, but $m$ should be tuned.

```python
from sklearn.ensemble import RandomForestClassifier
import numpy as np

rf = RandomForestClassifier(
    n_estimators=500,
    max_features='sqrt',       # m = sqrt(d), default for classification
    max_depth=None,            # fully grown trees
    min_samples_leaf=1,
    bootstrap=True,
    oob_score=True,
    n_jobs=-1,
    random_state=42
)
rf.fit(X_train, y_train)

print(f"OOB Score:  {rf.oob_score_:.4f}")
print(f"Test Score: {rf.score(X_test, y_test):.4f}")
```

### The Effect of $B$ (Number of Trees)

Unlike boosting, Random Forests **do not overfit as $B$ increases**. Adding more trees only:

1. Reduces the $\frac{1-\rho}{B}\sigma^2$ term → variance decreases
2. Makes the average converge to the true ensemble prediction

The error curve is monotonically decreasing and levels off. The practical question is: when does it level off? Usually around $B = 100$–$500$ for most datasets. Plot OOB error vs. $B$ to find this:

```python
import matplotlib.pyplot as plt

# Track OOB error as B increases
oob_errors = []
for n_est in range(1, 301, 5):
    rf_b = RandomForestClassifier(n_estimators=n_est, oob_score=True,
                                   max_features='sqrt', n_jobs=-1, random_state=42)
    rf_b.fit(X_train, y_train)
    oob_errors.append(1 - rf_b.oob_score_)

plt.plot(range(1, 301, 5), oob_errors)
plt.xlabel('Number of Trees'); plt.ylabel('OOB Error')
plt.title('OOB Error Converges as B Increases')
```

## ✦ Feature Importance

Random Forests provide two distinct notions of feature importance. They measure different things and can disagree.

### 1. Mean Decrease in Impurity (MDI)

For each feature $j$, sum the (weighted) impurity decrease over all splits on $j$, averaged over all trees:

$$\text{MDI}(j) = \frac{1}{B}\sum_{b=1}^B \sum_{\substack{t \in T_b \ \text{split on } j}} \frac{n_t}{n} \Delta I(t)$$

where $\Delta I(t) = I(t) - \frac{n_{t_L}}{n_t}I(t_L) - \frac{n_{t_R}}{n_t}I(t_R)$ is the impurity decrease at node $t$, weighted by the fraction of samples reaching $t$.

**Properties:**

- Computed for free during training
- Fast, no retraining needed
- **Known bias:** features with high cardinality (many unique values) or continuous features get artificially high importance, even if uninformative. This is because the tree has more opportunities to split on them.

### 2. Permutation Importance (MDA — Mean Decrease in Accuracy)

For each feature $j$:

1. Evaluate the trained forest on the validation set → baseline score $s$
2. **Randomly permute** the values of feature $j$ across all samples
3. Re-evaluate → permuted score $s_j^{\text{perm}}$
4. Importance of feature $j$: $\text{MDA}(j) = s - s_j^{\text{perm}}$

Permuting feature $j$ breaks its relationship with the target. If the score drops a lot, $j$ was important. If the score barely changes, $j$ was irrelevant or redundant.

**Properties:**

- Model-agnostic: works for any fitted model
- Unbiased with respect to feature cardinality
- Accounts for the feature's actual predictive contribution, not just its role in the training splits
- More reliable and preferred when cardinality varies across features
- **Caveat:** can give low importance to features that are correlated with other informative features — their information is "shared" and permuting one doesn't hurt much if the other carries the same signal

```python
from sklearn.inspection import permutation_importance

# MDI (fast, potentially biased)
mdi = rf.feature_importances_

# Permutation importance (slower, more reliable)
result = permutation_importance(
    rf, X_val, y_val,
    n_repeats=30,           # permute each feature 30 times
    random_state=42,
    n_jobs=-1
)
mda = result.importances_mean
mda_std = result.importances_std

# Compare
import pandas as pd
importance_df = pd.DataFrame({
    'feature': feature_names,
    'MDI': mdi,
    'MDA': mda,
    'MDA_std': mda_std
}).sort_values('MDA', ascending=False)
print(importance_df.head(10))
```

### When MDI and Permutation Importance Disagree

If a feature has high MDI but low permutation importance: it has many opportunities to split on (high cardinality / continuous) but its splits don't actually improve generalisation. **Trust permutation importance.**

If a feature has high permutation importance but low MDI: it's informative but competes with correlated features during training. Both measures can miss this.

## ✦ Extremely Randomised Trees (Extra-Trees)

Push the randomisation further: instead of finding the **optimal** threshold for each split among $m$ random features, draw the threshold **randomly** as well.

```
At each node in Extra-Trees:
    Sample m features
    For each of those m features:
        Draw a random threshold uniformly in [min_val, max_val]
    Pick the (feature, threshold) pair with best impurity reduction
```

**Effect:**

- Reduces variance further (more randomness → less correlation between trees)
- Increases bias slightly (random thresholds are suboptimal)
- **Much faster to train:** no sorting required to find optimal thresholds
- Often competitive with or better than standard Random Forests on large datasets

```python
from sklearn.ensemble import ExtraTreesClassifier

et = ExtraTreesClassifier(
    n_estimators=500,
    max_features='sqrt',
    n_jobs=-1,
    random_state=42
)
et.fit(X_train, y_train)
print(f"Extra-Trees Test Score: {et.score(X_test, y_test):.4f}")
```

## ✦ From Scratch: A Minimal Random Forest in NumPy

This implementation is intentionally minimal — the goal is to see the mechanics, not to reproduce sklearn. Assumes you have a `DecisionTreeClassifier` from [[05.Decision Trees|Decision Trees]].

```python
import numpy as np
from collections import Counter

# Assumes: DecisionTree class with .fit(X, y) and .predict(X) methods
# from your decision tree implementation

class RandomForest:
    def __init__(self, n_estimators=100, max_features='sqrt',
                 max_depth=None, min_samples_split=2, random_state=None):
        self.n_estimators = n_estimators
        self.max_features = max_features
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.rng = np.random.default_rng(random_state)
        self.trees = []
        self.feature_indices = []   # which m features each tree sees at root?
        # (In a full impl, m features are sampled per node, not globally)

    def _bootstrap(self, X, y):
        n = len(X)
        idx = self.rng.choice(n, size=n, replace=True)
        oob_mask = np.ones(n, dtype=bool)
        oob_mask[idx] = False
        return X[idx], y[idx], oob_mask

    def _get_m(self, d):
        if self.max_features == 'sqrt':
            return max(1, int(np.sqrt(d)))
        elif self.max_features == 'log2':
            return max(1, int(np.log2(d)))
        elif isinstance(self.max_features, float):
            return max(1, int(self.max_features * d))
        elif isinstance(self.max_features, int):
            return self.max_features
        return d  # 'all'

    def fit(self, X, y):
        n, d = X.shape
        m = self._get_m(d)
        self.oob_predictions = np.full((n, len(np.unique(y))), 0.0)
        self.oob_counts = np.zeros(n)

        for b in range(self.n_estimators):
            X_boot, y_boot, oob_mask = self._bootstrap(X, y)

            # Feature subsampling: in a real impl this happens per node.
            # Here we simulate by training on a feature subset (simplified).
            feat_idx = self.rng.choice(d, size=m, replace=False)
            self.feature_indices.append(feat_idx)

            # Train a decision tree on the bootstrap sample
            tree = DecisionTree(
                max_depth=self.max_depth,
                min_samples_split=self.min_samples_split
            )
            tree.fit(X_boot[:, feat_idx], y_boot)
            self.trees.append(tree)

            # Accumulate OOB predictions
            if oob_mask.any():
                oob_proba = tree.predict_proba(X[oob_mask][:, feat_idx])
                self.oob_predictions[oob_mask] += oob_proba
                self.oob_counts[oob_mask] += 1

        # OOB score
        valid = self.oob_counts > 0
        oob_pred = self.oob_predictions[valid] / self.oob_counts[valid, None]
        oob_labels = oob_pred.argmax(axis=1)
        self.oob_score_ = (oob_labels == y[valid]).mean()
        return self

    def predict_proba(self, X):
        probas = np.zeros((len(X), len(self.trees[0].classes_)))
        for tree, feat_idx in zip(self.trees, self.feature_indices):
            probas += tree.predict_proba(X[:, feat_idx])
        return probas / self.n_estimators

    def predict(self, X):
        return self.predict_proba(X).argmax(axis=1)

    def score(self, X, y):
        return (self.predict(X) == y).mean()
```

> **Note:** A production implementation samples $m$ features _per node_, not globally per tree. The above simplification captures the spirit but not the full mechanics. In sklearn's C implementation, feature sampling happens inside `_splitter.pyx` at each node.

## ✦ Worked Example: Bias-Variance on a Toy Problem

Let's make the variance reduction concrete. Consider a 1D regression problem $y = \sin(2\pi x) + \epsilon$, $\epsilon \sim \mathcal{N}(0, 0.1)$, with $n = 50$ training points.

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import BaggingRegressor, RandomForestRegressor

np.random.seed(42)
x_test = np.linspace(0, 1, 300).reshape(-1, 1)
true_f = np.sin(2 * np.pi * x_test.ravel())

n_simulations = 200
tree_preds = np.zeros((n_simulations, 300))
bag_preds  = np.zeros((n_simulations, 300))

for sim in range(n_simulations):
    # New training set each time
    X_train = np.random.rand(50, 1)
    y_train = np.sin(2 * np.pi * X_train.ravel()) + np.random.randn(50) * 0.1

    # Single tree
    tree = DecisionTreeRegressor()
    tree.fit(X_train, y_train)
    tree_preds[sim] = tree.predict(x_test)

    # Bagged forest
    bag = BaggingRegressor(DecisionTreeRegressor(), n_estimators=100, random_state=sim)
    bag.fit(X_train, y_train)
    bag_preds[sim] = bag.predict(x_test)

# Compute bias^2 and variance at each test point
tree_bias2 = (tree_preds.mean(0) - true_f)**2
tree_var   = tree_preds.var(0)
bag_bias2  = (bag_preds.mean(0) - true_f)**2
bag_var    = bag_preds.var(0)

print(f"Single Tree — Bias²: {tree_bias2.mean():.4f}, Variance: {tree_var.mean():.4f}")
print(f"Bagged Forest — Bias²: {bag_bias2.mean():.4f}, Variance: {bag_var.mean():.4f}")

# Expected output:
# Single Tree  — Bias²: ~0.012, Variance: ~0.080
# Bagged Forest — Bias²: ~0.012, Variance: ~0.008
# Variance drops ~10x. Bias is unchanged.
```

## ✦ Connection to AI / ML / Deep Learning

**Random Forests as a foundation for modern ensemble methods.** The feature-subsampling idea reappears in XGBoost (`colsample_bytree`, `colsample_bylevel`) and LightGBM (`feature_fraction`). These are not coincidental — Gradient Boosting with column subsampling is essentially applying the Random Forest decorrelation idea to the boosting framework.

**The Bias-Variance Tradeoff, operationalised.** Random Forests are perhaps the cleanest empirical demonstration that the bias-variance decomposition is not just theory — the formula $\rho\sigma^2 + \frac{1-\rho}{B}\sigma^2$ is _exactly_ what you observe when you vary $m$ and $B$. Deep learning has its own version: dropout (zero out random neurons during training) is analogous to Random Forest's feature subsampling, and ensemble training (training multiple neural networks and averaging) obeys the same $1/B$ variance reduction.

**Feature importance and model interpretability.** MDI and permutation importance from Random Forests are the precursors to SHAP values (see [[03.Model Interpretability|Model Interpretability]]). SHAP's TreeExplainer is optimised specifically for tree ensembles and computes exact Shapley values in $O(TLD^2)$ where $T$ is trees, $L$ leaves, $D$ depth — much faster than model-agnostic SHAP.

**Random Forests at scale.** In production, Random Forests are often the baseline against which deep models are compared on tabular data. A 2022 study ("Why do tree-based models still outperform deep learning on tabular data?" — Grinsztajn et al.) showed that Random Forests and gradient-boosted trees consistently beat MLP-style deep models on tabular benchmarks, especially for datasets $n < 10{,}000$. The inductive bias of axis-aligned splits is well-matched to tabular data with heterogeneous features.

**Uncertainty quantification.** The spread of predictions across trees in a Random Forest is a natural measure of uncertainty — akin to a Bayesian posterior. This is formalised in _Mondrian Forests_, which use a specific random splitting mechanism to provide valid confidence intervals. In deep learning, MC Dropout (Gal & Ghahramani, 2016) achieves a similar effect: dropout at test time produces an ensemble of subnetworks, analogous to the Random Forest ensemble.

## ✦ Common Pitfalls

**Pitfall 1: Scaling features.** Decision trees are scale-invariant — you don't need to standardise features. (Unlike SVMs or KNN which are distance-based.)

**Pitfall 2: Trusting MDI on mixed-cardinality data.** If you have a continuous feature and a binary feature, the continuous feature will almost always have higher MDI simply because there are more possible split points. Use permutation importance to verify.

**Pitfall 3: Using OOB score as your test score.** OOB gives an unbiased estimate, but it's still an estimate on training data. If you've already used OOB to tune hyperparameters, report the held-out test score, not OOB.

**Pitfall 4: Stopping $B$ too early.** Plotting OOB error vs. $B$ is cheap. Don't guess — verify convergence.

**Pitfall 5: Ignoring class imbalance.** Default Random Forests treat all samples equally. Use `class_weight='balanced'` or `class_weight={0: w_0, 1: w_1}` for imbalanced problems.

## ✦ Summary

|Property|Single Decision Tree|Bagged Trees|Random Forest|
|---|---|---|---|
|Bias|Low|Low|Slightly higher|
|Variance|High|Moderate|Low|
|Interpretability|High|None|None (but feature importances)|
|Training cost|$O(nd \log n)$|$B \times O(nd\log n)$|$B \times O(nm\log n)$|
|Parallel?|No|Yes|Yes|
|Overfits as $B \to \infty$?|—|No|No|
|OOB evaluation|No|Yes|Yes|

The variance floor of a bagged ensemble is $\rho\sigma^2$. The contribution of Random Forests is to drive $\rho$ toward zero by randomising the feature selection at each split, without substantially increasing the bias $\sigma^2$ of individual trees.
