> _"The agent does not know the rules of the game. It only knows what it did, and what happened next."_

## ✦ Why RL Is Different

In supervised learning, someone hands you $(x_i, y_i)$ pairs. A teacher has already labelled the correct answer. In unsupervised learning, you have $x_i$ alone and look for structure.

Reinforcement learning has neither. There is no teacher. There is only an **agent** living inside an **environment**, taking **actions**, and receiving **rewards**. The agent must figure out, entirely from this signal, what to do.

This changes everything:

|-|Supervised|Unsupervised|Reinforcement|
|---|---|---|---|
|Feedback|Correct label|None|Scalar reward (delayed, noisy)|
|i.i.d. data?|Yes|Yes|No — actions affect future observations|
|Exploration needed?|No|No|Yes — must try things to learn|
|Credit assignment|Immediate|N/A|Delayed — which action caused the reward?|

The **credit assignment problem** is the central difficulty: the reward you receive now may be the consequence of an action you took ten steps ago. How do you know which action to credit?

## ✦ 1. The Markov Decision Process (MDP)

Every RL problem is formalised as a **Markov Decision Process**. An MDP is a tuple:

$$\mathcal{M} = \langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle$$

|Symbol|Name|Meaning|
|---|---|---|
|$\mathcal{S}$|State space|All possible situations the agent can be in|
|$\mathcal{A}$|Action space|All possible actions the agent can take|
|$P(s' \| s, a)$|Transition kernel|Probability of landing in $s'$ after taking $a$ in $s$|
|$R(s, a, s')$|Reward function|Scalar feedback received after transition|
|$\gamma \in [0,1)$|Discount factor|How much the agent cares about future rewards|

**The Markov property** (the thing that makes MDPs tractable):

$$P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} | s_t, a_t)$$

The future depends on the present state and action only — not on the full history. This is the fundamental assumption. It is often violated in practice (partially observable environments), and we will note when this matters.

### 1.1 The Agent-Environment Loop

```
         ┌─────────────────────────────────────────┐
         │                                         │
         ▼                                         │
    ┌─────────┐   a_t    ┌─────────────┐           │
    │  Agent  │ ──────►  │ Environment │           │
    │         │          │             │           │
    │  π(a|s) │ ◄──────  │  P(s'|s,a)  │           │
    └─────────┘  s_{t+1} └─────────────┘           │
         │         r_{t+1}       │                 │
         │                       │                 │
         └───────────────────────┘─────────────────┘
                    one time step
```

At each time step $t$:

1. Agent observes state $s_t \in \mathcal{S}$
2. Agent samples action $a_t \sim \pi(\cdot | s_t)$
3. Environment transitions to $s_{t+1} \sim P(\cdot | s_t, a_t)$
4. Agent receives reward $r_{t+1} = R(s_t, a_t, s_{t+1})$
5. Repeat

The sequence $(s_0, a_0, r_1, s_1, a_1, r_2, s_2, \ldots)$ is called a **trajectory** or **episode**.

### 1.2 Concrete Examples

|Domain|$\mathcal{S}$|$\mathcal{A}$|$R$|
|---|---|---|---|
|Chess|Board position|Legal moves|$+1$ win, $-1$ loss, $0$ otherwise|
|CartPole|$(x, \dot{x}, \theta, \dot{\theta})$|Left/Right|$+1$ per timestep upright|
|Atari Breakout|Raw pixels (screen)|Joystick directions|Score increment|
|Robot arm|Joint angles, velocities|Torques|Distance to target (negated)|
|Recommender system|User history|Item to show|Click, purchase, dwell time|

Notice: in chess, reward is received only at the end of the game. In CartPole, reward is received at every step. The discount factor $\gamma$ is the mechanism for handling both cases.

## ✦ 2. The Return

The agent's goal is not to maximise the immediate reward. It is to maximise the **expected sum of future rewards**. Define the **return** (or **discounted cumulative reward**) from time $t$:

$$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

**Why discount?**

1. **Mathematical:** ensures $G_t$ is finite for bounded rewards. If $|r| \leq R_{\max}$, then $G_t \leq \frac{R_{\max}}{1 - \gamma}$.
2. **Economic:** future rewards are uncertain; a reward now is worth more than a reward later.
3. **Practical:** $\gamma < 1$ allows the algorithm to converge. At $\gamma = 1$, many algorithms diverge.

**The recursive structure of the return** — this is what makes dynamic programming possible: $$G_t = r_{t+1} + \gamma G_{t+1}$$

The return from $t$ equals the immediate reward plus the discounted return from $t+1$. This single equation is the seed from which Bellman equations grow.

### 2.1 Finite vs. Infinite Horizon

|Horizon|Setting|Return|
|---|---|---|
|**Finite** ($T < \infty$)|Episodes end (chess, Atari)|$G_t = \sum_{k=0}^{T-t} r_{t+k+1}$, $\gamma = 1$ possible|
|**Infinite** ($T = \infty$)|Continuing tasks (robot control)|$G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$, requires $\gamma < 1$|

For episodic tasks, it is common to define a **terminal state** $s_{\text{terminal}}$ that absorbs all transitions with zero reward — this makes finite and infinite horizon mathematically equivalent.

## ✦ 3. The Policy

The **policy** $\pi$ is the agent's behaviour — the mapping from states to actions.

**Stochastic policy:** $\pi(a | s) = P(A_t = a | S_t = s)$ — a probability distribution over actions given state.

**Deterministic policy:** $\pi(s) = a$ — a single action for each state.

Stochastic policies are more general and necessary for some environments (e.g., rock-paper-scissors — a deterministic policy is exploitable). Deterministic policies are a special case.

**The goal of RL:** find the optimal policy: $$\pi^* = \arg\max_\pi \mathbb{E}_\pi[G_0]$$

Everything in RL — value functions, Bellman equations, Q-learning, policy gradients — is a different approach to this optimisation.

## ✦ 4. Value Functions

Value functions answer the question: _"How good is it to be in state $s$ (or to take action $a$ in state $s$) when following policy $\pi$?"_

### 4.1 The State-Value Function $V^\pi$

$$V^\pi(s) = \mathbb{E}_\pi\left[G_t ,\Big|, S_t = s\right] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k+1} ,\Big|, S_t = s\right]$$

This is the **expected return** starting from state $s$, following policy $\pi$ thereafter.

**Intuition:** $V^\pi(s)$ is the "long-term value" of being in state $s$ under policy $\pi$. If $V^\pi(s) = 100$, state $s$ is great under $\pi$. If $V^\pi(s) = -50$, state $s$ is terrible.

### 4.2 The Action-Value Function $Q^\pi$

$$Q^\pi(s, a) = \mathbb{E}_\pi\left[G_t ,\Big|, S_t = s, A_t = a\right]$$

This is the expected return starting from state $s$, _first taking action $a$_, then following policy $\pi$ thereafter.

**The relationship between $V^\pi$ and $Q^\pi$:** $$V^\pi(s) = \sum_a \pi(a|s) \cdot Q^\pi(s, a)$$

$V^\pi(s)$ is the $\pi$-weighted average of $Q^\pi(s, a)$ over all actions — exactly what you'd expect. The value of a state is the expected value of the actions you'd take there.

**Why $Q^\pi$ is more useful for learning.** If you know $Q^\pi(s, a)$ for all $(s, a)$, you can improve the policy greedily: $$\pi'(s) = \arg\max_a Q^\pi(s, a)$$

without needing to know $P(s'|s,a)$ — the transition dynamics. This is the key to **model-free RL**.

### 4.3 The Advantage Function

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

The **advantage** measures how much better action $a$ is compared to the average action under $\pi$ in state $s$. If $A^\pi(s, a) > 0$, action $a$ is better than average. If $A^\pi(s, a) < 0$, it's worse.

The advantage function appears naturally in policy gradient methods (Advantage Actor-Critic, PPO) because it reduces variance: instead of asking "is the total return good?", we ask "is this action better than expected?"

## ✦ 5. Bellman Equations — The Engine of RL

The Bellman equations express a consistency condition: the value of a state must equal the immediate reward plus the discounted value of the next state. They are derived directly from the recursive structure of the return.

### 5.1 Bellman Expectation Equations

**For $V^\pi$:** Start from the definition and expand using $G_t = r_{t+1} + \gamma G_{t+1}$:

$$V^\pi(s) = \mathbb{E}_\pi[G_t | S_t = s]$$ $$= \mathbb{E}_\pi[r_{t+1} + \gamma G_{t+1} | S_t = s]$$ $$= \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma \mathbb{E}_\pi[G_{t+1}|S_{t+1}=s']\right]$$

$$\boxed{V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma V^\pi(s')\right]}$$

**For $Q^\pi$:** Same derivation, conditioning on $(s, a)$:

$$\boxed{Q^\pi(s,a) = \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma \sum_{a'} \pi(a'|s') Q^\pi(s',a')\right]}$$

These are **linear equations** in $V^\pi$ (or $Q^\pi$). Given $\pi$ and the model $P, R$, they can be solved exactly by linear algebra (for small state spaces) or by iterative methods.

### 5.2 The Backup Diagram

```
Bellman equation for V^π:

         s
        / \
       a₁  a₂        ← weighted by π(a|s)
      / \  / \
    s₁' s₂' s₃'      ← weighted by P(s'|s,a)
    
V^π(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a) [R + γ V^π(s')]

The value of s is computed by "backing up" from its successor states.
```

```
Bellman equation for Q^π:

        (s,a)
        /   \
      s₁'   s₂'       ← weighted by P(s'|s,a)
      / \   / \
    a₁' a₂' ...       ← weighted by π(a'|s')

Q^π(s,a) = Σ_{s'} P(s'|s,a) [R + γ Σ_{a'} π(a'|s') Q^π(s',a')]
```

### 5.3 Bellman Optimality Equations

The **optimal value functions** $V^_$ and $Q^_$ satisfy the Bellman optimality equations — the same structure, but with $\max$ in place of $\sum_a \pi(a|s)$:

$$\boxed{V^_(s) = \max_a \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma V^_(s')\right]}$$

$$\boxed{Q^_(s,a) = \sum_{s'} P(s'|s,a)\left[R(s,a,s') + \gamma \max_{a'} Q^_(s',a')\right]}$$

**Why the max?** The optimal policy always takes the best action. So instead of averaging over $\pi(a|s)$, we take the maximum.

**The optimal policy from $Q^*$:** $$\pi^_(s) = \arg\max_a Q^_(s,a)$$

If you have $Q^*$, the optimal policy is free — just act greedily. This is the core of Q-learning.

### 5.4 Bellman Equations Are Not Learning Algorithms

The Bellman equations are **fixed-point conditions** — they characterise what $V^\pi$ or $Q^*$ must look like. They are not algorithms by themselves. The algorithms (Value Iteration, Q-learning, TD learning) use these equations as update rules:

|Algorithm|Update rule|Comes from|
|---|---|---|
|Value Iteration|$V(s) \leftarrow \max_a \sum_{s'} P(s'\|s,a)[R + \gamma V(s')]$|Bellman optimality for $V^*$|
|Policy Iteration|Evaluate $V^\pi$ exactly, then $\pi \leftarrow \text{greedy}(V^\pi)$|Bellman expectation for $V^\pi$|
|Q-learning|$Q(s,a) \leftarrow Q(s,a) + \alpha[r + \gamma \max_{a'} Q(s',a') - Q(s,a)]$|Bellman optimality for $Q^*$|
|TD(0)|$V(s) \leftarrow V(s) + \alpha[r + \gamma V(s') - V(s)]$|Bellman expectation for $V^\pi$|

## ✦ 6. The Optimal Policy Exists — And We Can Find It

**Theorem (Existence of $\pi^*$).** For any finite MDP, there exists at least one deterministic optimal policy $\pi^_$ such that $V^{\pi^_}(s) \geq V^\pi(s)$ for all $s$ and all $\pi$.

_Proof sketch._ Define $V^_(s) = \max_\pi V^\pi(s)$ pointwise. One can show that the Bellman optimality equation has a unique fixed point $V^_$ (by contraction mapping — the Bellman operator is a $\gamma$-contraction in the $\ell^\infty$ norm). The greedy policy with respect to $V^*$ is then optimal.

**Theorem (Contraction of the Bellman operator).** Define the Bellman optimality operator $\mathcal{T}$: $$(\mathcal{T}V)(s) = \max_a \sum_{s'} P(s'|s,a)[R(s,a,s') + \gamma V(s')]$$

Then $|\mathcal{T}V - \mathcal{T}U|_\infty \leq \gamma |V - U|_\infty$ for all $V, U$.

By Banach's fixed-point theorem, iterating $\mathcal{T}$ converges to the unique fixed point $V^*$. This is **Value Iteration** — and the contraction guarantees convergence at rate $\gamma^k$ after $k$ iterations.

## ✦ 7. Key Dichotomies in RL

These distinctions determine which algorithm is appropriate for a given problem. Each is a design choice, not a fact about RL in general.

### 7.1 Model-Based vs. Model-Free

||Model-Based|Model-Free|
|---|---|---|
|Uses $P(s'\|s,a)$?|Yes (known or learned)|No|
|Planning|Yes — can simulate trajectories|No|
|Sample efficiency|High|Low|
|Computation|High (planning)|Low|
|Examples|Value Iteration, Dyna-Q, AlphaZero|Q-learning, REINFORCE, PPO|

**Model-free is the dominant paradigm** in deep RL because the transition model is often too complex to learn accurately (e.g., Atari pixels → next pixels). Model-based RL is preferred when the model is available (board games, physics simulators) or the sample budget is tiny (robotics).

### 7.2 On-Policy vs. Off-Policy

||On-Policy|Off-Policy|
|---|---|---|
|What data is used?|Data from the current policy $\pi$|Data from any policy (including old ones)|
|Sample efficiency|Lower (must discard old data)|Higher (can reuse data)|
|Convergence|Easier to guarantee|Harder (deadly triad)|
|Examples|SARSA, REINFORCE, PPO|Q-learning, DQN, SAC|

**Q-learning is off-policy** because the update uses $\max_{a'} Q(s', a')$ — the greedy action — regardless of what policy actually generated the data. This allows learning from a replay buffer of old experience.

### 7.3 Value-Based vs. Policy-Based vs. Actor-Critic

||Value-Based|Policy-Based|Actor-Critic|
|---|---|---|---|
|What is learned?|$Q(s,a)$ or $V(s)$|$\pi_\theta(a\|s)$ directly|Both $\pi_\theta$ and $V_\phi$|
|Policy|Implicit: $\pi = \arg\max_a Q$|Explicit|Explicit|
|Continuous actions?|Hard (requires $\arg\max$)|Natural|Natural|
|Variance|Low (bootstrapping)|High (Monte Carlo returns)|Medium|
|Examples|Q-learning, DQN|REINFORCE|A2C, PPO, SAC|

### 7.4 Tabular vs. Function Approximation

||Tabular|Function Approximation|
|---|---|---|
|Represents|$Q(s,a)$ as a table|$Q(s,a;\theta)$ as a neural network|
|State space|Small, discrete|Large, continuous, high-dimensional|
|Convergence guarantees|Strong|Weaker (deadly triad)|
|Expressiveness|Limited|High|

The **deadly triad** (Sutton & Barto): combining function approximation + bootstrapping + off-policy learning can cause divergence. DQN carefully avoids this with experience replay and target networks — see [[02.Model-Free Methods]].

## ✦ 8. Exploration vs. Exploitation — The Fundamental Tension

The agent must **explore** (try new actions to gain information) and **exploit** (take the best known action to maximise reward). These are in direct conflict.

**The multi-armed bandit** is the pure form of this problem (no states, no transitions) — see [[03.Exploration vs Exploitation]].

In full MDPs, exploration is harder because:

- The consequences of an action are delayed and entangled with future actions.
- Some states are only reachable through specific action sequences.

**Basic strategies:**

|Strategy|Idea|When to use|
|---|---|---|
|$\epsilon$-greedy|With prob $\epsilon$, take random action|Always a reasonable default|
|Boltzmann / Softmax|$\pi(a\|s) \propto \exp(Q(s,a)/\tau)$|When you want graded exploration|
|UCB (Upper Confidence Bound)|Favour actions with high uncertainty|Bandits, tabular RL|
|Thompson Sampling|Sample from posterior over $Q$ values|Bayesian RL|
|Intrinsic motivation|Reward for visiting novel states|Sparse reward environments|

## ✦ 9. Worked Example — GridWorld MDP

> **Setup.** A $4 \times 4$ grid. Agent starts at top-left. Goal state at bottom-right (reward $+1$). Cliff state at bottom-left (reward $-1$, terminal). All other transitions: reward $0$. Actions: ${$N, S, E, W$}$. $\gamma = 0.9$.

```
┌─────┬─────┬─────┬─────┐
│  S  │     │     │     │
├─────┼─────┼─────┼─────┤
│     │     │     │     │
├─────┼─────┼─────┼─────┤
│     │     │     │     │
├─────┼─────┼─────┼─────┤
│  C  │     │     │  G  │
└─────┴─────┴─────┴─────┘
S = Start, G = Goal (+1), C = Cliff (-1)
```

**MDP components:**

- $\mathcal{S}$: 16 grid positions (plus 2 terminal states)
- $\mathcal{A}$: ${$N, S, E, W$}$
- $P(s'|s,a)$: deterministic (actions succeed with prob 1, hit wall → stay)
- $R$: $+1$ at $G$, $-1$ at $C$, $0$ elsewhere
- $\gamma = 0.9$

**Value Iteration (first two sweeps, for the state adjacent to G):**

Let $s_G^-$ denote the state immediately left of G.

Sweep 1: $V(s_G^-) = \max_a \sum_{s'} P[R + \gamma V(s')] = \max(\ldots, 0 + 0.9 \cdot 0, \ldots, 1 + 0.9 \cdot 0) = 1$

_(Taking action East reaches G, reward $+1$, $V(G) = 0$ since terminal.)_

Sweep 2: The state two steps left of G updates: $V = 0 + 0.9 \cdot 1 = 0.9$

After convergence, $V^*(s)$ decreases with distance to G (and increases near C — the agent learns to avoid it). The optimal policy points toward G everywhere except near C, where it routes around.

## ✦ 10. Partial Observability — When the Markov Property Fails

The MDP assumes the agent observes the full state $s_t$. Often it doesn't:

- **Poker:** you cannot see opponents' cards.
- **Robot navigation:** the robot sees only its camera, not the full map.
- **Medical treatment:** the patient's true health state is hidden; only symptoms are observed.

The correct model is a **Partially Observable MDP (POMDP)**: $$\mathcal{M} = \langle \mathcal{S}, \mathcal{A}, \mathcal{O}, P, O, R, \gamma \rangle$$

where $\mathcal{O}$ is the observation space and $O(o | s', a)$ is the observation model.

The agent maintains a **belief state** $b_t = P(s_t | o_0, a_0, \ldots, o_t)$ — a probability distribution over states. The belief state is a sufficient statistic for the history, so $b_t$ is itself Markovian. But $|\mathcal{B}|$ (belief space) is infinite even if $|\mathcal{S}|$ is finite.

**Deep RL approximation:** use an RNN (LSTM, GRU) to maintain a hidden state that approximates the belief state. This is why RNNs appear in RL for partially observable environments.

## ✦ 11. Connection to AI / ML / Deep Learning

### 11.1 Deep RL — Approximating Value Functions with Neural Networks

In the tabular setting, $Q(s, a)$ is stored as a table. For large state spaces (Atari: $210 \times 160 \times 3$ pixel screens), this is impossible. Deep Q-Networks (DQN) approximate: $$Q(s, a) \approx Q(s, a; \theta)$$

where $\theta$ are the parameters of a convolutional neural network. The Bellman optimality equation becomes the training target: $$\mathcal{L}(\theta) = \mathbb{E}\left[\left(r + \gamma \max_{a'} Q(s', a'; \theta^-) - Q(s, a; \theta)\right)^2\right]$$

where $\theta^-$ is a **target network** — a periodically frozen copy of $\theta$ that stabilises training. See [[02.Model-Free Methods]].

### 11.2 RL as the Third Pillar of ML

Supervised learning: learn from labelled data. Unsupervised learning: find structure without labels. RL: learn to act from reward signals. These three are genuinely distinct and complementary:

- A speech recogniser is trained with supervised learning.
- A language model is pre-trained with self-supervised (unsupervised) learning.
- A dialogue agent is fine-tuned with RL (RLHF — Reinforcement Learning from Human Feedback).

**RLHF** is the mechanism behind GPT-4, Claude, and similar models — the MDP is: state = conversation history, action = next token, reward = human preference signal. The policy is the language model.

### 11.3 The Connection to Optimal Control

RL is essentially optimal control without a known model. Classical optimal control (LQR, MPC) assumes $P(s'|s,a)$ is known analytically. RL discovers the policy without knowing the model — making it applicable wherever simulation or interaction is possible but analytical models are not.

### 11.4 AlphaGo / AlphaZero

AlphaZero (Silver et al., 2017) is a model-based RL system for board games:

- State: board position.
- Action: legal moves.
- Model: known (rules of chess/Go).
- Value network: approximates $V^*(s)$ — trained by self-play.
- Policy network: approximates $\pi^*(a|s)$ — guides Monte Carlo Tree Search.

The Bellman equations are the foundation; the neural networks are the approximation; Monte Carlo Tree Search is the planning algorithm. The MDP formalism unifies all of it.
