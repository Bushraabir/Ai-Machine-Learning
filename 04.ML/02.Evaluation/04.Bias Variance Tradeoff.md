
> _"Every learning algorithm makes a bet on the shape of the world. Bias is what you lose when your bet is wrong. Variance is what you lose when you don't have enough data to be sure your bet was right. You cannot eliminate both simultaneously — you can only choose where to be wrong."_

---

## ✦ The Setup

You want to learn $f: \mathcal{X} \to \mathbb{R}$ from noisy observations:

$$y = f(x) + \epsilon, \quad \epsilon \sim \mathcal{N}(0, \sigma^2), \quad \mathbb{E}[\epsilon] = 0, \quad \epsilon \perp x$$

Given a training set $\mathcal{D} = {(x_i, y_i)}_{i=1}^n$ drawn i.i.d. from $\mathcal{P}$, your algorithm produces an estimate $\hat{f}_\mathcal{D}$. At a new test point $x$, the prediction is $\hat{f}_\mathcal{D}(x)$.

The quantity of interest is the **expected prediction error** at $x$, averaged over all possible training sets $\mathcal{D}$ of size $n$ drawn from $\mathcal{P}$:

$$\text{EPE}(x) = \mathbb{E}_{\mathcal{D}, \epsilon}\left[(y - \hat{f}_\mathcal{D}(x))^2\right]$$

The expectation is over both the randomness in $\mathcal{D}$ (which determines $\hat{f}_\mathcal{D}$) and the noise $\epsilon$ in the new observation $y = f(x) + \epsilon$.

---

## ✦ The Full Derivation

This is the central derivation of statistical learning theory. Do it once, by hand, until it is automatic.

Let $\bar{f}(x) = \mathbb{E}_\mathcal{D}[\hat{f}_\mathcal{D}(x)]$ denote the expected prediction at $x$ over all training sets.

**Step 1.** Expand the squared error:

$$\mathbb{E}_{\mathcal{D},\epsilon}\left[(y - \hat{f}_\mathcal{D}(x))^2\right] = \mathbb{E}\left[(f(x) + \epsilon - \hat{f}_\mathcal{D}(x))^2\right]$$

**Step 2.** Add and subtract $\bar{f}(x)$:

$$= \mathbb{E}\left[\left((f(x) - \bar{f}(x)) + (\bar{f}(x) - \hat{f}_\mathcal{D}(x)) + \epsilon\right)^2\right]$$

**Step 3.** Expand. The cross terms vanish:

- $\mathbb{E}[\epsilon] = 0$ eliminates all cross terms involving $\epsilon$
- $\mathbb{E}_\mathcal{D}[\bar{f}(x) - \hat{f}_\mathcal{D}(x)] = 0$ by definition of $\bar{f}$
- $(f(x) - \bar{f}(x))$ is a constant with respect to both $\mathcal{D}$ and $\epsilon$

$$= \underbrace{(f(x) - \bar{f}(x))^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}_\mathcal{D}\left[(\hat{f}_\mathcal{D}(x) - \bar{f}(x))^2\right]}_{\text{Variance}} + \underbrace{\mathbb{E}[\epsilon^2]}_{\text{Irreducible Noise}} $$

Therefore:

$$\boxed{\text{EPE}(x) = \text{Bias}^2[\hat{f}(x)] + \text{Var}[\hat{f}(x)] + \sigma^2}$$

This holds pointwise at every $x$. Integrating over the data distribution gives the **expected generalisation error**:

$$\mathbb{E}_x[\text{EPE}(x)] = \mathbb{E}_x\left[\text{Bias}^2[\hat{f}(x)]\right] + \mathbb{E}_x\left[\text{Var}[\hat{f}(x)]\right] + \sigma^2$$

### What Each Term Means

**Bias$^2$** $= (f(x) - \bar{f}(x))^2$

The squared difference between the true function and the _average_ prediction over all training sets. Bias measures **systematic error** — the error your algorithm makes even with infinite data. It captures the gap between the best function your hypothesis class can represent and the true $f$.

A high-bias model has the wrong inductive bias: it cannot fit $f$ no matter how much data you give it. A linear model applied to a quadratic $f$ has irreducible bias.

**Variance** $= \mathbb{E}_\mathcal{D}[(\hat{f}_\mathcal{D}(x) - \bar{f}(x))^2]$

The expected squared deviation of $\hat{f}_\mathcal{D}$ from its own mean. Variance measures **sensitivity to the training set** — how much the prediction at $x$ changes when you train on a different draw of $n$ samples. High variance means the model is fitting noise.

**Irreducible noise** $= \sigma^2$

The variance of $\epsilon$ — noise in the labels that no model can remove. Even the perfect predictor $\hat{f} = f$ incurs this error. It is a property of the data-generating process, not the model.

### The Key Insight

You cannot reduce $\sigma^2$. You can only trade Bias against Variance.

**Reducing bias** (making the model more flexible — add features, increase depth, reduce regularisation) tends to **increase variance**.

**Reducing variance** (making the model less flexible — regularise, constrain, use simpler hypothesis class) tends to **increase bias**.

The optimal model sits at the minimum of $\text{Bias}^2 + \text{Variance}$, not the minimum of either alone.

## ✦ Concrete Worked Example

### Polynomial Regression

True function: $f(x) = \sin(2\pi x)$ on $[0, 1]$. Noise: $\epsilon \sim \mathcal{N}(0, 0.1)$. Training set size: $n = 15$.

Fit polynomials of degree $d = 1, 3, 9$ to many training sets drawn from the same distribution. The figures below show the spread of fitted curves.

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(0)
n_datasets = 50
n_train = 15
sigma = 0.3
x_test = np.linspace(0, 1, 200)

fig, axes = plt.subplots(1, 3, figsize=(15, 5))

for ax, degree in zip(axes, [1, 3, 9]):
    predictions = []
    for _ in range(n_datasets):
        x_train = np.sort(np.random.uniform(0, 1, n_train))
        y_train = np.sin(2 * np.pi * x_train) + np.random.normal(0, sigma, n_train)
        
        coeffs = np.polyfit(x_train, y_train, degree)
        y_pred = np.polyval(coeffs, x_test)
        predictions.append(y_pred)
        ax.plot(x_test, y_pred, alpha=0.1, color='steelblue')
    
    predictions = np.array(predictions)
    mean_pred = predictions.mean(axis=0)
    y_true = np.sin(2 * np.pi * x_test)
    
    bias_sq  = np.mean((y_true - mean_pred) ** 2)
    variance = np.mean(predictions.var(axis=0))
    
    ax.plot(x_test, y_true, 'k-', lw=2, label='True f(x)')
    ax.plot(x_test, mean_pred, 'r-', lw=2, label=f'Mean prediction')
    ax.set_title(f'Degree {degree}\nBias²={bias_sq:.3f}  Var={variance:.3f}  '
                 f'Sum={bias_sq+variance:.3f}')
    ax.set_ylim(-2, 2); ax.legend()

plt.suptitle('Bias-Variance Tradeoff: Polynomial Regression', fontsize=14)
plt.tight_layout(); plt.show()
```

**Typical output:**

|Degree|Bias²|Variance|Bias² + Var|
|---|---|---|---|
|1 (line)|0.45|0.003|0.453|
|3 (cubic)|0.02|0.015|0.035|
|9 (overfit)|0.01|0.41|0.42|

Degree 3 is the sweet spot. Degree 1 has high bias (can't fit the sine wave). Degree 9 has high variance (fits each training set perfectly but predicts wildly on new data). The optimal degree minimises the sum — this is the U-shaped generalisation error curve.

### The U-Curve

```
Error
  │
  │  \         Total = Bias² + Var + σ²
  │   \       /
  │    \     /
  │     \   /   ← Variance
  │      \_/
  │      Bias²
  │
  └─────────────────────────
      Simple         Complex
      (underfit)     (overfit)
         Model Complexity
```

The minimum of the U-curve is the optimal model complexity. To the left: underfitting (high bias dominates). To the right: overfitting (high variance dominates). The bottom of the curve shifts right as $n$ increases — more data allows more complex models without variance exploding.

## ✦ Bias and Variance for Specific Models

### $k$-Nearest Neighbours (Analytical)

For KNN regression with $k$ neighbours, at a test point $x$:

$$\hat{f}_k(x) = \frac{1}{k} \sum_{x_j \in \mathcal{N}_k(x)} y_j$$

**Bias.** As $k$ increases, the neighbourhood expands and the average smooths over a larger region. For large $k$, the local average approximates the global mean — unable to track the local structure of $f$. Bias increases with $k$.

Formally, assuming $f$ is Lipschitz with constant $L$: $$\text{Bias}[\hat{f}_k(x)] \approx \frac{L \cdot r_k(x)}{d+2}$$ where $r_k(x)$ is the radius of the $k$-NN ball — which grows with $k$.

**Variance.** Each $y_j = f(x_j) + \epsilon_j$ with $\text{Var}[y_j] = \sigma^2$. Averaging $k$ i.i.d. noise terms:

$$\text{Var}[\hat{f}_k(x)] = \text{Var}!\left[\frac{1}{k}\sum_{j=1}^k y_j\right] = \frac{\sigma^2}{k}$$

Variance decreases as $1/k$. Small $k$ → high variance; large $k$ → low variance.

**The tradeoff:** $\text{EPE}(x) \approx \text{Bias}^2(k) + \frac{\sigma^2}{k} + \sigma^2$. The optimal $k^* = \arg\min_k [\text{Bias}^2(k) + \sigma^2/k]$.

This is exactly the same tension as the polynomial degree example — just parameterised differently. See [[07.k-Nearest Neighbours|k-Nearest Neighbours]].

### Linear Models (OLS)

For OLS with model $\hat{f}(x) = x^T\hat{w}$, $\hat{w} = (X^TX)^{-1}X^Ty$:

$$\mathbb{E}[\hat{w}] = w^* \implies \text{Bias} = 0 \quad \text{(when the true model is linear)}$$

OLS is **unbiased** when the true function is linear in the features. The Gauss-Markov theorem says OLS is the **B**est **L**inear **U**nbiased **E**stimator (BLUE) — minimum variance among all linear unbiased estimators.

$$\text{Var}[\hat{w}] = \sigma^2 (X^TX)^{-1}$$

The variance is large when $(X^TX)^{-1}$ is large — i.e., when features are nearly collinear (multicollinearity) or when $n$ is small relative to $d$.

**Ridge regression deliberately introduces bias to reduce variance:**

$$\hat{w}_\lambda = (X^TX + \lambda I)^{-1}X^Ty$$

$$\mathbb{E}[\hat{w}_\lambda] = (X^TX + \lambda I)^{-1}X^TX w^* \neq w^* \quad \text{for } \lambda > 0$$

The bias is $(X^TX + \lambda I)^{-1}X^TX w^* - w^* = -\lambda(X^TX + \lambda I)^{-1}w^*$ — nonzero but shrinking toward zero as $\lambda \to 0$.

The variance:

$$\text{Var}[\hat{w}_\lambda] = \sigma^2 (X^TX + \lambda I)^{-1} X^TX (X^TX + \lambda I)^{-1}$$

The $(X^TX + \lambda I)^{-1}$ terms shrink the variance relative to OLS. For large enough $\lambda$, the variance reduction outweighs the bias increase, and Ridge has lower MSE than OLS — the bias-variance tradeoff in its cleanest algebraic form.

## ✦ Diagnosing from Learning Curves

A **learning curve** plots training error and validation error as a function of training set size $n$. It is the most useful diagnostic tool for bias vs. variance.

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression, Ridge
from sklearn.model_selection import learning_curve
from sklearn.datasets import make_regression

X, y = make_regression(n_samples=500, n_features=1, noise=30, random_state=0)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

configs = [
    ("High Bias (degree=1)", Pipeline([
        ("poly", PolynomialFeatures(degree=1)),
        ("lr",   LinearRegression())
    ])),
    ("High Variance (degree=15)", Pipeline([
        ("poly", PolynomialFeatures(degree=15)),
        ("lr",   Ridge(alpha=0.0001))   # tiny regularisation
    ])),
]

train_sizes = np.linspace(0.1, 1.0, 10)

for ax, (title, model) in zip(axes, configs):
    sizes, train_scores, val_scores = learning_curve(
        model, X, y, train_sizes=train_sizes,
        cv=5, scoring="neg_mean_squared_error", n_jobs=-1
    )
    train_err = -train_scores.mean(axis=1)
    val_err   = -val_scores.mean(axis=1)
    
    ax.plot(sizes, train_err, 'o-', label='Train MSE',      color='steelblue')
    ax.plot(sizes, val_err,   's-', label='Validation MSE', color='darkorange')
    ax.set_xlabel('Training set size')
    ax.set_ylabel('MSE')
    ax.set_title(title)
    ax.legend()
    ax.set_ylim(bottom=0)

plt.tight_layout(); plt.show()
```

### Reading the Curves

**High Bias (Underfitting):**

```
MSE
 │  train ─────────────────  ← high and flat
 │  val   ─────────────────  ← high and flat, close to train
 │
 └──────────────────────────
       n (training set size)
```

- Both train and validation error are high
- They converge quickly and stay close together
- **Adding more data will not help** — the model is fundamentally limited by its hypothesis class
- **Fix:** increase model complexity, add features, reduce regularisation

**High Variance (Overfitting):**

```
MSE
 │  val   ╲___________________  ← starts high, slowly decreases
 │                              (large gap persists)
 │  train  ─────────────────── ← low, flat
 │
 └──────────────────────────────
        n (training set size)
```

- Training error is low; validation error is much higher
- Large, persistent gap between train and val
- Validation error decreases as $n$ increases (more data helps)
- **Fix:** reduce model complexity, add regularisation, get more data, feature selection

**Optimal Model:**

```
MSE
 │  val   ╲_______  ← converges to near train error
 │  train  ───────
 │
 └──────────────────────────────
```

- Small gap between train and val
- Both errors are acceptably low
- This is the target

## ✦ Regularisation as the Operational Solution

The bias-variance tradeoff is the diagnosis. Regularisation is the treatment.

Every regulariser constrains the hypothesis class, trading bias for variance.

### L2 Regularisation (Ridge / Weight Decay)

$$\mathcal{L}_{\text{Ridge}} = \underbrace{\frac{1}{n}\sum_i (y_i - \hat{y}_i)^2}_{\text{empirical risk}} + \underbrace{\lambda |w|_2^2}_{\text{regulariser}}$$

**Bayesian interpretation:** MAP estimation under a Gaussian prior $w \sim \mathcal{N}(0, \sigma_w^2 I)$ with $\lambda = \sigma^2 / \sigma_w^2$. Ridge shrinks all weights toward zero proportionally — never exactly zero.

**Effect on the bias-variance tradeoff:** increasing $\lambda$ shrinks $\hat{w}$ toward zero (increasing bias), while stabilising estimates across different training sets (decreasing variance).

**In neural networks:** L2 regularisation is called **weight decay**. In SGD:

$$w \leftarrow w - \alpha \nabla_w \mathcal{L} - 2\alpha\lambda w = (1 - 2\alpha\lambda) w - \alpha \nabla_w \mathcal{L}_{\text{data}}$$

The $(1 - 2\alpha\lambda)$ factor decays the weights at each step — hence the name.

### L1 Regularisation (Lasso)

$$\mathcal{L}_{\text{Lasso}} = \frac{1}{n}\sum_i (y_i - \hat{y}_i)^2 + \lambda |w|_1$$

**Bayesian interpretation:** MAP under a Laplace prior $w_j \sim \text{Laplace}(0, 1/\lambda)$.

**Why L1 produces sparsity — geometric argument.** The constraint region for L1 is a diamond (hypercube in $\mathbb{R}^d$) with corners on the axes. The loss function's level sets are ellipses (or hyperellipsoids). The intersection of an ellipse with a diamond tends to occur at a corner — where one or more $w_j = 0$ exactly.

For L2, the constraint region is a sphere. The intersection with an ellipse rarely occurs at a point of zero curvature — weights are shrunk but not zeroed.

```
w₂                      w₂
 │    ╲  L1 constraint   │       L2 constraint
 │     ╲                 │      ╭─────╮
 │      ◇   ←sparse      │     │       │ ←dense
 │     ╱ ╲  solution     │      ╰─────╯  solution
 │    ╱   ╲             │
─┼──────────────         ─┼──────────────
             w₁                        w₁
```

**Practical consequence:** Lasso performs **implicit feature selection**. The number of nonzero weights is at most $\min(n, d)$ — useful when you believe the true model is sparse.

### Elastic Net

$$\mathcal{L}_{\text{EN}} = \frac{1}{n}\sum_i (y_i - \hat{y}_i)^2 + \lambda_1 |w|_1 + \lambda_2 |w|_2^2$$

Combines L1 sparsity with L2 stability. Preferred over Lasso when features are correlated (Lasso arbitrarily picks one among correlated features; Elastic Net tends to include all of them with shared weight).

### Early Stopping — Implicit L2 Regularisation

For gradient descent on the empirical risk, stop training before convergence. This is not ad hoc — for linear models with squared loss and gradient descent with learning rate $\alpha$:

At step $t$, the GD estimate satisfies:

$$\hat{w}^{(t)} = (I - (I - \alpha X^TX)^t) (X^TX)^{-1} X^T y$$

The eigenvalue decomposition of $X^TX = V\Lambda V^T$ shows that component $j$ of $\hat{w}^{(t)}$ is multiplied by a factor $(1 - (1 - \alpha\lambda_j)^t)$. Small eigenvalues (corresponding to directions of low data variance) are suppressed more — exactly the behaviour of Ridge regression with $\lambda \propto 1/t$.

Early stopping is therefore equivalent to Ridge regularisation with $\lambda \approx 1/(\alpha t)$. More steps $\leftrightarrow$ less regularisation $\leftrightarrow$ higher variance.

### Dropout — Variance Reduction by Ensemble Approximation

Dropout (Srivastava et al., 2014) randomly sets each neuron's activation to zero with probability $p$ during training. At test time, weights are scaled by $(1-p)$.

**Bias-variance interpretation:** each forward pass uses a different sub-network (a different random mask). Dropout training is approximately optimising an ensemble of $2^d$ tied networks (where $d$ is the number of neurons). Since ensemble averaging reduces variance (see [[01.Bagging and Random Forests|Bagging]]), dropout reduces the effective variance of the deep network.

More precisely, dropout acts as a regulariser equivalent to L2 on the pre-activation weights (Wager et al., 2013) — but adapted to the geometry of the network.

### Data Augmentation — Variance Reduction by Increasing Effective $n$

By applying label-preserving transformations (flips, crops, colour jitter for images; back-translation for text), you effectively increase the training set size, pushing the U-curve's minimum rightward. More data reduces variance without increasing bias.

## ✦ The Double Descent Phenomenon

Classical statistical intuition says: increasing model complexity beyond the interpolation threshold (where training error = 0) increases variance and worsens generalisation. The U-curve ends there.

This turns out to be **wrong for modern overparameterised models**.

Belkin et al. (2019) showed that the risk curve has a **second descent** after the interpolation threshold:

```
Risk
  │
  │  \         Classical   │  Modern
  │   \          regime    │  regime
  │    \                   │
  │     \_____   ↑ peak    │ \___________
  │           ↑            │             (continues down)
  │   interpolation threshold
  └─────────────────────────────────────
              Model Parameters / n
```

**The interpolation threshold** (parameters $=$ training samples) is where the model can exactly fit the training data. Classical theory predicts this is the worst point. In practice, for neural networks and other sufficiently structured models, going beyond the threshold produces models that **interpolate the training data but still generalise** — because among all interpolating solutions, the optimiser (SGD) finds the minimum-norm one, which has low effective complexity.

**Why double descent occurs:**

1. Below interpolation: classical bias-variance tradeoff — complexity increases variance
2. At interpolation: variance spikes — the model barely fits, and is extremely sensitive to training data
3. Beyond interpolation: the model has excess capacity, finds a _simple_ interpolating solution (minimum norm in the case of gradient descent + infinitely wide networks), variance decreases again

**Implication for practice:** for neural networks, adding more parameters (width, depth) past the interpolation point continues to improve generalisation, as long as you use appropriate regularisation (weight decay, dropout, early stopping). The "right" amount of regularisation shifts, but the direction of model scaling is reliably beneficial — this is the empirical foundation of modern large model scaling laws.

## ✦ Ensemble Methods — Explicitly Decomposing the Tradeoff

Ensemble methods are the practical engineering solution to the bias-variance tradeoff. They manipulate the two terms explicitly:

**Bagging (Bootstrap Aggregating):**

If $M$ base learners each have variance $\sigma^2$ and pairwise correlation $\rho$, the ensemble variance is:

$$\text{Var}!\left[\frac{1}{M}\sum_{m=1}^M \hat{f}_m(x)\right] = \rho\sigma^2 + \frac{1-\rho}{M}\sigma^2$$

As $M \to \infty$: variance $\to \rho\sigma^2$. The irreducible variance floor is $\rho\sigma^2$ — determined entirely by the correlation between base learners. **Decorrelation is the core goal of bagging.** Random Forests achieve this by adding feature randomness at each split.

Bagging does **not** reduce bias — each learner is trained on roughly the same distribution, so the average has approximately the same bias as a single learner.

**Boosting:**

Fits models sequentially, each targeting the residuals of the previous. Boosting primarily **reduces bias** — it builds up an increasingly complex function that better approximates $f$. It can increase variance if the boosted model overfits. Shrinkage (small learning rate) plus subsampling controls this.

See [[01.Bagging and Random Forests|Bagging and Random Forests]] and [[02.Boosting|Boosting]] for full derivations.

## ✦ Bias-Variance in Classification

The decomposition above is for squared loss. For 0-1 loss (classification), the decomposition is more complex and less clean.

**Domingos (2000)** extended the decomposition to classification:

$$\text{Error} = \text{Noise} + \text{Bias} + \text{Variance}$$

But the terms interact multiplicatively near the decision boundary in a way that has no clean closed form. The qualitative conclusions are the same:

- Low-complexity classifiers (e.g., linear SVM, logistic regression): high bias if the true boundary is nonlinear
- High-complexity classifiers (e.g., unpruned decision tree): high variance — decision boundary wiggles around training points
- Ensembles of trees (Random Forest, XGBoost): reduce variance while controlling bias growth

In practice, even for classification, the MSE decomposition is used as a proxy and its intuitions transfer reliably.

## ✦ Connection to AI, Machine Learning, and Deep Learning

### The Scaling Laws Perspective

Kaplan et al. (2020) showed that for language models, test loss follows a power law in model size $N$, dataset size $D$, and compute $C$:

$$L(N, D, C) \approx \left(\frac{N_c}{N}\right)^{\alpha_N} + \left(\frac{D_c}{D}\right)^{\alpha_D} + L_\infty$$

where $L_\infty$ is the irreducible loss (analogous to $\sigma^2$) and the power law terms are the reducible bias + variance.

This is the bias-variance tradeoff at the scale of frontier models:

- **Model size $N$** controls expressivity → reducing bias
- **Dataset size $D$** controls overfitting → reducing variance
- **Compute $C$** determines where on the tradeoff you can afford to sit

The scaling laws tell you the optimal allocation of a compute budget between model size and data — the bias-variance tradeoff expressed as an engineering optimisation problem at scale.

### Underfitting and Overfitting in Practice — The Full Pipeline View

In a production ML pipeline, bias and variance do not only come from model complexity. They accumulate across the full system:

|Source|Bias|Variance|
|---|---|---|
|Label noise / annotation errors|✗|✓ (irreducible, contributes to $\sigma^2$)|
|Wrong features (missing signal)|✓|✗|
|Too simple a model|✓|✗|
|Too few training examples|✗|✓|
|Too much regularisation|✓|✗|
|Distribution shift (train ≠ test)|✓|✗|

**Distribution shift** is a form of bias that is invisible to the standard decomposition: the model's average prediction $\bar{f}(x)$ is measured over training distributions, but the test distribution differs. The bias term explodes in ways that more data from the training distribution cannot fix. This is why domain adaptation and out-of-distribution generalisation are active research areas.

## ✦ Implementation: Full Bias-Variance Estimator

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import BaggingRegressor

def bias_variance_decomp(model_fn, X_train_sets, X_test, y_test_true):
    """
    Estimate bias² and variance of a model.
    
    model_fn:       callable, returns a fresh unfitted model
    X_train_sets:   list of training sets (each a tuple (X_train, y_train))
    X_test:         fixed test points
    y_test_true:    f(X_test) — noiseless true values
    """
    predictions = []
    for X_tr, y_tr in X_train_sets:
        model = model_fn()
        model.fit(X_tr, y_tr)
        predictions.append(model.predict(X_test))
    
    predictions = np.array(predictions)   # shape: (n_datasets, n_test)
    mean_pred = predictions.mean(axis=0)  # shape: (n_test,)
    
    bias_sq  = np.mean((y_test_true - mean_pred) ** 2)
    variance = np.mean(predictions.var(axis=0))
    
    return bias_sq, variance

# Demonstration: decision tree depth vs. bias-variance
np.random.seed(42)
n_datasets = 200
n_train = 50

def gen_data(n):
    x = np.sort(np.random.uniform(0, 1, n))
    y = np.sin(2 * np.pi * x) + np.random.normal(0, 0.3, n)
    return x.reshape(-1, 1), y

train_sets = [gen_data(n_train) for _ in range(n_datasets)]
x_test = np.linspace(0, 1, 100).reshape(-1, 1)
y_true = np.sin(2 * np.pi * x_test.ravel())

print(f"{'Depth':>6} {'Bias²':>8} {'Var':>8} {'Sum':>8}")
print("-" * 36)
for depth in [1, 2, 3, 5, 10, None]:
    fn = lambda d=depth: DecisionTreeRegressor(max_depth=d)
    b2, v = bias_variance_decomp(fn, train_sets, x_test, y_true)
    print(f"{str(depth):>6} {b2:>8.4f} {v:>8.4f} {b2+v:>8.4f}")

# Adding bagging to see variance reduction
print("\nWith Bagging (depth=10, n_estimators=50):")
fn_bag = lambda: BaggingRegressor(
    DecisionTreeRegressor(max_depth=10), n_estimators=50, random_state=0
)
b2, v = bias_variance_decomp(fn_bag, train_sets, x_test, y_true)
print(f"{'bag':>6} {b2:>8.4f} {v:>8.4f} {b2+v:>8.4f}")
```

**Expected output pattern:**

```

------------------------------------
     1   0.4321   0.0051   0.4372  ← high bias, low variance
     2   0.1823   0.0089   0.1912
     3   0.0612   0.0183   0.0795
     5   0.0124   0.0521   0.0645  ← near optimal
    10   0.0021   0.1843   0.1864
  None   0.0008   0.3211   0.3219  ← low bias, high variance

With Bagging (depth=10, n_estimators=50):
   bag   0.0019   0.0312   0.0331  ← bagging collapses variance
```

Bagging a depth-10 tree reduces variance from 0.18 to 0.03 — an 83% reduction — without changing bias (both depth-10 variants have $\text{Bias}^2 \approx 0.002$). This is the mechanism of Random Forests in one table.
