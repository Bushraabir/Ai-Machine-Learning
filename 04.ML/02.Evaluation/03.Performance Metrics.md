
> _"A metric is a compressed summary of model behaviour. Every compression loses information. The art is knowing which information you can afford to lose — and which losses will cost you in production."_

---

## ✦ Why Metrics Matter More Than People Think

A metric does three things simultaneously:

1. **Defines what "better" means** — it is the objective function of model selection
2. **Summarises a distribution of errors** into a scalar — information is always lost
3. **Encodes a value judgement** about which errors cost more

Choosing the wrong metric is not a technical mistake — it is a _specification mistake_. A model that perfectly optimises the wrong metric can be actively harmful. Classic examples:

- **Accuracy on imbalanced data:** a cancer detector that predicts "no cancer" for everyone achieves 99.5% accuracy when prevalence is 0.5%. It has zero clinical utility.
- **MSE for demand forecasting:** penalises overestimation and underestimation equally. If overstock costs $1/unit and stockout costs $10/unit, MSE optimises the wrong objective.
- **BLEU score for translation:** maximises n-gram overlap. A model can score well by producing fluent but factually wrong translations.

The sections below derive each metric from first principles and make explicit what value judgement it encodes.

## ✦ Part I — Regression Metrics

Setting: $y \in \mathbb{R}$, predictions $\hat{y}_i$, residuals $e_i = y_i - \hat{y}_i$.

### Mean Squared Error (MSE)

$$\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 = \frac{1}{n} \sum_{i=1}^{n} e_i^2$$

**What it encodes:** squared error penalises large residuals quadratically. An error of 10 is penalised 100× more than an error of 1. MSE is the right metric when large errors are disproportionately costly.

**Probabilistic derivation:** MSE is the negative log-likelihood of a Gaussian noise model. If $y_i = f(x_i) + \epsilon_i$ with $\epsilon_i \sim \mathcal{N}(0, \sigma^2)$:

$$-\log P(\mathcal{D}|\theta) = \frac{n}{2}\log(2\pi\sigma^2) + \frac{1}{2\sigma^2}\sum_i e_i^2$$

Minimising MSE is therefore equivalent to maximum likelihood estimation under Gaussian noise. This is the probabilistic justification for OLS.

**Units:** squared units of $y$. Hard to interpret directly.

**Sensitivity to outliers:** extreme. A single outlier with $|e_i| = 100$ contributes $10^4$ to the sum. For robust regression, prefer MAE or Huber loss.

### Root Mean Squared Error (RMSE)

$$\text{RMSE} = \sqrt{\text{MSE}} = \sqrt{\frac{1}{n}\sum_i e_i^2}$$

Same units as $y$. Directly interpretable as "typical error magnitude" — but still outlier-sensitive. Preferred over MSE for reporting because of interpretability.

**RMSE vs. standard deviation of residuals:**

$$s_e = \sqrt{\frac{1}{n-p}\sum_i e_i^2}$$

RMSE divides by $n$; $s_e$ divides by $n-p$ (degrees of freedom, $p$ = number of parameters). For large $n$, the difference is negligible. $s_e$ is unbiased; RMSE is slightly biased downward.

### Mean Absolute Error (MAE)

$$\text{MAE} = \frac{1}{n}\sum_{i=1}^{n} |y_i - \hat{y}_i| = \frac{1}{n}\sum_i |e_i|$$

**What it encodes:** all errors penalised linearly. Robust to outliers — a single large $|e_i|$ contributes proportionally, not quadratically.

**Probabilistic derivation:** MAE is the negative log-likelihood under a Laplace noise model. If $\epsilon_i \sim \text{Laplace}(0, b)$:

$$P(\epsilon_i) = \frac{1}{2b}\exp!\left(-\frac{|\epsilon_i|}{b}\right)$$

$$-\log P(\mathcal{D}|\theta) \propto \sum_i |e_i|$$

Minimising MAE is MLE under Laplace noise. The optimal predictor under MAE is the **conditional median** $\hat{y} = \text{median}(y|x)$; under MSE it is the **conditional mean** $\hat{y} = \mathbb{E}[y|x]$.

**Gradient:** MAE is not differentiable at $e_i = 0$. The subgradient is $\text{sign}(e_i)$. In practice this causes instability near convergence for gradient-based optimisers — use Huber loss instead.

### Huber Loss

Combines MSE's smoothness near zero with MAE's robustness to outliers:

$$L_\delta(e) = \begin{cases} \frac{1}{2}e^2 & \text{if } |e| \leq \delta \ \delta|e| - \frac{1}{2}\delta^2 & \text{if } |e| > \delta \end{cases}$$

Quadratic for small residuals (smooth, fast convergence), linear for large residuals (robust to outliers). The hyperparameter $\delta$ controls the transition point.

**Why it matters in practice:** Huber loss is the default in robust regression and is used as the regression head loss in many object detection networks (e.g., Faster R-CNN uses smooth L1, which is Huber with $\delta=1$).

### $R^2$ (Coefficient of Determination)

$$R^2 = 1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}} = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2}$$

**Interpretation:** fraction of variance in $y$ explained by the model. $R^2 = 1$ means perfect prediction; $R^2 = 0$ means the model does no better than predicting the mean $\bar{y}$; $R^2 < 0$ means the model is worse than the mean predictor.

**Derivation of the decomposition.** For OLS specifically (and OLS only, when an intercept is included):

$$\text{SS}_{\text{tot}} = \text{SS}_{\text{reg}} + \text{SS}_{\text{res}}$$

$$\sum_i (y_i - \bar{y})^2 = \sum_i (\hat{y}_i - \bar{y})^2 + \sum_i (y_i - \hat{y}_i)^2$$

This follows from $\hat{\epsilon} \perp \hat{y}$ in OLS (the residuals are orthogonal to the fitted values — a consequence of the normal equations). This decomposition does **not** hold for arbitrary models or without an intercept.

**Pitfalls:**

- $R^2$ always increases as you add features (even noise features). Use **adjusted $R^2$**: $\bar{R}^2 = 1 - \frac{(1-R^2)(n-1)}{n-p-1}$, which penalises model complexity
- $R^2$ is not a measure of predictive accuracy for new data — a model can have high $R^2$ in-sample and poor generalisation
- For non-OLS models (random forests, NNs), $R^2$ can be negative on the test set even if the model performs well

### Mean Absolute Percentage Error (MAPE)

$$\text{MAPE} = \frac{100}{n}\sum_i \frac{|y_i - \hat{y}_i|}{|y_i|}$$

Scale-free: useful for comparing models across datasets with different units or magnitudes. Used heavily in forecasting (demand, finance).

**Critical failure mode:** undefined when $y_i = 0$; heavily penalises underestimates relative to overestimates (asymmetric). Prefer **sMAPE** (symmetric MAPE) or **MASE** (mean absolute scaled error) for forecasting.

## ✦ Part II — Classification Metrics

### The Confusion Matrix — The Foundation

For binary classification ($y \in {0, 1}$, positive class = 1):

$$\begin{array}{c|cc} & \hat{y} = 1 & \hat{y} = 0 \ \hline y = 1 & \text{TP} & \text{FN} \ y = 0 & \text{FP} & \text{TN} \end{array}$$

- **TP** (True Positive): correctly predicted positive
- **FP** (False Positive): incorrectly predicted positive — _Type I error_
- **FN** (False Negative): incorrectly predicted negative — _Type II error_
- **TN** (True Negative): correctly predicted negative

Every binary classification metric is a function of these four numbers. Understanding the matrix is understanding all classification metrics.

### Accuracy

$$\text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}} = \frac{\text{correct predictions}}{n}$$

**When it is meaningful:** balanced classes, equal cost of FP and FN.

**When it is misleading:** class imbalance. If 95% of samples are negative, the trivial classifier $\hat{y} = 0$ achieves 95% accuracy. This is the **accuracy paradox**.

**The deeper problem:** accuracy treats all errors equally. In most real problems, FP and FN have different costs. Accuracy ignores this.

### Precision and Recall — The Fundamental Tradeoff

$$\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}} \qquad \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}$$

**Precision** answers: _of all instances we predicted positive, what fraction actually are positive?_ It measures the quality of positive predictions. Low precision = many false alarms.

**Recall** (sensitivity, true positive rate) answers: _of all actual positives, what fraction did we catch?_ It measures coverage of the positive class. Low recall = many missed positives.

**The tradeoff is inherent and real.** For any probabilistic classifier, predictions are $\hat{p}(y=1|x)$, and a threshold $\tau$ determines the decision: $\hat{y} = \mathbf{1}[\hat{p} \geq \tau]$.

- Increase $\tau$: predict positive only when very confident → fewer FP (higher precision), more FN (lower recall)
- Decrease $\tau$: predict positive more liberally → more TP (higher recall), more FP (lower precision)

**Domain determines which matters more:**

|Domain|Costly Error|Preferred|Reasoning|
|---|---|---|---|
|Cancer screening|FN (missed cancer)|High recall|Missing cancer is catastrophic|
|Spam filtering|FP (lost legitimate email)|High precision|False alarms destroy user trust|
|Information retrieval|Depends on use case|PR curve|Plot the full tradeoff|
|Fraud detection|FN (missed fraud)|High recall|Financial losses from undetected fraud|

### F1-Score — Harmonic Mean of Precision and Recall

$$F_1 = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} = \frac{2\text{TP}}{2\text{TP} + \text{FP} + \text{FN}}$$

**Why harmonic mean, not arithmetic?** The harmonic mean penalises extreme imbalance between precision and recall. If precision = 1.0 and recall = 0.01:

$$\text{Arithmetic mean} = \frac{1.0 + 0.01}{2} = 0.505 \quad \text{(looks decent)}$$ $$F_1 = \frac{2 \times 1.0 \times 0.01}{1.0 + 0.01} = 0.020 \quad \text{(correctly poor)}$$

The harmonic mean is $\leq$ the arithmetic mean, with equality only when both terms are equal. It rewards balanced precision and recall.

**$F_\beta$ generalisation:** when FP and FN have different costs, weight recall $\beta$ times more than precision:

$$F_\beta = (1 + \beta^2) \cdot \frac{\text{Precision} \cdot \text{Recall}}{\beta^2 \cdot \text{Precision} + \text{Recall}}$$

- $\beta = 0.5$: precision matters twice as much as recall
- $\beta = 1$: $F_1$, equal weight
- $\beta = 2$: recall matters twice as much as precision (use for cancer detection)

### Specificity and False Positive Rate

$$\text{Specificity} = \frac{\text{TN}}{\text{TN} + \text{FP}} \qquad \text{FPR} = 1 - \text{Specificity} = \frac{\text{FP}}{\text{FP} + \text{TN}}$$

Specificity measures how well the classifier identifies negatives. FPR = probability of a false alarm given a truly negative instance. These appear on the x-axis of the ROC curve.

### ROC Curve and AUC

The **Receiver Operating Characteristic (ROC) curve** plots TPR (recall) vs. FPR at every possible threshold $\tau \in [0, 1]$.

$$\text{ROC} = {(\text{FPR}(\tau), \text{TPR}(\tau)) : \tau \in [0,1]}$$

```
TPR
1.0 |         ___________
    |       /            |
    |      /  Area = AUC |
    |     /              |
    |    /               |
    |   / (random)       |
0.0 |__/________________|
    0.0                 1.0  FPR
```

**Key reference points:**

- $(0, 0)$: threshold = 1, predict everything negative
- $(1, 1)$: threshold = 0, predict everything positive
- $(0, 1)$: perfect classifier
- Diagonal: random classifier

**AUC (Area Under the ROC Curve):**

$$\text{AUC} = \int_0^1 \text{TPR}(\text{FPR}), d(\text{FPR})$$

**Probabilistic interpretation (Mann-Whitney U statistic):** AUC equals the probability that the classifier assigns a higher score to a randomly chosen positive than to a randomly chosen negative:

$$\text{AUC} = P(\hat{p}(y=1|x^+) > \hat{p}(y=1|x^-))$$

where $x^+$ is a random positive and $x^-$ a random negative. This is a threshold-free measure of ranking quality.

**Derivation.** Let $n_+, n_-$ be the number of positives and negatives. Order all $n$ predictions by score. The Mann-Whitney U statistic counts concordant pairs:

$$U = \sum_{i: y_i=1} \sum_{j: y_j=0} \mathbf{1}[\hat{p}_i > \hat{p}_j]$$

Then $\text{AUC} = U / (n_+ \cdot n_-)$. This gives an $O(n \log n)$ algorithm for computing AUC exactly (sort by score, count inversions).

**When AUC is appropriate:** ranking quality matters, threshold is not fixed. AUC is threshold-invariant — good for comparing classifiers when the operating threshold will be chosen later.

**When AUC is misleading:** severe class imbalance. AUC can be high even when precision is catastrophically low. Consider a dataset with 1000 negatives and 10 positives. A classifier that ranks all positives above all negatives achieves AUC = 1.0, but at FPR = 1% it may have only 50% precision. Use the PR curve instead.

### Precision-Recall Curve

The **PR curve** plots Precision vs. Recall at every threshold $\tau$.

$$\text{PR curve} = {(\text{Recall}(\tau), \text{Precision}(\tau)) : \tau \in [0,1]}$$

**Average Precision (AP):**

$$\text{AP} = \sum_k (\text{Recall}_k - \text{Recall}_{k-1}) \cdot \text{Precision}_k$$

This is the area under the PR curve, approximated by the step function of threshold-sorted predictions.

**Why PR is preferred under class imbalance:** Precision = TP/(TP+FP) — FP is in the denominator. When negatives are abundant, a classifier that generates many FP is correctly penalised by low precision. The ROC curve's FPR = FP/(FP+TN) — TN is in the denominator — so large TN dilutes the penalty for many FP. The ROC curve is "too forgiving" when negatives vastly outnumber positives.

**Comparison:**

|Scenario|Use|
|---|---|
|Balanced classes, threshold flexibility|ROC / AUC|
|Imbalanced classes (fraud, disease, rare events)|PR curve / AP|
|Fixed operating point (specific threshold)|Precision + Recall at that threshold|
|Final report scalar|F1 (balanced) or $F_\beta$ (asymmetric costs)|

### Matthews Correlation Coefficient (MCC)

$$\text{MCC} = \frac{\text{TP} \cdot \text{TN} - \text{FP} \cdot \text{FN}}{\sqrt{(\text{TP}+\text{FP})(\text{TP}+\text{FN})(\text{TN}+\text{FP})(\text{TN}+\text{FN})}}$$

MCC ranges from $-1$ (perfect inverse prediction) to $+1$ (perfect prediction). MCC $= 0$ for a random predictor regardless of class balance.

**Why MCC is more informative than $F_1$:**

$F_1$ ignores TN entirely — it does not appear in the formula. This makes $F_1$ insensitive to how well the classifier identifies negatives. MCC uses all four cells of the confusion matrix. For imbalanced binary classification, MCC is now the recommended metric in computational biology and is increasingly standard in ML (Chicco & Jurman, 2020).

**Worked example:** dataset with 90 negatives, 10 positives. Classifier predicts all negative:

$$\text{TP}=0,\ \text{FP}=0,\ \text{FN}=10,\ \text{TN}=90$$

$$\text{Accuracy} = 90%$$ $$\text{Recall} = 0%,\quad \text{Precision} = \text{undefined (0/0)}$$ $$F_1 = 0%$$ $$\text{MCC} = \frac{0 - 0}{\sqrt{0 \cdot 10 \cdot 90 \cdot 90}} = 0$$

MCC = 0 correctly identifies this as a random-level classifier. Accuracy (90%) is wildly misleading.

## ✦ Part III — Multi-Class Classification

For $C$ classes, extend the binary confusion matrix to a $C \times C$ matrix where entry $(i,j)$ counts samples of true class $i$ predicted as class $j$.

**Macro-averaging:** compute the metric for each class independently, then take the unweighted mean. Treats all classes equally regardless of frequency. Sensitive to performance on rare classes.

**Micro-averaging:** aggregate TP, FP, FN across all classes first, then compute the metric once. Dominated by frequent classes. Equivalent to accuracy for F1.

**Weighted averaging:** weight each class by its support (number of true samples). Standard in sklearn's `classification_report`.

$$F_1^{\text{macro}} = \frac{1}{C}\sum_{c=1}^C F_1^{(c)} \qquad F_1^{\text{micro}} = \frac{2\sum_c \text{TP}_c}{2\sum_c \text{TP}_c + \sum_c \text{FP}_c + \sum_c \text{FN}_c}$$

**When to use which:**

- **Macro:** when every class matters equally (e.g., multi-disease diagnosis)
- **Micro:** when overall per-sample performance matters (e.g., tag prediction)
- **Weighted:** when class frequency should inform the metric (standard reporting)

## ✦ Part IV — Probabilistic Classifiers

Some classifiers output calibrated probabilities $\hat{p}(y=1|x)$ rather than hard labels. Evaluating these requires metrics that account for the full probability output.

### Log-Loss (Binary Cross-Entropy)

$$\mathcal{L}_{\log} = -\frac{1}{n}\sum_{i=1}^n \left[y_i \log \hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\right]$$

**What it measures:** the average negative log-likelihood of the true labels under the predicted distribution. A perfectly calibrated classifier achieves log-loss equal to the entropy of the label distribution. Lower is better.

**Why it penalises overconfident wrong predictions so harshly:** if the true label is $y=1$ and the model predicts $\hat{p} = 0.001$, the contribution is $-\log(0.001) \approx 6.9$. A confident wrong prediction is catastrophically penalised. This incentivises calibration.

Log-loss is the training objective for logistic regression and the output layer of classification neural networks. Using it as the evaluation metric aligns training and evaluation objectives perfectly.

### Brier Score

$$\text{BS} = \frac{1}{n}\sum_{i=1}^n (\hat{p}_i - y_i)^2$$

MSE for probability predictions. Ranges from 0 (perfect) to 1 (maximally wrong). Less sensitive to extreme predictions than log-loss (quadratic vs. logarithmic penalty).

**Brier score decomposition:**

$$\text{BS} = \underbrace{\frac{1}{n}\sum_k n_k (\bar{p}_k - \bar{y}_k)^2}_{\text{Calibration}} - \underbrace{\frac{1}{n}\sum_k n_k \bar{y}_k(1-\bar{y}_k)}_{\text{Resolution}} + \underbrace{\bar{y}(1-\bar{y})}_{\text{Uncertainty}}$$

where bins $k$ group predictions into intervals of $\hat{p}$. **Calibration** measures whether predicted probabilities match empirical frequencies; **Resolution** measures how much predicted probabilities differ from the base rate (informative classifiers have high resolution); **Uncertainty** is a property of the data, not the model.

### Calibration

A classifier is **calibrated** if among all instances where it predicts $\hat{p} = 0.7$, approximately 70% are truly positive.

**Calibration plot (reliability diagram):** bin predictions by $\hat{p}$, plot mean predicted probability vs. empirical positive rate per bin. A perfectly calibrated classifier lies on the diagonal $y=x$.

```
Empirical
positive
rate
1.0 |                    /  ← over-confident
    |               /
0.7 |          / ← well calibrated
    |      /
    |  /
0.0 |/________________________
    0.0    0.5    1.0   Predicted probability
```

**Common miscalibration patterns:**

- **Sigmoid-shaped curve (over-confident):** model predicts probabilities too close to 0 and 1. Common in SVMs and boosting
- **Inverse sigmoid (under-confident):** probabilities too moderate. Common in Naive Bayes

**Calibration methods:**

- **Platt scaling:** fit a logistic regression on top of the raw model scores. Works well for SVMs
- **Isotonic regression:** non-parametric, more flexible. Risk of overfitting on small datasets

Calibration matters whenever probabilities are used downstream — risk stratification, decision thresholds based on cost-benefit analysis, ensembling.

## ✦ Part V — Implementation

### Full Pipeline in sklearn

```python
import numpy as np
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split, StratifiedKFold, cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    roc_auc_score, average_precision_score, matthews_corrcoef,
    log_loss, brier_score_loss, classification_report,
    confusion_matrix, RocCurveDisplay, PrecisionRecallDisplay
)
import matplotlib.pyplot as plt

X, y = load_breast_cancer(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

pipe = Pipeline([("sc", StandardScaler()), ("clf", LogisticRegression(max_iter=1000))])
pipe.fit(X_train, y_train)

y_pred  = pipe.predict(X_test)
y_prob  = pipe.predict_proba(X_test)[:, 1]

# ── Threshold-dependent metrics ──────────────────────────────────────
print("Accuracy:  ", accuracy_score(y_test, y_pred))
print("Precision: ", precision_score(y_test, y_pred))
print("Recall:    ", recall_score(y_test, y_pred))
print("F1:        ", f1_score(y_test, y_pred))
print("MCC:       ", matthews_corrcoef(y_test, y_pred))
print()
print(classification_report(y_test, y_pred, target_names=["malignant", "benign"]))
print("Confusion matrix:\n", confusion_matrix(y_test, y_pred))

# ── Threshold-independent metrics ────────────────────────────────────
print("ROC-AUC:   ", roc_auc_score(y_test, y_prob))
print("Avg Prec:  ", average_precision_score(y_test, y_prob))

# ── Probabilistic metrics ─────────────────────────────────────────────
print("Log-loss:  ", log_loss(y_test, y_prob))
print("Brier:     ", brier_score_loss(y_test, y_prob))

# ── Curves ────────────────────────────────────────────────────────────
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 5))
RocCurveDisplay.from_estimator(pipe, X_test, y_test, ax=ax1)
ax1.plot([0,1],[0,1],'k--', label='Random'); ax1.legend()

PrecisionRecallDisplay.from_estimator(pipe, X_test, y_test, ax=ax2)
plt.tight_layout(); plt.show()
```

### Computing AUC from Scratch (Mann-Whitney U)

```python
def auc_from_scratch(y_true, y_score):
    """
    Exact AUC via Mann-Whitney U statistic.
    O(n_pos * n_neg) — use sort-based method for large n.
    """
    pos_scores = y_score[y_true == 1]
    neg_scores = y_score[y_true == 0]
    
    n_pos, n_neg = len(pos_scores), len(neg_scores)
    concordant = sum(
        1.0 if p > n else 0.5 if p == n else 0.0
        for p in pos_scores
        for n in neg_scores
    )
    return concordant / (n_pos * n_neg)

# Verify
from sklearn.metrics import roc_auc_score
auc_manual = auc_from_scratch(y_test, y_prob)
auc_sk     = roc_auc_score(y_test, y_prob)
print(f"Manual: {auc_manual:.6f}, sklearn: {auc_sk:.6f}")  # Should match
```

### Regression Metrics from Scratch

```python
def regression_metrics(y_true, y_pred):
    y_true, y_pred = np.array(y_true), np.array(y_pred)
    e = y_true - y_pred
    n = len(y_true)
    
    mse  = np.mean(e**2)
    rmse = np.sqrt(mse)
    mae  = np.mean(np.abs(e))
    ss_res = np.sum(e**2)
    ss_tot = np.sum((y_true - y_true.mean())**2)
    r2   = 1 - ss_res / ss_tot
    mape = np.mean(np.abs(e) / (np.abs(y_true) + 1e-10)) * 100
    
    return {"MSE": mse, "RMSE": rmse, "MAE": mae, "R2": r2, "MAPE": mape}
```

## ✦ Metric Selection Guide

```
Is the output a probability?
├── Yes → Log-loss + Brier score + calibration plot
└── No (hard labels or rankings)
    │
    Is it regression?
    ├── Yes
    │   ├── Outliers present / asymmetric costs? → MAE or Huber
    │   ├── Gaussian errors / large errors costly? → MSE / RMSE
    │   └── Explainability needed? → R²  (with adjusted R²)
    │
    └── Classification
        │
        Is the data imbalanced?
        ├── Yes → PR curve / AP / F_β / MCC
        └── No
            │
            Do FP and FN have equal costs?
            ├── Yes → Accuracy / F1 / ROC-AUC
            └── No → F_β  (choose β based on relative cost)
                      or MCC (uses all confusion matrix cells)
```

## ✦ Connection to AI, Machine Learning, and Deep Learning

### Loss Functions as Training Metrics

The training loss and the evaluation metric should ideally be the same — or at least tightly aligned. When they diverge, optimising the training loss does not optimise the evaluation metric:

|Evaluation Metric|Matching Training Loss|Notes|
|---|---|---|
|Log-loss|Binary cross-entropy|Perfect alignment|
|Brier score|MSE on probabilities|Good alignment|
|AUC|Surrogate losses (pairwise ranking losses, ListNet)|AUC is non-differentiable; surrogates approximate it|
|F1|Weighted cross-entropy / focal loss|F1 is non-differentiable; focal loss addresses class imbalance|
|BLEU (NLP)|Cross-entropy on tokens|Severe mismatch — active research in RLHF, minimum Bayes risk decoding|

The mismatch between training loss and evaluation metric is a fundamental tension in applied ML. It motivates reinforcement learning from human feedback (RLHF) in language models: the evaluation "metric" is human preference, which is non-differentiable, so a reward model is trained to approximate it, and policy gradient methods optimise the approximation.

### Focal Loss — Addressing Imbalance in Deep Learning

Lin et al. (2017, RetinaNet) introduced **focal loss** for object detection, where the class imbalance between background and objects is extreme ($\sim$ 1000:1):

$$\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

where $p_t$ is the model's probability for the true class, $\gamma > 0$ is the **focusing parameter**, and $\alpha_t$ is a class weight.

The term $(1-p_t)^\gamma$ down-weights easy negatives (where $p_t \approx 1$ for class 0, so the model is already confident). For $\gamma = 0$, focal loss reduces to standard cross-entropy. For $\gamma = 2$, a well-classified example with $p_t = 0.9$ contributes only $(0.1)^2 = 0.01\times$ its cross-entropy loss — effectively, easy examples barely affect the gradient.

This is a direct consequence of the precision-recall tradeoff baked into the loss function.

### Metrics in LLM Evaluation

Evaluating large language models is an open research problem precisely because no single metric captures the relevant properties:

|Property|Metric|Problem|
|---|---|---|
|Fluency|Perplexity|Low perplexity ≠ useful output|
|Translation quality|BLEU|Fails on paraphrases; doesn't measure factuality|
|Summarisation|ROUGE|Recall-oriented; rewards copying source|
|Factuality|FactScore, SAFE|Expensive; requires retrieval|
|Human preference|Win rate, Elo|No ground truth; expensive; inconsistent|
|Reasoning|Accuracy on benchmarks|Contamination, saturation issues|

The impossibility of a single metric for language model quality is why RLHF was developed: learn a reward model from human comparisons, then optimise it. The reward model is itself a learned metric.

```python
# Deliberately create imbalanced data to show metric divergence
from sklearn.datasets import make_classification
from sklearn.dummy import DummyClassifier

X, y = make_classification(
    n_samples=10000, n_features=20,
    weights=[0.95, 0.05],   # 95% negative, 5% positive
    random_state=42
)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.3, stratify=y)

# Model 1: trivial — predicts majority class always
dummy = DummyClassifier(strategy="most_frequent").fit(X_tr, y_tr)

# Model 2: logistic regression
pipe = Pipeline([("sc", StandardScaler()), ("clf", LogisticRegression())])
pipe.fit(X_tr, y_tr)

for name, model in [("Dummy", dummy), ("LogReg", pipe)]:
    y_pred = model.predict(X_te)
    print(f"\n── {name} ──")
    print(f"  Accuracy:  {accuracy_score(y_te, y_pred):.3f}")
    print(f"  F1:        {f1_score(y_te, y_pred, zero_division=0):.3f}")
    print(f"  MCC:       {matthews_corrcoef(y_te, y_pred):.3f}")
    if hasattr(model, "predict_proba"):
        y_prob = model.predict_proba(X_te)[:, 1]
        print(f"  ROC-AUC:   {roc_auc_score(y_te, y_prob):.3f}")
        print(f"  Avg Prec:  {average_precision_score(y_te, y_prob):.3f}")

# Expected output (approximately):
# ── Dummy ──
#   Accuracy:  0.950   ← looks great, is useless
#   F1:        0.000   ← correctly zero
#   MCC:       0.000   ← correctly zero
#
# ── LogReg ──
#   Accuracy:  0.960   ← only slightly better than dummy by accuracy
#   F1:        0.700   ← clearly better
#   MCC:       0.680   ← clearly better
#   ROC-AUC:   0.970   ← strong ranking ability
#   Avg Prec:  0.750   ← strong precision at high recall thresholds
```

This example demonstrates the metric divergence concretely: accuracy flatters the dummy classifier; F1, MCC, AUC, and AP correctly identify it as useless.
