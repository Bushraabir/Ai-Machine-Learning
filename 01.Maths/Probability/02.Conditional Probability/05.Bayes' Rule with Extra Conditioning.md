
> _This feels like a footnote the first time you see it. It isn't. The moment you start doing anything with random variables, inference, or hierarchical models, you're conditioning on background information constantly — and you need Bayes to still work inside that conditioned universe. So: does it? Yes. Trivially. But it's worth writing out once carefully so it becomes automatic._

---

## The Situation

We already know Bayes' Rule:

$$P(A \mid B) = \frac{P(B \mid A), P(A)}{P(B)}$$

But in practice, we're almost never reasoning in a vacuum. There's always some **background event** $E$ we're already conditioning on — maybe we know the patient is over 60, or we know the coin is from a particular batch, or we've already observed some earlier data.

The question is: **does Bayes still hold if we condition everything on $E$?**

---

## The Theorem

**Bayes' Rule with Extra Conditioning.** For events $A$, $B$, $E$ with $P(B \cap E) > 0$:

$$\boxed{P(A \mid B, E) = \frac{P(B \mid A, E), P(A \mid E)}{P(B \mid E)}}$$

where $P(A \mid B, E) := P(A \mid B \cap E)$ — standard shorthand.

---

## The Proof

This is one of those results where the proof is so short it almost feels like cheating. Everything follows from the **definition of conditional probability**, applied twice inside the universe $E$.

**Step 1.** Write out the definition of $P(A \mid B, E)$:

$$P(A \mid B, E) = \frac{P(A \cap B \cap E)}{P(B \cap E)}$$

**Step 2.** Rewrite the numerator by conditioning on $A \cap E$:

$$P(A \cap B \cap E) = P(B \mid A, E), P(A \cap E) = P(B \mid A, E), P(A \mid E), P(E)$$

**Step 3.** Rewrite the denominator:

$$P(B \cap E) = P(B \mid E), P(E)$$

**Step 4.** Divide:

$$P(A \mid B, E) = \frac{P(B \mid A, E), P(A \mid E), P(E)}{P(B \mid E), P(E)} = \frac{P(B \mid A, E), P(A \mid E)}{P(B \mid E)}$$

$\blacksquare$

> _The $P(E)$ factors cancel perfectly. That's the whole trick. Everything is just the ordinary definition of conditional probability — we never used anything deeper. Bayes inside a conditioned universe is Bayes, full stop._

---

## The Right Way to Read This

The formula says: **everything works exactly as before, but now $E$ is the ambient background that every probability inherits.**

|Ordinary Bayes|Conditioned Bayes|
|---|---|
|$P(A)$ — prior on $A$|$P(A \mid E)$ — prior on $A$, given $E$|
|$P(B \mid A)$ — likelihood|$P(B \mid A, E)$ — likelihood, given $E$|
|$P(B)$ — marginal|$P(B \mid E)$ — marginal, given $E$|
|$P(A \mid B)$ — posterior|$P(A \mid B, E)$ — posterior, given $E$|

Think of $E$ as **shrinking the sample space**. Once you've conditioned on $E$, every probability lives in the universe $\Omega_E = E$. And in that smaller universe, Bayes holds with exactly the same structure.

---

## LOTP with Extra Conditioning

Naturally, the **Law of Total Probability** also extends. If $A_1, A_2, \ldots, A_n$ partition the sample space and $P(A_i \cap E) > 0$ for all $i$, then:

$$P(B \mid E) = \sum_{i=1}^{n} P(B \mid A_i, E), P(A_i \mid E)$$

**Proof.** $$P(B \mid E) = \frac{P(B \cap E)}{P(E)} = \frac{\sum_i P(B \cap A_i \cap E)}{P(E)} = \sum_i \frac{P(B \mid A_i, E), P(A_i \mid E), \cancel{P(E)}}{\cancel{P(E)}}$$ $\blacksquare$

This is the denominator we plug into Bayes with extra conditioning when we need to compute $P(B \mid E)$ by summing over cases.

## A Worked Example: Two-Stage Disease Testing

Let's make this concrete and connect it back to [[Testing a Rare Disease]].

**Setup.** Suppose now we know that the patient is a **healthcare worker** (event $E$). Healthcare workers have a higher prior disease probability — say $P(D \mid E) = 0.01$ instead of $0.001$.

The test characteristics are the same: $$P(T^+ \mid D, E) = 0.99, \qquad P(T^+ \mid D^c, E) = 0.01$$

_(We condition on $E$ here too — the test works the same on healthcare workers as anyone else, but it's correct notation.)_

**Compute $P(D \mid T^+, E)$:**

$$P(T^+ \mid E) = P(T^+ \mid D, E),P(D \mid E) + P(T^+ \mid D^c, E),P(D^c \mid E)$$ $$= (0.99)(0.01) + (0.01)(0.99) = 0.0099 + 0.0099 = 0.0198$$

$$P(D \mid T^+, E) = \frac{(0.99)(0.01)}{0.0198} = \frac{0.0099}{0.0198} = \boxed{0.5}$$

Exactly 50%. The higher prior (1% vs 0.1%) made an enormous difference — we went from ~9% to 50% with the same test result. Same Bayes machine, different prior, radically different posterior.

> _This is the answer to "why does the prior matter so much?" It's not philosophical hand-waving — it's arithmetic. The prior is load-bearing in the formula. Change it, and the posterior moves dramatically, especially when evidence isn't overwhelming._

## A More Structural Example: Hierarchical Setup

Here's a setup that really forces you to use Bayes with extra conditioning.

**Setup.** A bag is chosen uniformly at random from two bags:

- Bag 1: 3 red, 7 blue
- Bag 2: 6 red, 4 blue

You draw two balls **with replacement**. The first is red (event $R_1$). Given this, what's the probability the second is also red (event $R_2$)?

Let $B_1$ = "Bag 1 chosen", $B_2$ = "Bag 2 chosen". We want:

$$P(R_2 \mid R_1)$$

**By LOTP with extra conditioning** (conditioning on which bag):

$$P(R_2 \mid R_1) = P(R_2 \mid R_1, B_1),P(B_1 \mid R_1) + P(R_2 \mid R_1, B_2),P(B_2 \mid R_1)$$

First, update the bag probabilities via Bayes:

$$P(B_1 \mid R_1) = \frac{P(R_1 \mid B_1),P(B_1)}{P(R_1)} = \frac{(0.3)(0.5)}{(0.3)(0.5)+(0.6)(0.5)} = \frac{0.15}{0.45} = \frac{1}{3}$$

$$P(B_2 \mid R_1) = 1 - \frac{1}{3} = \frac{2}{3}$$

Now, since draws are with replacement, $R_2 \perp R_1 \mid \text{Bag}$:

$$P(R_2 \mid R_1, B_1) = P(R_2 \mid B_1) = 0.3$$ $$P(R_2 \mid R_1, B_2) = P(R_2 \mid B_2) = 0.6$$

Therefore: $$P(R_2 \mid R_1) = (0.3)!\left(\frac{1}{3}\right) + (0.6)!\left(\frac{2}{3}\right) = 0.1 + 0.4 = \boxed{0.5}$$

Notice: $P(R_2) = (0.3)(0.5) + (0.6)(0.5) = 0.45 \neq 0.5$. So $R_1$ and $R_2$ are **not independent** — observing $R_1$ shifts our belief about which bag we're in, which shifts the probability of $R_2$.

> _This is one of my favourite examples because $R_1$ and $R_2$ are conditionally independent given the bag, but marginally dependent. The dependence is entirely mediated through the latent variable (which bag). This is the prototype of how hidden structure creates dependence — you'll see this pattern constantly in statistics and machine learning._

## The Key Conceptual Point

Bayes with extra conditioning is not a new theorem. It's the same theorem, applied in a restricted sample space. But it matters because:

1. **Real inference always has context.** You never reason in a vacuum. $E$ is the totality of everything you knew before seeing $B$.
    
2. **Sequential updating is just iterated Bayes with extra conditioning.** After seeing evidence $B_1$, your posterior $P(A \mid B_1)$ becomes the prior for the next update. When you then see $B_2$: $$P(A \mid B_1, B_2) = \frac{P(B_2 \mid A, B_1),P(A \mid B_1)}{P(B_2 \mid B_1)}$$ This is exactly Bayes with extra conditioning, where $E = B_1$.
    
3. **Conditional independence simplifies everything.** If $B_2 \perp B_1 \mid A$ (the new evidence is independent of old evidence, given the hypothesis), then $P(B_2 \mid A, B_1) = P(B_2 \mid A)$ and the formula simplifies. This is the **naïve Bayes** assumption — and it's the workhorse of an enormous amount of practical inference.
    

---

## Summary of the Formulas

$$P(A \mid B, E) = \frac{P(B \mid A, E), P(A \mid E)}{P(B \mid E)}$$

$$P(B \mid E) = \sum_{i} P(B \mid A_i, E), P(A_i \mid E) \qquad \text{(LOTP with extra conditioning)}$$

$$P(A \mid B_1, B_2) = \frac{P(B_2 \mid A, B_1),P(A \mid B_1)}{P(B_2 \mid B_1)} \qquad \text{(sequential updating)}$$
