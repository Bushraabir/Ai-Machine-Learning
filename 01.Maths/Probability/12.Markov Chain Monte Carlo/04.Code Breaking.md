

> _"Shannon's 1949 paper on secrecy systems is one of the most beautiful things I've ever read. He took a problem that had been purely combinatorial — how do you hide a message? — and recast it as a problem in probability theory. A cipher is a random variable. Security is a statement about conditional distributions. The perfect cipher is one where the ciphertext and plaintext are independent. And he proved, with elementary probability, exactly how much key you need to achieve this — and exactly how many ciphertext characters you need to read before the key is determined. The whole paper is just probability, applied with ruthless clarity."_

## The Setup: A Probabilistic Model of Secrecy

Let me fix notation. A **cryptosystem** consists of:

- $\mathcal{M}$: the **message space** (plaintexts). Each message $M \in \mathcal{M}$ is a random variable with some prior distribution $P(M = m)$.
- $\mathcal{K}$: the **key space**. Key $K \in \mathcal{K}$ drawn uniformly at random (or from some distribution), independent of $M$.
- $\mathcal{C}$: the **ciphertext space**.
- **Encryption:** $C = E_K(M)$ — a deterministic function of $M$ and $K$.
- **Decryption:** $M = D_K(C)$ — the inverse: $D_K(E_K(m)) = m$ for all $m, k$.

The **adversary** observes $C$ and wants to recover $M$ (or learn something about it). The question: how much does $C$ reveal about $M$?

This is a conditional distribution question:

$$P(M = m \mid C = c) = \frac{P(C = c \mid M = m) \cdot P(M = m)}{P(C = c)}$$

Security means the posterior $P(M \mid C)$ is close to the prior $P(M)$ — the ciphertext teaches the adversary nothing.

## Classical Ciphers

### The Caesar Cipher

**Encryption:** Shift each letter by a fixed key $k \in {0, 1, \ldots, 25}$:

$$E_k(m_i) = (m_i + k) \bmod 26$$

**Key space:** $|\mathcal{K}| = 26$.

**Breaking it:** Exhaustive search — try all 26 keys, find the one that produces readable English. Even by hand, this takes under a minute. Computationally, it's $O(26 \cdot n)$ where $n$ is the message length.

**Probabilistically:** If $M$ is a single letter drawn from English letter frequencies, the posterior after observing one ciphertext letter $C = c$ is:

$$P(K = k \mid C = c, M = m) = \mathbf{1}[c = (m + k) \bmod 26]$$

But I don't know $M$. I know $P(M = m)$ — the English letter frequencies. Bayes gives:

$$P(M = m \mid C = c) \propto P(C = c \mid M = m) P(M = m) = \mathbf{1}[E_k(m) = c] \cdot P(M = m)$$

For a fixed key $k$, the cipher is a bijection on letters, so each ciphertext letter corresponds to exactly one plaintext letter under that key. There's no probabilistic uncertainty — if I know $k$, I know $m$ exactly. The uncertainty is purely about $k$.

The Caesar cipher has essentially zero security: $H(K) = \log_2 26 \approx 4.7$ bits of key entropy — trivially exhausted.

### The Vigenère Cipher

**Encryption:** Use a key $\mathbf{k} = (k_1, \ldots, k_d) \in {0,\ldots,25}^d$ (period $d$). Encrypt the $i$-th letter by shifting by $k_{i \bmod d}$:

$$E_{\mathbf{k}}(m_i) = (m_i + k_{i \bmod d}) \bmod 26$$

**Key space:** $|\mathcal{K}| = 26^d$ — exponentially larger than Caesar.

**Naive analysis:** If the key length $d$ is unknown, brute force is infeasible for large $d$. Key entropy is $H(\mathbf{K}) = d \log_2 26 \approx 4.7d$ bits.

**But:** The Vigenère is just $d$ interleaved Caesar ciphers. Letters at positions $1, d+1, 2d+1, \ldots$ are all encrypted with the same shift $k_1$; letters at positions $2, d+2, \ldots$ with shift $k_2$; and so on.

**Breaking it — two steps:**

**Step 1: Determine $d$ (the key length).**

_Kasiski test:_ Look for repeated substrings in the ciphertext. If a plaintext substring appears at positions $i$ and $j$, and the key repeats with period $d$, then the ciphertext will repeat iff $d \mid (j - i)$. So: find all repeated trigrams in the ciphertext, compute the differences $j - i$, take $\gcd$ of all differences — that's likely $d$ (or a multiple of it).

_Index of Coincidence:_ More elegant. Define the **index of coincidence** of a text $c_1, \ldots, c_n$ as:

$$\text{IC} = \sum_{l=0}^{25} \frac{f_l(f_l - 1)}{n(n-1)}$$

where $f_l$ = frequency of letter $l$ in the text. This is an estimate of $P(\text{two random letters are equal}) = \sum_{l=0}^{25} p_l^2$, where $p_l$ is the true frequency of letter $l$.

For English text: $\text{IC}_{\text{English}} \approx 0.065$ (because English is non-uniform — 'e' appears $\sim 13%$ of the time, boosting $\sum p_l^2$).

For a uniformly random text: $\text{IC}_{\text{uniform}} = \sum_{l=0}^{25} (1/26)^2 = 1/26 \approx 0.038$.

For a Vigenère-encrypted English text with key length $d$: if I take every $d$-th character, I get a Caesar-shifted English text — IC $\approx 0.065$. If I take every $d'$-th character for $d' \neq d$, the letters mix like random — IC $\approx 0.038$.

**Algorithm:** For $d' = 1, 2, 3, \ldots$, compute IC of every $d'$-th subsequence. The true $d$ is the smallest $d'$ where IC jumps to $\approx 0.065$.

**Step 2: Break each Caesar cipher separately.**

Once I know $d$, split the ciphertext into $d$ streams. Each stream is a Caesar cipher on English text. For each stream, compare the observed letter frequencies (shifted by $k_j$) to the English frequency distribution — find the shift that best aligns them.

**Formally:** Let $\hat{p}_0, \ldots, \hat{p}_{25}$ be the observed letter frequencies in stream $j$. The true frequencies are $p_0, \ldots, p_{25}$ (English). The key $k_j$ satisfies $\hat{p}_{(l + k_j) \bmod 26} \approx p_l$ for all $l$. Find $k_j$ maximizing:

$$\sum_{l=0}^{25} \hat{p}_{(l+k_j) \bmod 26} \cdot p_l$$

(dot product of shifted observed with true frequencies — maximum at the correct shift). This is a **maximum likelihood estimate** under the assumption that letters are i.i.d. with English frequencies.

> _The IC is a beautiful object. It's just $\sum p_l^2$ estimated from data — the probability that two randomly drawn letters match. The fact that this single number distinguishes "Caesar-shifted English" (IC $\approx 0.065$) from "random noise" (IC $\approx 0.038$) is what makes the Vigenère breakable. The non-uniformity of English letter frequencies is the lever. The more non-uniform the source, the easier the cipher is to break._

## Frequency Analysis as Maximum Likelihood

Let me be precise about what frequency analysis is doing probabilistically.

**Model.** Assume plaintext letters $M_1, M_2, \ldots, M_n$ are i.i.d. with distribution $\mathbf{p} = (p_0, \ldots, p_{25})$ (English frequencies). Ciphertext is $C_i = (M_i + k) \bmod 26$ for unknown $k$.

**Likelihood** of observing ciphertext counts $\mathbf{f} = (f_0, \ldots, f_{25})$ given key $k$:

$$L(k) = P(\mathbf{f} \mid k) = \prod_{l=0}^{25} p_{(l-k) \bmod 26}^{f_l}$$

(Letter $l$ in the ciphertext came from plaintext letter $(l - k) \bmod 26$, which has probability $p_{(l-k) \bmod 26}$.)

**Log-likelihood:**

$$\log L(k) = \sum_{l=0}^{25} f_l \log p_{(l-k) \bmod 26} = n \sum_{l=0}^{25} \hat{p}_l \log p_{(l-k) \bmod 26}$$

where $\hat{p}_l = f_l / n$. Maximizing over $k \in {0, \ldots, 25}$:

$$\hat{k}_{\text{MLE}} = \arg\max_k \sum_{l=0}^{25} \hat{p}_l \log p_{(l-k) \bmod 26}$$

This is the **cross-entropy** between $\hat{\mathbf{p}}$ and the shifted English distribution — frequency analysis is maximum likelihood, nothing more.

**Connection to KL divergence:** Note $\sum_l \hat{p}_l \log p_{(l-k)\bmod 26} = -H(\hat{\mathbf{p}}) - D_{\text{KL}}(\hat{\mathbf{p}} | \mathbf{p}_{-k})$ where $\mathbf{p}_{-k}$ is $\mathbf{p}$ shifted by $-k$. Maximizing log-likelihood is equivalent to minimizing $D_{\text{KL}}(\hat{\mathbf{p}} | \mathbf{p}_{-k})$ — finding the shift that makes the English frequencies closest to the observed frequencies, in KL-divergence.

## Perfect Secrecy: Shannon's Definition

**Definition (Perfect Secrecy).** A cryptosystem has **perfect secrecy** if for every message $m \in \mathcal{M}$ and ciphertext $c \in \mathcal{C}$:

$$P(M = m \mid C = c) = P(M = m)$$

Equivalently (by Bayes): $P(C = c \mid M = m) = P(C = c)$ — $C$ and $M$ are **independent**. Observing the ciphertext gives zero information about the plaintext.

Equivalently (in terms of mutual information): $I(M; C) = 0$.

**Theorem (Shannon, 1949).** A cryptosystem with $|\mathcal{M}| = |\mathcal{C}| = |\mathcal{K}|$ achieves perfect secrecy if and only if:

1. Every key is used with equal probability: $K \sim \text{Uniform}(\mathcal{K})$.
2. For every plaintext $m$ and ciphertext $c$, there exists exactly one key $k$ such that $E_k(m) = c$.

**Proof.**

($\Rightarrow$) Assume perfect secrecy. Fix any ciphertext $c$. For every $m$:

$$P(C = c \mid M = m) = P(C = c)$$

Since $P(C = c \mid M = m) = \sum_k P(K = k) \cdot \mathbf{1}[E_k(m) = c]$, summing over all messages:

$$P(C = c) = \sum_m P(M = m) P(C = c \mid M = m) = P(C = c) \cdot \sum_m P(M = m) = P(C = c)$$

This is tautological, but the constraint comes from fixing $m$: for each $m$, $P(C = c \mid M = m) = P(C = c)$ must hold simultaneously for all $m$. This forces the map $k \mapsto E_k(m)$ to cover $\mathcal{C}$ with the same distribution regardless of $m$. Since $|\mathcal{M}| = |\mathcal{K}| = |\mathcal{C}|$, the function $k \mapsto E_k(m)$ is a bijection for each fixed $m$, and uniform $K$ gives uniform $C$ for any $m$. The key must be uniform, and the bijection must hold — conditions 1 and 2.

($\Leftarrow$) Assume conditions 1 and 2. Then for any $m, c$:

$$P(C = c \mid M = m) = P(E_K(m) = c) = P(K = \text{unique } k \text{ s.t. } E_k(m) = c) = \frac{1}{|\mathcal{K}|}$$

This is independent of $m$, so $C \perp M$. $\blacksquare$

**Corollary (Key Length Lower Bound).** For perfect secrecy: $|\mathcal{K}| \geq |\mathcal{M}|$. Equivalently, $H(K) \geq H(M)$. **The key must be at least as long as the message.**

_Proof:_ From condition 2, for each $c$, there are $|\mathcal{K}|$ pairs $(m, k)$ with $E_k(m) = c$ (one per key). These must cover all $|\mathcal{M}|$ messages. So $|\mathcal{K}| \geq |\mathcal{M}|$.

## The One-Time Pad: Perfect Secrecy Achieved

**Construction.** $\mathcal{M} = \mathcal{K} = \mathcal{C} = {0,1}^n$ (bitstrings of length $n$). Key $K \sim \text{Uniform}({0,1}^n)$. Encrypt:

$$C = M \oplus K \quad (\text{bitwise XOR})$$

Decrypt: $M = C \oplus K$ (XOR is its own inverse).

**Theorem.** The one-time pad has perfect secrecy.

**Proof.** For any $m, c \in {0,1}^n$:

$$P(C = c \mid M = m) = P(M \oplus K = c \mid M = m) = P(K = m \oplus c) = \frac{1}{2^n}$$

This is independent of $m$. So $C \perp M$. $\blacksquare$

The one-time pad is perfectly secure — and completely impractical. The key must be:

1. As long as the message ($n$ bits for $n$ bits of plaintext).
2. Truly random (not pseudorandom).
3. **Never reused.** (Hence "one-time.")

**Why no reuse?** If $C_1 = M_1 \oplus K$ and $C_2 = M_2 \oplus K$, then $C_1 \oplus C_2 = M_1 \oplus M_2$. The XOR of two ciphertexts equals the XOR of two plaintexts — and $M_1 \oplus M_2$ leaks massive information about both $M_1$ and $M_2$ (especially for natural language, where many bits are redundant). This is the "two-time pad" attack, and it completely breaks one-time pad reuse.

> _The two-time pad attack is what broke the Venona project intercepts — Soviet intelligence reused one-time pad pages due to production pressure during WWII, and the NSA spent decades exploiting the resulting $M_1 \oplus M_2$ relationships in intercepted cables. Perfect theoretical security, destroyed by a single implementation mistake._

## Unicity Distance: How Much Ciphertext Do I Need?

Shannon asked: given a ciphertext of length $n$, how many keys are consistent with it producing meaningful (English) plaintext?

**Definition.** The **unicity distance** $n_0$ is the minimum ciphertext length such that, in expectation, only one key is consistent with the ciphertext producing a valid plaintext. Formally, $n_0$ is the smallest $n$ such that the expected number of **spurious keys** — wrong keys that still decrypt to meaningful text — drops below 1.

**Derivation.** The number of meaningful $n$-letter English plaintexts is approximately $2^{nH_{\text{English}}}$ where $H_{\text{English}}$ is the per-letter entropy of English (empirically $\approx 1.0$–$1.5$ bits per letter, accounting for all correlations; often approximated as $1.0$ bits/letter for long texts).

The total number of $n$-letter strings is $26^n = 2^{n \log_2 26} \approx 2^{4.7n}$.

The **redundancy** of English per letter: $r = \log_2 26 - H_{\text{English}} \approx 4.7 - 1.5 = 3.2$ bits/letter.

For a key $k$ chosen uniformly from $|\mathcal{K}|$ keys, the expected number of spurious keys is:

$$\bar{s}(n) \approx |\mathcal{K}| \cdot \frac{2^{nH_{\text{English}}}}{26^n} = |\mathcal{K}| \cdot 2^{-nr}$$

Setting $\bar{s}(n_0) = 1$:

$$|\mathcal{K}| \cdot 2^{-n_0 r} = 1 \implies n_0 = \frac{\log_2 |\mathcal{K}|}{r} = \frac{H(K)}{r}$$

**For Caesar ($|\mathcal{K}| = 26$, $H(K) = \log_2 26 \approx 4.7$ bits):**

$$n_0 = \frac{4.7}{3.2} \approx 1.5 \text{ letters}$$

With just 2 ciphertext letters, in expectation fewer than 1 spurious key — essentially unique decryption. (This matches intuition: even with 3-4 letters of Caesar, the correct key is usually obvious.)

**For Vigenère with period $d$ ($|\mathcal{K}| = 26^d$, $H(K) = d \log_2 26 \approx 4.7d$ bits):**

$$n_0 = \frac{4.7d}{3.2} \approx 1.47d$$

With about $1.5d$ letters of ciphertext, the key is essentially determined. Impressively small — a Vigenère cipher with 10-letter key is broken with about 15 letters of ciphertext (in principle; in practice you need more for the statistical estimates to be accurate).

**For one-time pad ($|\mathcal{K}| = 26^n$, $H(K) = n \log_2 26$):**

$$n_0 = \frac{n \log_2 26}{r} = \frac{n \cdot 4.7}{3.2} > n$$

The unicity distance exceeds the message length — there are always multiple valid keys consistent with any ciphertext, so the key can never be uniquely determined from the ciphertext alone. This is perfect secrecy, recovered from the information-theoretic angle.

> _The unicity distance is the information-theoretic "point of no return" — below $n_0$ characters, there isn't enough ciphertext to pin down the key; above $n_0$, the key is essentially determined by the ciphertext (given a meaningful plaintext model). It's the boundary between "computationally hard to break" and "theoretically impossible to break." The OTP lives permanently below this boundary — by design._

## Entropy and the Security of Ciphers

Let me collect the information-theoretic picture.

**Key equivocation** $H(K \mid C^n)$: the remaining entropy in the key after observing $n$ ciphertext characters. As $n$ increases:

- For a finite-key cipher over redundant plaintext: $H(K \mid C^n) \to 0$ as $n \to \infty$. Eventually the key is determined.
- For the one-time pad: $H(K \mid C^n) = H(K)$ for all $n$. Observing ciphertext gives zero information about the key (and hence about the plaintext).

Shannon showed:

$$H(K \mid C^n) = H(K) + H(M^n) - H(C^n)$$

(from the chain rule, since $C^n$ is a deterministic function of $(M^n, K)$). For a "good" cipher, $H(C^n) \approx n \log_2 26$ (ciphertext looks uniform). So:

$$H(K \mid C^n) \approx H(K) + H(M^n) - n \log_2 26 = H(K) - n(\log_2 26 - H_{\text{English}}) = H(K) - nr$$

This decreases at rate $r$ per ciphertext character. It hits zero at $n = H(K)/r$ — the unicity distance.

**Mutual information $I(M^n; C^n)$:** How much the ciphertext reveals about the plaintext.

$$I(M^n; C^n) = H(M^n) - H(M^n \mid C^n)$$

For perfect secrecy: $I(M^n; C^n) = 0$ for all $n$.

For a finite-key cipher: $I(M^n; C^n) > 0$ and grows with $n$ past the unicity distance — the ciphertext increasingly pins down the plaintext.

## Frequency Analysis in Code

Let me make this concrete with a Python implementation of the full Vigenère attack.

```python
import numpy as np
from collections import Counter

# English letter frequencies (approximate)
ENGLISH_FREQ = np.array([
    0.0817, 0.0149, 0.0278, 0.0425, 0.1270, 0.0223, 0.0202,
    0.0609, 0.0697, 0.0015, 0.0077, 0.0402, 0.0241, 0.0675,
    0.0751, 0.0193, 0.0010, 0.0599, 0.0633, 0.0906, 0.0276,
    0.0098, 0.0236, 0.0015, 0.0197, 0.0007
])  # a=0, b=1, ..., z=25

def to_numbers(text):
    return [ord(c) - ord('a') for c in text.lower() if c.isalpha()]

def index_of_coincidence(nums):
    n = len(nums)
    if n < 2:
        return 0.0
    counts = np.bincount(nums, minlength=26)
    return np.sum(counts * (counts - 1)) / (n * (n - 1))

def find_key_length(ciphertext_nums, max_d=20):
    """Estimate key length via Index of Coincidence."""
    best_d, best_ic = 1, 0.0
    for d in range(1, max_d + 1):
        streams = [ciphertext_nums[i::d] for i in range(d)]
        avg_ic = np.mean([index_of_coincidence(s) for s in streams if len(s) >= 2])
        if avg_ic > best_ic:
            best_ic = avg_ic
            best_d = d
    return best_d

def break_caesar(stream):
    """MLE key for a Caesar-shifted stream."""
    counts = np.bincount(stream, minlength=26) / len(stream)
    scores = [np.dot(counts, np.roll(ENGLISH_FREQ, -k)) for k in range(26)]
    return np.argmax(scores)

def break_vigenere(ciphertext):
    nums = to_numbers(ciphertext)
    d    = find_key_length(nums)
    key  = [break_caesar(nums[i::d]) for i in range(d)]
    key_str = ''.join(chr(k + ord('a')) for k in key)
    plaintext = ''.join(
        chr((c - key[i % d]) % 26 + ord('a'))
        for i, c in enumerate(nums)
    )
    return key_str, plaintext

# --- Example ---
def vigenere_encrypt(plaintext, key):
    nums = to_numbers(plaintext)
    k    = to_numbers(key)
    return ''.join(
        chr((m + k[i % len(k)]) % 26 + ord('a'))
        for i, m in enumerate(nums)
    )

message    = "the quick brown fox jumps over the lazy dog" * 10
key        = "secret"
ciphertext = vigenere_encrypt(message, key)

recovered_key, recovered_plaintext = break_vigenere(ciphertext)
print(f"Key found:  {recovered_key}")
print(f"Plaintext:  {recovered_plaintext[:50]}...")
```

**What this does:**

1. `index_of_coincidence` estimates $\sum p_l^2$ from data.
2. `find_key_length` searches for the period $d$ that maximizes average IC across streams — the true $d$ makes each stream look like English.
3. `break_caesar` solves the MLE problem $\arg\max_k \sum_l \hat{p}_l \log p_{(l-k) \bmod 26}$ (equivalently, $\arg\max_k \hat{\mathbf{p}} \cdot \mathbf{p}_{-k}$).
4. `break_vigenere` combines them.

For messages of length $\geq 100$ characters, this recovers the key exactly. For shorter messages, the IC estimate is noisy and the key length detection can fail — consistent with the unicity distance analysis.

## The Redundancy of English: Why Ciphers Are Breakable

The entire analysis depends on English having $H_{\text{English}} \ll \log_2 26$. Let me make this precise.

**$n$-gram entropy.** Model English text as a stationary stochastic process. The per-character entropy rate:

$$H_\infty = \lim_{n \to \infty} \frac{H(M_1, M_2, \ldots, M_n)}{n}$$

Empirical estimates (Shannon's original 1951 paper on entropy of English):

- Unigram model (letters i.i.d. with English frequencies): $H \approx 4.03$ bits/letter.
- Bigram model: $H \approx 3.32$ bits/letter.
- Trigram model: $H \approx 3.1$ bits/letter.
- Full English (estimated via human prediction experiments): $H_\infty \approx 1.0$–$1.5$ bits/letter.

Each model captures more structure. The first-order frequency analysis only uses the unigram approximation — it treats letters as i.i.d. with English frequencies. This already gives enough redundancy ($r = 4.7 - 4.03 = 0.67$ bits/letter at the unigram level, or $r \approx 3.2$ bits/letter using the true entropy rate) to break the Vigenère.

**Higher-order attacks.** If I model English as a bigram or trigram process, I can use the transition probabilities for frequency analysis — this works with shorter ciphertexts and handles more complex ciphers. The principle is identical: maximum likelihood over the key space, using a richer language model.

> _This is the moment where code-breaking and natural language processing meet. A better language model (lower $H_\infty$ estimate, more accurate $n$-gram probabilities) gives a stronger attack. Modern cryptanalysis of classical ciphers often uses large $n$-gram frequency tables from billions of words of text. The math is the same as Shannon; the model is just better._

## Modern Implication: Why Modern Ciphers Are Different

Classical ciphers fail because:

1. **Short keys** relative to message: $H(K) < H(M)$, so unicity distance is finite.
2. **Structured operations** (shifts, substitutions) that preserve statistical structure.

Modern symmetric ciphers (AES) avoid both:

1. **Long keys:** AES-256 has $H(K) = 256$ bits. At $r \approx 1$ bit/bit for binary data, unicity distance $\approx 256$ bits of ciphertext — a fraction of one block. But:
2. **Pseudorandomness:** AES output is computationally indistinguishable from uniform random — not just statistically, but even for computationally bounded adversaries. Frequency analysis finds no structure because the cipher actively destroys it.

The distinction: Shannon's perfect secrecy is **information-theoretic** — even an adversary with unlimited computation can't break it. Modern cipher security is **computational** — an adversary with polynomial-time computation can't break it (under hardness assumptions like $P \neq NP$). The OTP is the only information-theoretically secure symmetric cipher. Everything else, including AES, is computationally secure.
