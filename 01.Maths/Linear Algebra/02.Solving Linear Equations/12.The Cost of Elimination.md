
## Why Counting Operations Matters

Gaussian elimination is the algorithm underlying almost everything in numerical linear algebra — solving $Ax = b$, computing $\det A$, factoring $A = LU$. Before trusting it on large problems, I need to know how its cost scales with $n$. The answer determines whether a $10000 \times 10000$ system is solvable in seconds or centuries.

The unit of measurement is a **flop** (floating point operation). One flop is one addition, subtraction, multiplication, or division on real numbers. The goal is to count flops for elimination on an $n \times n$ matrix as a function of $n$, keeping track of the leading term.

---

## Setup: What Elimination Does, Step by Step

Given $A \in \mathbb{R}^{n \times n}$, forward elimination produces an upper triangular matrix $U$ by zeroing out all entries below the diagonal. The process has $n - 1$ stages.

At stage $k$ (for $k = 1, 2, \ldots, n-1$), the pivot is $a_{kk}$ (the current $(k,k)$ entry). We eliminate the entries $a_{ik}$ for $i = k+1, \ldots, n$ by subtracting a multiple of row $k$ from each row $i$:

$$\text{Row } i \leftarrow \text{Row } i - \ell_{ik} \cdot \text{Row } k, \qquad \ell_{ik} = \frac{a_{ik}}{a_{kk}}$$

Let me count the cost of one such row operation.

---

## Counting Operations at Each Stage

### Stage $k$: Eliminating Column $k$

At stage $k$, the submatrix we are working on is $(n - k) \times (n - k + 1)$: there are $n - k$ rows to update, and each row has $n - k + 1$ entries that change (columns $k$ through $n$; column $k$ itself gets zeroed, but we still compute the multiplier).

**For each row $i > k$:**

- 1 division to compute $\ell_{ik} = a_{ik}/a_{kk}$
- For each column $j = k, k+1, \ldots, n$: one multiplication ($\ell_{ik} \cdot a_{kj}$) and one subtraction ($a_{ij} - \ell_{ik} a_{kj}$)
- That is $2(n - k + 1)$ operations per row, plus $1$ division

In practice the entry $a_{ij}$ for $j = k$ is set to zero and discarded (or stored as $\ell_{ik}$ if we want $L$), so we only update columns $k+1$ through $n$: that is $n - k$ useful updates per row, each costing $2$ flops (one multiply, one subtract). Plus $1$ division for the multiplier.

So per row: $1 + 2(n-k)$ flops.

There are $n - k$ rows to update at stage $k$.

**Cost of stage $k$:**

$$C_k = (n-k)\bigl(1 + 2(n-k)\bigr) = (n-k) + 2(n-k)^2$$

---

## Total Cost of Forward Elimination

Sum over all stages $k = 1$ to $n - 1$. Let $m = n - k$, so $m$ runs from $n-1$ down to $1$:

$$C = \sum_{k=1}^{n-1} \left[(n-k) + 2(n-k)^2\right] = \sum_{m=1}^{n-1} m + 2\sum_{m=1}^{n-1} m^2$$

Using the standard identities:

$$\sum_{m=1}^{n-1} m = \frac{(n-1)n}{2}$$

$$\sum_{m=1}^{n-1} m^2 = \frac{(n-1)n(2n-1)}{6}$$

So:

$$C = \frac{n(n-1)}{2} + 2 \cdot \frac{(n-1)n(2n-1)}{6} = \frac{n(n-1)}{2} + \frac{n(n-1)(2n-1)}{3}$$

Factor out $\frac{n(n-1)}{6}$:

$$C = \frac{n(n-1)}{6}\left[3 + 2(2n-1)\right] = \frac{n(n-1)}{6}(4n - 1 + 3 - 3 + 3)$$

Let me just expand directly:

$$C = \frac{n(n-1)}{2} + \frac{n(n-1)(2n-1)}{3}$$

$$= \frac{3n(n-1) + 2n(n-1)(2n-1)}{6}$$

$$= \frac{n(n-1)[3 + 2(2n-1)]}{6}$$

$$= \frac{n(n-1)(4n+1)}{6}$$

For large $n$:

$$C \approx \frac{4n^3}{6} = \frac{2n^3}{3}$$

> **The cost of forward elimination is $\dfrac{2}{3}n^3$ flops to leading order.**

---

## Back Substitution

After forward elimination we have $Ux = c$ (where $c = Pb$ after any row swaps). Back substitution solves this triangular system.

At step $k$ (solving for $x_k$, working from $k = n$ down to $1$):

- We compute $x_k = \frac{1}{u_{kk}}\left(c_k - \sum_{j=k+1}^{n} u_{kj} x_j\right)$
- The sum has $n - k$ multiplications and $n - k$ subtractions: $2(n-k)$ flops
- Plus 1 division

Cost of step $k$: $1 + 2(n - k)$ flops.

Total for back substitution:

$$\sum_{k=1}^{n} [1 + 2(n-k)] = n + 2\sum_{m=0}^{n-1} m = n + 2 \cdot \frac{(n-1)n}{2} = n + n(n-1) = n^2$$

> **Back substitution costs $n^2$ flops.**

---

## The Full Picture

$$\boxed{\text{Total cost of solving } Ax = b: \quad \frac{2}{3}n^3 + n^2 \approx \frac{2}{3}n^3}$$

The $n^2$ term from back substitution is negligible compared to $\frac{2}{3}n^3$ for large $n$. The bottleneck is always forward elimination.

|Step|Flops|
|---|---|
|Forward elimination ($A \to U$)|$\dfrac{2}{3}n^3 + O(n^2)$|
|Back substitution|$n^2$|
|Total|$\dfrac{2}{3}n^3$ (leading term)|

---

## Computing $A^{-1}$: Much More Expensive

If we want $A^{-1}$ explicitly (via Gauss-Jordan), we solve $n$ systems $Ax_i = e_i$ simultaneously, where $e_i$ are the standard basis vectors. This means running elimination once (cost $\frac{2}{3}n^3$) and then doing $n$ back substitutions (cost $n \cdot n^2 = n^3$).

Total cost of computing $A^{-1}$: approximately $\frac{2}{3}n^3 + n^3 = \frac{5}{3}n^3$ — about $\frac{5}{2}$ times more expensive than solving a single system.

> This is why one should **never explicitly compute $A^{-1}$** just to solve $Ax = b$. Factor once, substitute once: $\frac{2}{3}n^3$. Computing the inverse and multiplying: $\frac{5}{3}n^3 + n^2$. The ratio approaches $\frac{5}{2}$ as $n \to \infty$. The inverse is slower by a constant factor, and provides no additional accuracy — in fact, it is often numerically worse.

---

## $LU$ Factorization vs. Solving One System

What if we need to solve $Ax = b_1, Ax = b_2, \ldots, Ax = b_k$ for many right-hand sides?

- **Naive approach:** run full elimination $k$ times: cost $k \cdot \frac{2}{3}n^3$.
- **Smart approach:** factor $A = LU$ once (cost $\frac{2}{3}n^3$), then for each $b_i$ do two triangular solves (forward substitution on $Lc = b_i$, back substitution on $Ux = c$), each costing $n^2$.

Total smart approach: $\frac{2}{3}n^3 + 2kn^2$.

For $k \gg 1$ this is a dramatic saving: the $O(n^3)$ work is paid once, and each additional right-hand side costs only $O(n^2)$.

|Scenario|Cost|
|---|---|
|1 system, full elimination|$\frac{2}{3}n^3$|
|$k$ systems, naive|$\frac{2}{3}kn^3$|
|$k$ systems, factor once|$\frac{2}{3}n^3 + 2kn^2$|
|Explicit $A^{-1}$, then $k$ multiplies|$\frac{5}{3}n^3 + kn^2$|

The $LU$ factorization approach dominates for any $k \ge 1$.

## A Concrete Count: $n = 100$ vs $n = 1000$

$$n = 100: \quad \frac{2}{3}(100)^3 \approx 667{,}000 \text{ flops}$$

$$n = 1000: \quad \frac{2}{3}(1000)^3 \approx 667{,}000{,}000 \text{ flops}$$

Going from $n = 100$ to $n = 1000$ multiplies $n$ by $10$ and the cost by $10^3 = 1000$. At $10^9$ flops per second (a modest laptop), that is under a second for $n = 1000$. For $n = 10{,}000$: about $667$ seconds. This is why numerical methods for large problems exploit sparsity or structure to avoid full $O(n^3)$ elimination.

---

## Where the $\frac{2}{3}$ Comes From Geometrically

There is a nice way to see why the constant is $\frac{2}{3}$ and not $1$ or $\frac{1}{2}$. At stage $k$, we operate on an $(n-k) \times (n-k)$ submatrix. The total work is proportional to:

$$\sum_{k=0}^{n-1} (n-k)^2 = \sum_{j=1}^{n} j^2 = \frac{n(n+1)(2n+1)}{6} \approx \frac{n^3}{3}$$

Each operation in that submatrix costs $2$ flops (multiply + subtract), giving $\frac{2n^3}{3}$.

The factor $\frac{1}{3}$ is the fraction of an $n \times n \times n$ cube occupied by a tetrahedron — the triangular "staircase" of submatrices we work on during elimination. The factor of $2$ is the two flops per entry. That's the geometric picture: we are working on a triangular prism of operations.

---

## Pivoting and Its (Negligible) Cost

Partial pivoting — searching for the largest entry in column $k$ to use as pivot — requires scanning $n - k$ entries at stage $k$. Total cost:

$$\sum_{k=1}^{n-1}(n-k) = \frac{n(n-1)}{2} = O(n^2)$$

This is completely negligible compared to $\frac{2}{3}n^3$. Partial pivoting is essentially free, which is why it is always used in practice.

---

## Banded Matrices: When Structure Saves Everything

If $A$ is **banded** with bandwidth $w$ (meaning $a_{ij} = 0$ whenever $|i - j| > w$), elimination stays within the band. At stage $k$, only $w$ rows need updating, and only $w$ columns of each row change.

Cost of banded elimination: $O(nw^2)$ instead of $O(n^3)$.

For a tridiagonal matrix ($w = 1$): cost is $O(n)$, linear in $n$. This is the reason finite difference methods for PDEs are fast — they produce tridiagonal or banded systems.

## Summary

$$\text{Forward elimination on } n \times n: \quad \frac{2}{3}n^3 \text{ flops}$$

$$\text{Back substitution: } \quad n^2 \text{ flops}$$

$$\text{Full solve } Ax = b: \quad \frac{2}{3}n^3 \text{ flops (leading term)}$$

$$\text{Compute } A^{-1}: \quad \frac{5}{3}n^3 \text{ flops — never do this just to solve a system}$$

The dominant cost is always in forward elimination, it scales as $n^3$, and the constant $\frac{2}{3}$ comes from the triangular structure of the work. Every shortcut in numerical linear algebra — banding, sparsity, iterative methods, randomized algorithms — is an attempt to avoid paying the full $\frac{2}{3}n^3$.

## Things to Think About More

- The count here assumes exact arithmetic. In floating point, rounding errors accumulate. The backward error analysis (Wilkinson) says that elimination with partial pivoting solves a slightly perturbed system exactly — the perturbation is $O(u |A|)$ where $u$ is machine epsilon. The condition number $\kappa(A)$ determines how this affects the solution error. I should connect this to the note on condition numbers when I write it.
- Strassen's algorithm reduces matrix multiplication from $O(n^3)$ to $O(n^{2.807})$ by reusing products cleverly. Since elimination reduces to matrix multiplication in a certain sense, there exist $O(n^{2.807})$ algorithms for solving $Ax = b$. In practice these are never used below $n \approx 1000$ due to large constants and numerical instability.
- For sparse matrices (most entries zero), the fill-in during elimination can destroy sparsity. Reordering rows and columns before elimination — the AMD or nested dissection orderings — minimizes fill-in. This is a serious research area in numerical linear algebra.
