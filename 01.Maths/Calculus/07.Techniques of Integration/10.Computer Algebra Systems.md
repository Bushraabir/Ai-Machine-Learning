## 0. What Is a Computer Algebra System?

A **Computer Algebra System (CAS)** is software that manipulates mathematical expressions **symbolically** — treating $x$ as an unknown, not as a stored floating-point number. It can:

- Expand, factor, and simplify algebraic expressions exactly.
- Differentiate and integrate symbolically.
- Solve equations (algebraic, differential, integral) in closed form.
- Perform matrix algebra, series expansions, limit computations, and much more.

The key word is **exact**: a CAS computes $\int_0^1 x^2,dx = \frac{1}{3}$, not $0.333333\ldots$ A numerical solver computes the latter; a CAS computes the former.

### CAS vs Numerical Computing

|Property|CAS (e.g. Mathematica)|Numerical (e.g. NumPy)|
|---|---|---|
|Representation|Symbolic expression trees|Floating-point arrays|
|Output|Exact closed form|Approximate decimal|
|Speed|Slower (symbolic manipulation)|Much faster (hardware arithmetic)|
|Use case|Deriving formulas, verifying proofs|Large-scale computation, ML training|
|Precision|Infinite (exact)|~15 decimal digits (float64)|
|Failure mode|Cannot simplify / times out|Round-off / overflow error|

> **The central insight:** A CAS is a _proof assistant for computation_. When it returns a formula, that formula is correct within the assumptions given. This makes CAS output reliable for deriving identities, checking integration answers, and symbolic pre-processing before numerical work.

## 2. How a CAS Computes Integrals

Understanding the algorithms makes you a better user — and helps you interpret failures.

### 2.1 The Risch Algorithm

The **Risch algorithm** (1969) is the theoretical foundation of symbolic integration. It is a _decision procedure_ for elementary integration:

> **Theorem (Risch, 1969).** Given an elementary function $f(x)$ (built from rational functions, exponentials, logarithms, and their algebraic combinations), the Risch algorithm either:
> 
> - produces an elementary antiderivative $F(x)$ such that $F'(x) = f(x)$, or
> - **proves** that no such elementary $F$ exists.

This is remarkable: the algorithm does not just fail to find an antiderivative — it _proves impossibility_. This is why `integrate(exp(-x^2), x)` in Mathematica returns `(√π/2) erf(x)` rather than an elementary function — the Risch algorithm has proved no elementary form exists.

**Structure of the algorithm:**

The Risch algorithm works in a **differential field** — a field of functions closed under differentiation. It proceeds recursively:

1. If $f$ is rational, use partial fraction decomposition (§7 of [[Tables of Integrals]]).
2. If $f = P(x, e^{g(x)})$ for some rational $P$ and elementary $g$, use the exponential Risch step.
3. If $f = P(x, \ln g(x))$, use the logarithmic Risch step.
4. For algebraic extensions (square roots, etc.), use the algebraic Risch algorithm (harder; not fully implemented in all systems).

**Full implementation status:**

|System|Risch status|
|---|---|
|Mathematica|Nearly complete (handles most algebraic extensions)|
|Maple|Largely complete|
|SymPy|Partial (rational, exponential, logarithmic; algebraic incomplete)|
|Maxima|Partial|

---

### 2.2 Heuristic Methods in Practice

For many integrands, CAS systems use **heuristics before falling through to Risch**:

- **Pattern matching:** Check if the integrand matches a known table entry (§14 of [[Tables of Integrals]]).
- **$u$-substitution search:** Try common substitutions and check if the result simplifies.
- **Integration by parts automation:** Try IBP with each factor as $u$.
- **Parameter differentiation (Feynman):** Introduce a parameter, differentiate, and integrate back.
- **Algebraic simplification:** Expand, factor, apply trig identities first.

This is exactly the decision tree in [[Integration Strategy]], but applied exhaustively and at machine speed.

---

### 2.3 What Happens When the CAS Fails

A CAS can return:

- A **correct closed form** — the ideal case.
- An answer involving **special functions** ($\operatorname{erf}$, $\operatorname{Si}$, $\operatorname{Li}$, elliptic integrals) — still exact, but not elementary.
- **No answer** — the integral may be non-elementary, or the algorithm timed out, or the algebraic extension handling is incomplete.
- A **wrong answer** — rare but possible when simplification assumptions are violated (e.g., $\sqrt{x^2} = x$ assumed for $x > 0$ when $x$ could be negative).

> **Never trust a CAS answer blindly.** Always check: does the answer differentiate back to the integrand? For definite integrals, does it match a numerical estimate?

## 3. SymPy — The Mathematician's Python CAS

SymPy is the most relevant CAS for a mathematician who also works in ML. It lives in the Python ecosystem alongside NumPy, SciPy, PyTorch, and JAX.

### 3.1 Core Workflow

```python
import sympy as sp

# Declare symbolic variables
x, a, b, n = sp.symbols('x a b n', real=True)

# Define an expression
f = sp.sin(x)**2 * sp.exp(-a * x)

# Differentiate
df = sp.diff(f, x)

# Integrate (indefinite)
F = sp.integrate(f, x)

# Integrate (definite)
I = sp.integrate(sp.exp(-x**2), (x, -sp.oo, sp.oo))
# Returns: sqrt(pi)

# Simplify
s = sp.simplify(sp.sin(x)**2 + sp.cos(x)**2)
# Returns: 1

# Expand and factor
expr = x**3 - x
print(sp.factor(expr))   # x*(x - 1)*(x + 1)
print(sp.expand((x+1)**4))

# Solve equations
sol = sp.solve(x**2 - 3*x + 2, x)
# Returns: [1, 2]

# Limits
L = sp.limit(sp.sin(x)/x, x, 0)
# Returns: 1

# Series expansion (Taylor)
T = sp.series(sp.exp(x), x, 0, 6)
# Returns: 1 + x + x²/2 + x³/6 + x⁴/24 + x⁵/120 + O(x⁶)

# LaTeX output
print(sp.latex(F))
```

### 3.2 Integration Examples in SymPy

```python
import sympy as sp
x, a = sp.symbols('x a', positive=True)

# --- Basic ---
print(sp.integrate(x**3, x))
# x**4/4

print(sp.integrate(1/x, x))
# log(x)   [SymPy uses log for natural log]

# --- Trig ---
print(sp.integrate(sp.sin(x)**4, x))
# 3*x/8 - sin(2*x)/4 + sin(4*x)/32  [after simplification]

# --- Inverse trig form ---
print(sp.integrate(1/(a**2 + x**2), x))
# atan(x/a)/a

# --- Gaussian ---
print(sp.integrate(sp.exp(-x**2), (x, -sp.oo, sp.oo)))
# sqrt(pi)

# --- Non-elementary ---
print(sp.integrate(sp.exp(-x**2), x))
# sqrt(pi)*erf(x)/2   [returns erf, not elementary]

# --- Rationalising ---
print(sp.integrate(1/(1 + sp.sqrt(x)), x))
# 2*sqrt(x) - 2*log(sqrt(x) + 1)

# --- Product: exp × trig ---
print(sp.integrate(sp.exp(a*x)*sp.sin(x), x))
# (a*sin(x) - cos(x))*exp(a*x)/(a**2 + 1)
```

### 3.3 Assumptions Matter

SymPy's answers can depend critically on assumptions:

```python
x = sp.Symbol('x')             # no assumption
x_pos = sp.Symbol('x', positive=True)

print(sp.integrate(1/x, x))             # log(x)  [implicit |x|]
print(sp.sqrt(x**2))                     # sqrt(x**2)  [cannot simplify]
print(sp.sqrt(x_pos**2))                 # x         [correct under x > 0]

# Definite integral that needs assumption
a = sp.Symbol('a')
print(sp.integrate(sp.exp(-a*x**2), (x,-sp.oo,sp.oo)))
# Returns: Piecewise result or requires re(a) > 0
a_pos = sp.Symbol('a', positive=True)
print(sp.integrate(sp.exp(-a_pos*x**2), (x,-sp.oo,sp.oo)))
# sqrt(pi)/sqrt(a)    [correct!]
```

> **Lesson:** Declare assumptions explicitly with `positive=True`, `real=True`, `integer=True`, etc. Unqualified symbols are assumed complex, which restricts what SymPy can simplify.

### 3.4 Solving ODEs

```python
import sympy as sp
x = sp.Symbol('x')
y = sp.Function('y')

# dy/dx = y  →  y = Ce^x
ode = sp.Eq(y(x).diff(x), y(x))
print(sp.dsolve(ode))
# y(x) = C1*exp(x)

# Second-order: y'' + y = 0
ode2 = sp.Eq(y(x).diff(x,2) + y(x), 0)
print(sp.dsolve(ode2))
# y(x) = C1*sin(x) + C2*cos(x)

# With initial conditions
ode3 = sp.Eq(y(x).diff(x), -2*y(x))
sol = sp.dsolve(ode3, ics={y(0): 3})
# y(x) = 3*exp(-2*x)
```

## 5. Mathematica — Power User Reference

### 5.1 Basic Syntax

```mathematica
(* Integration *)
Integrate[x^2 Sin[x], x]
Integrate[Exp[-x^2], {x, -Infinity, Infinity}]   (* √π *)

(* Differentiation *)
D[Sin[x]^2, x]           (* 2 Sin[x] Cos[x] *)
D[f[x,y], x, y]          (* ∂²f/∂x∂y *)

(* Simplification *)
Simplify[Sin[x]^2 + Cos[x]^2]    (* 1 *)
FullSimplify[expr, Assumptions -> {x > 0}]

(* Solve *)
Solve[x^2 - 3x + 2 == 0, x]      (* {{x->1},{x->2}} *)
DSolve[y'[x] == y[x], y[x], x]   (* {{y[x] -> E^x C[1]}} *)

(* Series *)
Series[Exp[x], {x, 0, 5}]
(* 1 + x + x²/2! + x³/3! + x⁴/4! + x⁵/5! + O[x]⁶ *)

(* Numerical evaluation *)
N[Pi, 50]     (* 50 digits of π *)
NIntegrate[Exp[-x^2], {x, 0, 1}]  (* numerical when symbolic fails *)
```

### 5.2 Pattern Matching — The Wolfram Language Core

Mathematica is built on **pattern matching and rewriting rules**. Every computation is the repeated application of transformation rules to expression trees. This is why it is so powerful for symbolic work:

```mathematica
(* Define a rule *)
f[x_] := x^2 + 1

(* Pattern with condition *)
g[x_?Positive] := Sqrt[x]

(* Replace patterns *)
x^2 + y^2 /. x -> 3     (* 9 + y^2 *)

(* Apply transformation rules *)
expr = Sin[2x];
TrigExpand[expr]          (* 2 Sin[x] Cos[x] *)
TrigReduce[2 Sin[x] Cos[x]]  (* Sin[2x] *)
```

Understanding that Mathematica is a **term rewriting system** explains both its power and its occasional surprising simplifications.

## 6. CAS in the ML / Deep Learning Pipeline

This is where the CAS goes from being a calculus tool to being an active component of research infrastructure.

### 6.1 Symbolic Differentiation vs Automatic Differentiation

There are three ways to differentiate a function implemented in code:

|Method|How|Used in|
|---|---|---|
|**Symbolic diff**|Manipulate expression trees (like a CAS)|Mathematica, SymPy|
|**Numerical diff**|Finite differences: $\frac{f(x+h)-f(x)}{h}$|Debugging, gradient checking|
|**Automatic diff (AD)**|Forward/reverse accumulation through computation graph|PyTorch, JAX, TensorFlow|

**Automatic differentiation** is not symbolic (it does not produce a formula) but it is exact (no approximation error). PyTorch's `autograd` applies the chain rule in the reverse pass — it is the chain rule, executed mechanically on the computation graph. This is structurally identical to what a CAS does when applying IBP or $u$-substitution rules, but operating on numerical values rather than symbolic expressions.

```python
# PyTorch: automatic differentiation
import torch
x = torch.tensor(2.0, requires_grad=True)
y = x**3 + 2*x          # y = x³ + 2x
y.backward()             # reverse-mode AD
print(x.grad)            # dy/dx = 3x² + 2 = 14.0  ✓

# SymPy: symbolic differentiation
import sympy as sp
x = sp.Symbol('x')
y = x**3 + 2*x
print(sp.diff(y, x))     # 3*x**2 + 2  (a formula)
```

The key difference: PyTorch gives a number at a specific $x$; SymPy gives a formula valid for all $x$.

---

### 6.2 SymPy + PyTorch: Symbolic Pre-Processing

A common research pattern: use SymPy to derive a formula analytically, then convert it to PyTorch for efficient numerical evaluation.

```python
import sympy as sp
import torch

# Step 1: Derive formula symbolically
x, mu, sigma = sp.symbols('x mu sigma', real=True)
gaussian = sp.exp(-(x - mu)**2 / (2 * sigma**2))
log_gaussian = sp.log(gaussian)
score = sp.diff(log_gaussian, x)   # score function = ∇_x log p(x)
print(score)
# (-mu + x)/sigma**2    ← exact formula

# Step 2: Convert to numerical function
score_fn = sp.lambdify((x, mu, sigma), score, 'torch')

# Step 3: Use in training
x_val = torch.randn(100)
score_val = score_fn(x_val, 0.0, 1.0)   # exact score at 100 points
```

This pattern is used in:

- **Score-based diffusion models**: deriving the score $\nabla_x \log p_t(x)$ analytically.
- **Neural ODEs**: deriving the vector field equations symbolically before implementing the solver.
- **Physics-informed neural networks (PINNs)**: deriving PDE residuals symbolically, then evaluating them on collocation points.

### 6.3 JAX + SymPy: Symbolic-Numerical Bridge

JAX supports both automatic differentiation and just-in-time (JIT) compilation. SymPy can generate JAX-compatible code:

```python
import sympy as sp
x = sp.Symbol('x')
f = sp.sin(x)**2 * sp.exp(-x**2)

# Generate JAX code from SymPy expression
f_jax = sp.lambdify(x, f, modules='jax.numpy')

import jax
import jax.numpy as jnp
grad_f = jax.grad(f_jax)      # AD through the JAX function
print(grad_f(1.0))             # numerical value of f'(1)
```

### 6.4 CAS for Kernel Derivation in Gaussian Processes

Gaussian Process (GP) regression requires computing kernel matrices and their derivatives. A CAS is used to:

1. Derive the kernel function $k(x, x')$ symbolically.
2. Compute $\frac{\partial k}{\partial \theta}$ for hyperparameter gradients.
3. Simplify expressions to reduce numerical computation cost.

```python
import sympy as sp

x1, x2, l, sigma_f = sp.symbols('x1 x2 l sigma_f', positive=True)

# RBF (Squared Exponential) kernel
k = sigma_f**2 * sp.exp(-(x1-x2)**2 / (2*l**2))

# Derivative w.r.t. length scale l (needed for GP hyperparameter optimisation)
dk_dl = sp.diff(k, l)
print(sp.simplify(dk_dl))
# sigma_f**2*(x1 - x2)**2*exp(-(x1-x2)**2/(2*l**2)) / l**3
```

### 6.5 Verifying Backprop with SymPy

A classic debugging technique: compute the gradient symbolically in SymPy and compare to the PyTorch `autograd` result.

```python
import sympy as sp
import torch

# Symbolic ground truth
x_s = sp.Symbol('x')
f_s = sp.log(1 + sp.exp(x_s))   # softplus
df_s = sp.diff(f_s, x_s)
print(df_s)   # exp(x)/(exp(x) + 1) = sigmoid(x)

# PyTorch computation
x_t = torch.tensor(0.5, requires_grad=True)
f_t = torch.log(1 + torch.exp(x_t))
f_t.backward()
print(x_t.grad)   # 0.6225...

# Symbolic evaluation at x=0.5
print(float(df_s.subs(x_s, 0.5)))  # 0.6225...  ✓ match
```

### 6.6 Physics-Informed Neural Networks (PINNs)

PINNs solve PDEs by minimising a loss that penalises PDE residuals on collocation points. A CAS pre-computes the residual terms:

```python
import sympy as sp

x, t = sp.symbols('x t', real=True)
u = sp.Function('u')

# Heat equation: u_t = alpha * u_xx
alpha = sp.Symbol('alpha', positive=True)
heat_eq = u(x,t).diff(t) - alpha * u(x,t).diff(x,x)

# Verify that u = exp(-alpha*t)*sin(x) is an exact solution
u_exact = sp.exp(-alpha*t) * sp.sin(x)
residual = heat_eq.subs(u(x,t), u_exact)
residual_simplified = sp.simplify(residual.doit())
print(residual_simplified)   # 0  ← exact solution confirmed
```

## 7. Limitations and Pitfalls

### 7.1 Branch Cuts and Complex Logarithms

A CAS works over $\mathbb{C}$ by default. The identity $\ln(ab) = \ln a + \ln b$ holds only for $a, b > 0$; in general $\ln(ab) = \ln a + \ln b + 2\pi i k$ for some integer $k$.

```mathematica
(* Mathematica *)
Log[(-1)*(-1)]       (* 0 + 2πi  — not 0! *)
Simplify[Log[x^2]]   (* Log[x^2], not 2 Log[x] — needs x > 0 assumption *)
```

**Lesson:** Always declare assumptions. Without them, a CAS must be conservative.

### 7.2 Conditional Answers

```python
# SymPy
import sympy as sp
a, x = sp.symbols('a x')
print(sp.integrate(x**a, x))
# Piecewise((x**(a + 1)/(a + 1), Ne(a, -1)), (log(x), True))
# Correct! SymPy is handling the n = -1 case automatically.
```

### 7.3 Equivalent but Different Forms

CAS systems do not always produce the same form as a textbook. For example: $$\int \frac{dx}{\sqrt{x^2+1}}$$ may be returned as $\ln(x + \sqrt{x^2+1})$, or $\sinh^{-1}(x)$, or $\operatorname{arcsinh}(x)$ — all identical (§8 of [[Tables of Integrals]]). To verify two forms are equal:

```python
F1 = sp.log(x + sp.sqrt(x**2 + 1))
F2 = sp.asinh(x)
print(sp.simplify(F1 - F2))   # 0 ← same function
```

### 7.4 Definite Integrals and Singularities

```python
# This is an improper integral — converges, but SymPy must handle it carefully
print(sp.integrate(1/x, (x, -1, 1)))
# SymPy may return: zoo (complex infinity) — correct, the integral diverges (not Cauchy PV)
# Principal value:
print(sp.integrate(1/x, (x, -1, 1), conds='none'))   # 0 (Cauchy PV)
```

### 7.5 CAS Is Not Infallible

Known issues with SymPy:

- Incomplete algebraic Risch: some integrals with $\sqrt{\text{polynomial}}$ in the integrand are returned unevaluated even though an elementary antiderivative exists.
- Simplification: `simplify` is heuristic; `trigsimp`, `radsimp`, `expand`, `factor` are more targeted and often better.
- Speed: SymPy is pure Python — large matrix operations or high-degree polynomials can be slow.

## 8. Workflow Recommendations

### For Checking Integration Results

1. Compute the integral by hand using [[Integration Strategy]].
2. Check with SymPy: `sp.integrate(f, x)`.
3. Verify: `sp.diff(result, x) - f` should simplify to $0$.
4. For definite integrals: compare to `sp.N(sp.integrate(f, (x,a,b)))` or `scipy.integrate.quad`.

### For Exam Preparation

- Use Wolfram Alpha to **check specific integrals**, not to learn the method.
- After solving a problem, enter it into WA and compare — if the forms differ, prove they are equal analytically.
- Use SymPy's step-by-step hint by inspecting the `manualintegrate` module:

```python
from sympy.integrals.manualintegrate import manualintegrate, integral_steps
steps = integral_steps(sp.sin(x)**2 * sp.cos(x), x)
print(steps)   # prints the substitution u = sin(x) strategy
```

### For Research (ML Pipeline)

```
Derive formula symbolically (SymPy / Mathematica)
    ↓
Simplify / verify (sp.simplify, sp.diff check)
    ↓
Convert to fast numerical code (sp.lambdify → NumPy/JAX)
    ↓
Train / evaluate at scale (PyTorch / JAX)
    ↓
Debug gradients (compare autograd vs sp.diff at test points)
```

## 9. Visualisation — Expression Trees

A CAS represents every expression as a **tree of operations**. Understanding this explains why CAS simplification is hard (it is a tree rewriting problem, which is NP-hard in general).

```
Expression: x² sin(x) + e^x

         +
        / \
       *   e^x
      / \
    x²  sin
         |
         x

Tree rewriting rules:
  diff(+, x) → diff(left, x) + diff(right, x)
  diff(*, x) → IBP or product rule
  diff(sin, x) → cos
  diff(e^x, x) → e^x
```

Integration is the **inverse tree search problem**: given $f$, find $F$ such that $\text{diff}(F) = f$. This is underdetermined (many possible tree structures for $F$), which is why integration is harder than differentiation algorithmically.

```
SymPy integration pipeline:

  f(x)
   │
   ├─ manualintegrate: try known patterns
   │
   ├─ heurisch: heuristic Risch (most integrands)
   │
   ├─ risch_integrate: full Risch for elementary functions
   │
   └─ meijerg: Meijer G-functions for special functions
```

### SymPy Quick Reference

```python
import sympy as sp

x, y, a, b, n = sp.symbols('x y a b n')
```
