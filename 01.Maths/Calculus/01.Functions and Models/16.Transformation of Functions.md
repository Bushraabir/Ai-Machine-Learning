## 0. Motivation — Why Transformations?

Every function you will ever encounter in analysis, geometry, signal processing, or machine learning is, at its core, a **transformed version of something simpler**. The Gaussian $e^{-(x-\mu)^2/2\sigma^2}$ is a shifted, scaled, and compressed version of $e^{-x^2}$. The ReLU applied to a linear layer is a shifted threshold function. A convolutional filter is a translated version of a kernel.

The central insight is this: **you do not need to relearn a function from scratch when it is transformed. You need to understand how the transformation acts on the graph.**

There are exactly four elementary transformations — _translations_, _reflections_, _stretches/compressions_ — and every sophisticated transformation in analysis is a composition of these four. Learn them once, deeply, and you have a universal toolkit.

---

## 1. The Setup — Precise Language

Let $f: D \subseteq \mathbb{R} \to \mathbb{R}$ be a function with graph

$$\Gamma_f = {(x, f(x)) : x \in D} \subset \mathbb{R}^2.$$

A **transformation** is a map $T: \mathbb{R}^2 \to \mathbb{R}^2$ that sends the graph $\Gamma_f$ to a new graph $\Gamma_g$. The question is always: **given $f$, what is $g$?**

We classify transformations according to whether they act on the **input** (horizontal, affects $x$) or the **output** (vertical, affects $y = f(x)$). This distinction is the single most important conceptual divide.

$$g(x) = \underbrace{a \cdot f(\underbrace{bx + c}_{\text{input transformation}}) + d}_{\text{output transformation}}$$

Every transformation we study is a special case of this general form.

---

## 2. Vertical Translations — Output Shifts

### 2.1 Definition

$$g(x) = f(x) + d, \qquad d \in \mathbb{R}$$

The graph of $g$ is the graph of $f$ shifted **vertically by $d$ units**.

- $d > 0$: shift **up**.
- $d < 0$: shift **down**.

### 2.2 Why? — Precise Argument

For each $x$, the output value changes from $f(x)$ to $f(x) + d$. The $x$-coordinate of every point is unchanged; the $y$-coordinate increases by $d$. The transformation on $\mathbb{R}^2$ is $(x, y) \mapsto (x, y+d)$ — a pure vertical translation.

### 2.3 Effect on Domain and Range

- **Domain:** unchanged.
- **Range:** shifted by $d$. If $\text{ran}(f) = [m, M]$, then $\text{ran}(g) = [m+d, M+d]$.

### 2.4 Example

Let $f(x) = x^2$. Then $g(x) = x^2 + 3$ is the parabola shifted up 3 units. The vertex moves from $(0,0)$ to $(0,3)$.

```
        g(x) = x² + 3
y       │    *
        │   * *
   3────│──*───*──
        │ *     *
   0────┼─────────── x
        f(x) = x²
```

## 3. Horizontal Translations — Input Shifts

### 3.1 Definition

$$g(x) = f(x - c), \qquad c \in \mathbb{R}$$

The graph of $g$ is the graph of $f$ shifted **horizontally by $c$ units**.

- $c > 0$: shift **right**.
- $c < 0$: shift **left**.

### 3.2 The Crucial Subtlety — Sign Reversal

This is where students stumble. The intuition says: replacing $x$ with $x - c$ should shift _left_ (subtracting makes things smaller). It does the opposite.

**Rigorous explanation.** The graph of $f$ passes through $(x_0, f(x_0))$. The graph of $g(x) = f(x-c)$ passes through the point where $g(x) = f(x_0)$, i.e., where $x - c = x_0$, i.e., where $x = x_0 + c$. So the point $(x_0, f(x_0))$ on $\Gamma_f$ corresponds to the point $(x_0 + c, f(x_0))$ on $\Gamma_g$. The shift is **right by $c$**.

> **Mnemonic.** Think of it as a _delay_: $f(x-c)$ "waits" until $x = c$ before it starts behaving like $f$ does at $x=0$. It is delayed to the right.

### 3.3 Example

$f(x) = \sqrt{x}$, domain $[0, \infty)$.

- $g(x) = \sqrt{x-2}$: domain $[2,\infty)$, shifted right 2.
- $h(x) = \sqrt{x+3}$: domain $[-3,\infty)$, shifted left 3.

```
h(x)=√(x+3)    f(x)=√x    g(x)=√(x-2)
     ↗               ↗              ↗
─────────────────────────────────────── x
    -3           0         2
```

## 4. Vertical Stretches and Compressions

### 4.1 Definition

$$g(x) = a \cdot f(x), \qquad a > 0$$

- $a > 1$: **vertical stretch** by factor $a$ (graph pulled away from $x$-axis).
- $0 < a < 1$: **vertical compression** by factor $a$ (graph pushed toward $x$-axis).

### 4.2 Precise Geometry

Every $y$-coordinate is multiplied by $a$. The $x$-intercepts (where $f(x) = 0$) are **fixed points** of this transformation — they do not move. The transformation on $\mathbb{R}^2$ is $(x,y) \mapsto (x, ay)$.

### 4.3 Example

$f(x) = \sin x$.

- $g(x) = 3\sin x$: amplitude 3, stretched vertically.
- $h(x) = \tfrac{1}{2}\sin x$: amplitude $\tfrac{1}{2}$, compressed.

```
y
 3│   *               g(x) = 3sin(x)
  │  * *
 1│ *   *             f(x) = sin(x)
  │*     *
──┼────────────────────────────────── x
  │       *     *
-1│        *   *
  │         * *
-3│          *
```

## 5. Horizontal Stretches and Compressions

### 5.1 Definition

$$g(x) = f(bx), \qquad b > 0$$

- $b > 1$: **horizontal compression** by factor $1/b$ (graph squeezed toward $y$-axis).
- $0 < b < 1$: **horizontal stretch** by factor $1/b$ (graph pulled away from $y$-axis).

### 5.2 The Sign Paradox — Again

The horizontal case is always "backwards" from vertical:

- Multiplying $x$ by $b > 1$ **compresses** horizontally (counterintuitive but correct).

**Proof.** The feature of $f$ at $x = x_0$ now occurs when $bx = x_0$, i.e., at $x = x_0/b$. Since $b > 1$, $x_0/b < x_0$ — the feature moved closer to the origin. Compression.

### 5.3 Example — Sinusoidal Period

$f(x) = \sin x$ has period $2\pi$.

- $g(x) = \sin(2x)$: period $2\pi/2 = \pi$ — compressed.
- $h(x) = \sin(x/2)$: period $2\pi/(1/2) = 4\pi$ — stretched.

**General rule:** If $f$ has period $T$, then $f(bx)$ has period $T/b$.

```
f(x) = sin(x):     one full wave over [0, 2π]
g(x) = sin(2x):    two full waves over [0, 2π]  ← compressed
h(x) = sin(x/2):   half wave over [0, 2π]       ← stretched
```

## 6. Reflections

### 6.1 Reflection in the $x$-axis (Vertical Flip)

$$g(x) = -f(x)$$

Every $y$-coordinate is negated. The graph flips over the $x$-axis. The transformation on $\mathbb{R}^2$ is $(x,y) \mapsto (x,-y)$.

**Fixed points:** the $x$-intercepts of $f$.

### 6.2 Reflection in the $y$-axis (Horizontal Flip)

$$g(x) = f(-x)$$

Every $x$-coordinate is negated. The graph flips over the $y$-axis. The transformation on $\mathbb{R}^2$ is $(x,y) \mapsto (-x,y)$.

**Fixed points:** points on the $y$-axis (where $x=0$).

**Effect on domain:** if $D = [a,b]$, the new domain is $[-b,-a]$.

### 6.3 Reflection in Both Axes

$$g(x) = -f(-x)$$

This is equivalent to a $180°$ rotation about the origin. Note: if $f$ is an **odd function** ($f(-x) = -f(x)$), then $g = f$ — the graph is symmetric under this rotation.

### 6.4 Example

$f(x) = \sqrt{x}$, domain $[0,\infty)$.

- $-f(x) = -\sqrt{x}$: reflected over $x$-axis, domain unchanged.
- $f(-x) = \sqrt{-x}$: reflected over $y$-axis, domain $(-\infty, 0]$.

```
f(-x)=√(-x)   │    f(x)=√x
     ↗         │         ↗
───────────────┼─────────────── x
               │
    -f(-x)     │   -f(x)
     ↘         │        ↘
```

## 7. The General Form — Composing All Four

$$\boxed{g(x) = a \cdot f(b(x - h)) + k}$$

where:

|Parameter|Type|Effect|
|---|---|---|
|$h$|Horizontal translation|Shift right by $h$|
|$k$|Vertical translation|Shift up by $k$|
|$\|b\|$|Horizontal scale|Compress by $\|b\|$ if $\|b\|>1$, stretch if $\|b\|<1$|
|$\|a\|$|Vertical scale|Stretch by $\|a\|$ if $\|a\|>1$, compress if $\|a\|<1$|
|$a < 0$|Reflection|Flip over $x$-axis|
|$b < 0$|Reflection|Flip over $y$-axis|

### 7.1 Order of Operations Matters

To graph $g(x) = a \cdot f(b(x-h)) + k$ from $f$, apply transformations in this order:

1. **Horizontal shift**: replace $x$ with $x - h$.
2. **Horizontal scale/reflect**: replace $x$ with $bx$.
3. **Vertical scale/reflect**: multiply outputs by $a$.
4. **Vertical shift**: add $k$.

> **Warning.** Many textbooks write $f(bx + c)$ rather than $f(b(x-h))$. These are equivalent with $h = -c/b$, but the factored form $b(x-h)$ makes the shift explicit. Always factor out $b$ before reading off the horizontal shift.

### 7.2 Example — Full Analysis

Analyze $g(x) = 2\sin(3x - \pi) + 1$.

**Step 1: Factor.** $g(x) = 2\sin!\left(3!\left(x - \tfrac{\pi}{3}\right)\right) + 1$.

Now read off:

- $f(x) = \sin x$ (base function)
- $b = 3$: horizontal compression, period $= 2\pi/3$
- $h = \pi/3$: shift right by $\pi/3$
- $a = 2$: vertical stretch, amplitude $= 2$
- $k = 1$: shift up by $1$

**Key features of $g$:**

- Midline: $y = 1$
- Amplitude: $2$ (oscillates between $-1$ and $3$)
- Period: $2\pi/3$
- Phase shift: right $\pi/3$

```
y
 3│    *
  │   * *
 1│──*──────*──────── midline y=1
  │ *        *
-1│*          *
  │
  └──────────────────── x
     π/3   π/3+π/3
     (peak at x = π/3 + π/6 = π/2)
```

## 8. Absolute Value Transformations

### 8.1 $g(x) = |f(x)|$ — Reflect negative parts upward

The portions of the graph **below** the $x$-axis are reflected up. Points where $f(x) \geq 0$ are unchanged.

**Formally:** $|f(x)| = f(x)$ when $f(x) \geq 0$, and $|f(x)| = -f(x)$ when $f(x) < 0$.

**Example.** $f(x) = \sin x$. Then $|\sin x|$ takes all the negative half-arches and flips them up — the result has period $\pi$ instead of $2\pi$.

### 8.2 $g(x) = f(|x|)$ — Mirror right half onto left

The graph for $x \geq 0$ is kept; the graph for $x < 0$ is replaced by a mirror image of the right half.

**Formally:** $f(|x|) = f(x)$ for $x \geq 0$, and $f(|x|) = f(-x)$ for $x < 0$. The result is always an **even function**: $g(-x) = f(|-x|) = f(|x|) = g(x)$.

**Example.** $f(x) = e^x$. Then $f(|x|) = e^{|x|}$, which is V-shaped on a log scale and symmetric about the $y$-axis.

## 9. Even and Odd Functions — Symmetry as Transformation

### 9.1 Even Functions

$f(-x) = f(x)$ for all $x$ in the domain.

**Geometric meaning:** the graph is symmetric about the $y$-axis. Applying the transformation $g(x) = f(-x)$ gives back $f$ — the graph is invariant under reflection in the $y$-axis.

**Examples:** $x^2$, $\cos x$, $|x|$, $x^4 - 3x^2$.

### 9.2 Odd Functions

$f(-x) = -f(x)$ for all $x$.

**Geometric meaning:** the graph is symmetric about the origin (invariant under $180°$ rotation). Applying $g(x) = -f(-x)$ gives back $f$.

**Examples:** $x$, $x^3$, $\sin x$, $\tan x$.

### 9.3 Decomposition Theorem

**Every function** $f$ defined on a symmetric domain can be uniquely written as:

$$f(x) = \underbrace{\frac{f(x) + f(-x)}{2}}_{\text{even part}} + \underbrace{\frac{f(x) - f(-x)}{2}}_{\text{odd part}}$$

**Proof.** Let $E(x) = \frac{f(x)+f(-x)}{2}$ and $O(x) = \frac{f(x)-f(-x)}{2}$. Then $E(-x) = E(x)$ ✓, $O(-x) = -O(x)$ ✓, and $E(x)+O(x) = f(x)$ ✓. Uniqueness follows from the fact that the only function that is both even and odd is the zero function.

> **This decomposition matters in signal processing.** Every signal can be written as the sum of its even and odd parts — this is the real-variable analog of writing a complex exponential as cosine plus $i$ times sine.

## 10. Inverse Functions as Reflections

The graph of $f^{-1}$ (when it exists) is obtained by **reflecting $\Gamma_f$ over the line $y = x$**.

**Proof.** $(a, b) \in \Gamma_f \iff f(a) = b \iff a = f^{-1}(b) \iff (b, a) \in \Gamma_{f^{-1}}$. The map $(a,b) \mapsto (b,a)$ is exactly reflection over $y = x$. $\blacksquare$

```
y = x
  ↗
│   * (b,a) on f⁻¹
│  /
│ * (a,b) on f
│/
└──────────────────── x
```

This means: **a function and its inverse are mirror images across the diagonal**. Visually: if $f$ is increasing and concave up, then $f^{-1}$ is increasing and concave down (since curvature reflects over the line $y=x$).

**Example.** $f(x) = e^x$ and $f^{-1}(x) = \ln x$ are reflections of each other over $y = x$. Both are increasing; $e^x$ is convex (cups upward) and $\ln x$ is concave (cups downward).

## 11. Composition of Transformations — The Formal View

It is worth recording the transformations as maps on $\mathbb{R}^2$ so we can compose them algebraically.

|Transformation|Map on $\mathbb{R}^2$|
|---|---|
|Vertical shift up $k$|$(x,y) \mapsto (x, y+k)$|
|Horizontal shift right $h$|$(x,y) \mapsto (x+h, y)$|
|Vertical stretch by $a > 0$|$(x,y) \mapsto (x, ay)$|
|Horizontal compress by $b > 0$|$(x,y) \mapsto (x/b, y)$|
|Reflect over $x$-axis|$(x,y) \mapsto (x,-y)$|
|Reflect over $y$-axis|$(x,y) \mapsto (-x,y)$|
|Reflect over $y=x$|$(x,y) \mapsto (y,x)$|

These are all **affine maps** (linear maps plus translation). Their composition is again affine. This is the language of **linear algebra** — and it foreshadows why neural networks apply affine transformations before nonlinearities.

## 12. Connections to AI, Machine Learning, and Deep Learning

### 12.1 Neural Networks as Function Compositions

A single layer of a neural network computes:

$$\mathbf{h} = \sigma(W\mathbf{x} + \mathbf{b})$$

Break this down:

- $W\mathbf{x}$: a **linear transformation** — rotates and stretches the input space.
- $+ \mathbf{b}$: a **translation** — shifts the origin.
- $\sigma(\cdot)$: a **nonlinear transformation** applied elementwise (ReLU, sigmoid, tanh).

The whole network is a composition of affine transformations and nonlinearities. Understanding function transformations is literally understanding what a neural network computes at each layer.

### 12.2 Activation Functions as Shifted/Scaled Versions of Each Other

Consider the logistic sigmoid: $\sigma(x) = \frac{1}{1+e^{-x}}$.

Tanh is a rescaled, shifted sigmoid: $$\tanh(x) = 2\sigma(2x) - 1$$

This is a **vertical stretch by 2**, **horizontal compression by 2**, and **vertical shift down by 1** applied to $\sigma$. The two functions have the same shape; only their range differs ($[0,1]$ vs $[-1,1]$). In practice, tanh is preferred in some contexts precisely because it is **zero-centered** — a consequence of the vertical shift.

### 12.3 Data Normalization — Affine Transformation of Inputs

**Standardization** (z-score normalization) transforms data $x$ to:

$$\tilde{x} = \frac{x - \mu}{\sigma}$$

This is a **horizontal shift by $\mu$** followed by a **horizontal compression by $\sigma$** — but since $x$ is the input, in function terms:

$$f!\left(\frac{x-\mu}{\sigma}\right)$$

is $f$ shifted right by $\mu$ and stretched horizontally by $\sigma$. Standardization ensures the input distribution has mean 0 and variance 1, making gradient descent converge faster (the loss landscape is better conditioned). This is entirely a consequence of the transformation of functions.

### 12.4 Batch Normalization — Adaptive Affine Rescaling

Batch normalization in deep learning normalizes the activations of each layer and then applies a **learned affine transformation**:

$$\hat{z} = \gamma \cdot \frac{z - \mu_B}{\sigma_B} + \beta$$

- $\frac{z-\mu_B}{\sigma_B}$: normalize (shift and compress).
- $\gamma$: **learned vertical stretch**.
- $\beta$: **learned vertical shift**.

The network learns the optimal affine transformation for each layer's activations. The parameters $\gamma$ and $\beta$ are exactly the $a$ and $k$ from $g(x) = a \cdot f(x) + k$.

### 12.5 The ReLU and Its Translations

The Rectified Linear Unit is $\text{ReLU}(x) = \max(0, x)$. Notice:

$$\text{ReLU}(x - c) = \max(0, x-c)$$

This is ReLU **shifted right by $c$** — it "activates" at $x = c$ instead of $x = 0$. The **leaky ReLU** $\max(\alpha x, x)$ is a piecewise rescaling (vertical compression for $x < 0$). The **ELU** approximates a smooth version. All are transformations of the same underlying threshold idea.

### 12.6 Convolutional Neural Networks — Translation as a Core Operation

A **convolution** is:

$$(f * g)(x) = \int_{-\infty}^\infty f(t), g(x - t), dt$$

The term $g(x-t)$ is exactly $g$ **translated by $t$**. A convolutional neural network learns a filter $g$ and applies it at every translation $t$ — the fundamental operation is horizontal translation of a function. The property that makes convolutions powerful — **translation equivariance** — is precisely the statement that shifting the input by $c$ shifts the output by $c$:

$$\text{Conv}[f(x-c)](https://claude.ai/chat/x) = (f*g)(x-c).$$

This is not a coincidence. It is the algebraic consequence of the convolution using $g(x-t)$ (a translate of $g$) rather than $g(x+t)$.

### 12.7 Positional Encoding in Transformers

In the Transformer architecture, positional encodings use:

$$PE(pos, 2i) = \sin!\left(\frac{pos}{10000^{2i/d}}\right), \quad PE(pos, 2i+1) = \cos!\left(\frac{pos}{10000^{2i/d}}\right)$$

Each encoding is a sinusoid with a **different horizontal compression** ($b = 1/10000^{2i/d}$). Different dimensions of the encoding correspond to different frequencies. The model can learn to identify position by the _phase_ — which is exactly a horizontal translation of the sinusoid.

## 13. Worked Examples

### Example 13.1 — Identify All Transformations

Given $g(x) = -3\sqrt{2x + 6} + 1$, express relative to $f(x) = \sqrt{x}$ and describe all transformations.

**Step 1: Factor** the inside. $$g(x) = -3\sqrt{2(x+3)} + 1$$

**Step 2: Read off parameters** from $g(x) = a \cdot f(b(x-h)) + k$:

- $a = -3$: vertical stretch by 3, reflect over $x$-axis
- $b = 2$: horizontal compression by $1/2$
- $h = -3$: horizontal shift **left** by 3
- $k = 1$: vertical shift up 1

**Step 3: Key points.** $f$ has the point $(0,0)$. Applying transformations:

- Shift left 3: $(-3, 0)$
- Compress horizontally by $1/2$: $(-3/2, 0)$... wait — the shift happens _after_ factoring, so apply in order: first compress (applied to the factored form), then shift.

Actually, using $b(x-h)$: point $(x_0, f(x_0))$ on $f$ maps to $(x_0/b + h,; a\cdot f(x_0) + k)$.

|Point on $f$|Point on $g$|
|---|---|
|$(0, 0)$|$(0/2 + (-3),; -3 \cdot 0 + 1) = (-3, 1)$|
|$(1, 1)$|$(1/2 - 3,; -3 + 1) = (-5/2, -2)$|
|$(4, 2)$|$(2 - 3,; -6+1) = (-1, -5)$|

### Example 13.2 — From Graph to Formula

A function $g$ has the graph of $f(x) = x^2$ shifted left 2, compressed vertically by $1/3$, reflected over the $x$-axis, and shifted down 4. Find $g(x)$.

Apply in order:

1. Left 2: $f(x+2) = (x+2)^2$
2. Compress vertically by $1/3$: $\frac{1}{3}(x+2)^2$
3. Reflect over $x$-axis: $-\frac{1}{3}(x+2)^2$
4. Shift down 4: $g(x) = -\frac{1}{3}(x+2)^2 - 4$

### Example 13.3 — Even/Odd Decomposition

Decompose $f(x) = e^x$ into even and odd parts.

$$E(x) = \frac{e^x + e^{-x}}{2} = \cosh x$$ $$O(x) = \frac{e^x - e^{-x}}{2} = \sinh x$$

So $e^x = \cosh x + \sinh x$. This is the **hyperbolic function decomposition** — and it is exactly the real-axis version of Euler's formula $e^{ix} = \cos x + i\sin x$.

### Example 13.4 — Sinusoidal Model (Applied)

Model the temperature $T$ (°C) in a city: it oscillates between 10°C and 30°C with a period of 12 hours, reaching its maximum at $t=2$ pm (taken as $t=14$).

**Parameters:**

- Amplitude $= (30-10)/2 = 10$
- Midline $= (30+10)/2 = 20$
- Period $= 12 \Rightarrow b = 2\pi/12 = \pi/6$
- Maximum at $t = 14$: use cosine (max at $t=0$ for cosine), shift right 14.

$$T(t) = 10\cos!\left(\frac{\pi}{6}(t - 14)\right) + 20$$

This is $f(t) = \cos t$ with $b = \pi/6$ (stretch), $h = 14$ (shift right), $a=10$ (stretch), $k=20$ (shift up).

## 16. Personal Marginalia

> _The horizontal/vertical asymmetry is deep, not superficial._ In the formula $g(x) = f(bx+c)$, the input transformation $x \mapsto bx+c$ acts _before_ $f$ sees it. You are changing what you _feed_ to $f$, not what $f$ _produces_. So naturally, a large $b$ means you are feeding $f$ inputs that are "already large" — $f$ completes its oscillation sooner, hence compression. Vertical changes act _after_ $f$ finishes — they modify the output directly.

> _Every deep learning architecture is a transformation story._ ResNets add $f(x) + x$ — the identity is a skip connection, and the whole block is a perturbation of the identity transformation. Attention is a dynamic weighting of translated/scaled value vectors. Normalizing flows are stacks of invertible transformations (diffeomorphisms) applied to a simple base distribution. If you understand what transformation does to a graph, you are halfway to understanding what an architecture does to a distribution.

> _The even/odd decomposition is not a curiosity._ It is the $\mathbb{R}$-version of Fourier analysis. Every $L^2$ function on $[-L,L]$ decomposes into cosines (even) and sines (odd) — a decomposition into symmetric and antisymmetric pieces. The same logic, generalized.
