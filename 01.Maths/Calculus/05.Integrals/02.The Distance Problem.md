## 0. Why This Problem Matters

Before limits, before derivatives, before any of the formal machinery — there was a question so elementary it seems almost embarrassing:

> **If an object moves with non-constant velocity, how far does it travel?**

This is the **distance problem**. It sounds like a physics homework exercise. It is, in fact, the conceptual seed of integral calculus — and, as we will see, the philosophical ancestor of optimization in machine learning.

---

## 1. The Easy Case: Constant Velocity

If velocity is constant, say $v = 60$ km/h for $t = 3$ hours, then:

$$d = v \cdot t = 60 \times 3 = 180 \text{ km}$$

Geometrically, this is the **area of a rectangle** under the horizontal line $v = 60$ on a velocity-time graph.

```
v
|
60|---------------------
  |                    |
  |                    |
  +---------------------> t
  0                    3
```

Area $= 60 \times 3 = 180$. Distance $=$ Area under the $v$-$t$ curve. File this away — it is everything.

---

## 2. The Hard Case: Variable Velocity

Now suppose $v(t) = t^2$ m/s. What is the distance traveled from $t = 0$ to $t = 3$?

We cannot simply multiply, because $v$ is changing at every instant.

### The Key Idea: Approximate, then Refine

**Partition** the interval $[0, 3]$ into $n$ subintervals of width $\Delta t = 3/n$.

On each tiny subinterval $[t_{i-1}, t_i]$, **pretend** the velocity is constant — equal to $v(t_{i-1}) = t_{i-1}^2$ (left endpoint). Then the distance on that piece is approximately:

$$\Delta d_i \approx v(t_{i-1}) \cdot \Delta t = t_{i-1}^2 \cdot \Delta t$$

The **total approximate distance** is a **Riemann sum**:

$$D_n = \sum_{i=1}^{n} v(t_{i-1}) \cdot \Delta t = \sum_{i=1}^{n} t_{i-1}^2 \cdot \Delta t$$

### Worked Example: $n = 3$

With $\Delta t = 1$, sample points $t = 0, 1, 2$:

|Subinterval|$t_{i-1}$|$v(t_{i-1}) = t_{i-1}^2$|$\Delta d_i = v \cdot 1$|
|---|---|---|---|
|$[0,1]$|$0$|$0$|$0$|
|$[1,2]$|$1$|$1$|$1$|
|$[2,3]$|$2$|$4$|$4$|

$$D_3 = 0 + 1 + 4 = 5 \text{ m}$$

### Worked Example: $n = 6$

With $\Delta t = 0.5$, sample points $0, 0.5, 1, 1.5, 2, 2.5$:

$$D_6 = (0^2 + 0.5^2 + 1^2 + 1.5^2 + 2^2 + 2.5^2)(0.5) = (0 + 0.25 + 1 + 2.25 + 4 + 6.25)(0.5) = 13.75 \times 0.5 = 6.875$$

The true answer is $\int_0^3 t^2, dt = \left[\frac{t^3}{3}\right]_0^3 = 9$.

|$n$|Approximation|Error|
|---|---|---|
|3|5.000|4.000|
|6|6.875|2.125|
|12|7.875|1.125|
|100|8.910|0.090|
|∞|**9.000**|**0**|

The approximations converge. The error vanishes as $n \to \infty$.

Each rectangle is an **underestimate** here because $v(t) = t^2$ is increasing. If we used **right endpoints**, we'd overestimate. The true value is squeezed between:

$$L_n \leq \int_0^3 t^2, dt \leq R_n$$

and both $L_n, R_n \to 9$ as $n \to \infty$.

## 4. The Rigorous Limit

**Definition (Definite Integral — Stewart §5.2).** Let $f$ be defined on $[a,b]$. Partition $[a,b]$ into $n$ subintervals of equal width $\Delta x = (b-a)/n$. Choose sample points $x_i^*$ in $[x_{i-1}, x_i]$. Then:

$$\int_a^b f(x), dx = \lim_{n \to \infty} \sum_{i=1}^n f(x_i^*), \Delta x$$

provided this limit exists and is the same for **every** choice of sample points.

For the distance problem specifically:

$$\boxed{d = \int_a^b v(t), dt}$$

This is not a definition of distance — it is a _theorem_, grounded in the fact that the Riemann sums converge.

### Proof Sketch for $v(t) = t^2$ on $[0,b]$

Use right endpoints: $t_i = ib/n$, $\Delta t = b/n$.

$$D_n = \sum_{i=1}^n \left(\frac{ib}{n}\right)^2 \cdot \frac{b}{n} = \frac{b^3}{n^3} \sum_{i=1}^n i^2 = \frac{b^3}{n^3} \cdot \frac{n(n+1)(2n+1)}{6}$$

$$= \frac{b^3}{6} \cdot \frac{(n+1)(2n+1)}{n^2} = \frac{b^3}{6} \cdot \left(2 + \frac{3}{n} + \frac{1}{n^2}\right)$$

$$\lim_{n\to\infty} D_n = \frac{b^3}{6} \cdot 2 = \frac{b^3}{3}$$

For $b = 3$: $d = 27/3 = 9$. $\blacksquare$

---

## 5. The Fundamental Theorem Closes the Loop

The distance problem motivates the integral, but computing it via limits of Riemann sums is painful. The **Fundamental Theorem of Calculus** (FTC) tells us we can instead find an antiderivative:

$$\int_a^b v(t), dt = V(b) - V(a), \quad \text{where } V'(t) = v(t)$$

For $v(t) = t^2$: $V(t) = t^3/3$, so $\int_0^3 t^2, dt = 27/3 - 0 = 9$.

> The FTC is the miracle that makes calculus practical. The distance problem is why we went looking for it.

## 6. Connection to AI / Machine Learning / Deep Learning

This is not a detour. The conceptual architecture of the distance problem is load-bearing in modern ML.

### 6.1 Loss as an Integral — Expected Risk

In statistical learning theory, we don't minimize loss on a single point — we minimize **expected risk**:

$$R(f) = \int_{\mathcal{X} \times \mathcal{Y}} \ell(f(x), y), d\mathbb{P}(x, y)$$

This is an integral. In practice, $\mathbb{P}$ is unknown, so we approximate it with a **finite sum over the training set**:

$$\hat{R}(f) = \frac{1}{n} \sum_{i=1}^n \ell(f(x_i), y_i)$$

This is a **Riemann sum approximation to the integral** $R(f)$. The training loss is the empirical Riemann sum; the true risk is the integral. As $n \to \infty$, $\hat{R} \to R$ — exactly the story of the distance problem.

### 6.2 Gradient Descent as Continuous Limit

Gradient descent with step size $\eta$:

$$\theta_{k+1} = \theta_k - \eta \nabla_\theta \mathcal{L}(\theta_k)$$

In the limit $\eta \to 0$, this is the **Euler method** for the ODE:

$$\frac{d\theta}{dt} = -\nabla_\theta \mathcal{L}(\theta)$$

The **parameter trajectory** $\theta(t)$ satisfies:

$$\theta(T) - \theta(0) = -\int_0^T \nabla_\theta \mathcal{L}(\theta(t)), dt$$

Discrete gradient descent $\approx$ Riemann sum approximation to this integral. The distance problem lives here: total parameter displacement = integral of gradient velocity.

### 6.3 Backpropagation and the Chain Rule — Not a Coincidence

The chain rule of calculus is what makes differentiation composable. Backprop is the chain rule applied to computational graphs. The same derivative that tells us $\frac{dv}{dt} = $ acceleration in the distance problem is what backprop uses to compute $\frac{\partial \mathcal{L}}{\partial w}$ through layers.

### 6.4 Neural ODEs (Deep Learning $\to$ Integral)

A **ResNet** layer is:

$$h_{t+1} = h_t + f(h_t, \theta_t)$$

In the continuum limit (infinitely many layers), this becomes:

$$\frac{dh}{dt} = f(h(t), \theta(t)), \quad h(T) = h(0) + \int_0^T f(h(t), \theta(t)), dt$$

This is a **Neural ODE** (Chen et al., 2018). The forward pass of a deep network is literally numerically integrating an ODE — and the Riemann sums used to approximate that integral are exactly the architecture of a ResNet. The distance problem is a ResNet.

### Summary Table

|Distance Problem Concept|ML/DL Analog|
|---|---|
|Velocity $v(t)$|Gradient $\nabla_\theta \mathcal{L}$|
|Displacement $\int v, dt$|Parameter update $\Delta\theta$|
|Riemann sum approximation|Mini-batch gradient estimation|
|$n \to \infty$ convergence|Large dataset generalization|
|ODE $\dot{x} = v$|Gradient flow / Neural ODE|
|FTC (antiderivative)|Analytical loss minimization|

## 7. A Note on Rigor — What Are We Really Doing?

When we write $\lim_{n\to\infty} \sum_{i=1}^n f(x_i^*)\Delta x$, the word _limit_ is doing enormous work. Formally, this requires:

1. **Existence**: The limit must exist and be independent of the partition refinement and choice of $x_i^*$. This is guaranteed if $f$ is **Riemann integrable** — which holds for all continuous $f$, and more generally for bounded $f$ with at most countably many discontinuities.
    
2. **Uniqueness**: The integral is uniquely defined (up to measure-zero sets for Lebesgue integration, but that's a story for real analysis).
    
3. **The squeeze**: The rigorous proof that $L_n \leq \int \leq R_n$ and both $\to$ same limit uses the $\varepsilon$-$\delta$ definition of the limit. For a monotone $f$, the error of the $n$-th Riemann sum is bounded by $\frac{(b-a)(f(b)-f(a))}{n} \to 0$.
    

> **Instructor's note to self**: Students at this level often "know" the integral is an area without internalizing that the Riemann sum definition is both the _definition_ and the _computational algorithm_. The distance problem forces them to see both simultaneously.

## 9. Quick-Reference Cheatsheet

$$\text{Distance Problem:} \quad d = \lim_{n\to\infty} \sum_{i=1}^n v(t_i^*),\Delta t = \int_a^b v(t), dt$$

$$\text{Riemann Sum Error (monotone } f\text{):} \quad |E_n| \leq \frac{(b-a)(f(b)-f(a))}{n}$$

$$\text{FTC II:} \quad \int_a^b f(x), dx = F(b) - F(a), \quad F' = f$$

$$\text{ML Expected Risk:} \quad R(\theta) = \mathbb{E}_{(x,y)\sim\mathbb{P}}[\ell(f_\theta(x), y)] \approx \frac{1}{n}\sum_{i=1}^n \ell(f_\theta(x_i), y_i)$$

$$\text{Gradient Flow ODE:} \quad \dot{\theta} = -\nabla_\theta \mathcal{L}(\theta), \quad \Delta\theta = -\int_0^T \nabla_\theta \mathcal{L}, dt$$
