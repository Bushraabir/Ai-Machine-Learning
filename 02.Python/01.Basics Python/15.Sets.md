
## Core Idea

A set is an **unordered, mutable** collection of **unique, hashable** objects. It is implemented as a hash table with no associated values — essentially a dict where only keys exist.

- **Unordered** — no index, no position, no slicing
- **Mutable** — elements can be added or removed
- **Unique** — duplicates are silently discarded
- **Hashable elements only** — same constraint as dict keys

The two reasons to use a set:

1. **O(1) membership testing** — `x in my_set` is constant time regardless of size
2. **Set mathematics** — union, intersection, difference, symmetric difference

Everything else — ordering, indexing, duplicates — use a list or dict instead.

## Creating Sets

```python
# Literal syntax — curly braces with elements
fruits = {"apple", "banana", "cherry"}

# Empty set — MUST use set(), not {} (that creates an empty dict)
empty = set()
empty_dict = {}         # dict, not set
print(type(empty))      # <class 'set'>
print(type(empty_dict)) # <class 'dict'>

# From an iterable — duplicates removed automatically
nums = set([1, 2, 2, 3, 3, 3])
print(nums)             # {1, 2, 3}

chars = set("mississippi")
print(chars)            # {'m', 'i', 's', 'p'} — order not guaranteed

from_range = set(range(5))   # {0, 1, 2, 3, 4}

# Set comprehension
squares = {x**2 for x in range(10)}
# {0, 1, 4, 9, 16, 25, 36, 49, 64, 81}

even_squares = {x**2 for x in range(10) if x % 2 == 0}
# {0, 4, 16, 36, 64}
```

## Core Property — Uniqueness

```python
data = [1, 2, 2, 3, 3, 3, 4]
unique = set(data)
print(unique)    # {1, 2, 3, 4}

# Fast deduplication of a list
deduped = list(set(data))
# [1, 2, 3, 4] — order not guaranteed
```

If order matters after deduplication:

```python
# Preserve insertion order
data = [3, 1, 4, 1, 5, 9, 2, 6, 5]
seen = set()
deduped = []
for item in data:
    if item not in seen:
        deduped.append(item)
        seen.add(item)
# [3, 1, 4, 5, 9, 2, 6]

# One-liner using dict (Python 3.7+)
deduped = list(dict.fromkeys(data))
```

## Adding and Removing Elements

```python
s = {1, 2, 3}

# add — add a single element — O(1)
s.add(4)
print(s)        # {1, 2, 3, 4}

s.add(2)        # already exists — no error, no change
print(s)        # {1, 2, 3, 4}

# remove — remove element — KeyError if missing
s.remove(3)
print(s)        # {1, 2, 4}
s.remove(99)    # KeyError: 99

# discard — remove element — no error if missing
s.discard(99)   # no error
s.discard(2)
print(s)        # {1, 4}

# pop — remove and return an arbitrary element — O(1)
item = s.pop()
print(item)     # some element — order undefined
print(s)        # remaining elements

# clear — remove all elements
s.clear()
print(s)        # set()
```

Use `discard()` when you are not sure the element exists. Use `remove()` when its absence would indicate a bug — the `KeyError` then serves as a useful signal.

## Membership Testing

This is where sets outperform lists decisively.

```python
# O(1) — hash lookup
my_set = {1, 2, 3, 4, 5}
print(3 in my_set)      # True
print(9 in my_set)      # False
print(9 not in my_set)  # True

# O(n) — linear scan
my_list = [1, 2, 3, 4, 5]
print(3 in my_list)     # True — but scans up to 3 elements
```

For n = 1,000,000 elements:

```python
import time

large_list = list(range(1_000_000))
large_set  = set(range(1_000_000))

# List membership — O(n)
999_999 in large_list    # scans ~1,000,000 elements

# Set membership — O(1)
999_999 in large_set     # single hash lookup
```

**Pattern: convert to set once, check membership many times.**

```python
# Wrong — O(n) per lookup, O(n^2) total
valid_tokens = load_tokens()          # list
for token in user_tokens:
    if token in valid_tokens:         # O(n) each time
        process(token)

# Correct — O(n) once to build, O(1) per lookup
valid_token_set = set(load_tokens())  # convert once
for token in user_tokens:
    if token in valid_token_set:      # O(1) each time
        process(token)
```

This pattern appears constantly in ML preprocessing — vocabulary lookups, stop word filtering, label set validation.

## Set Operations — Mathematics

This is the primary reason sets exist. All operations work on any iterable, not just sets.

```python
A = {1, 2, 3, 4, 5}
B = {3, 4, 5, 6, 7}
```

### Union — all elements from both sets

```python
print(A | B)            # {1, 2, 3, 4, 5, 6, 7}
print(A.union(B))       # same
print(A.union([6, 7]))  # method accepts any iterable
```

### Intersection — elements in both sets

```python
print(A & B)                    # {3, 4, 5}
print(A.intersection(B))        # same
```

### Difference — elements in A but not in B

```python
print(A - B)                    # {1, 2}      — in A, not in B
print(A.difference(B))          # same
print(B - A)                    # {6, 7}      — in B, not in A
```

### Symmetric Difference — elements in either but not both

```python
print(A ^ B)                            # {1, 2, 6, 7}
print(A.symmetric_difference(B))        # same
```

### Subset and Superset

```python
C = {3, 4}

print(C.issubset(A))        # True  — all of C is in A
print(C <= A)               # True  — same

print(A.issuperset(C))      # True  — A contains all of C
print(A >= C)               # True  — same

print(C < A)                # True  — proper subset (C != A)
print(A < A)                # False — not a proper subset of itself

print(A.isdisjoint(B))      # False — they share {3, 4, 5}
print({1, 2}.isdisjoint({3, 4}))  # True — no elements in common
```

### In-Place Operations

Modify the set in place rather than creating a new one.

```python
A = {1, 2, 3, 4, 5}
B = {3, 4, 5, 6, 7}

A |= B          # A = A | B — union in place
A &= B          # A = A & B — intersection in place
A -= B          # A = A - B — difference in place
A ^= B          # A = A ^ B — symmetric difference in place
```

## Frozenset — Immutable Set

A `frozenset` is an immutable version of a set. It is hashable and can be used as a dictionary key or stored inside another set.

```python
fs = frozenset([1, 2, 3])
print(fs)               # frozenset({1, 2, 3})

# Can be used as dict key
graph_edge = frozenset({"A", "B"})
edge_weights = {frozenset({"A", "B"}): 5.0,
                frozenset({"B", "C"}): 3.0}

# Cannot be modified
fs.add(4)               # AttributeError — frozenset is immutable

# Can be stored inside a set
s = {frozenset({1, 2}), frozenset({3, 4})}
```

Frozensets are used when you need set semantics (uniqueness, set operations) but also need hashability — for example, representing undirected graph edges or immutable feature sets.

## Iteration

Sets are iterable but order is not guaranteed.

```python
s = {3, 1, 4, 1, 5, 9}

for item in s:
    print(item)
# some order — not necessarily insertion order

# If you need sorted output
for item in sorted(s):
    print(item)
# 1, 3, 4, 5, 9
```

Never write logic that depends on a specific iteration order from a set.

## Set Comprehensions

```python
# Basic
squares = {x**2 for x in range(10)}

# With condition
odd_squares = {x**2 for x in range(10) if x % 2 != 0}
# {1, 9, 25, 49, 81}

# Unique first characters
words = ["apple", "avocado", "banana", "blueberry", "cherry"]
first_chars = {w[0] for w in words}
# {'a', 'b', 'c'}
```

## Performance Characteristics

|Operation|Average Time|Notes|
|---|---|---|
|`x in s`|O(1)|Hash lookup|
|`s.add(x)`|O(1)||
|`s.remove(x)`|O(1)|KeyError if missing|
|`s.discard(x)`|O(1)|No error if missing|
|`s.pop()`|O(1)|Arbitrary element|
|`len(s)`|O(1)|Cached|
|`s \| t` (union)|O(len(s) + len(t))||
|`s & t` (intersection)|O(min(len(s), len(t)))||
|`s - t` (difference)|O(len(s))||
|Iteration|O(n)||

## Common Patterns in ML and CS

**Vocabulary building:**

```python
corpus = ["the cat sat", "the dog ran", "a cat ran"]
vocabulary = set()
for sentence in corpus:
    vocabulary.update(sentence.split())
print(vocabulary)
# {'the', 'cat', 'sat', 'dog', 'ran', 'a'}
```

**Finding common elements between datasets:**

```python
train_labels = {0, 1, 2, 3, 4}
test_labels  = {3, 4, 5, 6}

common    = train_labels & test_labels    # {3, 4}
only_test = test_labels - train_labels    # {5, 6} — unseen classes
all_labels = train_labels | test_labels  # {0, 1, 2, 3, 4, 5, 6}
```

**Removing stop words:**

```python
STOP_WORDS = {"the", "a", "an", "in", "is", "it", "of", "to"}

def remove_stop_words(tokens):
    return [t for t in tokens if t not in STOP_WORDS]
```

`STOP_WORDS` as a set makes each `in` check O(1). As a list it would be O(n) per token.

**Graph visited tracking in BFS / DFS:**

```python
def bfs(graph, start):
    visited = set()
    queue = [start]

    while queue:
        node = queue.pop(0)
        if node in visited:
            continue
        visited.add(node)
        process(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                queue.append(neighbor)
```

`visited` as a set gives O(1) membership check. As a list it degrades BFS from O(V + E) to O(V^2 + E). You will implement this exact algorithm in CLRS chapter 22 and AIMA chapter 3.

**Checking for duplicate entries:**

```python
def has_duplicates(data):
    return len(data) != len(set(data))

print(has_duplicates([1, 2, 3]))      # False
print(has_duplicates([1, 2, 2, 3]))   # True
```

## Set vs List vs Dict — Decision Guide

|Need|Use|
|---|---|
|O(1) membership check|Set|
|Unique elements|Set|
|Set operations (union, intersect)|Set|
|Ordered sequence, indexing|List|
|Duplicates allowed|List|
|Key-value mapping|Dict|
|Ordered unique sequence|list + set together|
|Immutable set (as dict key)|Frozenset|
|Counting occurrences|Counter (dict subclass)|

## Summary

|Feature|Behavior|
|---|---|
|Ordered|No|
|Mutable|Yes (frozenset is immutable)|
|Duplicates|Not allowed — silently ignored|
|Elements|Must be hashable|
|Indexing|Not supported|
|Membership|O(1) — primary advantage|
|Set operations|Union, intersection, difference, symmetric difference|
|Empty set|`set()` — NOT `{}`|
|Comprehension|`{expr for x in it if cond}`|

## Common Mistakes to Avoid

```python
# Mistake 1: using {} for empty set
empty = {}          # this is a dict
empty = set()       # correct

# Mistake 2: storing mutable objects in a set
s = {[1, 2], [3, 4]}    # TypeError — lists are not hashable
s = {(1, 2), (3, 4)}    # correct — tuples are hashable

# Mistake 3: relying on iteration order
for item in my_set:
    # order is not guaranteed — never depend on it
    ...

# Mistake 4: using a list for repeated membership checks
stop_words = ["the", "a", "an"]   # O(n) per check
stop_words = {"the", "a", "an"}   # O(1) per check — always use set here

# Mistake 5: using remove when discard is safer
s.remove("x")       # KeyError if "x" not in s
s.discard("x")      # safe — no error

# Mistake 6: forgetting set operations return new sets
A = {1, 2, 3}
B = {3, 4, 5}
C = A & B           # new set {3} — A and B unchanged
A &= B              # modifies A in place — A is now {3}

# Mistake 7: converting to set when order matters
data = [3, 1, 4, 1, 5]
unique = list(set(data))    # order lost — may be [1, 3, 4, 5] or anything
# use dict.fromkeys() to preserve order
unique = list(dict.fromkeys(data))   # [3, 1, 4, 5]
```
