## Core Idea

Nesting means placing one control structure inside another. A loop inside a loop, a conditional inside a loop, a loop inside a conditional — any combination is valid. Python has no hard limit on nesting depth, but human cognition does.

The central skill with nesting is not writing it — it is knowing when to flatten it, extract it, or restructure it entirely.

---

## Nested Loops

### Basic Structure

The inner loop runs to completion for every single iteration of the outer loop.

```python
for i in range(3):
    for j in range(3):
        print(i, j)

# (0,0) (0,1) (0,2)
# (1,0) (1,1) (1,2)
# (2,0) (2,1) (2,2)
```

Total iterations = outer count * inner count. Here: 3 * 3 = 9.

For large ranges this compounds fast:

```python
for i in range(1000):
    for j in range(1000):
        compute(i, j)    # 1,000,000 iterations
```

Nested loops are O(n^2) at minimum. In algorithm analysis — which you will do extensively in CLRS — recognizing nested loops is how you identify quadratic time complexity.

### Matrix Traversal

The canonical use case for nested loops. A 2D matrix is a list of lists.

```python
matrix = [
    [1,  2,  3],
    [4,  5,  6],
    [7,  8,  9]
]

# Row by row
for row in matrix:
    for val in row:
        print(val, end=" ")
    print()

# 1 2 3
# 4 5 6
# 7 8 9
```

Accessing a specific element:

```python
rows = len(matrix)       # 3
cols = len(matrix[0])    # 3

for i in range(rows):
    for j in range(cols):
        print(f"matrix[{i}][{j}] = {matrix[i][j]}")
```

Transposing a matrix — rows become columns:

```python
matrix = [[1, 2, 3],
          [4, 5, 6]]

rows = len(matrix)
cols = len(matrix[0])

transposed = [[matrix[r][c] for r in range(rows)] for c in range(cols)]

# [[1, 4],
#  [2, 5],
#  [3, 6]]
```

### Generating Pairs and Combinations

```python
# All ordered pairs (i, j)
pairs = [(i, j) for i in range(4) for j in range(4)]

# All pairs where i != j
pairs = [(i, j) for i in range(4) for j in range(4) if i != j]

# All pairs where j > i (upper triangle — avoid duplicates)
pairs = [(i, j) for i in range(4) for j in range(i+1, 4)]
# (0,1), (0,2), (0,3), (1,2), (1,3), (2,3)
```

The upper triangle pattern appears in distance matrices, similarity computations, and graph edge enumeration — all of which you will encounter in ML and AI.

### Nested while Loops

```python
i = 0
while i < 3:
    j = 0
    while j < 3:
        print(i, j)
        j += 1
    i += 1
```

**Critical: reset the inner variable at the start of each outer iteration.** `j = 0` must be inside the outer loop. If placed outside, `j` is already 3 after the first outer iteration and the inner loop never runs again.

```python
# Bug — j is never reset
i = 0
j = 0
while i < 3:
    while j < 3:      # j is 3 after first outer iteration — inner loop skipped
        print(i, j)
        j += 1
    i += 1

# Only prints: (0,0) (0,1) (0,2) — inner loop never runs for i=1, i=2
```

### break and continue in Nested Loops

`break` exits only the innermost loop:

```python
for i in range(4):
    for j in range(4):
        if j == 2:
            break           # exits inner loop
        print(i, j)

# (0,0) (0,1)
# (1,0) (1,1)
# (2,0) (2,1)
# (3,0) (3,1)
# outer loop continues for all values of i
```

`continue` affects only the innermost loop:

```python
for i in range(3):
    for j in range(3):
        if j == 1:
            continue        # skip j=1, continue inner loop
        print(i, j)

# (0,0) (0,2)
# (1,0) (1,2)
# (2,0) (2,2)
```

**To exit all nested loops — use a function:**

```python
# Flag approach — works but cluttered
found = False
for i in range(n):
    for j in range(n):
        if condition(i, j):
            found = True
            break
    if found:
        break

# Function approach — clean, preferred
def find_target(matrix, target):
    for i, row in enumerate(matrix):
        for j, val in enumerate(row):
            if val == target:
                return (i, j)    # exits all loops immediately
    return None
```

Return from a function is the cleanest way to exit multiple loop levels. When you find yourself writing a flag to propagate a `break`, extract the loops into a function instead.

## Nested Conditionals

### Basic Structure

An `if` inside another `if`. Each level adds a condition that must also be true.

```python
x = 15

if x > 0:
    if x > 10:
        print("large positive")
    else:
        print("small positive")
else:
    print("non-positive")
```

This is equivalent to:

```python
if x > 10:
    print("large positive")
elif x > 0:
    print("small positive")
else:
    print("non-positive")
```

The flattened version is almost always preferable. Nested conditionals that can be expressed as `elif` chains should be.

---

### Conditionals Inside Loops

Filter, transform, or branch on each item as you iterate.

```python
data = [3, -1, 7, 0, -4, 9, 2]

for n in data:
    if n > 0:
        print(f"{n} is positive")
    elif n < 0:
        print(f"{n} is negative")
    else:
        print(f"{n} is zero")
```

**Guard clause style — use continue to reduce nesting:**

```python
# Nested — harder to follow
for record in records:
    if record is not None:
        if record.is_valid():
            if record.value > threshold:
                process(record)

# Flat with continue — easy to follow
for record in records:
    if record is None:
        continue
    if not record.is_valid():
        continue
    if record.value <= threshold:
        continue
    process(record)
```

Both do the same thing. The second version reads like a checklist — each condition is a gate. The main logic (`process(record)`) is reached only after all gates pass. This is the guard clause pattern applied to loops.

---

### Loops Inside Conditionals

Conditionally execute an entire loop based on outer state.

```python
mode = "train"

if mode == "train":
    for epoch in range(100):
        train_one_epoch()
elif mode == "eval":
    for batch in eval_data:
        evaluate_batch(batch)
else:
    raise ValueError(f"unknown mode: {mode}")
```

---

### Nested Loops with Conditionals — Combined

```python
matrix = [
    [1,  -2,  3],
    [-4,  5, -6],
    [7,  -8,  9]
]

# Replace all negative values with 0
for i in range(len(matrix)):
    for j in range(len(matrix[0])):
        if matrix[i][j] < 0:
            matrix[i][j] = 0

# [[1, 0, 3],
#  [0, 5, 0],
#  [7, 0, 9]]
```

```python
# Count elements matching a condition
count = 0
for row in matrix:
    for val in row:
        if val > 5:
            count += 1
```

## Flattening Nested Logic — Refactoring Patterns

Deep nesting is a code smell. It means your function is doing too many things or your conditions are not organized clearly. Here are the standard ways to fix it.

---

### Pattern 1 — Invert Conditions and Use continue / return

```python
# Deeply nested
def process_user(user):
    if user is not None:
        if user.is_active:
            if user.age >= 18:
                send_email(user)

# Flat — guard clauses
def process_user(user):
    if user is None:
        return
    if not user.is_active:
        return
    if user.age < 18:
        return
    send_email(user)
```

---

### Pattern 2 — Extract Inner Loop into a Function

```python
# Hard to read — two nested loops plus conditions
def find_anomaly(dataset):
    for batch in dataset:
        for record in batch:
            if record.value > threshold:
                if record.source == "sensor_A":
                    return record
    return None

# Clean — inner loop extracted
def scan_batch(batch):
    for record in batch:
        if record.value > threshold and record.source == "sensor_A":
            return record
    return None

def find_anomaly(dataset):
    for batch in dataset:
        result = scan_batch(batch)
        if result is not None:
            return result
    return None
```

Each function does one thing. The logic is testable independently.

---

### Pattern 3 — Combine Conditions with and / or

```python
# Nested conditions
for item in data:
    if item is not None:
        if isinstance(item, (int, float)):
            if item > 0:
                process(item)

# Combined
for item in data:
    if item is not None and isinstance(item, (int, float)) and item > 0:
        process(item)
```

Combine when conditions are all about the same decision. Split when each condition has a different consequence or error message.

---

### Pattern 4 — Use List Comprehensions for Nested Loops

```python
# Nested loops to build a list
result = []
for i in range(5):
    for j in range(5):
        if i != j:
            result.append((i, j))

# Comprehension — one line
result = [(i, j) for i in range(5) for j in range(5) if i != j]
```

Use this when the logic is simple enough to fit on one line. If the condition is complex or the body has multiple steps, keep the explicit loop.

## Complexity Awareness

Every level of nesting multiplies iteration count. Know what you are writing before you write it.

```python
# O(n) — one loop
for item in data:
    process(item)

# O(n^2) — two nested loops over the same data
for i in data:
    for j in data:
        compare(i, j)

# O(n^3) — three nested loops
for i in data:
    for j in data:
        for k in data:
            compute(i, j, k)
```

For n = 1000:

- O(n) = 1,000 operations
- O(n^2) = 1,000,000 operations
- O(n^3) = 1,000,000,000 operations

Before writing nested loops over large data, ask whether a vectorized operation (NumPy), a mathematical shortcut, or a better algorithm eliminates the need for nesting entirely. This is a core skill in ML engineering — nested Python loops over large arrays are almost always replaced by NumPy operations that run in optimized C.

## Real Patterns You Will Use in ML

**Computing a distance matrix (pairwise distances):**

```python
n = len(points)
distance_matrix = [[0.0] * n for _ in range(n)]

for i in range(n):
    for j in range(i + 1, n):    # upper triangle only
        d = euclidean(points[i], points[j])
        distance_matrix[i][j] = d
        distance_matrix[j][i] = d    # symmetric
```

**Mini-batch training loop:**

```python
for epoch in range(num_epochs):
    for batch in dataloader:
        inputs, labels = batch
        predictions = model(inputs)
        loss = criterion(predictions, labels)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

**Grid search over hyperparameters:**

```python
results = []

for lr in [0.001, 0.01, 0.1]:
    for batch_size in [32, 64, 128]:
        for dropout in [0.2, 0.5]:
            acc = train_and_evaluate(lr, batch_size, dropout)
            results.append((lr, batch_size, dropout, acc))

best = max(results, key=lambda x: x[3])
print(f"Best config: lr={best[0]}, batch={best[1]}, dropout={best[2]}, acc={best[3]}")
```

## Maximum Nesting Depth Guideline

```
1 level  — fine
2 levels — acceptable, think about clarity
3 levels — refactor if possible
4+ levels — always refactor
```

If you are 4 levels deep, you have at least one function that needs to be broken into smaller pieces. No exceptions.

---

## Summary

|Pattern|Guideline|
|---|---|
|Nested loops|Total iterations multiply — know the complexity|
|Nested while|Always reset inner variable inside outer loop|
|break in nested|Exits innermost loop only — use function + return to exit all|
|Nested conditionals|Flatten with elif or guard clauses when possible|
|Conditionals in loops|Use continue for guard clauses — keep main logic un-nested|
|3+ levels deep|Extract inner block into a named function|
|Large data nested loops|Replace with NumPy vectorization when possible|

## Common Mistakes to Avoid

```python
# Mistake 1: forgetting to reset inner while variable
i = 0
j = 0             # should be inside outer loop
while i < 3:
    while j < 3:
        j += 1
    i += 1        # j is already 3 — inner loop never runs again

# Mistake 2: expecting break to exit all nested loops
for i in range(5):
    for j in range(5):
        if condition:
            break     # only breaks inner loop
# fix: use a function and return

# Mistake 3: nesting when elif suffices
if x > 0:
    if x > 10:      # unnecessary nesting
        ...
# fix:
if x > 10:
    ...
elif x > 0:
    ...

# Mistake 4: writing O(n^2) loops over large data without thinking
for a in million_item_list:
    for b in million_item_list:
        compare(a, b)    # 10^12 operations — do not do this

# Mistake 5: not extracting deeply nested logic into functions
# leads to untestable, unreadable, unmaintainable code
```
