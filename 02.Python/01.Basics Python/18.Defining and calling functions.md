
## Core Idea

A function is a named, reusable block of code that takes inputs, performs a computation, and optionally returns a result. Functions are the primary unit of abstraction in Python — they let you name a concept, hide implementation details, and reuse logic without repetition.

In Python, functions are **first-class objects**. They can be assigned to variables, passed as arguments, returned from other functions, and stored in data structures. This is not a curiosity — it is the foundation of decorators, callbacks, higher-order functions, and much of the ML ecosystem.

---

## Defining a Function

```python
def function_name(parameters):
    """Docstring — what this function does."""
    # body
    return value
```

```python
def square(x):
    """Return the square of x."""
    return x ** 2
```

- `def` — keyword that starts a function definition
- `function_name` — follows the same rules as variable names, snake_case by convention
- `parameters` — comma-separated variable names that receive arguments when called
- `return` — sends a value back to the caller and exits the function
- body — indented block, any valid Python code

---

## Calling a Function

```python
result = square(5)
print(result)       # 25

# Inline
print(square(4))    # 16

# In expressions
total = square(3) + square(4)   # 9 + 16 = 25
```

Arguments are the values passed at call time. Parameters are the variable names in the definition. The terms are often used interchangeably but the distinction matters when discussing how values are passed.

## Return Statement

### Explicit Return

```python
def add(a, b):
    return a + b

result = add(3, 4)    # 7
```

`return` immediately exits the function. Any code after it in the same scope is unreachable.

```python
def check(x):
    if x > 0:
        return "positive"     # exits here if x > 0
    return "non-positive"     # only reached if x <= 0
```

### No Return Statement — Returns None

```python
def greet(name):
    print(f"Hello, {name}")

result = greet("Alice")
print(result)    # None — function has no return statement
```

Every Python function returns something. If no `return` is written, Python returns `None` implicitly.

### Returning Multiple Values

```python
def min_max(data):
    return min(data), max(data)    # returns a tuple

low, high = min_max([3, 1, 4, 1, 5, 9])
print(low, high)    # 1 9

# Or capture as tuple
result = min_max([3, 1, 4, 1, 5, 9])
print(result)       # (1, 9)
print(result[0])    # 1
```

Multiple return values are packed into a tuple automatically. Unpack immediately at the call site — do not carry around unexploded tuples.

### Early Return — Guard Clause Pattern

```python
def process(data):
    if data is None:
        return None
    if len(data) == 0:
        return []
    if not isinstance(data, list):
        raise TypeError(f"expected list, got {type(data).__name__}")

    # main logic — reached only if all guards pass
    return [transform(x) for x in data]
```

Return early to handle edge cases. Keep the main logic at the lowest indentation level — not buried inside nested conditionals.

## Parameters and Arguments

### Positional Parameters

Matched by position. Order matters.

```python
def power(base, exponent):
    return base ** exponent

power(2, 10)     # base=2, exponent=10 -> 1024
power(10, 2)     # base=10, exponent=2 -> 100
```

### Keyword Arguments

Passed by name at call time. Order does not matter.

```python
power(exponent=10, base=2)    # 1024 — same as power(2, 10)
power(base=2, exponent=10)    # 1024
```

You can mix positional and keyword — positional must come first:

```python
power(2, exponent=10)         # valid
power(base=2, 10)             # SyntaxError — positional after keyword
```

### Default Parameter Values

Parameters can have defaults. They become optional at call time.

```python
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))            # Hello, Alice!
print(greet("Alice", "Hi"))      # Hi, Alice!
print(greet("Alice", greeting="Hey"))   # Hey, Alice!
```

Parameters with defaults must come after parameters without defaults:

```python
def func(a, b=2, c=3): ...     # valid
def func(a=1, b, c=3): ...     # SyntaxError
```

### Mutable Default Argument — The Classic Trap

**Never use a mutable object as a default argument.**

```python
# Wrong — the list is created ONCE when the function is defined
# and shared across all calls
def append_to(item, lst=[]):
    lst.append(item)
    return lst

print(append_to(1))    # [1]
print(append_to(2))    # [1, 2]  — not [2], the list persists
print(append_to(3))    # [1, 2, 3]  — still accumulating
```

```python
# Correct — use None as sentinel, create fresh object each call
def append_to(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

print(append_to(1))    # [1]
print(append_to(2))    # [2]  — fresh list each time
```

This applies to all mutable defaults: lists, dicts, sets. Always use `None` as the sentinel.

## *args — Variable Positional Arguments

Collects all extra positional arguments into a tuple.

```python
def total(*args):
    return sum(args)

print(total(1, 2, 3))           # 6
print(total(1, 2, 3, 4, 5))    # 15
print(total())                  # 0
```

```python
def log(level, *messages):
    for msg in messages:
        print(f"[{level}] {msg}")

log("INFO", "Server started", "Listening on port 8080")
# [INFO] Server started
# [INFO] Listening on port 8080
```

Inside the function, `args` is a tuple. You can iterate it, index it, pass it to `sum()`, etc.

---

## **kwargs — Variable Keyword Arguments

Collects all extra keyword arguments into a dict.

```python
def display(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

display(name="Alice", age=30, role="Engineer")
# name: Alice
# age: 30
# role: Engineer
```

```python
def configure(host="localhost", **kwargs):
    print(f"Host: {host}")
    for k, v in kwargs.items():
        print(f"  {k}: {v}")

configure(host="prod.server.com", port=8080, ssl=True, timeout=30)
# Host: prod.server.com
#   port: 8080
#   ssl: True
#   timeout: 30
```

Inside the function, `kwargs` is a regular dict.

## Parameter Order — The Full Signature

When all parameter types are combined, they must follow this order:

```
def f(positional, /, standard, *args, keyword_only, **kwargs):
```

```python
def full(a, b, c=3, *args, kw_only, **kwargs):
    print(a, b, c, args, kw_only, kwargs)

full(1, 2, 3, 4, 5, kw_only="x", extra="y")
# a=1, b=2, c=3, args=(4, 5), kw_only="x", kwargs={"extra": "y"}
```

In practice, you rarely use all of these together. The common combinations:

```python
def f(a, b, c=10): ...          # fixed + default
def f(*args): ...               # variable positional
def f(**kwargs): ...            # variable keyword
def f(*args, **kwargs): ...     # both — forward all arguments
def f(a, b, *args, **kwargs): . # fixed + variable both
```

The `*args, **kwargs` combination is used constantly for wrapper functions and decorators.

## Positional-Only and Keyword-Only Parameters

### Positional-Only (Python 3.8+) — `/`

Parameters before `/` can only be passed positionally:

```python
def greet(name, /, greeting="Hello"):
    return f"{greeting}, {name}!"

greet("Alice")                  # valid
greet("Alice", "Hi")            # valid
greet(name="Alice")             # TypeError — name is positional-only
```

### Keyword-Only — `*`

Parameters after `*` (or `*args`) can only be passed by keyword:

```python
def connect(host, port, *, timeout=30, ssl=False):
    print(host, port, timeout, ssl)

connect("localhost", 8080)                     # uses defaults
connect("localhost", 8080, timeout=60)         # keyword only
connect("localhost", 8080, 60)                 # TypeError — timeout is keyword-only
```

Keyword-only parameters prevent callers from accidentally passing positional arguments to the wrong parameter — particularly valuable in functions with many parameters like ML model builders.

## Unpacking Arguments at Call Time

You can unpack a list or dict into function arguments with `*` and `**`.

```python
def add(a, b, c):
    return a + b + c

args = [1, 2, 3]
print(add(*args))       # 6 — unpacks list as positional args

kwargs = {"a": 1, "b": 2, "c": 3}
print(add(**kwargs))    # 6 — unpacks dict as keyword args
```

```python
# Combining
def f(a, b, c, d):
    print(a, b, c, d)

f(*[1, 2], **{"c": 3, "d": 4})    # 1 2 3 4
```

This is how many ML frameworks pass configuration dicts directly to functions — `model(**config)`, `optimizer(**optim_params)`.

## Scope — LEGB Rule

Python resolves variable names in this order:

1. **L**ocal — inside the current function
2. **E**nclosing — inside any enclosing function (for nested functions)
3. **G**lobal — module-level
4. **B**uilt-in — Python's built-in names (`len`, `print`, `range`, etc.)

```python
x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)    # local

    inner()
    print(x)        # enclosing

outer()
print(x)            # global
```

### global Keyword

Allows a function to read and modify a global variable.

```python
counter = 0

def increment():
    global counter
    counter += 1

increment()
increment()
print(counter)    # 2
```

Avoid `global` except in very specific situations — it creates hidden coupling between functions and module state.

### nonlocal Keyword

Allows an inner function to modify a variable in its enclosing (but non-global) scope.

```python
def make_counter():
    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

counter = make_counter()
print(counter())    # 1
print(counter())    # 2
print(counter())    # 3
```

This is a closure — `increment` captures and remembers `count` from its enclosing scope. Closures are the foundation of decorators and stateful callbacks.

## Functions as First-Class Objects

Functions are objects. They can be assigned, passed, and returned like any other value.

```python
def square(x):
    return x ** 2

# Assign to variable
f = square
print(f(5))         # 25

# Pass as argument
def apply(func, value):
    return func(value)

print(apply(square, 4))    # 16

# Store in data structure
operations = {
    "square": square,
    "double": lambda x: x * 2,
    "negate": lambda x: -x,
}
print(operations["square"](5))    # 25
print(operations["double"](5))    # 10

# Return from function
def make_multiplier(n):
    def multiply(x):
        return x * n
    return multiply

triple = make_multiplier(3)
print(triple(7))    # 21
```

## Lambda Functions

Anonymous, single-expression functions. Useful for short callbacks and sort keys.

```python
# Syntax: lambda parameters: expression
square = lambda x: x ** 2
add    = lambda x, y: x + y

print(square(5))     # 25
print(add(3, 4))     # 7
```

```python
# Common use — sort key
data = [("alice", 30), ("bob", 25), ("carol", 28)]
data.sort(key=lambda person: person[1])   # sort by age
print(data)    # [('bob', 25), ('carol', 28), ('alice', 30)]

# With sorted()
words = ["banana", "apple", "cherry"]
sorted_words = sorted(words, key=lambda w: len(w))
# ["apple", "banana", "cherry"]

# With map() and filter()
nums = [1, 2, 3, 4, 5]
doubled  = list(map(lambda x: x * 2, nums))
evens    = list(filter(lambda x: x % 2 == 0, nums))
```

Lambdas are limited to a single expression. For anything more complex, use a named function with `def`. Do not write complex logic in a lambda — it becomes unreadable and untestable.

## Pure Functions and Side Effects

A **pure function** depends only on its inputs and has no side effects — same input always produces same output, no external state is modified.

```python
# Pure — output determined entirely by input
def add(a, b):
    return a + b

# Impure — modifies external state (side effect)
total = 0
def add_to_total(x):
    global total
    total += x    # modifies external variable
```

Prefer pure functions:

- Easier to test — no setup, no teardown, no mocking
- Easier to reason about — no hidden dependencies
- Safe to parallelize — no shared mutable state
- Composable — can be chained and combined freely

In ML, loss functions, activation functions, and metric calculations should all be pure. Training loops that update weights are necessarily impure — that is acceptable when the impurity is intentional and contained.

## Recursive Functions

A function that calls itself. Every recursive function needs:

1. A base case — the condition that stops recursion
2. A recursive case — the call that moves toward the base case

```python
def factorial(n):
    if n == 0:          # base case
        return 1
    return n * factorial(n - 1)   # recursive case

print(factorial(5))    # 120
print(factorial(0))    # 1
```

```python
def fibonacci(n):
    if n <= 1:          # base case
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(10))   # 55
```

Python's default recursion limit is 1000 calls deep (`sys.getrecursionlimit()`). For deeply recursive problems, use iteration or increase the limit with `sys.setrecursionlimit()`. The naive recursive fibonacci above is O(2^n) — for production use memoization or iteration.

## Docstrings in Functions

Every non-trivial function should have a docstring. See note 06 for full format details.

```python
def train_step(model, batch, optimizer, criterion):
    """
    Perform one training step.

    Parameters
    ----------
    model : nn.Module
        The neural network model.
    batch : tuple of (Tensor, Tensor)
        Input data and target labels.
    optimizer : torch.optim.Optimizer
        The optimizer to update weights.
    criterion : callable
        The loss function.

    Returns
    -------
    float
        The scalar loss value for this step.
    """
    inputs, targets = batch
    outputs = model(inputs)
    loss = criterion(outputs, targets)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    return loss.item()
```

## Summary

|Concept|Syntax / Rule|
|---|---|
|Define|`def name(params): ...`|
|Call|`name(args)`|
|Return|`return value` — exits immediately|
|No return|Implicitly returns `None`|
|Multiple return|`return a, b` — packed as tuple|
|Positional args|Matched by order|
|Keyword args|Matched by name at call site|
|Default values|`def f(x, n=10)` — must come after non-defaults|
|Mutable default|Never — use `None` as sentinel|
|`*args`|Collects extra positional into tuple|
|`**kwargs`|Collects extra keyword into dict|
|`*` in signature|Forces following params to be keyword-only|
|`/` in signature|Forces preceding params to be positional-only|
|Unpack at call|`f(*list)`, `f(**dict)`|
|First-class|Functions can be assigned, passed, returned|
|Lambda|`lambda x: expr` — anonymous, single expression|
|Scope|LEGB — Local, Enclosing, Global, Built-in|
|`global`|Modify module-level variable from inside function|
|`nonlocal`|Modify enclosing function variable from inner function|
|Pure function|No side effects, output depends only on input|
|Recursion|Base case + recursive case, default limit 1000|

## Common Mistakes to Avoid

```python
# Mistake 1: mutable default argument
def f(data=[]):         # wrong — list shared across calls
    data.append(1)
    return data

def f(data=None):       # correct
    if data is None:
        data = []
    data.append(1)
    return data

# Mistake 2: ignoring return value
result = my_list.sort()     # sort() returns None
result = sorted(my_list)    # correct

# Mistake 3: using global when a return would suffice
count = 0
def increment():
    global count            # hidden dependency
    count += 1

def increment(count):       # explicit, testable, pure
    return count + 1

# Mistake 4: returning inside a loop when you want to collect all results
def get_evens(data):
    for x in data:
        if x % 2 == 0:
            return x        # returns first even only, exits immediately
    # correct:
    return [x for x in data if x % 2 == 0]

# Mistake 5: complex logic in a lambda
f = lambda x: x**2 if x > 0 else (-x)**2 if x < 0 else 0  # unreadable
# use def for anything beyond a simple expression

# Mistake 6: keyword argument after positional in call
f(1, 2, a=3, 4)   # SyntaxError — positional after keyword

# Mistake 7: not unpacking multiple return values
result = min_max(data)    # result is a tuple — easy to misuse
low, high = min_max(data) # unpack immediately
```
