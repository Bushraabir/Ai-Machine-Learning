
## What is a Variable

A variable is a name bound to a value stored in memory. Python does not require you to declare a type — the interpreter infers it at runtime. This is called **dynamic typing**.

```python
x = 10        # x is bound to the integer 10
x = "hello"   # x is now rebound to a string — perfectly valid in Python
```

Under the hood, a variable is not a container. It is a **label pointing to an object** in memory. This distinction matters when you get to mutability.

---

## The 5 Core Data Types

### int — Integer

Whole numbers, positive or negative, with no size limit in Python.

```python
a = 42
b = -7
c = 0
big = 10_000_000   # underscores allowed for readability

print(type(a))     # <class 'int'>
```

Operations on integers always return exact results. There is no overflow in Python integers (unlike C/C++).

```python
print(2 ** 100)    # works fine — arbitrarily large
```

---

### float — Floating Point Number

Numbers with a decimal point. Stored internally as IEEE 754 double-precision (64-bit).

```python
x = 3.14
y = -0.001
z = 2.0        # still a float, not an int
sci = 1.5e10   # scientific notation: 1.5 x 10^10

print(type(x)) # <class 'float'>
```

**Critical caveat — floating point is not exact:**

```python
print(0.1 + 0.2)          # 0.30000000000000004
print(0.1 + 0.2 == 0.3)   # False
```

This is not a Python bug. It is a fundamental property of binary floating point representation. You will encounter this constantly in numerical computing and ML. Use `round()` or `math.isclose()` for comparisons.

```python
import math
print(math.isclose(0.1 + 0.2, 0.3))  # True
```

---

### str — String

An ordered, immutable sequence of Unicode characters.

```python
name = "Alice"
greeting = 'hello'
multi = """This is
a multiline string"""

print(type(name))  # <class 'str'>
```

**Strings are immutable** — you cannot change a character in place.

```python
s = "hello"
s[0] = "H"    # TypeError: 'str' object does not support item assignment
```

**Common string operations:**

```python
s = "hello"

print(len(s))          # 5
print(s[0])            # 'h'
print(s[-1])           # 'o'
print(s[1:4])          # 'ell'  (slicing)
print(s.upper())       # 'HELLO'
print(s + " world")    # 'hello world'  (concatenation)
print(s * 3)           # 'hellohellohello'
```

**f-strings (preferred formatting method):**

```python
name = "Alice"
score = 98.5
print(f"Name: {name}, Score: {score:.2f}")
# Name: Alice, Score: 98.50
```

---

### bool — Boolean

Represents one of two values: `True` or `False`. In Python, bool is a subclass of int.

```python
a = True
b = False

print(type(a))   # <class 'bool'>
print(int(True)) # 1
print(int(False))# 0
```

**Truthiness — every object in Python has a boolean value:**

```python
# Falsy values
bool(0)       # False
bool(0.0)     # False
bool("")      # False
bool([])      # False
bool(None)    # False

# Everything else is truthy
bool(1)       # True
bool("hi")    # True
bool([1,2])   # True
```

This is used extensively in conditionals:

```python
data = []
if not data:
    print("list is empty")
```

**Boolean operators:**

```python
print(True and False)   # False
print(True or False)    # True
print(not True)         # False
```

---

### None — The Null Value

`None` is Python's way of representing the absence of a value. It is its own type: `NoneType`.

```python
x = None
print(type(x))    # <class 'NoneType'>
print(x is None)  # True  (use 'is', not '==')
```

**Common use cases:**

```python
# Default function return
def do_nothing():
    pass

result = do_nothing()
print(result)   # None

# Placeholder variable
model = None
# ... later assigned when ready
model = load_model()

# Optional function parameter
def connect(host, port=None):
    if port is None:
        port = 8080
```

Always check for `None` using `is None` or `is not None`, not `== None`.

## Type Checking

```python
x = 42

print(type(x))              # <class 'int'>
print(type(x) == int)       # True

print(isinstance(x, int))   # True
print(isinstance(x, bool))  # False
```

Prefer `isinstance()` over `type() ==` because isinstance handles inheritance correctly.

```python
print(isinstance(True, int))   # True  — bool is a subclass of int
print(type(True) == int)       # False — type() is exact, no inheritance
```

## Type Conversion (Casting)

```python
# int conversions
int("42")       # 42
int(3.99)       # 3  — truncates, does not round
int(True)       # 1
int(False)      # 0

# float conversions
float("3.14")   # 3.14
float(10)       # 10.0

# str conversions
str(100)        # "100"
str(3.14)       # "3.14"
str(True)       # "True"

# bool conversions
bool(0)         # False
bool(42)        # True
bool("")        # False
bool("hello")   # True
```

**Conversion that will raise an error:**

```python
int("hello")    # ValueError: invalid literal for int()
int("3.14")     # ValueError — must cast to float first
int(float("3.14"))  # 3 — correct approach
```

## Variable Naming Rules

```python
# Valid
my_variable = 1
_private = 2
camelCase = 3      # valid but not Pythonic
name2 = 4

# Invalid
2name = 5          # SyntaxError: cannot start with digit
my-var = 6         # SyntaxError: hyphen is subtraction operator
class = 7          # SyntaxError: reserved keyword
```

**Python convention (PEP 8):**

|Use case|Convention|Example|
|---|---|---|
|Variables and functions|snake_case|`learning_rate`|
|Constants|UPPER_SNAKE_CASE|`MAX_EPOCHS`|
|Classes|PascalCase|`NeuralNetwork`|
|Private (convention only)|_leading_underscore|`_internal`|

## Multiple Assignment

```python
# Assign same value to multiple variables
a = b = c = 0

# Unpack a sequence
x, y, z = 1, 2, 3

# Swap without temp variable
a, b = b, a

# Ignore values with underscore
first, _, last = (1, 2, 3)
```

---

## Identity vs Equality

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)    # True  — same value
print(a is b)    # False — different objects in memory
print(a is c)    # True  — same object (c points to a)
```

`==` checks value equality. `is` checks object identity (same memory address). For `None`, always use `is`.
