## Core Idea

A conditional statement evaluates a boolean expression and executes a block of code only if that expression is truthy. This is the primary mechanism for **branching** — making your program take different paths based on data.

Python uses indentation (4 spaces) to define blocks. There are no curly braces. The indentation is not style — it is syntax. Wrong indentation is a runtime error.

---

## Basic Structure

```python
if condition:
    # block executes if condition is truthy
elif another_condition:
    # block executes if above was falsy and this is truthy
else:
    # block executes if all above were falsy
```

Only one branch executes. Once a condition matches, the rest are skipped entirely.

---

## if — Single Branch

```python
x = 10

if x > 5:
    print("x is greater than 5")
```

If the condition is falsy, nothing happens. No error, no output — the block is simply skipped.

```python
x = 3

if x > 5:
    print("this does not print")

print("this always prints")   # outside the if block
```

---

## if / else — Two Branches

```python
age = 17

if age >= 18:
    print("adult")
else:
    print("minor")
```

`else` has no condition. It catches everything the `if` did not.

---

## if / elif / else — Multiple Branches

```python
score = 75

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(grade)    # C
```

**Evaluation is top-down and stops at the first match.** The order of your conditions matters.

```python
# Wrong order — logic error, not a syntax error
score = 95

if score >= 60:
    grade = "D"    # this matches first — 95 >= 60 is True
elif score >= 90:
    grade = "A"    # never reached

print(grade)    # D — incorrect
```

Always order from most specific (strictest) to least specific (broadest).

---

## Condition Evaluation — What Counts as True

Python does not require `condition == True`. It evaluates **truthiness**. Any object can be a condition.

```python
# Falsy — these all take the else branch
if 0: ...
if 0.0: ...
if "": ...
if []: ...
if {}: ...
if (): ...
if None: ...

# Truthy — everything else takes the if branch
if 1: ...
if -1: ...
if "hello": ...
if [0]: ...         # non-empty list, even if contents are falsy
if " ": ...         # whitespace string is non-empty
```

```python
data = []

if data:
    process(data)
else:
    print("no data")    # runs — empty list is falsy
```

This is idiomatic Python. Do not write `if len(data) > 0` or `if data != []` — write `if data`.

## Comparison Operators in Conditions

```python
x = 7

if x == 7: ...          # equal
if x != 7: ...          # not equal
if x > 5: ...           # greater than
if x < 10: ...          # less than
if x >= 7: ...          # greater than or equal
if x <= 7: ...          # less than or equal
```

**Chained comparisons:**

```python
x = 5

if 0 < x < 10:
    print("x is between 0 and 10")    # runs
```

This is equivalent to `x > 0 and x < 10` but is more readable and mathematically natural. Use it.

---

## Logical Operators in Conditions

```python
age = 25
income = 50000

# and — both must be true
if age >= 18 and income >= 30000:
    print("eligible")

# or — at least one must be true
if age < 18 or income < 10000:
    print("not eligible")

# not — negation
if not is_banned:
    print("access granted")
```

**Short-circuit behavior affects performance and safety:**

```python
# The second condition is only evaluated if the first is True
if data is not None and len(data) > 0:
    process(data)

# If data is None, len(data) is never called — avoids TypeError
# Reversed order would crash:
if len(data) > 0 and data is not None:   # TypeError if data is None
    process(data)
```

Order your conditions from cheapest and most likely to fail, to most expensive.

---

## Nested if Statements

```python
x = 15

if x > 0:
    if x > 10:
        print("large positive")
    else:
        print("small positive")
else:
    print("non-positive")
```

Nesting works but increases cognitive load. Prefer flattening with `and` where possible:

```python
# Equivalent and cleaner
if x > 0 and x > 10:
    print("large positive")
elif x > 0:
    print("small positive")
else:
    print("non-positive")
```

If you are more than 2-3 levels deep, refactor into functions. Deep nesting is a design problem.

---

## Ternary Expression — Inline if/else

A one-line conditional that returns a value. Used for simple assignments.

```python
# Syntax: value_if_true if condition else value_if_false

x = 10
label = "even" if x % 2 == 0 else "odd"
print(label)    # even
```

```python
age = 20
status = "adult" if age >= 18 else "minor"

score = 85
result = "pass" if score >= 50 else "fail"

print("yes" if flag else "no")
```

**Do not nest ternary expressions — it destroys readability:**

```python
# Avoid
result = "A" if score >= 90 else "B" if score >= 80 else "C"

# Use if/elif instead
if score >= 90:
    result = "A"
elif score >= 80:
    result = "B"
else:
    result = "C"
```

## Identity and Membership in Conditions

```python
x = None

# Identity — use is for None, True, False
if x is None:
    print("no value")

if x is not None:
    print("has value")

# Membership — check if value exists in collection
fruits = ["apple", "banana", "cherry"]

if "apple" in fruits:
    print("found")

if "mango" not in fruits:
    print("not found")

# Dict membership checks keys only
config = {"debug": True, "verbose": False}

if "debug" in config:
    print(config["debug"])
```

## pass — Empty Branch

`pass` is a no-op. Use it when syntax requires a block but you have nothing to execute yet.

```python
x = 5

if x > 10:
    pass    # placeholder — will implement later
else:
    print("x is small")
```

Without `pass`, an empty block is a `SyntaxError`.

## Common Patterns

**Guard clauses — reject bad input early, keep main logic flat:**

```python
# Deep nesting — hard to follow
def process(data):
    if data is not None:
        if len(data) > 0:
            if isinstance(data, list):
                return result

# Guard clauses — flat and readable
def process(data):
    if data is None:
        return None
    if len(data) == 0:
        return []
    if not isinstance(data, list):
        raise TypeError("expected list")

    # actual logic here — no nesting
    return result
```

Guard clauses are a fundamental pattern in clean code. Get used to writing them.

**Checking multiple values with in:**

```python
# Verbose
if status == "active" or status == "pending" or status == "review":
    process()

# Clean
if status in ("active", "pending", "review"):
    process()

# For large collections, use a set for O(1) lookup
VALID_STATUSES = {"active", "pending", "review"}

if status in VALID_STATUSES:
    process()
```

**Condition assigned to a variable for reuse:**

```python
is_valid = 0 <= score <= 100

if is_valid:
    compute_grade(score)

if is_valid:
    log_score(score)
```

## match / case — Python 3.10+

Python has no `switch` statement before 3.10. Python 3.10 introduced structural pattern matching:

```python
command = "quit"

match command:
    case "quit":
        exit()
    case "help":
        show_help()
    case "start":
        start_program()
    case _:
        print("unknown command")   # _ is the wildcard, equivalent to else
```

Pattern matching can also match on types and structure, not just values:

```python
match point:
    case (0, 0):
        print("origin")
    case (x, 0):
        print(f"on x-axis at {x}")
    case (0, y):
        print(f"on y-axis at {y}")
    case (x, y):
        print(f"at ({x}, {y})")
```

Not essential right now — but worth knowing it exists.

## Indentation Rules

```python
# Correct — 4 spaces per level
if x > 0:
    print("positive")
    print("still in block")
print("outside block")

# IndentationError — inconsistent indentation
if x > 0:
    print("positive")
  print("wrong indent")

# SyntaxError — empty block without pass
if x > 0:
# nothing here            # use pass
```

Configure your editor to insert 4 spaces on Tab. Never mix tabs and spaces — Python 3 raises `TabError`.

## Summary

|Construct|Use case|
|---|---|
|`if`|Single condition, optional execution|
|`if / else`|Binary choice|
|`if / elif / else`|Multiple mutually exclusive branches|
|Ternary `x if c else y`|Inline single-line assignment|
|`pass`|Empty block placeholder|
|`match / case`|Python 3.10+, structural pattern matching|

## Common Mistakes to Avoid

```python
# Mistake 1: wrong condition order in elif chains
if score >= 60:      # matches score=95 immediately
    grade = "D"
elif score >= 90:    # never reached
    grade = "A"

# Mistake 2: using == for None
if x == None: ...      # wrong
if x is None: ...      # correct

# Mistake 3: verbose truthiness check
if len(data) > 0: ...  # unnecessary
if data: ...           # idiomatic

# Mistake 4: nested ternary
x = "A" if a else "B" if b else "C"    # unreadable

# Mistake 5: wrong short-circuit order
if len(data) > 0 and data is not None:   # crashes if data is None
if data is not None and len(data) > 0:   # correct

# Mistake 6: checking multiple values with or
if x == "a" or x == "b" or x == "c":   # verbose
if x in ("a", "b", "c"):               # clean
```
