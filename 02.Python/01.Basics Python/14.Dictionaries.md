
## Core Idea

A dictionary is an **unordered** (insertion-ordered since Python 3.7), **mutable** mapping of unique **keys** to **values**. It is the primary way to associate data in Python.

- **Keys** — must be hashable (immutable): strings, ints, floats, tuples of hashables
- **Values** — can be anything: any type, any depth of nesting
- **Lookup by key** — O(1) average, implemented as a hash table

A dict is not a sequence — you do not access values by position. You access them by key. This distinction determines when a dict is the right tool.

```python
# List — access by position
scores = [88, 92, 75]
scores[0]               # 88 — what does index 0 mean?

# Dict — access by meaning
scores = {"alice": 88, "bob": 92, "carol": 75}
scores["alice"]         # 88 — immediately clear
```

## Creating Dictionaries

```python
# Empty dict
empty = {}
empty = dict()

# Literal syntax
person = {"name": "Alice", "age": 30, "role": "Engineer"}

# dict() constructor with keyword arguments
person = dict(name="Alice", age=30, role="Engineer")
# keys become strings automatically

# From a list of (key, value) pairs
pairs = [("a", 1), ("b", 2), ("c", 3)]
d = dict(pairs)                  # {"a": 1, "b": 2, "c": 3}

# From two lists using zip
keys = ["x", "y", "z"]
values = [10, 20, 30]
d = dict(zip(keys, values))      # {"x": 10, "y": 20, "z": 30}

# Dict comprehension
squares = {x: x**2 for x in range(6)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}
```

## Accessing Values

```python
config = {"host": "localhost", "port": 8080, "debug": True}

# Direct access by key
print(config["host"])        # localhost
print(config["port"])        # 8080

# KeyError if key does not exist
print(config["timeout"])     # KeyError: 'timeout'
```

### get() — Safe Access with Default

```python
# get(key) returns None if key missing — no exception
print(config.get("timeout"))          # None

# get(key, default) returns default if key missing
print(config.get("timeout", 30))      # 30
print(config.get("port", 80))         # 8080 — key exists, returns value
```

Use `get()` whenever the key may not exist. Use direct `[]` access when the key must exist — the `KeyError` is then a useful signal that something is wrong upstream.

## Adding and Updating

```python
user = {"name": "Bob", "age": 25}

# Add new key
user["email"] = "bob@example.com"
print(user)    # {"name": "Bob", "age": 25, "email": "bob@example.com"}

# Update existing key
user["age"] = 26
print(user)    # {"name": "Bob", "age": 26, "email": "bob@example.com"}

# update() — merge another dict or iterable of pairs
user.update({"role": "Admin", "age": 27})
print(user)    # age updated, role added

user.update(role="Developer", active=True)   # keyword arguments also work
```

### Merge Operators (Python 3.9+)

```python
defaults = {"timeout": 30, "retries": 3}
overrides = {"timeout": 60, "debug": True}

# | creates a new merged dict
merged = defaults | overrides
print(merged)    # {"timeout": 60, "retries": 3, "debug": True}
# right side wins on conflict

# |= merges in place
defaults |= overrides
```

## Removing Keys

```python
data = {"a": 1, "b": 2, "c": 3, "d": 4}

# pop(key) — remove and return value — KeyError if missing
val = data.pop("b")
print(val)      # 2
print(data)     # {"a": 1, "c": 3, "d": 4}

# pop(key, default) — safe removal
val = data.pop("z", None)   # returns None instead of raising KeyError
print(val)      # None

# popitem() — remove and return last inserted (key, value) pair
item = data.popitem()
print(item)     # ("d", 4) — last inserted in Python 3.7+
print(data)     # {"a": 1, "c": 3}

# del — remove by key
del data["a"]
print(data)     # {"c": 3}

# del on missing key — KeyError
del data["z"]   # KeyError: 'z'

# clear — remove all entries
data.clear()
print(data)     # {}
```

## Checking Keys

```python
config = {"host": "localhost", "port": 8080}

# in — checks keys only — O(1)
print("host" in config)       # True
print("timeout" in config)    # False
print("localhost" in config)  # False — values not checked

# not in
print("timeout" not in config)   # True
```

## Iterating

```python
scores = {"alice": 92, "bob": 85, "carol": 78}

# Iterate keys (default)
for name in scores:
    print(name)
# alice, bob, carol

# Iterate keys explicitly
for name in scores.keys():
    print(name)

# Iterate values
for score in scores.values():
    print(score)
# 92, 85, 78

# Iterate key-value pairs — most common
for name, score in scores.items():
    print(f"{name}: {score}")
# alice: 92
# bob: 85
# carol: 78
```

`.keys()`, `.values()`, and `.items()` return **view objects** — they reflect changes to the dict in real time and do not create copies. Convert to `list()` if you need a snapshot.

```python
keys_view = scores.keys()
scores["dave"] = 88
print(keys_view)    # dict_keys(['alice', 'bob', 'carol', 'dave']) — live update
```

**Do not modify a dict while iterating over it** — raises `RuntimeError`. Iterate over a copy if mutation is needed:

```python
for key in list(scores.keys()):    # list() creates a snapshot
    if scores[key] < 80:
        del scores[key]
```

## Dict Comprehensions

```python
# Basic
squares = {x: x**2 for x in range(6)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# With condition
even_squares = {x: x**2 for x in range(10) if x % 2 == 0}
# {0: 0, 2: 4, 4: 16, 6: 36, 8: 64}

# Invert a dict (swap keys and values)
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
# {1: "a", 2: "b", 3: "c"}
# safe only if values are unique and hashable

# Filter dict entries
scores = {"alice": 92, "bob": 45, "carol": 78, "dave": 55}
passing = {name: score for name, score in scores.items() if score >= 60}
# {"alice": 92, "carol": 78}

# Transform values
normalized = {k: v / 100 for k, v in scores.items()}
```

## Nested Dictionaries

Dicts can contain dicts as values — common for structured data, configs, and JSON.

```python
users = {
    "alice": {"age": 30, "role": "admin", "active": True},
    "bob":   {"age": 25, "role": "user",  "active": False},
}

# Access nested value
print(users["alice"]["role"])       # admin

# Modify nested value
users["bob"]["active"] = True

# Add new nested entry
users["carol"] = {"age": 28, "role": "user", "active": True}

# Safe nested access
role = users.get("dave", {}).get("role", "unknown")   # unknown — dave missing
```

**Chained `.get()` for safe nested access** is a critical pattern. Direct `[]` access on nested dicts fails with `KeyError` if any level is missing.

## setdefault() and defaultdict

### setdefault()

Returns the value for a key if it exists. If not, inserts the key with a default value and returns it.

```python
data = {}

# Without setdefault
if "count" not in data:
    data["count"] = 0
data["count"] += 1

# With setdefault — cleaner
data.setdefault("count", 0)
data["count"] += 1
```

Classic use case — building a dict of lists (grouping):

```python
words = ["apple", "banana", "avocado", "blueberry", "cherry"]

grouped = {}
for word in words:
    key = word[0]                        # first letter
    grouped.setdefault(key, [])          # initialize if missing
    grouped[key].append(word)

# {"a": ["apple", "avocado"], "b": ["banana", "blueberry"], "c": ["cherry"]}
```

### defaultdict

From the `collections` module. Automatically initializes missing keys with a factory function.

```python
from collections import defaultdict

# List as default — grouping
grouped = defaultdict(list)
for word in words:
    grouped[word[0]].append(word)
# No KeyError, no setdefault needed

# Int as default — counting
counter = defaultdict(int)
for char in "mississippi":
    counter[char] += 1
# {"m": 1, "i": 4, "s": 4, "p": 2}

# Set as default — unique grouping
tag_map = defaultdict(set)
tag_map["python"].add("language")
tag_map["python"].add("snake")
```

`defaultdict` is cleaner than `setdefault` when you always need the same default type. Know both.

## Counter

`collections.Counter` is a dict subclass specifically for counting hashable objects.

```python
from collections import Counter

words = ["apple", "banana", "apple", "cherry", "banana", "apple"]
count = Counter(words)
print(count)
# Counter({'apple': 3, 'banana': 2, 'cherry': 1})

# Most common
print(count.most_common(2))    # [('apple', 3), ('banana', 2)]

# Count characters
letter_count = Counter("mississippi")
print(letter_count)
# Counter({'i': 4, 's': 4, 'p': 2, 'm': 1})

# Counter arithmetic
a = Counter(["x", "y", "x"])
b = Counter(["y", "z", "y"])
print(a + b)    # Counter({'y': 3, 'x': 2, 'z': 1})
print(a - b)    # Counter({'x': 2})  — subtracts, drops zeros and negatives
```

## Performance Characteristics

|Operation|Average Time|Notes|
|---|---|---|
|`d[key]`|O(1)|Hash lookup|
|`d[key] = v`|O(1)|Hash insertion|
|`del d[key]`|O(1)|Hash deletion|
|`key in d`|O(1)|Hash lookup|
|`d.get(key)`|O(1)||
|`len(d)`|O(1)|Cached|
|Iteration|O(n)|All keys/values/items|
|`d.copy()`|O(n)|Shallow copy|

Worst case for hash operations is O(n) due to hash collisions, but this is extremely rare with Python's hash implementation.

## Common Patterns in ML and CS

**Hyperparameter config:**

```python
config = {
    "learning_rate": 0.001,
    "batch_size": 32,
    "epochs": 100,
    "optimizer": "adam",
    "layers": [128, 64, 32],
}

lr = config.get("learning_rate", 0.01)
```

**Memoization / caching:**

```python
cache = {}

def fibonacci(n):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    result = fibonacci(n - 1) + fibonacci(n - 2)
    cache[n] = result
    return result
```

**Frequency counting:**

```python
def word_frequency(text):
    freq = {}
    for word in text.lower().split():
        freq[word] = freq.get(word, 0) + 1
    return freq
```

**Lookup table / dispatch table:**

```python
# Instead of long if/elif chains
def add(a, b): return a + b
def sub(a, b): return a - b
def mul(a, b): return a * b

operations = {
    "+": add,
    "-": sub,
    "*": mul,
}

result = operations["+"](10, 5)   # 15
```

**Graph as adjacency list:**

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A", "D"],
    "D": ["B", "C"],
}

# Neighbors of A
print(graph["A"])    # ["B", "C"]
```

You will use this exact representation when studying graph algorithms in CLRS and AI search algorithms in AIMA.

## Copying Dictionaries

```python
original = {"a": 1, "b": [2, 3]}

# Assignment — same object
ref = original
ref["c"] = 4
print(original)    # {"a": 1, "b": [2, 3], "c": 4}  — affected

# Shallow copy — new dict, shared inner objects
shallow = original.copy()     # or dict(original)
shallow["a"] = 99
print(original["a"])          # 1 — outer key unaffected

shallow["b"].append(99)
print(original["b"])          # [2, 3, 99] — inner list shared

# Deep copy — fully independent
import copy
deep = copy.deepcopy(original)
deep["b"].append(0)
print(original["b"])          # [2, 3, 99]  — unaffected
```

Same shallow vs deep rules as lists. If values are mutable objects, use `deepcopy`.

## Summary

|Feature|Behavior|
|---|---|
|Ordered|Yes — insertion order preserved (Python 3.7+)|
|Mutable|Yes|
|Key constraint|Must be hashable — str, int, float, tuple|
|Value constraint|Any type|
|Duplicate keys|Not allowed — later assignment overwrites|
|Lookup|O(1) average — hash table|
|Safe access|`.get(key, default)`|
|Iteration|`.keys()`, `.values()`, `.items()`|
|Comprehension|`{k: v for k, v in ...}`|
|Grouping|`defaultdict(list)` or `setdefault`|
|Counting|`Counter`|
|Merging|`.update()`, `|

## Common Mistakes to Avoid

```python
# Mistake 1: direct access on potentially missing key
config["timeout"]           # KeyError if missing
config.get("timeout", 30)   # correct

# Mistake 2: iterating and modifying dict simultaneously
for key in d:
    del d[key]              # RuntimeError
for key in list(d.keys()):
    del d[key]              # correct — iterate over snapshot

# Mistake 3: using mutable object as key
d[[1, 2]] = "value"         # TypeError — list is not hashable
d[(1, 2)] = "value"         # correct — tuple is hashable

# Mistake 4: assuming values are checked by 'in'
"localhost" in config       # False — only keys are checked
"localhost" in config.values()  # True — explicit value check

# Mistake 5: ignoring that .copy() is shallow
original = {"data": [1, 2, 3]}
copy = original.copy()
copy["data"].append(4)
print(original["data"])     # [1, 2, 3, 4] — shared inner list

# Mistake 6: reinventing Counter
freq = {}
for item in data:
    if item in freq:
        freq[item] += 1
    else:
        freq[item] = 0      # bug — should be 1 on first occurrence
# use: freq = Counter(data)

# Mistake 7: overwriting keys silently
d = {"a": 1}
d["a"] = 2                  # silently overwrites — no warning
# always check with 'in' if overwrite is not intended
```
