
## What is an Operator

An operator is a symbol that performs an operation on one or more operands and produces a result. Python operators are not magic syntax — under the hood, every operator maps to a **dunder method** on the object. This means operator behavior can be customized per class.

```python
a + b       # calls a.__add__(b)
a == b      # calls a.__eq__(b)
a > b       # calls a.__gt__(b)
```

Understanding this becomes critical when you build custom classes in ML (e.g. custom tensor types, loss objects).

## 1. Arithmetic Operators

|Operator|Name|Example|Result|
|---|---|---|---|
|`+`|Addition|`3 + 2`|`5`|
|`-`|Subtraction|`3 - 2`|`1`|
|`*`|Multiplication|`3 * 2`|`6`|
|`/`|Division|`7 / 2`|`3.5`|
|`//`|Floor Division|`7 // 2`|`3`|
|`%`|Modulo|`7 % 2`|`1`|
|`**`|Exponentiation|`2 ** 10`|`1024`|

```python
print(7 / 2)     # 3.5   — always returns float in Python 3
print(7 // 2)    # 3     — floor division, truncates toward negative infinity
print(-7 // 2)   # -4    — NOT -3 (floors toward negative infinity)
print(7 % 2)     # 1
print(-7 % 2)    # 1     — result has same sign as divisor in Python
print(2 ** 10)   # 1024
```

**Floor division vs truncation — a critical distinction:**

```python
import math

# Floor division floors toward negative infinity
-7 // 2          # -4   (floor of -3.5 is -4)

# int() truncates toward zero
int(-7 / 2)      # -3   (truncates -3.5 toward zero)

# math.floor always floors
math.floor(-3.5) # -4
```

This difference causes bugs in index calculations and modular arithmetic if you assume `//` behaves like C integer division (which truncates).

**Division always returns float:**

```python
print(4 / 2)          # 2.0  — float, not int
print(type(4 / 2))    # <class 'float'>
print(type(4 // 2))   # <class 'int'>
```

**Arithmetic with mixed types:**

```python
3 + 1.5       # 4.5   — int + float -> float (implicit promotion)
True + 4      # 5     — bool treated as int (True = 1)
False + 4     # 4
True + True   # 2
```

## 2. Comparison Operators

Return `True` or `False`. Used in conditionals and filtering.

|Operator|Name|Example|Result|
|---|---|---|---|
|`==`|Equal|`3 == 3`|`True`|
|`!=`|Not equal|`3 != 4`|`True`|
|`>`|Greater than|`5 > 3`|`True`|
|`<`|Less than|`3 < 5`|`True`|
|`>=`|Greater or equal|`3 >= 3`|`True`|
|`<=`|Less or equal|`2 <= 3`|`True`|

```python
print(3 == 3.0)    # True  — value equality across int and float
print(3 is 3.0)    # False — different objects, different types
```

**Python allows chained comparisons — unique to Python:**

```python
x = 5

# Instead of this:
if x > 0 and x < 10:
    print("in range")

# Write this:
if 0 < x < 10:
    print("in range")

# More examples
print(1 < 2 < 3)        # True
print(1 < 2 > 0)        # True
print(10 > 5 > 3 > 1)   # True
print(1 < 2 < 2)        # False
```

Chained comparisons are evaluated left to right and short-circuit. Use them — they are more readable and mathematically natural.

**Comparing different types:**

```python
print(1 == True)     # True   — bool is subclass of int
print(0 == False)    # True
print(1 == "1")      # False  — no implicit conversion
print(1 < "2")       # TypeError in Python 3 — cannot compare int and str
```

## 3. Logical Operators

Operate on boolean values (or any truthy/falsy values). Return the actual operand, not just True/False.

|Operator|Meaning|Returns|
|---|---|---|
|`and`|Both must be true|First falsy value, or last value|
|`or`|At least one must be true|First truthy value, or last value|
|`not`|Negation|`True` or `False`|

```python
print(True and False)    # False
print(True or False)     # True
print(not True)          # False
```

**Short-circuit evaluation — the actual return behavior:**

`and` returns the first falsy value it encounters, or the last value if all are truthy. `or` returns the first truthy value it encounters, or the last value if all are falsy.

```python
# and
print(0 and 5)       # 0      — 0 is falsy, returns immediately
print(3 and 5)       # 5      — 3 is truthy, evaluates 5, returns it
print(3 and 0)       # 0
print("" and "hi")  # ""

# or
print(0 or 5)        # 5      — 0 is falsy, tries next, returns 5
print(3 or 5)        # 3      — 3 is truthy, returns immediately
print("" or "hi")   # "hi"
print(0 or "")      # ""      — both falsy, returns last
```

**Practical patterns using short-circuit:**

```python
# Default value pattern
name = user_input or "Anonymous"

# Guard pattern — avoids calling function if condition fails
data = get_data()
result = data and process(data)   # process() only called if data is truthy

# Safe attribute access
config = None
value = config and config.get("key")   # avoids AttributeError
```

**not always returns bool:**

```python
print(not 0)        # True
print(not 42)       # False
print(not "")       # True
print(not "hello")  # False
print(not [])       # True
```

## 4. Bitwise Operators

Operate on the binary representation of integers, bit by bit.

|Operator|Name|Example|Result|
|---|---|---|---|
|`&`|AND|`5 & 3`|`1`|
|`\|`|OR|`5 \| 3`|`7`|
|`^`|XOR|`5 ^ 3`|`6`|
|`~`|NOT (complement)|`~5`|`-6`|
|`<<`|Left shift|`2 << 3`|`16`|
|`>>`|Right shift|`16 >> 2`|`4`|

```python
# 5 = 0b0101
# 3 = 0b0011

print(5 & 3)    # 1    -> 0b0001  (AND: both bits must be 1)
print(5 | 3)    # 7    -> 0b0111  (OR: either bit is 1)
print(5 ^ 3)    # 6    -> 0b0110  (XOR: bits differ)
print(~5)       # -6   -> -(5+1)  (bitwise NOT, inverts all bits)
```

**Bit shifting:**

```python
print(2 << 3)    # 16   — left shift by 3: 2 * (2^3) = 16
print(16 >> 2)   # 4    — right shift by 2: 16 / (2^2) = 4
```

Left shift by n = multiply by 2^n. Right shift by n = integer divide by 2^n. These are faster than multiplication/division for powers of 2 and appear in low-level ML and systems code.

**Bitwise vs logical — do not confuse them:**

```python
# & and | are bitwise — no short-circuit, always evaluate both sides
# and / or are logical — short-circuit

print(True & False)    # False  — works on bool, but no short-circuit
print(True and False)  # False  — short-circuits

# This matters when the right side has side effects or is expensive
get_data() and process()   # process() not called if get_data() is falsy
get_data() & process()     # process() ALWAYS called — dangerous
```

## 5. Assignment Operators

|Operator|Equivalent to|
|---|---|
|`x = 5`|assign|
|`x += 3`|`x = x + 3`|
|`x -= 3`|`x = x - 3`|
|`x *= 3`|`x = x * 3`|
|`x /= 3`|`x = x / 3`|
|`x //= 3`|`x = x // 3`|
|`x %= 3`|`x = x % 3`|
|`x **= 3`|`x = x ** 3`|
|`x &= 3`|`x = x & 3`|
|`x \|= 3`|`x = x \| 3`|
|`x ^= 3`|`x = x ^ 3`|
|`x <<= 1`|`x = x << 1`|
|`x >>= 1`|`x = x >> 1`|

```python
x = 10
x += 5     # x = 15
x -= 3     # x = 12
x *= 2     # x = 24
x /= 4     # x = 6.0
x //= 2    # x = 3.0
x **= 3    # x = 27.0
x %= 5     # x = 2.0
```

**Important: augmented assignment on immutable types still creates a new object:**

```python
x = 5
print(id(x))    # some memory address

x += 1
print(id(x))    # different address — x now points to a new int object
```

**But for mutable types like lists, += mutates in place:**

```python
a = [1, 2]
b = a
a += [3]       # mutates the list in place
print(b)       # [1, 2, 3]  — b sees the change because they share the object

a = a + [4]    # creates a NEW list, rebinds a
print(b)       # [1, 2, 3]  — b does not see this change
```

This is a frequent source of bugs. Know it cold.

## 6. Identity Operators

|Operator|Meaning|
|---|---|
|`is`|Same object in memory|
|`is not`|Different objects in memory|

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)      # True  — same values
print(a is b)      # False — different objects
print(a is c)      # True  — same object

print(a is not b)  # True
```

**Use `is` only for:**

- Comparing against `None`
- Comparing against singletons (`True`, `False`)

```python
x = None
if x is None: ...       # correct
if x == None: ...       # works but semantically wrong — avoids __eq__

if flag is True: ...    # fine but usually just: if flag:
```

**Integer caching — a CPython implementation detail:**

```python
a = 256
b = 256
print(a is b)    # True  — CPython caches small integers (-5 to 256)

a = 257
b = 257
print(a is b)    # False — outside cache range, different objects
```

Do not rely on `is` for integer equality. This behavior is CPython-specific and version-dependent.

## 7. Membership Operators

|Operator|Meaning|
|---|---|
|`in`|Value exists in sequence|
|`not in`|Value does not exist in sequence|

```python
print(3 in [1, 2, 3])         # True
print(5 in [1, 2, 3])         # False
print("a" in "cat")           # True
print("x" not in "cat")       # True
print("key" in {"key": 1})    # True  — checks dict keys
```

**Performance difference matters:**

```python
# List: O(n) — scans every element
item in [1, 2, 3, ..., 1000000]

# Set: O(1) — hash lookup
item in {1, 2, 3, ..., 1000000}

# Dict: O(1) — hash lookup on keys
key in {"a": 1, "b": 2}
```

When checking membership in large collections repeatedly, use a set or dict — not a list. This matters enormously in ML preprocessing pipelines.

## 8. Walrus Operator (`:=`) — Python 3.8+

Assigns a value and returns it in a single expression. Called the assignment expression operator.

```python
# Without walrus
data = get_data()
if data:
    process(data)

# With walrus
if data := get_data():
    process(data)
```

```python
# Useful in while loops
while chunk := file.read(8192):
    process(chunk)

# Useful in list comprehensions
results = [y for x in data if (y := transform(x)) > 0]
```

Use it when the assignment and condition are tightly coupled. Do not use it just to save a line — clarity matters.

## Operator Precedence

Higher in the table = evaluated first.

|Precedence|Operator(s)|
|---|---|
|1 (highest)|`()` parentheses|
|2|`**`|
|3|`+x`, `-x`, `~x` (unary)|
|4|`*`, `/`, `//`, `%`|
|5|`+`, `-`|
|6|`<<`, `>>`|
|7|`&`|
|8|`^`|
|9|`\|`|
|10|`==`, `!=`, `>`, `<`, `>=`, `<=`, `is`, `is not`, `in`, `not in`|
|11|`not`|
|12|`and`|
|13 (lowest)|`or`|

```python
print(2 + 3 * 4)         # 14  — * before +
print((2 + 3) * 4)       # 20  — () overrides
print(2 ** 3 ** 2)       # 512 — ** is right-associative: 2 ** (3**2) = 2**9
print(True or False and False)  # True — and before or
```

**Practical rule: use parentheses liberally.** Do not rely on precedence memory in complex expressions — parentheses make intent explicit and prevent bugs.

```python
# Ambiguous — reader must recall precedence
result = a + b * c ** 2 - d / e

# Clear
result = a + (b * (c ** 2)) - (d / e)
```

## Operator Overloading (Preview)

Every operator calls a dunder method. You can define these in your own classes:

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

v1 = Vector(1, 2)
v2 = Vector(3, 4)
print(v1 + v2)    # Vector(4, 6)
```

This is how NumPy arrays support `+`, `-`, `*` etc. directly. When you write `array1 + array2` in NumPy, it calls `ndarray.__add__()`, which dispatches to optimized C code.

## Summary

|Category|Operators|Key Behavior|
|---|---|---|
|Arithmetic|`+ - * / // % **`|`/` always float, `//` floors toward -inf|
|Comparison|`== != > < >= <=`|Chainable, return bool|
|Logical|`and or not`|Short-circuit, return operand not bool|
|Bitwise|`& \| ^ ~ << >>`|On binary representation of ints|
|Assignment|`= += -= *=` etc.|Augmented forms mutate mutable objects|
|Identity|`is is not`|Object identity, use only for None/singletons|
|Membership|`in not in`|O(n) for list, O(1) for set/dict|
|Walrus|`:=`|Assign and return in one expression|

## Common Mistakes to Avoid

```python
# Mistake 1: using == instead of is for None
if x == None: ...       # wrong
if x is None: ...       # correct

# Mistake 2: expecting // to truncate like C
-7 // 2                 # -4, not -3 (floors, not truncates)

# Mistake 3: confusing and/or with &/|
if x > 0 & x < 10:     # wrong — & has higher precedence than >
if x > 0 and x < 10:   # correct

# Mistake 4: not using parentheses in complex expressions
2 ** 3 ** 2             # 512, not 64 — right associative

# Mistake 5: using 'in' on large lists instead of sets
if item in large_list:  # O(n) — slow
if item in large_set:   # O(1) — fast
```
