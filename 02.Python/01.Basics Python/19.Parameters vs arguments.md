
## Core Idea

These two terms are used interchangeably in casual conversation but they refer to different things. Conflating them causes confusion when reading documentation, error messages, and technical literature — especially in ML frameworks where the distinction is precise.

- **Parameter** — the variable name declared in the function definition
- **Argument** — the actual value passed to the function at call time

```python
def add(a, b):        # a and b are PARAMETERS
    return a + b

add(3, 4)             # 3 and 4 are ARGUMENTS
```

Parameters live in the function signature. Arguments live at the call site. When the function is called, each argument is bound to its corresponding parameter.

---

## The Binding Mechanism

When a function is called, Python creates a new local namespace and binds each argument to its parameter name. This is not copying the value — it is binding a name to an object.

```python
def inspect(x):
    print(id(x))

n = 42
print(id(n))        # some address, e.g. 140234567890
inspect(n)          # same address — x and n point to the same object
```

Python passes by **object reference** — sometimes called pass-by-assignment. The parameter becomes a new name pointing to the same object the argument points to.

This single fact explains all of the following behavior.

---

## Pass by Object Reference — What It Means

### Immutable Objects — Reassignment Has No Effect Outside

```python
def double(x):
    x = x * 2      # rebinds x to a new object — original is untouched
    print(x)

n = 5
double(n)           # 10
print(n)            # 5  — unchanged
```

`x = x * 2` creates a new integer object and rebinds the local name `x` to it. The caller's variable `n` still points to `5`. Nothing was mutated — a new object was created and `x` was redirected.

This applies to all immutable types: `int`, `float`, `str`, `bool`, `tuple`.

### Mutable Objects — Mutation Is Visible Outside

```python
def append_one(lst):
    lst.append(1)   # mutates the object lst points to

data = [10, 20, 30]
append_one(data)
print(data)         # [10, 20, 30, 1]  — caller sees the change
```

`lst` and `data` point to the same list object. Calling `.append()` mutates the object in place — there is no rebinding, no new object, so the caller sees the change.

### Rebinding a Mutable Parameter Has No Effect Outside

```python
def replace(lst):
    lst = [99, 99, 99]   # rebinds local name lst — original untouched

data = [1, 2, 3]
replace(data)
print(data)              # [1, 2, 3]  — unchanged
```

`lst = [99, 99, 99]` creates a new list and rebinds the local name `lst`. The caller's `data` still points to `[1, 2, 3]`. Rebinding never affects the caller — only mutation does.

### The Complete Picture

```python
def f(x):
    # This mutates — caller sees it
    x.append(99)

    # This rebinds — caller does NOT see it
    x = [0, 0, 0]

data = [1, 2, 3]
f(data)
print(data)    # [1, 2, 3, 99]
               # append visible, rebinding not visible
```

## Four Kinds of Parameters

### 1. Positional Parameters

Bound by position. The order of arguments at call time determines which parameter receives which value.

```python
def power(base, exponent):
    return base ** exponent

power(2, 10)    # base=2, exponent=10
power(10, 2)    # base=10, exponent=2
```

Swapping the argument order changes the result — no error is raised.

---

### 2. Default Parameters

Parameters with a pre-assigned value. The argument becomes optional at call time.

```python
def greet(name, greeting="Hello", punctuation="!"):
    return f"{greeting}, {name}{punctuation}"

greet("Alice")                          # Hello, Alice!
greet("Alice", "Hi")                    # Hi, Alice!
greet("Alice", punctuation=".")         # Hello, Alice.
greet("Alice", "Hey", "?")             # Hey, Alice?
```

**Rule:** parameters with defaults must appear after parameters without defaults.

```python
def f(a, b=2, c=3): ...    # valid
def f(a=1, b, c=3): ...    # SyntaxError
```

**The mutable default trap — reviewed from a parameter perspective:**

```python
# The default value is evaluated ONCE when Python reads the def statement
# not once per call

def f(data=[]):
    data.append(1)
    return data

f()    # [1]
f()    # [1, 1]  — same list object reused
f()    # [1, 1, 1]

# Correct pattern — None as sentinel, create fresh object inside
def f(data=None):
    if data is None:
        data = []
    data.append(1)
    return data

f()    # [1]
f()    # [1]  — fresh list each call
```

---

### 3. *args — Variadic Positional Parameters

Collects all extra positional arguments into a **tuple** inside the function. The name `args` is a convention — the `*` is what matters.

```python
def total(*args):
    print(type(args))    # <class 'tuple'>
    return sum(args)

total(1, 2, 3)         # args = (1, 2, 3)     -> 6
total(1, 2, 3, 4, 5)   # args = (1, 2, 3, 4, 5) -> 15
total()                # args = ()             -> 0
```

```python
def f(a, b, *args):
    print(a, b, args)

f(1, 2, 3, 4, 5)    # a=1, b=2, args=(3, 4, 5)
f(1, 2)             # a=1, b=2, args=()
```

`a` and `b` consume the first two arguments. `*args` captures everything remaining.

---

### 4. **kwargs — Variadic Keyword Parameters

Collects all extra keyword arguments into a **dict** inside the function. The name `kwargs` is a convention — the `**` is what matters.

```python
def display(**kwargs):
    print(type(kwargs))    # <class 'dict'>
    for k, v in kwargs.items():
        print(f"  {k}: {v}")

display(name="Alice", age=30, role="Engineer")
# kwargs = {"name": "Alice", "age": 30, "role": "Engineer"}
```

```python
def f(a, b=10, **kwargs):
    print(a, b, kwargs)

f(1)                         # a=1, b=10, kwargs={}
f(1, b=20)                   # a=1, b=20, kwargs={}
f(1, b=20, extra="yes")      # a=1, b=20, kwargs={"extra": "yes"}
f(1, 2, x=3, y=4)            # a=1, b=2,  kwargs={"x": 3, "y": 4}
```

## Parameter Order in a Signature

When all parameter types appear together, they must follow this strict order:

```
def f(positional_only, /, standard, *args, keyword_only, **kwargs):
```

```python
def f(a, b, c=10, *args, kw=None, **kwargs):
    print(f"a={a}, b={b}, c={c}, args={args}, kw={kw}, kwargs={kwargs}")

f(1, 2)
# a=1, b=2, c=10, args=(), kw=None, kwargs={}

f(1, 2, 3, 4, 5, kw="yes", extra=True)
# a=1, b=2, c=3, args=(4, 5), kw="yes", kwargs={"extra": True}
```

In practice, most functions use only one or two of these types. The full combination appears mainly in wrapper and decorator patterns.

---

## Positional-Only Parameters — `/`

Parameters before `/` must be passed by position. They cannot be named at the call site.

```python
def f(a, b, /, c):
    print(a, b, c)

f(1, 2, 3)          # valid
f(1, 2, c=3)        # valid — c is not positional-only
f(1, b=2, c=3)      # TypeError — b is positional-only
f(a=1, b=2, c=3)    # TypeError — a and b are positional-only
```

This is used in built-in functions and library APIs to prevent reliance on parameter names that might change:

```python
# Built-in divmod uses positional-only
help(divmod)    # divmod(x, y, /)
```

---

## Keyword-Only Parameters — `*`

Parameters after `*` (bare star) or after `*args` must be passed by name. They cannot be passed positionally.

```python
def connect(host, port, *, timeout=30, ssl=False):
    print(host, port, timeout, ssl)

connect("localhost", 8080)                    # defaults used
connect("localhost", 8080, timeout=60)        # keyword only
connect("localhost", 8080, 60)               # TypeError — timeout is keyword-only
```

The bare `*` consumes no arguments — it only acts as a divider in the signature.

```python
def f(a, b, *args, kw_only):
    print(a, b, args, kw_only)

f(1, 2, 3, 4, kw_only="x")    # a=1, b=2, args=(3,4), kw_only="x"
f(1, 2, kw_only="x")          # a=1, b=2, args=(), kw_only="x"
f(1, 2)                        # TypeError — kw_only has no default and was not provided
```

Keyword-only parameters are valuable when functions have many parameters and positional ordering would be ambiguous or error-prone — common in ML model and optimizer constructors.

## Argument Unpacking at Call Time

You can unpack a sequence or mapping into arguments using `*` and `**` at the call site.

```python
def add(a, b, c):
    return a + b + c

# Unpack list/tuple as positional arguments
args = [1, 2, 3]
add(*args)          # equivalent to add(1, 2, 3)

t = (10, 20, 30)
add(*t)             # equivalent to add(10, 20, 30)

# Unpack dict as keyword arguments
kwargs = {"a": 1, "b": 2, "c": 3}
add(**kwargs)       # equivalent to add(a=1, b=2, c=3)

# Mix both
add(*[1, 2], **{"c": 3})    # 6
```

```python
# Forwarding arguments to another function
def wrapper(*args, **kwargs):
    print("before")
    result = original(*args, **kwargs)   # pass everything through
    print("after")
    return result
```

The `*args, **kwargs` forwarding pattern is how Python decorators work — they wrap a function and pass all arguments through unchanged.

## Argument Passing — Type Behavior Summary

```python
# int — immutable, reassignment does not affect caller
def f(x):
    x = 99
n = 5
f(n)
print(n)    # 5

# str — immutable, reassignment does not affect caller
def f(s):
    s = s.upper()
name = "alice"
f(name)
print(name)    # alice

# list — mutable, mutation affects caller
def f(lst):
    lst.append(99)
data = [1, 2, 3]
f(data)
print(data)    # [1, 2, 3, 99]

# list — mutable, but rebinding does not affect caller
def f(lst):
    lst = [99, 99]
data = [1, 2, 3]
f(data)
print(data)    # [1, 2, 3]

# dict — mutable, mutation affects caller
def f(d):
    d["new_key"] = 99
config = {"a": 1}
f(config)
print(config)    # {"a": 1, "new_key": 99}
```

## Defensive Copying Inside Functions

When a function should not modify its caller's data, copy inside the function.

```python
# Modifies caller's data — dangerous
def normalize(data):
    data.sort()
    # ... continues to modify data

# Safe — works on a copy
def normalize(data):
    data = data.copy()    # or sorted(data) for a new sorted list
    data.sort()
    return data
```

```python
import copy

# For nested structures
def process(config):
    config = copy.deepcopy(config)    # fully independent copy
    config["modified"] = True
    return config
```

As a rule: if your function takes a mutable argument and should not change the caller's object, copy at the top of the function before doing anything else.

## Introspecting Parameters — inspect Module

Python lets you examine a function's signature programmatically.

```python
import inspect

def train(model, lr=0.001, epochs=10, *, verbose=False):
    pass

sig = inspect.signature(train)
print(sig)
# (model, lr=0.001, epochs=10, *, verbose=False)

for name, param in sig.parameters.items():
    print(f"{name}: kind={param.kind.name}, default={param.default}")

# model:   kind=POSITIONAL_OR_KEYWORD, default=<class 'inspect._empty'>
# lr:      kind=POSITIONAL_OR_KEYWORD, default=0.001
# epochs:  kind=POSITIONAL_OR_KEYWORD, default=10
# verbose: kind=KEYWORD_ONLY, default=False
```

The `inspect` module is used internally by frameworks like PyTorch, FastAPI, and pytest to inspect function signatures and auto-wire arguments. Understanding it demystifies how those frameworks work.

## Summary

|Term|Definition|
|---|---|
|Parameter|Variable name in the function definition|
|Argument|Value passed at the call site|
|Positional parameter|Matched by order|
|Default parameter|Has a fallback value — argument is optional|
|`*args`|Captures extra positional args into a tuple|
|`**kwargs`|Captures extra keyword args into a dict|
|`/`|Parameters before it are positional-only|
|`*` (bare)|Parameters after it are keyword-only|
|Pass by reference?|No — pass by object reference (assignment)|
|Immutable arg|Reassignment inside function never affects caller|
|Mutable arg|Mutation inside function IS visible to caller|
|Rebind mutable arg|Never affects caller — only mutation does|
|Unpack at call site|`f(*list)`, `f(**dict)`|

## Common Mistakes to Avoid

```python
# Mistake 1: expecting reassignment of a list to affect the caller
def f(lst):
    lst = [99, 99]    # rebinds local name only — caller unchanged
data = [1, 2]
f(data)
print(data)    # [1, 2]

# Mistake 2: mutable default argument
def f(data=[]):
    data.append(1)
    return data
f()    # [1]
f()    # [1, 1]  — shared object
# use None sentinel

# Mistake 3: passing keyword argument positionally
def connect(host, port, *, timeout=30):
    ...
connect("localhost", 8080, 60)   # TypeError — timeout is keyword-only

# Mistake 4: confusing *args (tuple) with **kwargs (dict)
def f(*args):
    args["key"]    # TypeError — args is a tuple, not a dict

def f(**kwargs):
    kwargs[0]      # KeyError — kwargs is a dict with string keys (usually)

# Mistake 5: modifying caller's data unintentionally
def process(data):
    data.sort()    # mutates caller's list
    return data
# fix: data = sorted(data) or data = data.copy(); data.sort()

# Mistake 6: positional after keyword at call site
f(1, 2, key="x", 3)    # SyntaxError — positional after keyword
```
