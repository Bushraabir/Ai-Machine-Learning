## Core Idea

A `while` loop executes a block of code repeatedly as long as a condition remains truthy. Unlike a `for` loop — which iterates over a known sequence — a `while` loop runs for an **unknown number of iterations** determined by runtime state.

Use a `for` loop when you know what you are iterating over. Use a `while` loop when you are waiting for a condition to change.

```python
# for loop — known sequence
for item in data:
    process(item)

# while loop — unknown repetition count
while not converged:
    update_model()
```

---

## Basic Syntax

```python
while condition:
    # block executes as long as condition is truthy
    # something inside must eventually make condition falsy
    # or you must break out — otherwise infinite loop
```

```python
count = 0

while count < 5:
    print(count)
    count += 1

# 0, 1, 2, 3, 4
```

Three parts every while loop must have:

1. An initial state before the loop (`count = 0`)
2. A condition that will eventually become false (`count < 5`)
3. Something inside the loop that moves toward that condition (`count += 1`)

Missing any one of these produces an infinite loop or a loop that never runs.

---

## Condition Evaluation

The condition is evaluated before each iteration. If it is falsy on the very first check, the loop body never runs.

```python
x = 10

while x < 5:
    print(x)    # never executes — condition is false immediately

print("done")   # runs
```

The condition can be any truthy expression — not just a comparison:

```python
items = [1, 2, 3]

while items:             # truthy as long as list is non-empty
    item = items.pop()
    print(item)

# 3, 2, 1
```

---

## break — Exit the Loop Early

Exits the loop immediately, regardless of the condition.

```python
x = 0

while True:              # infinite loop — relies entirely on break
    if x >= 5:
        break
    print(x)
    x += 1

# 0, 1, 2, 3, 4
```

`while True` with a `break` is a deliberate and common pattern when the exit condition is complex or must be checked in the middle of the loop body, not at the top.

```python
# Interactive input loop
while True:
    command = input("Enter command: ")
    if command == "quit":
        break
    execute(command)
```

---

## continue — Skip to the Next Iteration

Skips the rest of the current iteration and re-evaluates the condition.

```python
x = 0

while x < 10:
    x += 1
    if x % 2 == 0:
        continue        # skip even numbers
    print(x)

# 1, 3, 5, 7, 9
```

**Critical: increment before continue, or you get an infinite loop.**

```python
# Infinite loop — x never increments when condition is true
x = 0
while x < 10:
    if x % 2 == 0:
        continue        # x stays 0 forever
    x += 1
```

When using `continue` in a `while` loop, ensure the loop variable is always updated before the `continue` statement.

---

## else Clause on a while Loop

The `else` block runs after the loop exits normally — only if the loop condition became false. It does NOT run if the loop exited via `break`.

```python
x = 0

while x < 5:
    print(x)
    x += 1
else:
    print("loop finished normally")

# 0, 1, 2, 3, 4
# loop finished normally
```

```python
x = 0

while x < 5:
    if x == 3:
        break
    print(x)
    x += 1
else:
    print("this does not run")    # break was hit

# 0, 1, 2
```

Same semantics as `for/else`. Useful for search and retry loops where you need to distinguish between "condition met" and "exhausted all attempts."

## Infinite Loops

A loop whose condition never becomes false. Either intentional (with a `break`) or a bug.

**Intentional — event loops, servers, interactive programs:**

```python
while True:
    event = get_next_event()
    if event.type == "quit":
        break
    handle(event)
```

**Bug — forgetting to update the loop variable:**

```python
x = 0
while x < 10:
    print(x)
    # x never increments — infinite loop
```

If your program freezes, an infinite loop is the first thing to check. In a terminal, `Ctrl+C` sends a `KeyboardInterrupt` to stop it.

## Common Patterns

### Countdown

```python
n = 10

while n > 0:
    print(n)
    n -= 1

print("done")
```

---

### Input Validation Loop

Repeatedly ask for input until valid data is provided:

```python
while True:
    raw = input("Enter a positive integer: ")
    if raw.isdigit() and int(raw) > 0:
        value = int(raw)
        break
    print("Invalid input. Try again.")

print(f"You entered: {value}")
```

This is cleaner than nested conditionals for validation. The loop keeps asking until the condition is satisfied, then breaks with the valid value.

---

### Retry with Limit

Attempt an operation up to N times before giving up:

```python
MAX_RETRIES = 5
attempts = 0
success = False

while attempts < MAX_RETRIES:
    try:
        result = connect_to_server()
        success = True
        break
    except ConnectionError:
        attempts += 1
        print(f"Attempt {attempts} failed. Retrying...")

if not success:
    print("All attempts failed.")
```

---

### Convergence Loop — Core ML Pattern

Run until a value stops changing significantly. This is how gradient descent and iterative solvers work.

```python
TOLERANCE = 1e-6

prev_loss = float("inf")
loss = compute_loss()

while abs(prev_loss - loss) > TOLERANCE:
    prev_loss = loss
    update_weights()
    loss = compute_loss()

print(f"Converged at loss: {loss}")
```

This pattern appears constantly in optimization algorithms. The loop has no fixed count — it runs until the mathematical condition is met.

---

### Draining a Queue or Stack

Process items until a collection is empty:

```python
stack = [1, 2, 3, 4, 5]

while stack:
    item = stack.pop()
    process(item)
```

```python
from collections import deque

queue = deque([1, 2, 3, 4, 5])

while queue:
    item = queue.popleft()
    process(item)
```

---

### Binary Search (Manual Loop)

```python
def binary_search(arr, target):
    low, high = 0, len(arr) - 1

    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1    # not found
```

Binary search is a canonical while loop — it does not iterate over a known sequence. It narrows a range based on runtime comparisons until the range collapses.

## while vs for — When to Use Which

|Situation|Use|
|---|---|
|Iterating over a list, tuple, string, dict, set|`for`|
|Iterating a fixed number of times|`for` with `range()`|
|Waiting for a condition to change|`while`|
|Unknown number of iterations|`while`|
|Input validation|`while True` with `break`|
|Convergence / optimization loops|`while`|
|Retry logic|`while` with counter|
|Event loops|`while True` with `break`|

If you can express it with `for`, use `for`. It is more readable and less prone to infinite loop bugs because the termination is built into the iterable.

---

## Nested while Loops

```python
i = 0
while i < 3:
    j = 0
    while j < 3:
        print(i, j)
        j += 1
    i += 1
```

`break` inside a nested while only breaks the inner loop. The outer loop continues.

Mix of for and while is perfectly valid:

```python
for epoch in range(100):
    while not batch_exhausted():
        train_on_batch()
```

---

## else with break — Full Pattern

```python
attempts = 0

while attempts < 3:
    password = input("Enter password: ")
    if password == "secret":
        print("Access granted")
        break
    attempts += 1
else:
    print("Too many failed attempts. Locked out.")
```

The `else` branch only runs if all 3 attempts were exhausted without a correct password. This is cleaner than a flag variable.

## Safety: Adding a Loop Guard

For loops that should converge but might not (due to floating point or logic errors), add a maximum iteration guard:

```python
MAX_ITER = 10_000
iteration = 0

while not converged and iteration < MAX_ITER:
    update()
    check_convergence()
    iteration += 1

if iteration == MAX_ITER:
    print("Warning: did not converge within iteration limit")
```

This is standard practice in numerical methods and ML training loops. Never trust a convergence loop without a safety cap.

---

## Summary

|Construct|Behavior|
|---|---|
|`while condition`|Runs as long as condition is truthy|
|`while True`|Infinite loop — must use `break` to exit|
|`break`|Exit loop immediately|
|`continue`|Skip to next iteration, re-check condition|
|`while ... else`|`else` runs only if loop exited without `break`|

## Common Mistakes to Avoid

```python
# Mistake 1: forgetting to update the loop variable — infinite loop
x = 0
while x < 10:
    print(x)
    # x += 1 missing

# Mistake 2: continue before increment — infinite loop
x = 0
while x < 10:
    if x % 2 == 0:
        continue      # x never advances past 0
    x += 1

# Mistake 3: using while when for is cleaner
i = 0
while i < len(data):     # verbose
    print(data[i])
    i += 1

for item in data:         # correct
    print(item)

# Mistake 4: no iteration cap on convergence loops
while not converged:
    update()
# if convergence never happens, this runs forever
# always add: and iteration < MAX_ITER

# Mistake 5: condition that can never become false
flag = True
while flag:
    do_work()
    # flag is never set to False — infinite loop

# Mistake 6: modifying condition variable inside continue without updating
x = 0
while x < 5:
    x += 1
    if x == 3:
        continue      # fine — x already incremented above continue
    print(x)
```
